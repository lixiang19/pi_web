# 认证部署与单用户服务器

## 用户模型

第一版是个人单用户服务器。

不做：

- 多用户；
- 团队；
- 用户级权限隔离；
- 多工作空间租户。

原因：AI 可以读写文件、运行命令、操作项目和终端，多用户会显著增加隔离、审计和越权风险。

## Web 登录

Web 使用管理密码登录。

- 管理密码通过环境变量配置。
- 登录成功后使用 HttpOnly Cookie 保存服务器会话。
- 前端不直接持有长期 token。

## 桌面登录与绑定

桌面端首次连接服务器时：

```txt
输入服务器 URL
→ 输入管理密码
→ 登录成功
→ 保存服务器地址
→ 保存桌面设备本地持久 ID
→ 保存长期设备 token
```

之后桌面端使用设备 token 自动连接服务器，建立设备长连接并上报心跳。

## 部署目标

第一版主要面向个人 VPS 部署。

- Node 服务运行在 VPS。
- Web 由服务器提供。
- 桌面端连接服务器。
- 移动端通过浏览器连接服务器。

公网访问默认假设 HTTPS。

HTTPS 可以由应用自身或反向代理提供，具体实现阶段确定。

## 安全边界

- 所有服务器 API 默认需要登录。
- 桌面设备必须完成绑定后才能注册设备和上报心跳。
- 桌面设备必须在线，服务器才能调度其本机项目会话。
- 项目会话运行必须受设备在线状态和项目注册约束。
- 多用户和团队权限不在第一版范围。
