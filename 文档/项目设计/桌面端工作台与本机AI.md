# 桌面端工作台与本机 AI

## 定位

桌面端是连接服务器的本机工作台。

- 桌面端必须登录服务器后使用。
- 工作空间仍属于服务器。
- 工作空间会话永远在服务器运行。
- 服务器项目在服务器运行。
- 桌面本机项目在对应桌面设备运行。
- Web 可以操作在线桌面设备上的本机项目。

## 本机项目

桌面端可以注册本机文件夹为项目。

- 项目绑定当前桌面设备。
- 项目文件不复制到服务器工作空间。
- 项目会话由桌面本机 Pi 运行。
- 项目会话正文保存在桌面设备的 Pi 会话存储中。
- 服务器保存项目注册和会话索引。
- 启动本机项目会话时，必须在线连接服务器获取 ridge runtime bundle。
- 桌面设备离线时，项目不能新建、继续或查看会话。

## Runtime Bundle

桌面本机 Pi 使用默认 `~/.pi/agent`，ridge 以覆盖写入方式管理配置。

每次启动本机项目会话前，服务器生成完整 ridge runtime bundle 并下发给桌面端。

桌面端把 bundle 物化到 ridge 管理目录，再启动 Pi。

bundle 至少包含：

- 全局 Agent；
- 全局 Skill；
- Mac-only 等设备专属 Skill；
- MCP 配置；
- 工具配置；
- 权限配置；
- 模型配置；
- `MEMORY.md` 和 `Wiki/index.md` 启动上下文；
- 必要脚本和资源文件。

服务器下发的 bundle 覆盖桌面本地 ridge 运行配置。

桌面端不能自行保留或选择本地 Skill。

项目目录中的 `.pi/agents`、`.pi/skills` 仍按 Pi 原机制读取，并覆盖 bundle 中的全局资源。

## Web 远程操作

Web 端操作桌面本机项目时，服务器负责调度。

- Web 请求先到服务器。
- 服务器校验登录、项目、设备在线状态。
- 服务器通过桌面长连接发送运行请求。
- 桌面端在本机项目目录启动 Pi。
- 桌面端消费本机 Pi SSE。
- 桌面端把事件通过长连接转发给服务器。
- 服务器再向 Web 暴露统一 SSE。

服务器不把桌面项目会话正文镜像到 `ridge.db`。

## 会话索引

所有项目会话都进入服务器会话索引。

- 服务器项目会话有索引。
- 桌面项目会话也有索引。
- 索引用于左侧列表、路由、任务关联和归档状态。
- 消息正文和过程仍以运行设备上的 Pi 会话为真源。
- 设备离线时，不打开该设备项目的会话历史。

## 桌面采集

桌面端提供菜单栏常驻采集入口。

第一版采集：

- 文字；
- 区域截图；
- 窗口截图；
- 全屏截图；
- 文件；
- 剪贴板；
- 当前选区；
- 浏览器网址；
- 录音。

采集时不选择项目，统一进入闪念。

## 浏览器网址

浏览器网址采集第一版走系统权限读取。

- 读取当前浏览器标签 URL；
- 能读取标题就保存标题；
- 能读取选中文本就保存选中文本；
- 权限不足或浏览器不支持时明确提示失败。

## 录音

录音先作为音频闪念保存。

录音采集不立即转写，不立即进入 RAG。

转写、结构化和知识化由处理链路决定。
