# 设备模型与在线状态

## 设备类型

第一版设备类型：

- 服务器设备；
- 桌面设备。

服务器设备由系统内置，代表当前 ridge 服务器。

桌面设备由桌面端首次登录服务器时自动注册。

## 设备身份

- 服务器设备使用系统内置固定身份。
- 桌面端首次启动生成本地持久设备 ID。
- 桌面设备 ID 保存在桌面本机应用数据中。
- 桌面端每次登录服务器时注册或更新设备信息。

## 在线状态

桌面设备通过长连接和心跳上报在线状态。

- 服务器根据最近心跳判断在线或离线。
- 桌面长连接用于服务器向桌面发送运行请求。
- 桌面长连接也用于转发桌面本机 Pi 产生的 SSE 事件。
- 设备 UI 显示名称、类型、在线/离线、最近在线时间。
- 设备离线时，其项目进入离线区。
- 离线项目已有会话索引仍保留。
- 离线项目不展开会话列表。
- 离线项目不能新建或继续项目会话。

## 设备能力

设备天然代表可运行能力，不单独维护能力表。

- 服务器设备可运行服务器项目、服务器终端和服务器侧 AI。
- 桌面设备可运行本机项目、本机终端和本机侧 AI。
- 具体能力由设备类型和项目位置决定。
- 桌面本机项目 AI 运行也依赖服务器在线，因为需要服务器下发 ridge runtime bundle。

## 设备专属能力

有些能力只在特定设备可用。

例如：

- 操作 Mac；
- 操作 Chrome；
- 本机截图；
- 本机窗口控制。

这些能力对应的 Skill 和工具配置仍由服务器管理。

服务器根据设备类型和能力生成 runtime bundle。

桌面端只负责执行服务器下发的能力，不负责本地选择。

## Web 操控设备项目

Web 端可以操控在线设备上的项目运行 AI。

规则：

- 项目必须已注册。
- 项目所属设备必须在线。
- Web 端打开项目会话时，请求先到服务器。
- 如果项目在服务器设备上，服务器直接运行 AI。
- 如果项目在桌面设备上，服务器把运行请求转发到桌面端执行。
- 桌面端消费本机 Pi SSE，再通过长连接转发给服务器。
- 服务器向 Web 暴露统一 SSE。
- 桌面设备离线时，Web 端不能新建或继续该项目会话。
- 桌面端无法连接服务器时，本机项目 Agent 也不能离线启动。

## 设备管理

第一版不做独立设备管理页。

- 设置中可以查看设备。
- 桌面设备允许重命名。
- 服务器设备不需要用户重命名。
- 设备不允许删除。
