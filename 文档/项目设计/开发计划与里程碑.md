# 开发计划与里程碑

## 定位

本文件把项目设计收束成第一版开发路线。

目标不是列完所有想法，而是形成可以逐步实现、验证和交付的里程碑。

第一版先做单用户服务器工作空间。

桌面端先作为连接服务器的采集端和本机项目运行设备。

## 总原则

- 先打通主闭环，再补体验。
- 先服务器工作空间，再桌面本机项目。
- 先 Markdown 文件资产，再 RAG、图谱、Wiki。
- 先可运行，再自动维护。
- 后台 Agent 不阻塞主对话。
- 外部仓库文件不复制进工作空间。
- 桌面本机 Pi 运行必须依赖服务器 runtime bundle。

## 第一版闭环

第一版必须打通：

```txt
输入
→ 会话 / 闪念 / 文件 / 任务
→ 处理和确认
→ Markdown / 任务 / 项目 / 空间 / 记忆
→ RAG / 图谱 / Wiki
→ 下一次会话可继续使用
```

## 里程碑 0：设计冻结与数据基座

目标：把产品边界和数据真源固定下来。

范围：

- 确认唯一工作空间。
- 确认 `ridge.db` 结构化真源。
- 确认 `~/ridge-workspace` 文件真源。
- 创建工作空间预制目录。
- 初始化隐藏 `.ridge`。
- 初始化 `记忆/MEMORY.md` 和 `Wiki/index.md`。
- 建立基础迁移框架。

验收：

- 新服务器启动后能创建完整工作空间结构。
- DB 可记录设备、项目、会话索引、任务、闪念、后台任务、通知、搜索状态。
- 不污染源码目录和用户真实 Pi 配置。

## 里程碑 1：工作台 Shell 与会话基础

目标：完成 ridge 的基本使用入口。

范围：

- 左侧固定入口。
- 右侧标签系统。
- 工作空间主页。
- 工作空间会话创建。
- 会话索引。
- 会话打开、归档、只读状态。
- 会话界面基础消息流。
- 右侧工作侧栏基础结构。

验收：

- 用户可以从主页发起工作空间会话。
- 会话出现在左侧工作空间会话列表。
- 同一个会话只能打开一个标签。
- 空主页关闭不产生会话。

## 里程碑 2：任务、里程碑与任务 Agent

目标：让工作可以落到可管理任务。

范围：

- 任务和里程碑 DB 模型。
- 系统默认里程碑。
- 任务列表和详情。
- 任务状态流转。
- 任务处理会话一对一。
- 任务 Agent。
- 规划工具。
- 完成必须用户确认。

验收：

- 用户可以创建任务和里程碑。
- 任务可以绑定项目或保持工作空间。
- 任务处理会话按绑定位置运行。
- Agent 不能把任务或里程碑改为完成。

## 里程碑 3：闪念与采集

目标：完成快速捕获入口。

范围：

- 闪念 DB。
- Web 闪念入口。
- 桌面菜单栏采集入口。
- 文字、截图、文件、剪贴板、选区、浏览器网址、录音入口。
- 闪念临时附件目录。
- fleeting agent 分析建议。
- 用户确认后处理为正式对象。

验收：

- 捕获阶段不分类、不选项目、不进入 RAG。
- 未处理闪念和临时附件不进入 RAG。
- 处理成功后删除原闪念。
- 处理失败不删除原闪念。

## 里程碑 4：文件、附件与空间

目标：让长期文件资产可以沉淀和预览。

范围：

- 文件页。
- 正式附件目录。
- 会话临时上传边界。
- 文件处理状态。
- PDF、Word、音频、图片标准化转换。
- Markdown、assets、metadata、`.originals`。
- 空间 HTML 作品目录。
- 私有空间 `iframe srcdoc` 预览。

验收：

- 正式文件位于工作空间可见目录。
- 临时文件位于 `.ridge`，不进入 RAG，不给 MCP 读取。
- PDF/Word 转换后以 Markdown 为 RAG 真源。
- 私有空间作品不生成公开 URL。

## 里程碑 5：RAG 与全局搜索

目标：让工作空间资产可找回。

范围：

- Markdown 全文索引。
- RAG chunk 构建。
- embedding 写入。
- RAG metadata。
- 上传立即索引。
- 普通变更夜间索引。
- 手动刷新索引。
- 全局搜索页面。
- 搜索聚合文件、任务、项目、会话索引、RAG、图谱节点。

验收：

- RAG 结果能回到来源文件和位置。
- 文件删除会删除 RAG chunk。
- 文件移动会更新 metadata。
- 全局搜索不启动 AI、不搜索外部仓库文件。

## 里程碑 6：记忆、daily 与后台会话链路

目标：让会话产生可持续长期记忆。

范围：

- summary agent。
- memory agent。
- `记忆/daily/YYYY/MM/YYYY-MM-DD.md`。
- `记忆/MEMORY.md`。
- Pi 会话结束钩子。
- 后台任务队列。
- 后台失败重试。
- MEMORY 和 Wiki 注入。

验收：

- 会话结束后 summary agent 追加 daily。
- memory agent 从 daily 和 MEMORY 更新长期记忆。
- MEMORY 短小、自然短句、不分区、不写来源。
- MEMORY 和 Wiki 空内容不注入。

## 里程碑 7：图谱与 Wiki 夜间维护

目标：让知识结构化并形成轻量入口。

范围：

- Kuzu 图谱库。
- graph agent。
- wiki agent。
- 夜间维护链路：RAG -> 图谱 -> Wiki。
- 图谱实体和关系抽取。
- 图谱证据短摘录。
- Wiki canonical 页面。
- Wiki/index 固定入口。

验收：

- 图谱不直接读取混杂原文件。
- 外部仓库不进入图谱。
- Wiki 是少量 Markdown 入口页。
- Wiki Agent 可以更新和删除低价值页面。

## 里程碑 8：项目注册与设备运行

目标：打通服务器项目和桌面本机项目。

范围：

- 内部项目。
- 外部仓库（服务器文件夹）。
- GitHub 拉取项目。
- 桌面本机文件夹项目。
- 设备注册和在线状态。
- Web 调度在线桌面项目。
- 桌面 Pi SSE 转发。
- workspace MCP。
- ridge runtime bundle。
- 设备专属 Skill。

验收：

- 项目必须绑定设备。
- 外部仓库文件不复制进工作空间。
- 桌面设备离线时不能新建、继续或查看项目会话。
- 桌面本机 Pi 不读取用户真实 `~/.pi` 作为 ridge 全局配置。
- 服务器下发 runtime bundle 覆盖桌面本地 ridge 运行配置。

## 里程碑 9：通知、任务回顾与自动化

目标：让后台系统可被用户处理。

范围：

- 通知与建议中心。
- notification_events。
- task review agent。
- 后台失败通知。
- 文件处理失败通知。
- 自动化规则。
- 自动化运行和跳过记录。
- 自动化失败通知。

验收：

- 需要用户处理的后台结果进入通知中心。
- 后台成功且无用户动作不打扰。
- task review agent 只提出建议，不自动完成任务。
- 项目设备离线时自动化跳过并记录原因。

## 里程碑 10：备份、恢复与收尾

目标：让第一版可长期使用。

范围：

- 服务器全量备份。
- 整包恢复。
- 备份校验信息。
- 恢复前快照。
- 设置页。
- 主题。
- 数据路径展示。
- 基础错误边界。

验收：

- 备份包含 `ridge.db`、工作空间、正式附件、Kuzu 图谱、隐藏版本、设置、注册信息和索引。
- 备份不包含外部仓库文件、桌面本机项目文件、桌面 Pi 会话正文、RAG 向量库和转换缓存。
- 第一版只做整包恢复。

## 实现顺序

推荐顺序：

1. 里程碑 0；
2. 里程碑 1；
3. 里程碑 2；
4. 里程碑 3；
5. 里程碑 4；
6. 里程碑 5；
7. 里程碑 6；
8. 里程碑 7；
9. 里程碑 8；
10. 里程碑 9；
11. 里程碑 10。

桌面本机项目不要提前做。

必须先把服务器工作空间闭环做通。

## 第一版不抢跑

第一版不做：

- 多用户；
- 多工作空间；
- 团队协作；
- 桌面离线运行；
- 外部仓库 RAG；
- 外部仓库文件备份；
- 复杂图谱画布；
- 空间公开发布；
- 通知规则编辑器；
- 选择性恢复。
