# V2 最终上线计划

## 定位

V2 是 ridge 的最终上线版。

V2 不再扩散新方向，不做“以后再补”的核心闭环。做完 V2 后，产品应达到可长期使用、可本机运行、可恢复、可诊断、可上线发布的状态。

Task 36 自动化规则运行与跳过按用户确认在其他 worktree 正常推进，本计划不把它作为当前 worktree 的阻塞项；合并时只做契约验收和回归。

## 上线目标

V2 上线必须同时满足：

- 主会话、任务、闪念、文件、空间、搜索、记忆、图谱、Wiki、通知、设置都能从 UI 真实使用。
- 服务器工作空间与桌面本机项目都能按设计运行。
- 数据可以备份、恢复、迁移到新机器，并有恢复前快照。
- 出错时用户能看到原因、重试入口和相关对象，不靠日志猜。
- 所有正式入口都有测试和 `npm run check` 通过记录。
- 不保留会误导用户的占位页、空按钮、伪功能或旧实现入口。

## V2 范围冻结

### 必做

1. 主会话资源选择器接入。
2. 文件页与正式附件管理完整闭环。
3. 桌面端 runtime bundle 物化与本机项目端到端运行。
4. 备份、恢复、设置和错误边界完整闭环。
5. 隐藏版本管理与空间作品保存点关系收口。
6. 搜索、RAG、图谱、Wiki、MCP 的用户可见诊断入口。
7. 上线安全、认证、数据目录、生产启动和打包流程。
8. 全量验收脚本、冒烟用例、手工验收清单。

### 合并验收但不在当前 worktree 主攻

- Task 36 自动化规则运行与跳过。

合并时必须验证：

- 自动化运行记录真实写入。
- 设备离线跳过有原因。
- 失败进入通知中心。
- 设置页或自动化页能看到运行历史。

### 不做

- 多用户。
- 多工作空间。
- 团队协作。
- 桌面离线运行。
- 外部仓库 RAG。
- 外部仓库文件备份。
- 复杂图谱画布。
- 空间公开发布。
- 插件市场。
- 选择性恢复。

## 阶段 1：主会话资源与 Agent 运行体验

目标：让主工作空间会话具备最终版 Agent 运行入口。

范围：

- `WorkspaceChatTab` 接入 commands、prompts、skills。
- 资源选择器使用 `GET /api/resources` 当前真源。
- Agent 选择、Skill 注入、prompt 插入、extension command 触发在主会话中真实可用。
- 旧 `SessionTabContent` / `SessionTabArea` 主会话双轨已删除，资源选择器只走 `WorkspaceChatTab`。
- 会话运行时显示模型、Agent、cwd、资源注入状态。
- 权限请求、ask、取消、归档只读、任务会话禁编辑等状态统一。

验收：

- 用户能在主工作空间会话选择 Skill / prompt / command 并发送，组件测试与 `usePerSessionChat` payload 测试覆盖该链路。
- 任务处理会话仍保持任务 Agent 和完成确认边界。
- resource catalog 后端返回空时 UI 显示真实空状态，不显示假入口。
- 有组件测试、composable 测试和会话集成测试。阶段 1 当前阻塞项已收口完成。

## 阶段 2：文件、附件、空间与隐藏版本

目标：文件资产从上传、转换、预览、编辑、沉淀、版本点到 RAG 全闭环。

范围：

- `FilesView` 增加上传、创建文件夹、重命名、移动、删除、重试转换入口。
- 正式附件目录与闪念附件迁移后的文件管理统一。
- 文件处理状态在文件页、通知中心、搜索结果中一致展示。
- 音频、图片 Markdown 标准产物按 Task 20 收口归档。
- 空间作品保存点进入隐藏版本管理，预览仍保持私有 `srcdoc`。
- Markdown 编辑保存、RAG deferred、手动刷新和失败通知链路可见。

验收：

- 从文件页上传 PDF/Word/图片/音频后能看到状态、产物和 RAG 结果。
- 文件移动/删除后 RAG、处理状态、通知关联同步更新。
- 空间作品修改后能形成可追溯保存点。
- 有服务端集成测试、组件测试和文件页 E2E。

## 阶段 3：桌面端与本机项目上线闭环

目标：桌面端不只是壳，而是可靠连接服务器的本机运行设备。

范围：

- 设备注册、token、心跳、离线清理、bundle ack 在桌面端真实使用。
- runtime bundle 下载、校验、物化到 ridge 管理目录。
- 桌面本机项目会话通过本机 Pi 运行，SSE 转发回 Web。
- 离线设备禁止新建、继续或查看需要设备的项目会话。
- 桌面采集入口完成文字、截图、文件、剪贴板、浏览器网址、录音。
- 桌面端不读取用户真实 `~/.pi` 作为 ridge 全局配置。

验收：

- 新设备首次启动能注册并出现在 Web。
- Web 对在线桌面项目发起会话，消息流能回到当前标签。
- 断网或关闭桌面端后，项目入口给出明确不可运行原因。
- 有 desktop bridge 单元测试、WebSocket 集成测试和手工验收记录。

## 阶段 4：备份、恢复、设置与错误边界

目标：上线后数据可长期维护，故障可恢复。

范围：

- 设置页展示数据目录、数据库路径、默认工作空间、服务状态、设备状态。
- 服务器全量备份包含 `ridge.db`、工作空间、正式附件、Kuzu 图谱、隐藏版本、设置、注册信息和索引。
- 备份排除外部仓库文件、桌面本机项目文件、桌面 Pi 会话正文、RAG 向量库和转换缓存。
- 整包恢复前自动创建恢复前快照。
- 恢复后重建可重建索引，并清楚显示重建状态。
- 全局错误边界覆盖工作台、会话、文件、任务、设置。

验收：

- 在干净目录能从备份恢复出可打开的工作空间。
- 恢复失败不破坏原数据，能回退到恢复前快照。
- 设置页能触发备份、选择恢复包、查看校验信息。
- 有备份恢复 API 测试、设置页组件测试和恢复手工验收。

## 阶段 5：知识系统可诊断化

目标：RAG、图谱、Wiki、MCP 不只是后台跑通，用户能判断当前知识是否健康。

范围：

- 搜索页显示索引队列、失败文件、最近索引时间和刷新入口。
- 记忆页或设置页显示 `MEMORY.md`、daily、Wiki 注入状态。
- 图谱维护状态、最近抽取时间、纠错入口和失败通知可见。
- workspace MCP 工具调用边界写入诊断页或设置页。
- 后台任务队列状态与通知中心关联可追踪。

验收：

- 用户能找到“为什么搜不到某个文件”的原因。
- 用户能手动刷新失败索引并看到结果。
- 图谱纠错后能看到对应状态变化或失败原因。
- 有搜索/RAG/通知联动测试。

## 阶段 6：上线质量门禁

目标：从“功能完成”推进到“可上线”。

范围：

- 删除所有无真实能力的占位入口。
- 全部文档、模块梳理、功能矩阵同步。
- 跑通 `npm run check`、`pnpm test`、关键 E2E。
- 生产启动检查：`RIDGE_ADMIN_PASSWORD`、端口、数据目录权限、数据库迁移。
- 桌面打包检查：Tauri 只打包，不承载业务功能。
- 上线验收文档：安装、首次启动、备份、恢复、桌面注册、主会话、任务、文件、搜索、通知。

验收：

- 根目录 `npm run check` 通过。
- 根目录 `pnpm test` 通过；若有环境型阻塞，记录精确阻塞日志并提供替代验证。
- 关键 E2E 通过或有明确人工验收记录。
- `文档/功能开发/` 当前任务全部归档。
- `文档/模块梳理/` 与最终代码一致。

## 上线验收清单

### 首次启动

- 启动服务后自动创建 `~/ridge-workspace`。
- 自动创建 `~/.pi/ridge.db`。
- 登录、主题、设置页正常。

### 主会话

- 从主页创建会话。
- 选择 Agent、Skill、prompt、command。
- ask、permission、cancel、归档只读均可用。

### 任务

- 创建里程碑和任务。
- 从任务开始处理会话。
- Agent 不能直接完成任务。
- 用户确认后完成任务。

### 闪念

- Web 捕获文字和文件。
- 桌面捕获截图、剪贴板、浏览器网址、录音。
- 闪念可处理为日记、剪藏、任务、里程碑、附件。

### 文件与空间

- 上传 PDF/Word/图片/音频。
- 转换产物可查看。
- 文件移动、删除、重命名后索引同步。
- 空间作品可私有预览且不生成公开 URL。

### 搜索与知识

- 搜索文件、任务、会话、记忆、Wiki、空间、RAG。
- 手动刷新 RAG。
- 记忆、Wiki、图谱夜间维护可诊断。

### 桌面项目

- 注册桌面设备。
- 外部仓库绑定设备。
- 在线时可运行本机项目会话。
- 离线时明确拒绝。

### 通知与自动化

- 文件失败、RAG 失败、后台任务失败进入通知中心。
- task review 只生成建议。
- Task 36 合并后，自动化运行、跳过、失败通知可见。

### 备份恢复

- 创建全量备份。
- 在新数据目录恢复。
- 恢复失败时保留恢复前快照。
