# 项目创建注册与归档删除

## 入口

项目创建和注册入口在左侧项目区。

## 来源

Web 端显示三种来源：

- 创建内部项目；
- 注册服务器文件夹；
- 从 GitHub URL 拉取。

桌面端显示四种来源：

- 创建内部项目；
- 注册服务器文件夹；
- 从 GitHub URL 拉取；
- 注册本机文件夹。

本机文件夹项目在桌面端创建，绑定当前桌面设备。

## 内部项目

内部项目只需要输入名称。

- 创建目录：`~/ridge-workspace/项目/<name>`。
- 项目名使用目录名。
- 内部项目默认进入工作空间 RAG。
- 内部项目可执行命令。
- 内部项目不是独立 Git 项目。

## 服务器文件夹项目

服务器文件夹通过服务器目录选择器注册。

- 目录选择器限制在服务器用户 Home 下。
- 注册时校验目录存在、可访问、同设备同路径未注册。
- 项目名使用文件夹名。
- 自动检测是否 Git 仓库。

## GitHub 项目

GitHub 项目通过粘贴 URL 拉取。

- 不做 GitHub 登录。
- 无权限时直接报错。
- 默认拉取到 `~/ridge-projects/<repo-name>`。
- 项目名使用仓库名。
- 拉取后自动检测 Git 仓库。

## 本机文件夹项目

桌面端支持选择本机文件夹。

- 本机项目绑定桌面设备。
- AI 在本机项目所在桌面设备运行。
- 本机项目需要访问服务器工作空间时，通过 MCP。
- Web 端不能创建本机文件夹项目。
- Web 端可以操作已注册且在线的本机项目。

## 注册规则

- 项目必须绑定设备。
- 项目就是设备上的文件夹。
- 同一设备同一路径不能重复注册。
- 注册后不能换设备。
- 注册后不能改路径。
- 注册后不能改名。
- 删除注册不删除项目文件。

## 归档与删除

- 项目有会话时禁止删除。
- 项目没有会话时，可以删除项目注册；只删除注册记录，不删除文件。
- 项目可以归档。
- 归档项目不显示在普通项目列表。
- 归档项目从左侧归档入口查看。
- 归档项目下的会话只读。
