# 空间 HTML 作品

## 定位

空间是工作空间里的 HTML 作品空间。

它用于保存和预览 AI 生成的 HTML 页面。

典型内容包括：

- 原型；
- 可视化；
- 报告；
- 交互 demo；
- 临时网页作品。

空间不是外部项目。

空间不是普通附件。

空间第一版只做预览，不做编辑器。

## 目录

空间是工作空间一级目录：

```txt
空间/
```

每个作品一个目录：

```txt
空间/<作品名>/index.html
```

第一版私有作品只要求支持单文件 HTML。

后续公开作品可以支持目录资源。

## 创建

空间作品主要由会话中的 AI 生成。

用户在任意会话要求生成页面、报告、可视化或原型时，AI 可以把产物保存为空间作品。

保存路径：

```txt
空间/<作品名>/index.html
```

空间作品是长期文件资产，进入工作空间隐藏版本管理。

## 预览入口

用户点击空间作品后，打开 ridge 内部的空间预览标签。

空间预览标签只负责展示 HTML。

HTML 源文件编辑仍走普通文件能力，第一版不在空间预览里提供编辑器。

## 私有预览

私有空间作品只在主站内渲染。

它不生成可复制的独立访问 URL。

实现方式：

1. 主站后端读取 `空间/<作品名>/index.html`；
2. 前端打开空间预览标签；
3. 使用受控 iframe 渲染；
4. iframe 使用 `srcdoc` 写入 HTML；
5. iframe 使用 sandbox 限制能力。

私有预览第一版只支持单文件 HTML。

单文件 HTML 可以包含内联 CSS 和内联 JS。

私有预览不允许：

- 读取工作空间文件；
- 调用 ridge 主站 API；
- 获取主站登录态；
- 访问隐藏目录；
- 产生公开链接。

私有预览是“主站内部展示”，不是“发布网页”。

## 公开作品

公开作品是后续能力。

公开作品可以生成公开只读链接。

公开作品才进入独立 preview/public 静态访问体系。

公开作品可以支持：

- `index.html`；
- 同作品目录 assets；
- CSS；
- JS；
- 图片。

公开作品和私有作品必须有明确状态区分。

## 安全边界

空间预览的核心安全原则：

- 私有作品不直接暴露静态 URL；
- 私有作品不使用主站同源权限；
- 私有作品不携带主站 cookie；
- 私有作品不能调用 ridge API；
- 私有作品第一版不读取工作空间文件；
- 公开作品才使用可分享 URL。

私有预览 iframe 必须使用 sandbox。

第一版 sandbox 默认不授予主站权限。

如果 HTML 需要更强能力，应该转为公开作品或后续设计专用权限。

## 与 RAG

空间作品是工作空间可见文件。

`空间/` 可以进入 RAG。

RAG 主要索引 HTML 文本内容和 metadata。

空间预览本身不触发 RAG 更新。

上传或生成空间作品后，按文件处理流程进入索引策略。

## 与文件处理

空间作品本质是工作空间文件。

AI 生成的 HTML 保存后成为正式文件。

普通变更等待夜间索引。

用户可以手动触发立即更新索引。

空间作品不放入 `附件/`。
