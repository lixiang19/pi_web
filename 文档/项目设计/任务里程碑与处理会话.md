# 任务里程碑与处理会话

## 基础模型

任务和里程碑默认属于唯一工作空间。

- 任务可选绑定项目。
- 里程碑可选绑定项目。
- 里程碑是更大的任务，但不创建处理会话。

任务和里程碑使用同一组状态：

- `pending`；
- `in_progress`；
- `blocked`；
- `reviewing`；
- `completed`。

新建任务和里程碑默认是 `pending`。

Agent 可以设置到 `pending`、`in_progress`、`blocked`、`reviewing`，不能设置到 `completed`。

`completed` 必须由用户人工确认。

## 字段

任务至少包含：

- 标题；
- 状态；
- 优先级：`normal`、`important`、`urgent`；
- 完成标准；
- 截止日期；
- 阻塞原因；
- 所属里程碑；
- 可选项目；
- 处理会话 ID；
- 排序；
- 创建和更新时间。

里程碑至少包含：

- 标题；
- 目标；
- 完成标准；
- 状态；
- 截止日期；
- 可选项目；
- 是否系统里程碑；
- 颜色；
- 排序；
- 创建和更新时间。

## 项目绑定

项目绑定是可选字段。

- 创建任务和里程碑时都提供项目下拉。
- 项目筛选位于任务系统顶部。
- 项目筛选包含全部、无项目、具体项目。
- 新任务默认继承所属里程碑的项目。
- 用户和 Agent 都可以覆盖继承结果，改为无项目或其他项目。

## 处理会话

任务和处理会话严格一对一。

- 一个任务最多一个处理会话。
- 任务已有处理会话时，入口只打开已有会话。
- 任务处理会话不允许通过编辑旧消息或重试回复创建分叉。

处理会话运行位置：

- 任务绑定项目时，创建项目会话。
- 任务未绑定项目时，创建工作空间会话。
- 任务绑定离线项目时，禁止启动或继续处理会话。

## 任务 Agent

任务处理入口强制使用任务 Agent。

- 模型使用全局默认模型。
- 思路强度使用全局默认思路强度。
- 任务 Agent 可使用规划工具。
- 任务 Agent 不能把任务或里程碑改为完成。
- 完成必须由用户人工确认。

## 任务回顾 Agent

task review agent 是后台 Agent。

它定期或由用户触发，回顾任务、里程碑、处理会话和最近 daily。

它只提出建议，不自动执行任务。

它不能把任务或里程碑改为完成。

它可以建议：

- 标记阻塞；
- 拆分任务；
- 调整优先级；
- 推进下一步；
- 请求用户确认完成；
- 提醒过期或长期无进展。

用户确认后，建议才写入正式任务或里程碑。

## 规划工具

任务 Agent 的规划工具可以创建或更新任务和里程碑。

- Agent 可显式指定项目。
- 不指定项目时，继承当前任务或里程碑上下文。
- Agent 创建任务时仍必须挂到某个里程碑。
- Agent 创建或更新里程碑时也可以选择项目。
- Agent 不能批量隐式修改某个里程碑下所有任务状态；多任务更新必须逐项明确。
- 工具成功时只返回必要结果；失败时必须返回可自我修正的明确原因。

## API 候选

任务和里程碑 API 保持工作空间边界。

- `GET /api/workspace/milestones`
- `POST /api/workspace/milestones`
- `GET /api/workspace/milestones/:milestoneId`
- `PATCH /api/workspace/milestones/:milestoneId`
- `DELETE /api/workspace/milestones/:milestoneId`
- `GET /api/workspace/tasks`
- `POST /api/workspace/tasks`
- `GET /api/workspace/tasks/:taskId`
- `PATCH /api/workspace/tasks/:taskId`
- `DELETE /api/workspace/tasks/:taskId`

实现时可以按新项目模型补充项目筛选参数。

## 验收点

- 首次查询可创建系统默认里程碑。
- 创建任务未指定里程碑时进入系统默认里程碑。
- 删除仍有任务的里程碑必须失败。
- Agent 不能完成任务或里程碑。
- 用户可以完成任务或里程碑。
- 一个任务只能绑定一个处理会话。

## Todo 工具

todo 是 Pi 自定义工具，不属于正式任务系统。

- 任何会话都可以使用 todo 工具。
- todo 内容保存在 Pi 会话内。
- 右侧摘要显示 todo 工具产生的内容。
- todo 不自动创建正式任务或里程碑。
