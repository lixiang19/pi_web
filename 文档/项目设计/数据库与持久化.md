# 数据库与持久化

## 真源

服务器 `~/.pi/ridge.db` 是 ridge 产品层结构化数据真源。

它保存：

- 设备；
- 项目；
- 任务和里程碑；
- 闪念；
- 自动化规则和简短运行记录；
- 会话索引；
- 搜索索引状态；
- 通知事件；
- 设置；
- 其他 ridge 产品状态。

Pi 原始会话正文不复制进 `ridge.db`。

## Pi 会话存储

Pi 原始会话跟随运行设备。

- 服务器会话存储在服务器 Pi 会话存储中。
- 桌面项目会话存储在桌面设备的 Pi 会话存储中。
- `ridge.db` 只保存会话索引和关联信息。

Web 查看桌面设备项目会话历史时，必须通过在线桌面设备读取 Pi 原始会话。

设备离线时，该设备项目的会话索引仍在服务器保留，但不展开会话列表，也不能打开历史。

桌面项目实时流也不镜像到服务器。

- 桌面端消费本机 Pi SSE。
- 桌面端通过设备长连接转发事件。
- 服务器再向 Web 暴露统一 SSE。
- 服务器只保存会话索引，不保存桌面项目消息正文。

## 会话索引

ridge 会话索引至少保存：

- sessionId；
- 标题；
- 会话类型；
- 工作空间或项目上下文；
- 任务关联；
- 设备关联；
- 运行位置；
- 归档状态；
- 只读状态；
- 创建时间；
- 更新时间。

消息正文仍以 Pi 原始会话为真源。

## 文件根目录

服务器主要目录：

- 工作空间：`~/ridge-workspace`；
- 服务器外部项目默认根：`~/ridge-projects`；
- 应用数据：`~/.pi`；
- 结构化数据库：`~/.pi/ridge.db`。

正式附件目录固定为：

```txt
~/ridge-workspace/附件
```

## 缓存和索引

RAG 向量库、Kuzu 图谱、转换缓存等放在工作空间 `.ridge` 内。

示例：

```txt
~/ridge-workspace/.ridge/
```

Kuzu 图谱是结构化理解资产，进入服务器完整备份。

RAG 向量库和转换缓存属于可重建缓存，不进入服务器完整备份。

这些内容都不进入隐藏版本管理。

表名可以在实现阶段按现有命名调整，但上述对象必须存在。
