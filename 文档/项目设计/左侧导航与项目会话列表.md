# 左侧导航与项目会话列表

## 定位

左侧导航负责打开固定功能、工作空间会话、项目会话和归档会话。

## 顶部固定入口

顶部固定入口全部平铺。

第一版入口：

- 闪念；
- 搜索；
- 通知；
- 任务；
- 文件；
- 终端；
- 自动化；
- Skill；
- 设置。

点击固定入口在右侧打开标签页。

- 闪念、搜索、通知、任务、文件、自动化、Skill、设置为单例标签，重复点击只激活。
- 终端不是单例，可以打开多个终端标签。
- 固定入口不通过旧独立页面路由跳转。
- 固定入口只操作工作台标签页。

旧的 `chat`、`terminal`、`automations`、`settings` 等独立页面路由不作为新设计入口。

## 工作空间会话

左侧显示工作空间历史会话列表。

- 会话按最近更新排序。
- 不支持置顶。
- 点击会话时，如果右侧已打开该会话标签，则激活；否则新开标签。
- 同一个会话只能存在于一个标签内。

工作空间新主页标签不从左侧创建，由右侧标签栏加号创建。

## 主页标签

右侧标签栏加号创建新的工作空间主页标签。

- 主页标签只有输入框和轻量摘要。
- 首次发送后才创建 Pi 会话和 ridge 会话索引。
- 空主页标签关闭时不产生会话记录。

## 项目列表

左侧显示项目列表，不按类型分组。

每个项目项显示：

- 项目名；
- 设备；
- 内部 / 外部；
- GitHub 来源；
- 在线、离线、归档状态；
- 新建项目会话加号。

项目下默认显示最近 3 个会话，并提供展开更多。

点击项目加号时，右侧打开该项目的新 AI 输入标签；首次发送后才创建项目会话和索引。

## 离线与归档

- 设备离线时，相关项目进入离线区。
- 离线项目已有会话索引仍保留。
- 离线项目不展开会话列表，不能打开历史，不能新建会话。
- 归档项目下的会话只读。
- 会话支持归档。
- 归档会话默认不显示在普通列表，从左侧底部“归档”入口查看。

## 会话索引

ridge 保存会话索引和元信息，Pi 原始会话数据仍是消息正文真源。

索引用于支持：

- 工作空间会话列表；
- 项目会话列表；
- 会话所属项目；
- 会话所属任务；
- 归档状态；
- 标题；
- 最近更新时间。
