# 项目管理与会话任务

## 项目来源

第一版支持三种项目创建方式。

- 工作空间内部项目：创建在 `~/ridge-workspace/项目/<name>`。
- 外部仓库（服务器文件夹）：注册服务器上的已有目录。
- GitHub 仓库：粘贴 URL 拉取到 `~/ridge-projects/<repo-name>`。

GitHub 不做登录；无权限时直接报错。

## 项目规则

- 项目必须绑定设备。
- 项目名自动使用文件夹名或仓库名。
- 同一设备同一路径不能重复注册。
- 项目有会话时禁止删除。
- 项目可以归档；归档项目下的会话只读。
- 设备离线时，相关项目进入离线区；索引保留，但不展开会话列表，也不能新建会话。

## 内部项目

内部项目服务研究、分类组织、执行和 RAG。

- 内部项目可以执行命令。
- 内部项目不是独立 Git 项目。
- 工作空间隐藏版本管理覆盖内部项目。
- 内部项目默认进入工作空间 RAG。
- **内部项目是组织/关注对象，不作为 pi 运行目录**。

外部仓库（含 GitHub、服务器文件夹、本机文件夹）不进入 RAG。

## 左侧项目列表

- 左侧显示已注册项目，按 `updatedAt` 降序排列。
- 每个项目显示名称、路径、设备名、类型（项目/外部仓库/工作空间）、来源（GitHub/本地文件夹）、在线/离线/归档状态。
- 项目下默认显示最近 3 个非归档会话，提供"展开更多/收起"查看剩余。
- 离线项目不展开会话列表，不能新建会话；历史会话不可点击打开。
- 归档项目只读，不可新建会话。
- **内部项目新建会话**：点击加号打开 `home` 标签，`cwd` 设为 **工作空间目录**（内部项目是组织对象，不作为 pi 运行目录）。
- **外部仓库新建会话**：点击加号打开 `home` 标签，`cwd` 设为 **外部仓库路径**（外部仓库可作为 pi 运行目录）。

## 会话

- 在工作空间分区新建的会话是工作空间会话。
- 在项目下新建的会话是项目会话（但内部项目下新建时 cwd 仍为工作空间目录）。
- 项目会话的 AI 在项目所在设备运行。
- 会话右侧文件列表显示当前运行位置：工作空间或外部仓库。
- 会话上下文创建后绝对不能迁移。

## 任务与项目

任务系统通过项目筛选查看绑定项目的任务。

- 任务和里程碑都可选绑定项目。
- 任务默认继承所属里程碑的项目。
- 用户和 Agent 创建任务、里程碑时都可以选择项目。
- **任务绑定内部项目时，处理会话 cwd = 工作空间目录**。
- **任务绑定外部仓库时，处理会话 cwd = 外部仓库路径**。
- 任务未绑定项目时，处理会话创建为工作空间会话。
- 里程碑不创建处理会话。

## 终端

终端按设备管理，不按会话管理。

- 服务器设备终端默认目录为 `~/ridge-workspace`。
- 桌面设备终端在桌面设备运行。
