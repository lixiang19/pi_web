# 非目标与边界

## 定位

本文件记录第一版明确不做或不展开的内容。

边界用于防止实现阶段范围漂移。

## 第一版不做

- 多用户；
- 团队空间；
- 多工作空间；
- 用户级权限隔离；
- 项目级设置页面；
- 移动端完整设计；
- 工作流编排引擎；
- 复杂自动化 DAG；
- 选择性备份恢复；
- 普通文件“导入”；
- 外部项目文件备份；
- 桌面本机项目文件自动同步到服务器；
- 桌面本机 Pi 会话正文镜像到服务器；
- 删除项目注册时删除真实项目文件；
- 工作空间会话切换到桌面设备运行；
- 会话创建后迁移工作空间或项目上下文。

## 不是产品概念

ridge 不把这些作为第一版产品概念：

- 多租户；
- 团队成员；
- 项目所有者；
- 项目内独立权限系统；
- 收件箱；
- 文件导入；
- 会话内任务面板；
- 用户可见 Git 工作空间。

## 保留但不展开

以下方向保留，但第一版不写细设计：

- 移动端独立设计；
- APK 形态；
- 桌面端浏览器扩展；
- RAG 的高级检索策略；
- 复杂知识图谱画布；
- 多模型并行运行；
- 多设备冲突合并。

## 强边界

- 工作空间只有一个。
- 工作空间属于服务器。
- 项目必须注册和管理。
- 项目文件不复制进工作空间。
- 工作空间会话在服务器运行。
- 项目会话在项目所在设备运行。
- Pi 原始会话正文以运行设备为真源。
- `ridge.db` 保存 ridge 结构化状态和会话索引。
- 文件上传不等于系统导入。
