# 会话界面与右侧工作侧栏

## 会话布局

会话标签页主体由聊天区和右侧工作侧栏组成。

- 聊天区承载消息过程、统一输入框、发送与中断。
- 右侧工作侧栏承载当前会话的文件、版本、产物和来源。
- 右侧工作侧栏可折叠。

移动端独立重新设计，不按桌面端自适应压缩。

## 会话流

会话流保留现有轮次结构。

- 每轮由用户消息、折叠的执行过程、最终 AI 回复组成。
- 执行过程默认折叠，可展开查看工具、思考、工具结果等过程消息。
- 思考内容作为执行过程中的折叠卡片展示；如果模型隐藏思考，只显示隐藏提示。
- Ask 提问和权限审批以会话流内卡片呈现，用户在上下文中直接回答或批准。
- 默认只显示最近若干轮，用户可展开更早历史或加载更早历史。
- 流式回复时，如果用户停留在底部则自动跟随；用户上滑阅读后不抢滚动，只显示回到底部按钮。
- 会话运行时事件通过 `GET /api/sessions/:id/events` 订阅；取消运行走 `POST /api/sessions/:id/cancel`。
- 归档会话是只读会话，统一输入框禁用并显示只读原因；服务端发送入口仍必须返回 403。

## 消息操作

消息支持完整操作，但不直接破坏原始历史。

- 支持复制消息。
- 支持编辑旧用户消息。
- 支持重试 AI 回复。
- 不支持删除单条消息。

编辑旧用户消息或重试 AI 回复时，创建新的分叉会话，不覆盖原会话历史。

分叉规则：

- 分叉继承原会话上下文。
- 工作空间会话分叉仍是工作空间会话。
- 项目会话分叉仍绑定同一项目。
- 任务会话分叉仍绑定同一任务。
- 分叉会话在左侧作为普通会话显示，不展示树状父子关系。
- 分叉标题先临时沿用来源语义，之后由 AI 自动生成。

## 右侧固定标签

右侧工作侧栏第一版默认显示三个标签：

- 摘要；
- 文件树；
- 版本。

只有当前运行目录是真实 Git 仓库时，才额外显示 Git 操作标签。

默认打开“摘要”标签。

## 摘要

摘要是系统自动收集的事实投影，不允许手动编辑。

第一版包含：

- 产物：会话创建或修改过的文件；
- 来源：使用过的 Skills、Exa 搜索或打开过的网址、被会话引用过的本地文件；
- Todo：todo 工具产生的结构化内容。

摘要不记录完整工具调用流水；过程由会话正文承载。

## 文件树

文件树显示当前会话运行位置。

- 工作空间会话显示 `~/ridge-workspace`。
- 内部项目会话显示工作空间目录。
- 外部仓库会话显示外部仓库目录。
- 桌面项目会话显示桌面设备上的项目目录。

第一版操作：

- 浏览文件树；
- 打开文件到右侧主标签页；
- 将文件拖入统一输入框作为文件引用。

文件树不承担完整文件管理。

## Git 操作

Git 操作只对 Git 项目真正可用。

- GitHub 拉取项目通常可用。
- 外部 Git 仓库可用。
- 外部非 Git 仓库显示不可用。
- 工作空间和内部项目不作为显式 Git 项目操作。
- ridge 内部隐藏版本库不能出现在 Git 面板，也不能伪装成 Git 仓库。

第一版 Git 操作包含常用能力：

- 状态；
- 提交；
- 分支；
- 拉取；
- 推送；
- stash。

推送是外部副作用，每次都需要用户明确确认。

## 版本

版本是 ridge 的工作空间隐藏版本，不是用户项目 Git。

版本页第一版只做：

- 查看工作空间隐藏版本状态；
- 查看文件 diff；
- 提交版本点。

版本页不提供：

- 分支；
- 远程；
- 拉取；
- 推送；
- stash。

Git 项目继续使用项目自身 Git 工作区 diff；工作空间和内部项目使用版本页查看隐藏版本 diff。
