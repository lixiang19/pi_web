# 工作空间隐藏版本管理

## 定位

工作空间隐藏版本管理基于 Git 实现，但不向用户暴露 Git 概念。

用户看到的是：

- 历史；
- Diff；
- 提交版本；
- 恢复。

## 覆盖范围

隐藏版本管理覆盖整个 `~/ridge-workspace`。

包含：

- 用户可见内容；
- 内部项目；
- 正式附件；
- 必要系统状态。

排除：

- macOS 元数据 `.DS_Store`；
- `node_modules`、构建产物和工具缓存；
- RAG 向量库；
- Kuzu 图谱库；
- 运行缓存；
- 临时附件；
- 其他缓存和大体积可重建产物。

外部项目不受工作空间隐藏版本管理覆盖。

## 版本点

会话或用户操作完成后创建版本点。

典型触发：

- AI 会话修改工作空间文件；
- 用户编辑文件；
- 闪念处理为正式对象；
- RAG 转换产物写入；
- 附件迁移到正式附件目录。

版本说明由系统根据操作类型生成。

## 会话 Diff

工作空间会话和内部项目会话使用隐藏版本管理提供 Diff。

规则：

- 会话开始时记录基线；
- 会话 Diff 对比基线到当前状态；
- 目标是展示当前会话相关文件变化。

外部 Git 项目使用项目自身 Git diff。

外部非 Git 项目不提供可靠 Diff，显示不可用。

## 版本页

会话右侧“版本”页是隐藏版本管理的轻量操作面。

第一版能力：

- 查看当前工作空间相对上一版本点的变更；
- 查看单文件 diff；
- 提交版本点。

版本页不暴露 Git 概念：

- 不显示分支；
- 不显示远程；
- 不支持 push/pull；
- 不支持 worktree。

## 恢复

底层基于 Git 支持恢复。

第一版 UI：

- 常规支持文件级恢复；
- 整体恢复作为高级危险操作；
- 整体恢复必须二次确认。

恢复操作不暴露 Git 命令。
