# 通知与建议中心

## 定位

通知与建议中心是待处理系统事件入口。

它不替代聊天。

它不用于高频打扰。

它集中承接后台 Agent、文件处理、索引、自动化和任务系统产生的需要用户知道或确认的事项。

## 入口

通知与建议中心是左侧固定入口。

点击后在右侧打开单例标签页。

重复点击只激活已有标签。

左侧入口可以显示未处理数量或红点。

## 事件来源

第一版事件来源包括：

- fleeting agent 的闪念处理建议；
- task review agent 的任务回顾建议；
- 后台 Agent 失败；
- 后台 Agent 自动重试失败；
- 文件转换失败；
- RAG 索引失败；
- 图谱维护失败；
- Wiki 维护失败；
- 自动化跳过；
- 自动化失败；
- 任务或里程碑需要用户确认完成；
- 设备离线导致项目运行失败。

summary agent 和 memory agent 成功运行不产生通知。

普通索引成功不产生通知。

## 事件类型

通知事件分为：

- 建议；
- 待确认；
- 失败；
- 警告；
- 信息。

第一版重点是建议、待确认和失败。

信息类事件默认不主动显示，只在历史中可查。

## 用户动作

每条事件可以有零个或多个动作。

常见动作：

- 查看；
- 忽略；
- 重试；
- 接受建议；
- 拒绝建议；
- 打开相关对象；
- 标记已处理。

接受建议时，才写入正式对象。

例如 task review agent 建议修改任务状态，用户接受后才更新任务。

## 展示

通知中心按时间倒序展示。

默认只显示未处理事件。

支持切换：

- 未处理；
- 全部；
- 失败；
- 建议；
- 已处理。

每条事件显示：

- 标题；
- 类型；
- 来源；
- 关联对象；
- 时间；
- 简短说明；
- 可执行动作。

## 关联对象

事件必须尽量关联到真实对象。

对象类型包括：

- 闪念；
- 任务；
- 里程碑；
- 文件；
- 项目；
- 会话；
- 自动化；
- 后台任务；
- 图谱维护任务；
- Wiki 页面。

点击关联对象应打开对应标签或定位到对应页面。

## 与后台 Agent 的关系

后台 Agent 不向原会话追加消息。

需要用户处理的后台结果进入通知与建议中心。

不需要用户处理的后台结果只记录状态。

后台 Agent 失败进入通知中心。

后台 Agent 成功但无用户动作，不进入通知中心。

## 与任务系统的关系

任务回顾建议进入通知中心。

任务或里程碑完成仍必须由用户确认。

通知中心可以承载“确认完成”的动作。

接受后调用正式任务工具或任务 API。

通知中心本身不是任务列表。

## 与文件处理的关系

文件转换失败、索引失败进入通知中心。

用户可以从通知中心打开文件或重试处理。

文件处理成功不产生通知。

## 与自动化的关系

自动化失败或跳过进入通知中心。

项目设备离线导致自动化跳过时，生成一条警告事件。

用户可以打开自动化详情或重试。

## 数据模型

通知事件进入 `ridge.db`。

至少包含：

- id；
- type；
- source；
- severity；
- status；
- title；
- body；
- related_type；
- related_id；
- actions；
- created_at；
- updated_at；
- handled_at。

状态包括：

- unread；
- pending；
- handled；
- dismissed；
- failed。

## 保留策略

通知事件不是长期知识资产。

已处理和已忽略事件可以长期留在 DB，但不进入 RAG。

第一版不做复杂清理策略。

后续可以按时间清理信息类事件。

## 非目标

第一版不做：

- 系统级推送通知；
- 邮件通知；
- 多用户通知；
- 聊天式通知助手；
- 通知规则编辑器；
- 复杂订阅系统。
