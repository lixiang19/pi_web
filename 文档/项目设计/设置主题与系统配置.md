# 设置主题与系统配置

## 定位

设置中心管理 ridge 全局配置。

第一版不做项目级设置页面。

项目级覆盖来自项目自身 `.pi` 资源和 Pi 运行时发现。

## 配置范围

第一版设置包含：

- 默认模型；
- 默认思考强度；
- 默认 Agent；
- Agent 权限；
- 主题；
- 设备；
- 服务器连接；
- 数据路径展示。

## 默认会话配置

全局保存：

- 默认模型；
- 默认思考强度；
- 默认 Agent。

新会话继承全局默认。

会话中临时切换模型、思考强度或 Agent，只影响后续消息，不回写全局默认。

## Agent 与权限

权限按 Agent 配置。

权限值：

- allow；
- ask；
- deny。

运行时权限还必须受当前会话位置约束。

- 工作空间会话只能作用于工作空间边界。
- 项目会话只能作用于对应项目边界。
- 桌面项目会话只能在对应桌面设备上执行。

任务 Agent、闪念 Agent 是系统 Agent，可以在设置中覆盖模板。

## 主题

主题第一版支持：

- 浅色；
- 深色；
- 跟随系统。

主题是全局设置。

界面实现必须走主题文件和 shadcn-vue，不在组件中散落手写基础颜色。

## 设备设置

设备设置第一版只做查看和命名。

- 查看服务器设备；
- 查看桌面设备；
- 查看在线状态；
- 重命名桌面设备；
- 不允许删除设备。

## 服务器覆盖同步

服务器 `~/.pi/ridge.db` 是全局设置真源。

桌面设备连接服务器后：

```txt
登录服务器
→ 获取全局设置快照
→ 覆盖本地 ridge 设置
→ 建立设备长连接
```

桌面端不把本地全局设置反向合并到服务器。

项目自己的 `.pi` 配置不属于 ridge 全局设置，不被服务器覆盖。

## 数据路径展示

设置中展示关键路径，但第一版不提供复杂迁移。

- 工作空间：`~/ridge-workspace`
- 服务器项目根：`~/ridge-projects`
- 应用数据：`~/.pi`
- 数据库：`~/.pi/ridge.db`
