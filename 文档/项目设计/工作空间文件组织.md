# 工作空间文件组织

## 定位

工作空间文件组织负责给用户和 AI 一个稳定、简单的文件秩序。

工作空间根目录是：

```txt
~/ridge-workspace
```

## 预制一级目录

最终用户可见预制一级目录固定为八个：

```txt
项目/
笔记/
日记/
剪藏/
附件/
记忆/
Wiki/
空间/
```

系统隐藏目录：

```txt
.ridge/
```

`.ridge/` 不在普通文件界面中作为用户目录展示。

## 项目

`项目/` 是工作分类和内部项目目录。

- 项目既是分类，也是需要时的执行现场。
- 内部项目创建在 `项目/<项目名>`。
- 外部项目和桌面本机项目不复制进工作空间。
- 项目可以收纳相关笔记、剪藏、附件引用、任务和会话索引。
- 项目需要执行时，绑定设备和路径。

## 笔记

`笔记/` 是用户主要自由组织的目录。

- 用户可以在 `笔记/` 下自由创建目录。
- 新建普通笔记默认进入 `笔记/`。
- 系统不强制笔记分类。
- AI 可以建议整理方式，但不能自动重排用户笔记结构。

## 日记

`日记/` 是按时间组织的轻量记录。

默认路径：

```txt
日记/YYYY/MM/YYYY-MM-DD.md
```

日记用于沉淀每日观察、总结、状态和回顾。

日记不是复杂日记应用，只是稳定文件约定。

## 剪藏

`剪藏/` 是外部信息收集落点。

包含：

- 网页；
- 摘录；
- PDF；
- 外部文章；
- 截图；
- 浏览器网址；
- 其他从外部收集的信息。

剪藏不是最终分类。

需要长期使用的剪藏，可以移动或关联到对应项目。

## 附件

`附件/` 是正式附件目录。

- 闪念处理后的截图或文件迁入这里。
- 会话沉淀出的正式附件迁入这里。
- 附件可以被项目、笔记、日记、任务或会话引用。
- 附件进入 RAG。

## 记忆

`记忆/` 是后台 Agent 维护的长期记忆目录。

- `记忆/MEMORY.md` 是全局长期记忆。
- `记忆/daily/` 保存每日会话记忆。
- 记忆文件是普通 Markdown。
- 用户可以直接打开和编辑。
- 记忆进入 RAG。
- `MEMORY.md` 会在会话启动时通过 Pi 钩子注入。

## Wiki

`Wiki/` 是 AI 维护的轻量知识入口目录。

- `Wiki/index.md` 是固定入口。
- Wiki 文件是普通 Markdown。
- Wiki 不是第二个文件库。
- Wiki 用于维护少量 canonical 入口页。
- Wiki 进入 RAG。
- `Wiki/index.md` 会在会话启动时通过 Pi 钩子注入。

## 空间

`空间/` 是 HTML 作品空间。

- 用于保存 AI 生成的 HTML 页面、原型、可视化、报告和交互 demo。
- 每个作品一个目录。
- 默认入口是 `空间/<作品名>/index.html`。
- 点击作品打开 ridge 内部空间预览标签。
- 私有作品只在主站内预览，不生成独立访问 URL。
- 公开作品后续可以生成公开只读链接。
- 空间作品进入隐藏版本管理。
- 空间作品可以进入 RAG。

## 自定义一级目录

根目录允许用户创建自定义一级目录，但系统不鼓励。

- 自定义一级目录作为普通可见文件进入 RAG。
- 自定义一级目录不获得系统语义。
- 预制一级目录名称受保护。
- 预制一级目录不能改名或删除。

## 标签与领域

最终设计不引入标签系统。

最终设计不预制 `领域/` 目录。

- 分类优先由项目承载。
- 语义找回依赖 RAG。
- 长期说明可以写成笔记、日记或项目说明。
