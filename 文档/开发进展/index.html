功能任务列表

最近发现与关键决策

已同步到设计
本机 Pi 不能靠纯内存注入 Skill

Pi 读取 Skill 时需要真实文件系统路径，因此桌面项目运行前必须由服务器下发完整 runtime bundle，并物化到桌面 ridge 管理目录。

长期知识优先是 Markdown

记忆、daily、Wiki、空间作品、笔记、日记、剪藏都应落成可见文件；DB 只保存任务、索引、状态、通知、设备等结构化对象。

私有 HTML 预览不是公开链接

空间私有作品第一版只在主站内部用 iframe srcdoc 预览，不生成可复制 URL；公开发布后续再设计。

文件处理本身不是后台 Agent

PDF、Word、音频、图片先标准化成 Markdown、assets 和 metadata；需要 AI 判断时再交给对应后台 Agent。

桌面本机项目不能离线运行

桌面本机 Agent 需要服务器下发 runtime bundle、记忆、Wiki 和 workspace MCP，服务器不可达时禁止启动。

技术栈与机制选型

待实现验证
结构化真源

SQLite `~/.pi/ridge.db`，保存设备、项目、任务、会话索引、后台任务、通知、搜索状态和设置。

文件真源

`~/ridge-workspace`，包含项目、笔记、日记、剪藏、附件、记忆、Wiki、空间八个一级目录。

隐藏系统区

`.ridge/` 保存临时附件、RAG 缓存、Kuzu 图谱、runtime 缓存和系统中间产物。

文件解析

Docling 或同类解析能力，把 PDF、Word、音频和图片转为 Markdown、assets、metadata。

检索

RAG 只索引标准产物，chunk 保留路径、标题、序号、hash 和文件类型。

图谱

Kuzu 存储结构化理解资产，夜间从标准产物和 daily 抽取实体关系。

Agent 运行

Pi 会话仍是运行核心；桌面本机项目通过服务器下发 runtime bundle 获得一致环境。

工作空间 MCP

提供 `rag_search`、`graph_search`、`file_search`、`read_workspace_file`，只查读不写。

前端

Vue、shadcn-vue、主题文件和明暗模式；Tauri 只负责打包，不做业务功能。
