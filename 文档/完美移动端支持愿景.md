# 完美移动端支持愿景

## 一句话定义

ridge 在移动端不是"桌面版的妥协"，而是**原生为触控和碎片场景设计的 AI 伴侣**。

---

## 设计原则

| 原则 | 说明 |
|------|------|
| **触控优先** | 所有交互为手指设计，无悬停依赖 |
| **碎片场景** | 支持随时打断、快速回复、稍后继续 |
| **单手操作** | 核心功能在屏幕下半区完成 |
| **离线意识** | 弱网环境下优先本地功能，联网后同步 |

---

## 移动端专属特性

### 输入革命
- **语音优先**：长按输入框即说话，边说边看转录
- **快捷输入**：滑动手势触发常用指令（/continue /summarize）
- **照片/截图**：直接粘贴图片询问，OCR 自动提取文字
- **分享接入**：从任意 App 分享文本到 ridge 快速提问

### 会话管理
- **卡片流**：会话以卡片形式呈现，左右滑动归档/收藏
- **稍后阅读**：消息长按"标记待办"，生成统一任务列表
- **会话恢复**：自动记住阅读位置，打断后可无缝继续

### 通知与打断
- **智能通知**：仅当 assistant 完成时推送，流式过程静默
- **快速回复**：通知栏直接回复，不打开 App
- **后台播放**：语音回复可后台听，锁屏也能继续对话

---

## 桌面→移动端的功能取舍

| 桌面功能 | 移动端策略 | 原因 |
|---------|-----------|------|
| 三栏布局 | 单栏卡片流 | 屏幕宽度限制 |
| 文件树 | 仅查看已关联文件，不支持浏览 | 触控选择困难 |
| 终端 | ❌ 完全隐藏 | 无触控友好方案 |
| Git 面板 | ❌ 完全隐藏 | 移动场景不 commit |
| 代码编辑 | ⚪ 只读查看，触发"在桌面继续" | 不尝试在手机上写代码 |
| 助手模式 | ✅ 完全保留 | 移动端主场 |
| 代码模式 | ⚪ 只读审查，建议切桌面实现 | 紧急时可查看 |

---

## 跨端协同

### 桌面发起 → 移动端继续
- 桌面上的会话标记"移动端可读"
- 手机收到推送，继续阅读 assistant 的长回复
- 简单回复/确认可在手机完成

### 移动端发起 → 桌面实现
- 手机上想到点子，语音快速创建会话草稿
- 桌面端同步后显示"从移动端继续"
- 复杂实现自动在桌面端执行

### 会话状态同步
- 阅读位置、折叠状态、草稿文字实时同步
- 移动端发送消息后，桌面端自动刷新

## 技术方案

| 层面 | 方案 |
|------|------|
| **响应式** | 桌面 Web + 移动 Web 同一套代码，布局断点切换 |
| **PWA** | 可添加到主屏，支持离线访问已同步会话 |
| **原生壳** | 可选：Capacitor 打包 iOS/Android，访问原生分享/相机 |
| **输入优化** | 接入系统语音 API，本地预处理转录 |

---

## 演进阶段

- **当前**：桌面 Web 响应式基础（能看不能用）
- **一阶段**：移动 Web 可用（单栏布局、触控优化、语音输入）
- **二阶段**：PWA 离线能力（缓存会话、后台同步）
- **三阶段**：原生壳增强（系统分享、相机 OCR、推送优化）
