# 笔记编辑可靠性 Spec

> 对应范围：`文档/功能开发/2026-05-01_笔记编辑体验Level2范围与验收标准.md` B1  
> 日期：2026-05-01  
> 目标：补齐笔记编辑“不丢内容”的可靠性保护与测试，不重写编辑器，不带入 B2 文件组织能力。

## 1. 背景

当前 `.md` 文件已通过 `WorkspaceMarkdownEditor` 编辑，具备 2s 自动保存和 `unsaved/saving/saved/error` 状态；多标签切换通过 `v-show` 保持组件挂载；非 markdown 文件走只读预览或外部打开。

B1 不重做编辑器，而是补齐可靠性短板：

- 保存失败对用户可见，并保留当前编辑内容。
- 保存失败后有重试路径。
- 关闭标签或组件卸载前，不静默丢失 `unsaved/saving/error` 内容。
- 自动保存状态对用户可感知，并同步到标签状态。
- 测试覆盖 `.md` 编辑和非 `.md` 不误编辑。

## 2. 范围

### 本阶段做

- 打开 `.md` 文件进入 `WorkspaceMarkdownEditor`。
- 编辑后状态变化可见：`unsaved` → `saving` → `saved`，失败进入 `error`。
- 自动保存成功后内容持久化，重新打开能读到新内容。
- 自动保存失败时显示明确错误提示，当前编辑内容保留。
- 保存失败后支持重试，或用户再次编辑/触发保存时能重新保存。
- 关闭标签或组件卸载前处理未保存内容：优先 flush；无法保存时阻止/确认或保留明确风险提示。
- 非 markdown 文件不进入 markdown 编辑器。
- 补充组件/接线测试。

### 本阶段不做

- 不做新建笔记默认 `笔记/`，放 B2。
- 不做目录选择、重命名、删除、新建文件夹体验，放 B2。
- 不做 checkbox 聚合到待办。
- 不做双链、搜索、模板、附件、双栏预览、block editor。
- 不改 Dashboard。
- 不改 AI 会话、Agent、Pi session 接线。
- 不占用 `项目/` 目录语义。

## 3. 用户故事与验收

### Story 1：打开 markdown 并编辑保存

作为用户，我能打开 `.md` 文件并编辑，系统自动保存，重新打开后内容仍存在。

验收：
- `.md` 文件使用 `WorkspaceMarkdownEditor` 打开。
- 编辑内容后显示未保存状态。
- 自动保存时显示保存中状态。
- 保存成功后显示已保存状态，或状态消退但用户可感知。
- 重新打开或刷新后能读到保存后的内容。

### Story 2：保存失败不丢内容

作为用户，如果保存失败，我能看到错误，并且当前输入不会被覆盖或丢失。

验收：
- `saveNoteContent` 失败时显示可见错误提示。
- 编辑器中仍保留用户当前输入内容。
- 标签或编辑器状态进入失败/风险状态。
- 用户能重试保存，或再次编辑后重新触发保存。

### Story 3：离开前不静默丢内容

作为用户，我切换或关闭标签时，如果内容还没保存，系统不能静默丢弃。

验收：
- 多标签切换不卸载编辑器，不丢当前输入。
- 关闭标签时，如果存在 `unsaved/saving/error`，系统需要先 flush 成功再关闭，或阻止/确认关闭。
- 组件卸载前清理自动保存 timer，避免悬挂保存。
- 如果暂时无法完整支持关闭保护，必须在实现说明中写明降级策略，并由产品/审查确认。

### Story 4：非 markdown 不误编辑

作为用户，我打开非 markdown 文件时，不会进入 markdown 编辑器导致误编辑。

验收：
- `.md` 使用编辑器。
- 非 `.md` 文本文件走只读预览。
- 不支持文件走外部打开或明确不可编辑提示。

## 4. 测试计划

### 组件测试

- `WorkspaceMarkdownEditor` 编辑后触发 `unsaved` 状态。
- debounce 到期后调用 `saveNoteContent`，状态进入 `saving`，成功后进入 `saved`。
- `saveNoteContent` 失败后进入 `error`，显示错误提示，当前输入内容仍在。
- 失败后重试路径能再次调用保存。
- 组件卸载前清理 pending timer；如有 flush，断言 flush 行为。

### 接线测试

- `WorkspaceContentArea` 或相关渲染层：`.md` 文件渲染 `WorkspaceMarkdownEditor`。
- 非 `.md` 文件不渲染 `WorkspaceMarkdownEditor`，走只读预览/外部打开分支。
- 标签状态能接收编辑器保存状态变化。
- 关闭 unsaved/saving/error 标签触发保护逻辑。

### 回归测试

- 不影响 Dashboard / AI session 相关测试。
- 不影响非 markdown 文件打开。
- 不改变 `项目/` 目录行为。

## 5. 开发注意事项

- 优先手术式补齐可靠性保护，不重写 Milkdown/Markdown 编辑器。
- 保存状态必须来自真实保存流程，不做纯视觉假状态。
- 保存失败不能通过重新加载文件覆盖用户当前输入。
- 如果关闭标签保护依赖现有 tab close 架构，先做最小可测实现；复杂关闭确认 UI 可作为后续小任务。
