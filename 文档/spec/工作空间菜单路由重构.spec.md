# 工作空间菜单路由重构测试 Spec

## 背景

本轮将 ridge 从“左侧全局菜单 + 多独立路由页面”调整为“工作空间唯一主界面 + 顶部系统菜单 + 工作空间标签页”。

用户确认的产品决策：

1. 独立路由删除：只保留根路由/工作空间路由，`terminal`、`automations`、`settings`、`chat`、`search` 等独立页面路由移除。
2. 顶部系统菜单独立一行：`终端 / 自动化 / AI 会话`，下方仍是工作空间分屏面板自己的标签栏。
3. 终端打开为工作空间标签页：允许多个终端标签页，每个标签页一个终端实例，终端内部不再有二级多标签栏。
4. 自动化打开为工作空间标签页。
5. 设置与终端、自动化一样，点击后打开工作空间标签页。
6. 聊天和项目入口从当前侧栏删除。AI 会话后续会在工作空间主页标签中启动；本轮不设计项目选择能力。

## 测试目标

验证路由、工作空间顶部菜单、标签页内容、终端单实例标签、自动化迁移、设置迁移、旧导航删除、数据流断点处理均符合新信息架构。

## 前端测试用例

### 1. 路由精简

- 访问根路径 `/` 时进入工作空间主页面。
- 路由表不再注册以下独立页面：
  - `/chat`
  - `/terminal`
  - `/automations`
  - `/settings`
  - `/search`
  - `/spaces`
  - `/datasets`
- 不存在通过左侧全局菜单切换页面的入口。

### 2. 工作空间顶部系统菜单

- 工作空间页面顶部存在独立系统菜单行。
- 菜单至少包含：
  - 终端
  - 自动化
  - 设置
- `AI 会话`入口如保留展示，应指向工作空间主页标签的后续承载位，不触发旧 `/chat` 路由。
- 顶部系统菜单点击不会调用 `router.push` 跳转独立功能路由。

### 3. 工作空间标签页类型扩展

- 工作空间分屏标签系统支持以下标签类型：
  - `home`
  - `view`
  - `file`
  - `terminal`
  - `automation`
  - `settings`
- 点击“终端”在当前活动面板打开一个新的 `terminal` 标签页。
- 点击“自动化”在当前活动面板打开或激活 `automation` 标签页。
- 点击“设置”在当前活动面板打开或激活 `settings` 标签页。
- 原有 `home/view/file` 标签页能力不回退。

### 4. 终端标签页行为

- 每次从顶部菜单打开“终端”时，创建一个工作空间终端标签页。
- 每个终端标签页绑定一个独立终端实例。
- 终端标签页内容区不渲染 `TerminalTabBar`。
- 终端内部没有“新建终端/切换终端”的二级标签栏。
- 关闭终端工作空间标签页时，按既定终端生命周期销毁或关闭对应终端实例。
- 切换工作空间标签页时，不因隐藏/显示导致终端输入异常。

### 5. 自动化标签页行为

- 点击顶部“自动化”打开工作空间 `automation` 标签页。
- 自动化规则列表、规则编辑、保存、启用/禁用仍可使用。
- 自动化立即运行不再依赖旧 `openChatSession()` 跳转 `/chat`。
- 自动化立即运行的结果处理不触发旧聊天路由。

### 6. 设置标签页行为

- 点击顶部“设置”打开工作空间 `settings` 标签页。
- 设置页原有主题、模式、通知、语言等设置项仍可读写。
- 设置仍通过现有 settings store 与服务端存储持久化。
- 关闭设置标签页不会丢失已保存设置。

### 7. 旧侧栏与项目/聊天入口删除

- 页面不再渲染旧 `WorkbenchSidebar`。
- 不再展示旧底部聊天入口。
- 不再展示旧项目列表、项目添加、worktree 操作入口。
- 不再从旧侧栏触发新建聊天、打开聊天会话、搜索会话。

### 8. 无旧路由依赖

- `useWorkbenchPrimaryNavigation` 不再承担旧一级路由跳转职责。
- 自动化、终端、设置入口不再依赖 `navigateToRoute()`。
- 移除旧路由后，前端构建不出现未解析路由名或组件引用错误。

## 后端/服务层测试用例

本轮主要是前端信息架构迁移。后端终端、自动化、设置存储接口原则上不变，但需要验证：

- 终端创建、连接、关闭接口仍可被工作空间终端标签页调用。
- 自动化 CRUD 与立即运行接口仍可被工作空间自动化标签页调用。
- 设置读写接口仍可被工作空间设置标签页调用。

## 验收命令

- `cd packages/web && pnpm test`
- `cd packages/web && pnpm lint`
- `cd packages/web && pnpm typecheck`
- 如服务端类型受影响：`cd packages/server && pnpm test` 与对应类型检查命令
- 根目录全量：`pnpm test`

## 非目标

- 不实现新的项目引入/项目选择系统。
- 不实现新的 AI 会话启动完整设计；后续由工作空间主页标签承载。
- 不保留旧独立路由重定向兼容。
- 不保留终端内部二级多标签栏。
