# 单用户固定密码认证 Spec

## 背景

ridge 后续会部署在 VPS 上，当前只面向个人自用，不需要注册、多用户、角色权限和页面改密。

## 目标

- 访问工作台前必须登录。
- 除认证接口外，所有 `/api/*` 必须校验登录态。
- 终端 WebSocket `/api/terminals/:id/stream` 必须校验登录态。
- 登录成功后使用服务端 Session 和 `HttpOnly Cookie`。
- 前端不保存密码，不保存 token 到 `localStorage`。
- 未登录 API 返回 `401`，页面跳登录页。

## 非目标

- 不做多用户、注册、找回密码、角色权限。
- 不做初始化设置页。
- 不做页面修改密码。
- 不做持久化 Session。
- 不开放匿名只读模式。

## 密码来源

第一版使用服务端固定密码常量，真实密码只允许存在后端运行时代码中。

约束：

- 禁止写入前端代码。
- 禁止通过接口返回。
- 禁止打印到日志。
- 禁止放入浏览器存储。
- 修改密码需要改后端代码或部署配置并重启服务。
- 如果仓库或镜像泄露，密码视为泄露，这是本方案已接受风险。

## Session 模型

- 登录成功后服务端生成随机 Session ID。
- Session ID 存入服务端内存 `Map`。
- 浏览器通过 Cookie 保存 Session ID。
- Cookie 名称：`ridge_session`。
- Cookie 属性：`HttpOnly`、`SameSite=Lax`、`Path=/`。
- 生产环境 HTTPS 下启用 `Secure`。
- 默认有效期：7 天。
- 服务重启后 Session 失效，需要重新登录。
- 登出时删除服务端 Session 并清除 Cookie。

## 公开接口

### `GET /api/auth/session`

返回当前登录状态。

```json
{
  "authenticated": true
}
```

未登录时：

```json
{
  "authenticated": false
}
```

### `POST /api/auth/login`

请求：

```json
{
  "password": "string"
}
```

成功响应：

```json
{
  "ok": true
}
```

密码错误返回 `401`。

### `POST /api/auth/logout`

清除当前 Session，成功响应：

```json
{
  "ok": true
}
```

## 受保护范围

除以下接口外，所有 `/api/*` 默认受保护：

- `GET /api/auth/session`
- `POST /api/auth/login`
- `POST /api/auth/logout`

必须保护：

- `/api/notes/*`
- `/api/files/*`
- `/api/filesystem/*`
- `/api/workspace/*`
- `/api/sessions/*`
- `/api/projects/*`
- `/api/git/*`
- `/api/terminals/*`
- `/api/storage/*`
- `/api/system/*`
- `/api/automations/*`
- `/api/fleeting/*`

## WebSocket 鉴权

`/api/terminals/:id/stream` 在 `upgrade` 阶段读取 Cookie。

- 没有有效 `ridge_session` 时返回 `401 Unauthorized` 并关闭 socket。
- 不能只校验 REST 终端接口，因为攻击者可以直接连接 WebSocket。

## 前端路由

- 新增 `/login` 登录页。
- 未登录访问 `/` 时跳转 `/login`。
- 已登录访问 `/login` 时跳转 `/`。
- 登录成功后回到原目标路径或 `/`。
- App 初始化时调用 `GET /api/auth/session`。
- 检查中显示轻量 loading。
- 未登录不加载 workspace 数据。
- 任意 API 返回 `401` 时清理前端登录态并跳转 `/login`。

## 登录限流

- 同一 IP 连续失败 5 次后锁定 5 分钟。
- 登录成功后清空失败计数。
- 锁定时返回 `429`。
- 不记录明文密码。

## 验收标准

1. 未登录访问 `/` 会进入 `/login`。
2. 未登录请求任意业务 API 返回 `401`。
3. 密码正确登录后能进入工作台。
4. 登录后刷新页面仍保持登录。
5. 登出后不能访问任何业务 API。
6. 终端 WebSocket 未登录不能连接。
7. 连续密码错误触发限流。
8. 前端包里搜不到固定密码。
9. API 响应和日志不出现固定密码。
10. `npm run check` 通过。

## 测试计划

- 服务端集成测试覆盖登录、登出、Session、受保护 API、限流、WebSocket upgrade 鉴权。
- 前端测试覆盖登录页、路由守卫、`401` 跳转、登录成功进入工作台。
- 修改 `.ts` / `.vue` 后必须运行根目录 `npm run check`。
