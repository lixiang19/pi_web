# 笔记新建、目录选择与组织体验 Spec

> 对应任务：task #11 B2: 笔记新建、目录选择与组织体验  
> 日期：2026-05-01  
> 前置：B1 笔记编辑可靠性已完成。B2 必须保持 B1 的保存状态、失败重试、关闭保护不被破坏。

## 1. 目标

让用户能在非 `项目/` 目录里稳定创建和组织 markdown 笔记：选择目录、填写标题、新建后自动打开、文件树同步、标签页标题同步，并保持 B1 的可靠编辑体验。

## 2. 范围

### 本阶段做

- 新建笔记默认进入 `笔记/`。
- 新建笔记时可选择目录。
- 可选目录不推荐/默认 `项目/`。
- 标题为空时阻止创建并提示。
- 新建成功后自动打开笔记标签页。
- 新建文件夹后文件树刷新，并可继续在该目录创建笔记。
- 重命名笔记后，文件树和标签页标题同步。
- 删除笔记有确认或明确危险反馈；删除后关闭对应标签或显示文件不存在状态。
- 保持 B1 保存状态、失败重试、关闭保护不被破坏。

### 本阶段不做

- 不做 task DB / JSON / SQLite。
- 不做 AI 会话、Agent、AI 写入。
- 不改 Dashboard 新设计。
- 不做 checkbox 聚合到待办。
- 不做双链、搜索、模板、附件、图片粘贴。
- 不占用 `项目/` 目录语义；`项目/` 保留给未来特殊“创建项目”能力。

## 3. 目录规则

### 默认目录

- 默认新建笔记目录：`笔记/`。
- 如果 `笔记/` 不存在，系统可自动创建。

### 可选目录

允许：
- `笔记/`
- `阅读/`
- `归档/`
- 用户自建的非冻结目录

冻结：
- `项目/` 不作为默认目录；
- `项目/` 不作为推荐目录；
- 如果文件树里已有 `项目/`，本阶段不删除、不改名、不默认进入。

## 4. 用户故事与验收

### Story 1：默认新建笔记

作为用户，我可以快速新建一篇普通笔记，它默认进入 `笔记/` 并自动打开。

验收：
- 点击新建笔记后，默认目录为 `笔记/`。
- 输入标题后创建 `笔记/<title>.md`。
- 创建成功后文件树刷新。
- 创建成功后自动打开该笔记标签页。
- 打开后可直接编辑，并沿用 B1 保存状态。

### Story 2：选择目录新建

作为用户，我可以选择 `阅读/`、`归档/` 或自建目录来创建笔记。

验收：
- 新建表单/对话框支持选择目录。
- 目录选择不默认推荐 `项目/`。
- 选择目录后创建到对应路径。
- 创建后自动打开对应笔记。

### Story 3：标题校验

作为用户，我不会因为空标题或非法标题创建奇怪文件。

验收：
- 标题为空时不创建文件，并显示提示。
- 标题包含不安全路径字符时有提示或自动清理。
- 同名文件冲突时有明确提示或自动生成不冲突名称。

### Story 4：重命名同步

作为用户，我重命名笔记后，文件树和标签页标题都同步。

验收：
- 文件树重命名成功后刷新显示新名称。
- 已打开标签页标题同步为新名称。
- 内容区仍指向新路径，不出现旧路径空白。
- B1 保存状态不被重置成错误状态。

### Story 5：删除同步

作为用户，我删除笔记时，系统不会留下坏标签或误导我继续编辑已删除文件。

验收：
- 删除前有确认或足够明确的危险反馈。
- 删除成功后文件树刷新。
- 如果文件已打开，对应标签关闭，或内容区显示文件已删除且不能继续编辑。
- 不影响其他打开标签。

### Story 6：新建文件夹后继续组织

作为用户，我可以新建文件夹，并继续在该文件夹里创建笔记。

验收：
- 新建文件夹后文件树刷新。
- 新目录可在新建笔记目录选择里出现。
- 可在新目录中创建笔记并自动打开。

## 5. 测试计划

### 组件 / Composable

- 新建笔记默认目录为 `笔记/`。
- 空标题阻止创建。
- 目录选择排除/不推荐 `项目/`。
- 同名冲突处理。

### 页面接线

- 新建成功后调用 createNote，并打开标签页。
- 文件树刷新。
- 重命名后更新 tab title/path。
- 删除已打开文件后关闭 tab 或显示删除态。
- 新建文件夹后可继续创建笔记。

### 回归

- B1 保存状态、保存失败重试、关闭保护仍通过。
- 不触碰 task DB / JSON / SQLite 相关代码。
- 不触碰 AI session / Dashboard。

## 6. 开发注意事项

- 优先复用现有 FileTreePanel、WorkspacePage、useSplitPanes、createNote / move / rename / delete API。
- 手术式修改，不重写文件树或编辑器。
- 如果现有删除/重命名 API 缺失或能力不足，先在 spec 中明确最小 API 变更，不扩大到完整文件管理器。
