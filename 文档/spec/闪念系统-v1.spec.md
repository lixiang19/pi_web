# 闪念系统 v1 Spec

> 来源：`文档/功能开发/闪念系统PRD-v0.1.md` 及 2026-05-08 讨论结论。  
> 状态：待实现。  
> 原则：全部以 PRD 为准；旧 `收件箱/*.md` 不迁移、不兼容、不作为新系统数据源。

## 1. 背景

闪念是 ridge 的临时捕获队列。它不是笔记系统，也不是长期知识库，而是让用户在任意工作场景中快速丢下内容，由系统后台分析最合适的处理方式，再由用户点击处理。处理成功后，原始闪念必须从队列中删除。

当前仓库已有文件型“收件箱”体验，但其数据源是 `收件箱/*.md` 文件，不符合 PRD 中“DB 临时队列 + 后台分析 + 用户点击处理 + 成功删除”的核心模型。v1 将以 DB 队列替代文件型收件箱语义。

## 2. 目标

1. 网页端提供右下角常驻闪念入口。
2. 用户可以快速输入任意内容并保存为闪念。
3. 闪念保存到 DB 临时队列。
4. 保存成功后只显示轻提示，不跳转收件箱。
5. 保存成功后后台调用 pi agent 进行分析。
6. 每条闪念最多生成一个 AI 推荐处理。
7. 收件箱页面展示所有未处理闪念。
8. 用户可以按 AI 建议处理。
9. 用户可以通过固定按钮处理为日记、任务、剪藏，或删除。
10. 日记处理写入今天日记。
11. 剪藏处理写入 DB 剪藏系统。
12. 任务处理入口保留，但当前 worktree 暂不创建任务。
13. 处理成功后删除原始闪念。
14. 处理失败时保留原始闪念。
15. 所有闪念处理或删除后，收件箱显示“闪念已清空”。

## 3. 非目标

1. 不兼容旧 `收件箱/*.md` 闪念文件。
2. 不迁移旧收件箱文件。
3. 不把已处理闪念长期保留在 DB。
4. 不做闪念历史审计。
5. 不允许 AI 自动处理闪念。
6. 不在捕获时要求用户选择分类、标签、重要级别或目标系统。
7. 不在当前 worktree 实现完整任务系统。
8. 不实现浏览器插件、网页全文抓取、剪藏高级搜索、剪藏标签系统。
9. 不实现多端同步入口、桌面全局快捷键、安卓通知栏入口。
10. 不支持选择日记日期；v1 只写入今天。

## 4. 范围

### 4.1 包含

- 闪念 DB 队列。
- 闪念创建、列表、删除。
- 闪念分析状态。
- pi agent 后台分析。
- AI 单一推荐。
- 日记确认表单与写入今天日记。
- 剪藏确认表单与 DB 存储。
- 任务按钮待接入行为。
- 收件箱队列 UI。
- 全局右下角闪念入口。
- 后端 API 测试、前端 composable/组件测试。

### 4.2 不包含

- 旧文件收件箱迁移。
- 任务系统创建逻辑。
- 日记完整系统重构。
- 剪藏完整产品体系。
- AI 多建议排序。
- 自动处理正式对象。

## 5. 术语

| 术语 | 定义 |
| --- | --- |
| 闪念 | 用户快速捕获的临时内容，保存于 DB 队列。 |
| 收件箱 | 展示未处理闪念的清空队列页面。 |
| AI 推荐 | pi agent 对单条闪念生成的唯一最佳处理建议。 |
| 固定处理按钮 | 不依赖 AI 的日记、任务、剪藏、删除按钮。 |
| 日记处理 | 将闪念内容追加到今天的日记 Markdown 文件。 |
| 剪藏处理 | 将闪念保存为结构化剪藏 DB 记录。 |
| 任务处理 | 当前仅展示入口和待接入提示，不创建正式任务。 |

## 6. 状态模型

### 6.1 闪念主状态

闪念主状态只表达是否仍在队列中。

```txt
待处理
处理中
```

处理成功后不进入“已处理”状态，而是直接删除记录。

### 6.2 分析状态

```txt
未分析
分析中
已建议
```

失败不作为面向用户的常驻状态。系统可以在后台记录失败并重试。用户仍然可以使用固定按钮处理。

### 6.3 删除规则

以下情况删除闪念：

- 用户主动点击删除。
- 用户按 AI 建议处理成功。
- 用户按日记固定按钮处理成功。
- 用户按剪藏固定按钮处理成功。

以下情况不得删除闪念：

- AI 分析成功。
- AI 分析失败。
- 处理动作开始但未完成。
- 处理失败。
- 任务按钮被点击但任务系统暂未接入。
- 目标系统返回失败。

## 7. 数据模型要求

> 具体字段名由实现阶段决定，但必须满足以下业务信息。不要把 pi session 当业务数据库。

### 7.1 闪念队列记录

每条未处理闪念至少需要保存：

- 唯一标识。
- 原始内容。
- 创建时间。
- 更新时间。
- 主状态。
- 分析状态。
- AI 推荐处理类型。
- AI 推荐文案。
- AI 处理草稿。
- 是否需要用户补充。
- pi session 关联信息。
- 分析重试信息。
- 最近错误信息，仅供后台诊断。

### 7.2 AI 推荐类型

v1 支持：

```txt
日记
剪藏
任务
删除或保守处理
```

推荐只能有一个。

### 7.3 剪藏记录

剪藏使用 DB 存储。每条剪藏至少保存：

- 唯一标识。
- 标题。
- URL，可为空。
- 摘录或正文。
- 来源说明，可为空。
- 创建时间。
- 更新时间。

### 7.4 日记存储

日记继续使用现有 Markdown 文件路径：

```txt
日记/YYYY/MM/YYYY-MM-DD.md
```

v1 只写入今天，不提供日期选择。

## 8. API 契约要求

> 端点命名由实现阶段决定，但能力必须完整覆盖以下契约。

### 8.1 创建闪念

输入：

- 原始内容。

规则：

- 空内容拒绝。
- 保存时不要求分类、标签、目标系统。
- 创建成功后进入未分析状态。
- 创建成功后触发后台分析。

输出：

- 新闪念记录。

### 8.2 获取闪念列表

输出：

- 所有未处理闪念。
- 按创建时间或更新时间排序。
- 包含分析状态和 AI 推荐信息。

规则：

- 不读取旧 `收件箱/*.md`。
- 不返回已处理历史。

### 8.3 删除闪念

输入：

- 闪念标识。

规则：

- 用户确认后删除。
- 删除后不保留历史。
- 如果后台分析仍在运行，应取消或忽略迟到结果。

### 8.4 写入今天日记

输入：

- 闪念标识。
- 用户确认后的写入内容。

规则：

- 只写入今天。
- 如果今天日记不存在，自动创建。
- 如果今天日记存在，追加到文件末尾。
- 写入成功后删除闪念。
- 写入失败时保留闪念。

建议追加格式：

```md
## 闪念

- HH:mm  
  闪念内容
```

实现可以避免重复创建多个连续空标题，但不得破坏用户已有内容。

### 8.5 创建剪藏

输入：

- 闪念标识。
- 用户确认后的标题。
- URL，可为空。
- 摘录或正文。
- 来源说明，可为空。

规则：

- 创建剪藏 DB 记录成功后删除闪念。
- 创建失败时保留闪念。
- 如果闪念中包含 URL，确认表单应自动填充 URL。
- 纯文本闪念应作为摘录保存。

### 8.6 任务待接入

输入：

- 闪念标识。

规则：

- 当前 worktree 不创建任务。
- 返回“任务系统待接入”的可显示结果。
- 不删除闪念。
- 不改变闪念为已处理。

### 8.7 AI 分析写回

输入：

- 闪念标识。
- 单一推荐。
- 推荐文案。
- 草稿内容。
- pi session 关联信息。

规则：

- 只能写回仍存在、仍未处理的闪念。
- 如果用户已经处理或删除，应忽略迟到结果。
- 同一闪念只能保留一个当前推荐。

## 9. pi agent 分析要求

### 9.1 接入方式

- 使用项目已有的 `@mariozechner/pi-coding-agent` SDK 能力。
- 优先使用 `createAgentSession`、`SessionManager`、`DefaultResourceLoader`、`session.subscribe` 等真实 pi 能力。
- 不允许生产路径 mock agent。
- 不允许自造 pi session 文件格式。
- 不允许把 pi session 当业务状态 DB。

### 9.2 分析触发

创建闪念成功后，后端自动排队分析。

### 9.3 分析输入

至少包含：

- 闪念原始内容。
- 可用处理目标：日记、剪藏、任务、删除或保守处理。
- 当前任务系统暂未接入的事实。
- 日记只能写入今天的事实。
- 剪藏支持链接和纯文本摘录的事实。

### 9.4 分析输出

只允许一个最佳建议，包含：

- 推荐类型。
- 面向用户的推荐文案。
- 处理草稿。
- 是否需要用户补充。
- 无法判断时的保守建议。

### 9.5 推荐规则

1. 明确行动项优先推荐任务。
2. 明确个人记录、感受、复盘优先推荐日记。
3. 明确外部资料、链接、摘录优先推荐剪藏。
4. 无法判断时，给出保守且不会污染正式系统的建议。

### 9.6 并发与取消

- 用户手动处理优先于 AI 分析。
- 用户删除或处理成功后，正在运行的分析应取消；无法取消时必须忽略迟到写回。
- 同一闪念不得并发运行多个有效分析结果写回。

## 10. 前端交互要求

### 10.1 全局闪念入口

- 网页端右下角常驻按钮。
- 点击后显示轻量输入浮层。
- 浮层只包含输入和保存能力。
- 空内容不能保存。
- 保存成功后显示 toast。
- 保存成功后不打开收件箱。
- 保存失败时保留用户输入。

### 10.2 收件箱页面

收件箱页面用于清空队列。

顶部展示：

- 待处理数量。
- 分析中数量。

主体展示闪念卡片列表。

每张卡片展示：

- 原始内容。
- 创建时间。
- AI 推荐处理状态。
- AI 推荐处理文案。
- 按建议处理按钮。
- 固定处理按钮：日记、任务、剪藏、删除。

空状态展示：

```txt
闪念已清空
```

### 10.3 日记确认表单

用户点击日记固定按钮或 AI 推荐日记时打开。

展示：

- 将写入“今日日记”。
- 可编辑写入内容。
- 确认按钮。
- 取消按钮。

规则：

- 确认成功后删除闪念。
- 取消不改变闪念。
- 写入失败显示错误并保留闪念。

### 10.4 剪藏确认表单

用户点击剪藏固定按钮或 AI 推荐剪藏时打开。

展示：

- 标题。
- URL。
- 摘录或正文。
- 来源说明。
- 确认按钮。
- 取消按钮。

规则：

- 确认成功后删除闪念。
- 取消不改变闪念。
- 创建失败显示错误并保留闪念。

### 10.5 任务按钮

用户点击任务固定按钮或 AI 推荐任务时：

- 显示“任务系统正在接入中，暂不能从闪念创建任务”。
- 不删除闪念。
- 不改变闪念状态。

## 11. 处理流程

### 11.1 快速捕获

```txt
用户点击右下角闪念按钮
↓
输入内容
↓
保存
↓
写入 DB
↓
轻提示
↓
后台启动 pi agent 分析
```

### 11.2 后台分析

```txt
闪念进入 DB
↓
进入待分析队列
↓
pi agent 分析内容
↓
生成一个最佳处理建议
↓
写回闪念记录
↓
收件箱展示建议
```

### 11.3 写入日记

```txt
用户点击日记
↓
确认写入内容
↓
追加到今日日记
↓
写入成功
↓
删除原闪念
```

### 11.4 创建剪藏

```txt
用户点击剪藏
↓
确认标题、URL、摘录、来源
↓
创建剪藏 DB 记录
↓
创建成功
↓
删除原闪念
```

### 11.5 任务待接入

```txt
用户点击任务
↓
显示任务系统待接入提示
↓
闪念保留
```

## 12. 错误处理

1. 创建闪念失败：输入框保留内容，展示错误提示。
2. 分析失败：不打扰用户，后台可重试，固定处理按钮仍可用。
3. 日记写入失败：展示错误提示，闪念保留。
4. 剪藏创建失败：展示错误提示，闪念保留。
5. 删除失败：展示错误提示，闪念保留。
6. 任务待接入：展示提示，闪念保留。
7. AI 迟到结果：如果闪念不存在或已处理，忽略。

## 13. 测试用例

### 13.1 后端 API 测试

- 创建闪念：有效内容创建成功。
- 创建闪念：空内容返回错误。
- 创建闪念：成功后分析状态为未分析或进入分析中。
- 获取列表：只返回未处理闪念。
- 获取列表：不读取旧 `收件箱/*.md`。
- 删除闪念：成功后列表不再返回。
- 删除闪念：不存在的闪念返回错误。
- AI 写回：写回单一建议成功。
- AI 写回：已删除闪念的迟到结果被忽略。
- 日记处理：今天日记不存在时自动创建并写入。
- 日记处理：今天日记存在时追加内容。
- 日记处理：写入成功后删除闪念。
- 日记处理：写入失败时保留闪念。
- 剪藏处理：链接型闪念可创建剪藏。
- 剪藏处理：纯文本闪念可创建剪藏。
- 剪藏处理：创建成功后删除闪念。
- 剪藏处理：创建失败时保留闪念。
- 任务处理：返回待接入提示。
- 任务处理：不删除闪念。

### 13.2 pi agent 分析测试

- 创建闪念后会触发分析任务。
- 分析结果只包含一个推荐。
- 行动项优先推荐任务。
- 个人记录优先推荐日记。
- 链接或摘录优先推荐剪藏。
- 无法判断时返回保守建议。
- 用户删除后，分析迟到结果不写回。
- 用户处理成功后，分析迟到结果不写回。

测试可以 mock pi SDK 边界，但生产实现必须使用真实 pi SDK 路径。

### 13.3 前端 composable 测试

- 加载闪念列表。
- 创建闪念成功后刷新或本地插入。
- 创建失败时保留输入。
- 删除成功后本地移除。
- 删除失败时恢复或重新加载。
- 日记处理成功后本地移除。
- 日记处理失败时保留。
- 剪藏处理成功后本地移除。
- 剪藏处理失败时保留。
- 任务按钮不移除闪念。

### 13.4 前端组件测试

- 全局入口可打开输入浮层。
- 空内容保存按钮不可用或保存时报错。
- 保存成功显示 toast 且不跳转收件箱。
- 收件箱展示待处理数量。
- 收件箱展示分析中数量。
- 闪念卡片展示原始内容、时间、推荐状态、固定按钮。
- 点击日记打开确认表单。
- 点击剪藏打开确认表单。
- 点击任务显示待接入提示。
- 点击删除触发删除确认或删除动作。
- 空列表展示“闪念已清空”。
