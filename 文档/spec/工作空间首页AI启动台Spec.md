# 工作空间首页 AI 启动台 Spec

## 背景

工作空间 `/workspace` 目前存在 `HomePage` 空壳与 `DashboardView` 仪表盘定位重复的问题。新首页要成为工作空间内的 AI 启动台，并把会话能力纳入工作空间 SplitGrid 多标签体系。

## 用户故事

用户进入工作空间后，首先看到居中的 AI 输入框。用户输入第一句话后，当前首页标签原地变成完整 AI 会话标签；后续文件、首页、会话都作为同级标签在工作空间内打开和分屏。

## 输入

1. 首页中央输入框文本。
2. 首页输入框聚焦/展开状态。
3. 标签栏“+”操作。
4. 最近事情条目点击。
5. 最近文件条目点击。

## 输出

1. 首页初始态：
   - 中央 AI 启动输入框，输入框位于首页上半屏的视觉中心，上方保留留白。
   - 模型、Agent、Thinking 默认展示在输入框底部工具栏，不依赖聚焦才出现。
   - 模型、Agent、Thinking 以 AI 对话输入工具栏形式呈现，仍使用真实 shadcn-vue Select 控件。
   - 下方信息区整体位于输入框下方：左侧混合时间线为主，右侧最近文件与 AI 建议占位。

2. 首条消息提交：
   - 触发时机是发送消息，不是输入、聚焦或展开输入框。
   - 当前首页标签原地转换为 AI 会话标签。
   - 标签标题变为会话标题。
   - 内容区显示完整会话的消息流 + 输入框。
   - 发送失败时仍保留会话标签，并在会话内显示错误态。
   - 不显示旧 `/chat` 页面的右侧文件预览面板和项目文件面板。

3. 新建标签：
   - 点击当前工作空间标签栏“+”总是创建新的首页启动台。
   - 新首页再次输入后同样原地转换为会话标签。

4. 混合时间线：
   - 把最近文件、任务/闪念、日记、会话等统一成最近事情列表。
   - 点击条目直接在当前分屏中新开对应标签：文件开文件标签，会话开会话标签，任务/闪念开详情。

5. AI 建议：
   - 首版只展示占位，不做真实 AI 分析。

## 边界

1. 只专注 `/workspace`；旧 `/chat` 路由暂不调整。
2. 不把旧 chat 页右侧 OperationPanel / ProjectFilePanel 搬入工作空间会话标签。
3. 仪表盘入口废弃；其有价值内容并入首页下方区域。
4. 会话、文件、首页是 SplitGrid 同级标签，允许分屏。
5. 不新增后台 AI 分析任务、不新增建议持久化。

## 验收标准

1. 打开 `/workspace` 默认可看到首页启动台。
2. 首页输入第一句话后，当前标签原地变成会话标签，并开始发送消息。
3. 会话标签只包含消息流与输入框，文件打开走工作空间标签。
4. 点击标签栏“+”创建新的首页启动台。
5. 最近事情点击能打开对应标签。
6. 仪表盘入口不再与首页重复出现。
7. UI 使用 shadcn-vue 与 ridge 主题变量，禁止硬编码颜色与手写基础组件。
8. 修改 `.ts` / `.vue` / `.js` 后，根目录 `npm run check` 必须通过。
9. 首页初始态无需聚焦即可看到模型、Agent、思考级别三个选择入口。
10. 首页模型列表与默认模型异步加载时，选择器必须在 `models/defaultModel` 到达后同步为有效默认模型；若用户当前已选择的模型仍在列表中，不得被后续默认值覆盖。
