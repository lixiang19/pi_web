# 双工作模式愿景

## 一句话定义

ridge 支持 **代码模式** 与 **助手模式** 两种工作流，一键切换，各尽其能。

---

## 模式对比

| 维度 | 代码模式 | 助手模式 |
|------|---------|---------|
| **定位** | 开发 IDE | 通用助手 |
| **核心任务** | 写代码、改文件、调试、审查 | 问答、写作、分析、学习 |
| **界面重心** | 文件树 + 编辑器 + 对话 | 对话区为主，文件树可选 |
| **工具集** | 全工具（read/bash/edit/write） | 受限工具（read/有限 bash） |
| **Agent 默认** | 编程专家型 | 通用对话型 |
| **快捷键** | IDE 风格（Cmd+Enter 发送） | 聊天风格（Enter 发送） |
| **输入框** | 底部固定，支持多行 | 底部自适应，简洁 |

---

## 模式专属特性

### 代码模式（工程导向）
- **Git 集成**：status/diff/commit/branch 可视化面板
- **终端**：底部终端面板，直接执行命令
- **Worktree 管理**：多分支并行开发切换
- **文件树**：项目目录浏览与文件编辑
- **Plan/Execute 阶段**：计划 → 执行 两阶段工作流
- **子代理委派**：并行代码审查、多文件处理

### 助手模式（知识导向）
- **无 Git/终端/Worktree**：剥离工程复杂度，专注对话
- **知识库检索**：接入个人文档/笔记/网页收藏
- **上下文管理**：自动压缩、摘要、会话整理
- **Markdown 预览**：输入区支持实时预览
- **独立工作空间**：可选关联轻量文件夹，不强制项目结构

---

## 切换机制

### 全局切换
- 设置页选择默认模式
- 快捷键 `Cmd/Ctrl + Shift + 1/2` 快速切换

### 会话级切换
- 新建会话时选择本次模式
- 会话头部显示当前模式图标
- 模式切换时提示："切换模式将创建新会话以隔离上下文"

---

## 架构影响

| 层 | 变化 |
|---|------|
| **UI** | 工作台布局组件化，根据模式动态装配 |
| **Agent** | 默认 Agent 推荐根据模式筛选 |
| **工具权限** | 助手模式工具白名单限制 |
| **系统提示词** | 模式级基础提示词追加 |
| **会话类型** | Session 表增加 `mode: 'code' \| 'assistant'` 字段 |

### 功能差异化实现

```
┌─────────────────────────────────────────────────────┐
│                    工作台壳层                        │
│  根据 session.mode 动态决定右侧栏内容                    │
└──────────────────┬──────────────────────────────────┘
                   │
        ┌──────────┴──────────┐
        │                     │
        ▼                     ▼
┌───────────────┐    ┌────────────────┐
│   代码模式     │    │    助手模式     │
│               │    │                │
│  右栏: Git    │    │  右栏: 知识库    │
│      + 终端   │    │      + 收藏     │
│      + 文件树 │    │      (可选文件) │
│               │    │                │
│  可用工具: 全量 │    │  可用工具: 受限  │
│  (edit/write) │    │  (仅 read/简单bash)│
│               │    │                │
│  cwd: 项目目录 │    │  cwd: 轻量目录或null│
└───────────────┘    └────────────────┘
```

### 助手模式的关键简化

| 功能 | 代码模式 | 助手模式 | 原因 |
|------|---------|---------|------|
| Git 面板 | ✅ 完整 | ❌ 无 | 非工程场景不需要版本控制 |
| 终端 | ✅ 底部面板 | ❌ 无 | 减少误操作风险 |
| Worktree | ✅ 管理 | ❌ 无 | 单分支对话足够 |
| 文件树 | ✅ 项目级 | ⚪ 可选轻量 | 可能只需参考文档 |
| edit/write 工具 | ✅ 启用 | ❌ 禁用 | 助手模式只读咨询 |
| 子代理 | ✅ 代码审查 | ✅ 知识检索 | 两者都需要 |
| 会话压缩 | ✅ 自动 | ✅ 更激进 | 长对话更常见 |

---

## 演进阶段

- **当前**：代码模式 MVP（三栏工作台）
- **一阶段**：助手模式 MVP（对话区为主，隐藏文件树）
- **二阶段**：模式间会话可引用（助手分析 → 代码模式实现）
- **三阶段**：智能模式推荐（根据输入内容自动建议模式）
