# 记忆与后台会话整理

## 职责

在 Pi 会话结束后，把本次会话整理为 daily Markdown 记忆，并基于 daily 维护工作空间级 `记忆/MEMORY.md`；在新会话启动时把 `MEMORY.md` 与 `Wiki/index.md` 注入系统上下文。

## 入口

- `packages/server/src/workspace-memory.ts` — summary agent、memory agent、显式记忆命令和注入格式。
- `packages/server/src/index.ts` — `POST /api/sessions/:sessionId/end` 会话结束入口、消息入口显式“记住/忘掉”处理、后台 worker 启动与后台模型设置解析。
- `packages/server/src/session-context.ts` — `DefaultResourceLoader.appendSystemPromptOverride` 注入最新记忆与 Wiki。
- `packages/web/src/lib/api.ts` — `endSession(sessionId)`。
- `packages/web/src/pages/WorkspacePage.vue` — 关闭会话 tab、页面卸载和 `pagehide` 时调用会话结束入口。
- `packages/web/src/components/workspace/SettingsTabContent.vue` — 配置后台整理模型与后台思考强度。
- `packages/server/src/background-jobs.ts` — `summary.daily` 按 payload `dailyDate` 串行，`memory.maintain` 全局串行。

## 文件真源

```txt
记忆/MEMORY.md
记忆/daily/YYYY/MM/YYYY-MM-DD.md
Wiki/index.md
```

- `MEMORY.md` 是全局长期记忆，只有标题时视为空内容，不注入。
- `Wiki/index.md` 只有标题时视为空内容，不注入。
- daily 文件按服务器本地日期分目录，只追加，不生成单会话摘要文件。

## 后台任务

### `summary.daily`

触发：`POST /api/sessions/:sessionId/end`。

幂等：

- 同一 session 已存在 pending/running/completed 且未 failed/cancelled 的 `summary.daily` job 时，入口返回原 job，不重复入队。
- active session 仍在 streaming 时返回 409，不做整理。

payload 记录：

- `sessionId`
- `sessionFile`
- `title`
- `cwd`
- `workspaceDir`
- `projectLabel`
- `projectRoot`
- `dailyDate`
- `dailyYear`
- `dailyMonth`
- `dailyTime`

行为：

- 只读 `sessionFile` 中本次会话记录。
- 调用 Pi SDK 创建后台会话，禁用工具，让 agent 只返回结构化 JSON。
- 服务端负责把结构化结果追加到 `记忆/daily/YYYY/MM/YYYY-MM-DD.md`。
- 服务端二次归一产物路径：工作空间产物写 workspace 相对路径；外部项目绝对路径写成 `项目名: 项目内相对路径`。
- 非 active session 通过 `session_contexts` 补 `projectLabel/projectRoot`，避免外部项目路径归一化降级。
- 完成后排队同一工作空间的 `memory.maintain`。
- 不写 `记忆/<sessionId>.md` 之类单会话摘要文件。
- 只写 `payload.dailyDate` 对应 daily；跨日结束不会修改旧日期 daily。

## 后台模型

- settings 增加 `backgroundAgentModel` 与 `backgroundAgentThinkingLevel`。
- `backgroundAgentModel` 格式为 `provider/model`，由 Pi `ModelRegistry.find(provider, model)` 解析。
- 未配置 `backgroundAgentModel` 时不强行复用前台会话选择，交给 Pi SDK 默认模型选择。
- summary agent 和 memory agent 共用这组后台配置，并始终传 `noTools: "all"`。
- 设置页提供后台整理模型与后台思考强度入口。

### `memory.maintain`

触发：`summary.daily` 完成后。

行为：

- 读取当天 daily 与当前 `MEMORY.md`。
- 调用后台 agent 产出短句记忆数组。
- 服务端过滤 token、密码、私钥、密钥等敏感内容。
- 去重并按长度预算写回 `MEMORY.md`；无变化时只完成 job，不改文件。

## 显式记忆命令

消息入口在发送给 Pi 前处理：

- `记住 ...` / `请记住 ...`：立即写入 `MEMORY.md`，排在现有记忆前。
- `忘掉 ...` / `别再记 ...` / `不要再记 ...`：立即从 `MEMORY.md` 删除包含该内容的记忆。
- 命中敏感信息规则时不写入。

显式命令只维护 Markdown 文件，不依赖 RAG 或图谱。

## 注入格式

新会话或资源 reload 时同步读取当前文件：

```xml
<ridge_memory>
记忆可能过时，当前用户最新话语和当前文件事实优先。
...
</ridge_memory>

<ridge_wiki_index>
记忆可能过时，当前用户最新话语和当前文件事实优先。
...
</ridge_wiki_index>
```

空 `MEMORY.md` 或空 `Wiki/index.md` 不生成对应 XML 块。

## 并发边界

- `summary.daily` 以 `payload.dailyDate` 为串行 scope，同一天 daily 写入不会并发。
- `summary.daily` 的去重键是 `relatedType=session, relatedId=sessionId`，因此同一天不同会话不会被误去重。
- `memory.maintain` 全局串行，避免多个后台任务同时改写 `MEMORY.md`。

## 测试覆盖

- `workspace-memory.test.ts`
  - 会话结束后生成当天 daily。
  - 跨日结束只写新日期 daily，不改旧日期 daily。
  - 外部项目产物路径归一为项目名和项目内相对路径。
  - 后台 agent 创建时带独立模型和思考级别设置。
  - 不生成单会话摘要文件。
  - summary 完成后排队 memory 维护。
  - 空 MEMORY/Wiki 不注入。
  - 注入包含过时提醒。
  - 显式“记住/忘掉”立即改写。
  - token 等敏感信息不进入 MEMORY。
- `background-jobs.test.ts`
  - session 级 summary job 不误去重。
  - 同一 daily date 的 summary job 串行。
- `security-guards.test.ts`
  - `/end` 会为 indexed session 入队。
  - completed summary job 后重复 `/end` 不重复入队。
  - 非 active 外部 session 入队时保留 `projectLabel/projectRoot`。
- `WorkspacePage.test.ts`
  - 页面卸载时结束仍打开的会话标签。
  - `pagehide` 时优先使用 `sendBeacon`。
- `SettingsTabContent.test.ts`
  - 设置页渲染并保存后台整理模型和后台思考强度。
