# 记忆与后台会话整理

## 职责

在 Pi 会话结束后，把本次会话整理为按日期追加的 daily Markdown；daily 是 L1 真源。随后由不可见内置 `memory-agent` 维护场景级 L2 Scenario 与启动注入用 L3 `MEMORY.md`；在新会话启动时把当前有效 `MEMORY.md` 与 `Wiki/index.md` 注入系统上下文。

## 入口

- `packages/server/src/workspace-memory.ts` — summary agent、memory agent、显式记忆命令和注入格式。
- `packages/server/src/wiki-agent.ts` — Wiki Agent 来源收集、严格 JSON 解析、Wiki 页面写入/删除和 RAG 待索引标记。
- `packages/server/src/index.ts` — `POST /api/sessions/:sessionId/end` 会话结束入口、消息入口显式“记住/忘掉”处理、后台 worker 启动与后台模型设置解析。
- `packages/server/src/session-context.ts` — `DefaultResourceLoader.appendSystemPromptOverride` 注入最新记忆与 Wiki；当前记忆注入不依赖 Pi `settings.json` 的 `instructions` 字段。
- `packages/web/src/lib/api.ts` — `endSession(sessionId)`。
- `packages/web/src/pages/WorkspacePage.vue` — 关闭会话 tab、页面卸载和 `pagehide` 时调用会话结束入口。
- `packages/web/src/components/workspace/SettingsTabContent.vue` — 配置后台整理模型与后台思考强度。
- `packages/server/src/background-jobs.ts` — `summary.daily` 按 payload `dailyDate` 串行，`memory.maintain` 全局串行。
- `packages/server/src/rag-worker.ts` — 夜间维护入口，顺序固定为 RAG deferred 索引 -> graph agent -> Wiki agent -> Wiki immediate RAG 索引。

## 文件真源

```txt
记忆/MEMORY.md
记忆/daily/YYYY/MM/YYYY-MM-DD.md
记忆/scenarios/<scenario-slug>.md
Wiki/index.md
```

- L1 真源是 `记忆/daily/YYYY/MM/YYYY-MM-DD.md`，按日期和会话追加 Markdown 摘要；日期是一等可信度与冲突裁决信号。
- L2 真源是 `记忆/scenarios/<scenario-slug>.md`，按场景聚合 daily 中沉淀的事实、决策和约束，保存当前结论与历史覆盖。
- L3 `MEMORY.md` 只保存启动注入需要的当前有效结论；每条必须带 `[scope][date]`，只有标题时视为空内容，不注入。
- `Wiki/index.md` 只有标题时视为空内容，不注入。
- daily 文件按服务器本地日期分目录，只追加，不生成单会话摘要文件。
- 新会话启动时同步读取 `~/ridge-workspace/记忆/MEMORY.md` 与 `~/ridge-workspace/Wiki/index.md`，格式化为 `<ridge_memory>` / `<ridge_wiki_index>` 追加到系统提示；`packages/server/pi-default-config/settings.json` 不配置 `instructions`。

## 后台任务

### `summary.daily`

触发：`POST /api/sessions/:sessionId/end`。

幂等：

- 同一 session 已存在 pending/running/completed 且未 failed/cancelled 的 `summary.daily` job 时，入口返回原 job，不重复入队。
- active session 仍在 streaming 时返回 409，不做整理。

payload 记录：

- `sessionId`
- `sessionFile`
- `title`
- `cwd`
- `workspaceDir`
- `projectLabel`
- `projectRoot`
- `dailyDate`
- `dailyYear`
- `dailyMonth`
- `dailyTime`

行为：

- 只读 `sessionFile` 中本次会话记录。
- 如果 `sessionFile` 已不存在，job 以 `{ skipped: true, reason: "session_file_missing" }` 完成，不重试、不写失败通知；会话文件缺失通常表示会话已被清理，用户无可操作项。
- 调用 `@mariozechner/pi-ai complete()` 做一次基础模型调用，不创建 `AgentSession`，不开放工具。
- summary agent 只输出可直接追加到 daily 的 Markdown 正文，不要求 JSON。
- 服务端把 Markdown 追加到 `记忆/daily/YYYY/MM/YYYY-MM-DD.md` 的当前会话小节。
- 服务端二次归一产物路径：工作空间产物写 workspace 相对路径；外部项目绝对路径写成 `项目名: 项目内相对路径`。
- 非 active session 通过 `session_contexts` 补 `projectLabel/projectRoot`，避免外部项目路径归一化降级。
- 完成后排队同一工作空间的 `memory.maintain`。
- 不写 `记忆/<sessionId>.md` 之类单会话摘要文件。
- 只写 `payload.dailyDate` 对应 daily；跨日结束不会修改旧日期 daily。

## 后台模型

- settings 增加 `backgroundAgentModel` 与 `backgroundAgentThinkingLevel`。
- `backgroundAgentModel` 格式为 `provider/model`，由 Pi `ModelRegistry.find(provider, model)` 解析。
- 未配置 `backgroundAgentModel` 时不强行复用前台会话选择，交给 Pi SDK 默认模型选择。
- summary agent 和 memory agent 共用这组后台配置；summary 使用基础 `complete()` 调用，memory 使用真实内部 `AgentSession`。
- 设置页提供后台整理模型与后台思考强度入口。

### `memory.maintain`

触发：`summary.daily` 完成后。

行为：

- 读取当天 daily、当前 `MEMORY.md` 和现有 Scenario 语义，由不可见内置 `memory-agent` 真实运行维护。
- `memory-agent` 使用普通 Agent 文件工具读写 `记忆/MEMORY.md` 与 `记忆/scenarios/*.md`，可以合并重复主题、删除被覆盖的旧记忆。
- `memory-agent` 的权限固定收窄：`ask: deny`、`subagent: deny`、`bash: deny`，`edit` 只允许 `记忆/MEMORY.md` 与 `记忆/scenarios/*`。
- 完成后调用统一工具 `complete_internal_task`，后台任务结果直接使用该工具的 `status/summary/error`。
- 同主题冲突由当前用户最新话语、显式程度、daily 日期新近程度和长期稳定偏好共同裁决；被覆盖的旧结论不必保留在 L3，可按需要写入 L2 的“历史覆盖”。

### Wiki 夜间维护

触发：RAG worker 每日夜间入口。

顺序：

1. `indexAllPending({ includeDeferred: true, event: "nightly" })`
2. `graphRunner.runNightlyOnce()`
3. `wikiRunner.runNightlyOnce()`
4. `indexAllPending({ event: "nightly" })`

输入：

- 当前 `Wiki/**/*.md`，其中 `current_wiki` 是用户可编辑真源。
- `记忆/MEMORY.md`。
- `记忆/daily/**/*.md`。
- 已索引 Markdown RAG chunk，包括正式附件转换产物。
- `.ridge/graph.kuzu` 图谱快照。

行为：

- Wiki agent 只返回严格 JSON：`pages` 与 `deletePaths`。
- `path` 必须是相对 `Wiki/` 的 `.md` 路径，禁止绝对路径、`..`、`Wiki/` 前缀、隐藏/系统路径段和符号链接写入路径。
- `Wiki/index.md` 是固定入口，缺失时补回只有标题的空入口。
- 可以新增、改写、删除低价值或过时页面；不能删除 `index.md`。
- 写入页面后调用 `markRagTargetPending(..., { refreshPolicy: "immediate", event: "nightly" })`；内容未变但未索引的保留页也会补标记。Wiki agent 完成后夜间链路立即再跑一次 non-deferred RAG 扫描，让 Wiki 页面在本轮夜间维护内进入 RAG；删除页面后无论文件是否已存在都同步 `removeRagTarget`。
- 不把 Wiki 做成第二个文件库，只维护少量 canonical Markdown 页面。

## 显式记忆命令

消息入口在发送给 Pi 前处理：

- `记住 ...` / `请记住 ...`：立即写入 `MEMORY.md`，排在现有记忆前。
- 显式写入使用 `global` scope 和当前日期，格式为 `- [global][YYYY-MM-DD] ...`。
- `忘掉 ...` / `别再记 ...` / `不要再记 ...`：立即从 `MEMORY.md` 删除包含该内容的记忆。
- 命中敏感信息规则时不写入。

显式命令只维护 Markdown 文件，不依赖 RAG 或图谱。

## 注入格式

新会话或资源 reload 时同步读取当前文件：

```xml
<ridge_memory>
记忆可能过时，当前用户最新话语和当前文件事实优先。
...
</ridge_memory>

<ridge_wiki_index>
记忆可能过时，当前用户最新话语和当前文件事实优先。
...
</ridge_wiki_index>
```

空 `MEMORY.md` 或空 `Wiki/index.md` 不生成对应 XML 块。

## 并发边界

- `summary.daily` 以 `payload.dailyDate` 为串行 scope，同一天 daily 写入不会并发。
- `summary.daily` 的去重键是 `relatedType=session, relatedId=sessionId`，因此同一天不同会话不会被误去重。
- `memory.maintain` 全局串行，避免多个后台任务同时改写 `MEMORY.md`。
