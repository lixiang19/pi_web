# 运行时环境配置

## 职责

运行时环境配置负责加载 ridge server 需要的密钥、外部服务地址和部署参数。此类配置统一来自环境变量，可写在仓库根目录 `.env` 或 `.env.local`，启动 server 时由 `packages/server/src/env.ts` 加载。

用户偏好、收藏、项目列表等产品状态继续由 `app_settings` 等数据库表保存；它们不是运行时密钥配置。

## 加载入口

- `packages/server/src/env.ts`
  - 启动时自动加载 `.env` 与 `.env.local`。
  - 从仓库根目录到当前 server 运行目录依次查找。
  - shell 已提供的环境变量优先级最高。
  - 同一目录下 `.env.local` 覆盖 `.env`。
- `packages/server/src/index.ts`
  - 第一行导入 `./env.js`，保证后续模块读取 `process.env` 前已经加载环境文件。

## 变量清单

| 变量 | 用途 | 必填 |
|------|------|------|
| `EXA_API_KEY` | Exa Contents API 鉴权 | URL 闪念/网页剪藏必填 |
| `EXA_BASE_URL` | Exa API base URL | 可选，默认 `https://api.exa.ai` |
| `PYTHON_CONVERTER_BASE_URL` | ridge Python Converter client base URL | 文件转 Markdown 必填 |
| `PYTHON_CONVERTER_API_KEY` | ridge Node 调用 Python Converter 的 Bearer key | 文件转 Markdown 必填 |
| `PYTHON_CONVERTER_CALLBACK_TOKEN` | Converter webhook callback token | 可选 |
| `PYTHON_CONVERTER_CALLBACK_BASE_URL` | Converter 回调 ridge server 的公开 base URL | 可选 |
| `SILICONFLOW_EMBEDDING_API_KEY` / `SILICONFLOW_API_KEY` | RAG embedding API key | RAG embedding 必填 |
| `SILICONFLOW_EMBEDDING_BASE_URL` / `SILICONFLOW_BASE_URL` | SiliconFlow-compatible embedding endpoint | 可选 |
| `SILICONFLOW_EMBEDDING_MODEL` | embedding 模型 | 可选 |
| `SILICONFLOW_EMBEDDING_DIMENSIONS` | embedding 维度参数 | 可选 |
| `SILICONFLOW_EMBEDDING_TIMEOUT_MS` | embedding 超时 | 可选 |
| `SILICONFLOW_EMBEDDING_MAX_RETRIES` | embedding 重试次数 | 可选 |
| `RIDGE_ADMIN_PASSWORD` | production 登录密码 | production 必填 |
| `RIDGE_PUBLIC_BASE_URL` / `RIDGE_SERVER_BASE_URL` | runtime bundle 公网 origin | 公网部署必填 |
| `RIDGE_DB_PATH` | ridge SQLite DB 路径 | 可选 |

## 边界

- `.env` 和 `.env.local` 不提交真实密钥；仓库只提交 `.env.example`。
- shell 环境变量优先级高于 `.env` 文件，便于部署系统注入。
- `app_settings` 只保存产品状态，不保存 Exa、Converter、SiliconFlow 这类服务密钥。
