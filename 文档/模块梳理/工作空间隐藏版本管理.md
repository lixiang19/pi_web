# 工作空间隐藏版本管理

## 职责

工作空间隐藏版本管理为用户可见工作空间内容创建保存点。底层使用 isomorphic-git，但 UI 和业务文案不暴露 Git 概念。

## 存储位置

- 工作树：当前工作空间目录。
- gitdir：`~/.pi/ridge-workspace-git/<workspace-hash>`。
- 内置 exclude 包含 `.ridge`、`.git`、`node_modules`、构建产物和缓存目录。

## 保存点入口

- `workspace-version.ts`
  - `getWorkspaceVersionContext(workspaceDir)`：解析隐藏 gitdir。
  - `commitWorkspaceVersionPoint({ workspaceDir, files, message })`：校验文件仍在工作空间内，过滤 `.ridge`，提交指定文件。
- `iso-git-service.ts`
  - `ensureInit` 初始化隐藏仓库并写 exclude。
  - `commit` 支持新增、修改和删除文件。
  - `getStatus` 返回状态时过滤 `.ridge` 内部 gitdir 变化。

## 当前触发点

- `/api/files/upload` 上传文件后创建保存点。
- `/api/files/entries` 新建文件或目录后尝试创建保存点；空目录无 Git 内容时返回空保存点。
- `/api/files/entries/path` 移动或重命名后提交旧路径和新路径。
- `DELETE /api/files/entries` 删除后提交删除路径。
- `PUT /api/files/content` 保存 Markdown 或空间 `index.html` 后创建保存点。

## 边界

- `.ridge`、RAG 缓存、Kuzu 图谱、运行缓存、临时附件不进入隐藏版本。
- 文件路径必须在工作空间内；realpath 越界会拒绝。
- 文件页操作如果目标是已注册外部仓库，只执行真实文件操作和外部项目边界校验，不创建工作空间隐藏版本点。
- 没有实际变更时返回 `hash: null`，不制造空提交。
