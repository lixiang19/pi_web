# 工作空间隐藏版本管理

## 职责

工作空间隐藏版本管理为用户可见工作空间内容创建保存点。底层使用 isomorphic-git，但 UI 和业务文案不暴露 Git 概念。

## 存储位置

- 工作树：当前工作空间目录。
- gitdir：`~/.pi/ridge-workspace-git/<workspace-hash>`。
- 内置 ignore 包含 `.DS_Store`、`.ridge`、`.git`、`node_modules`、构建产物和缓存目录。

## 保存点入口

- `workspace-version.ts`
  - `getWorkspaceVersionContext(workspaceDir)`：解析隐藏 gitdir。
  - `commitWorkspaceVersionPoint({ workspaceDir, files, message })`：校验文件仍在工作空间内，过滤隐藏版本 ignore 路径，提交指定文件。
- `iso-git-service.ts`
  - `ensureInit` 初始化隐藏仓库并写 exclude。
  - `commit` 支持新增、修改和删除文件。
  - `isWorkspaceVersionIgnoredPath` 是隐藏版本 ignore 的统一判断。
  - `getStatus` 返回状态时过滤 `.DS_Store`、`.ridge`、依赖目录、构建产物和缓存目录。
  - `getFileDiff` 读取隐藏版本 HEAD 与当前工作树，生成工作空间版本 diff；ignore 路径返回空 diff。
- `routes/workspace-version.ts`
  - `GET /api/workspace/version/status`：读取隐藏版本状态。
  - `GET /api/workspace/version/diff`：读取单文件 diff。
  - `POST /api/workspace/version/commit`：提交用户手动版本点。

## 当前触发点

- `/api/files/upload` 上传文件后创建保存点。
- `/api/files/entries` 新建文件或目录后尝试创建保存点；空目录无 Git 内容时返回空保存点。
- `/api/files/entries/path` 移动或重命名后提交旧路径和新路径。
- `DELETE /api/files/entries` 删除后提交删除路径。
- `PUT /api/files/content` 保存 Markdown 或空间 `index.html` 后创建保存点。

## 边界

- `.DS_Store`、`.ridge`、依赖目录、构建产物、RAG 缓存、Kuzu 图谱、运行缓存、临时附件不进入隐藏版本。
- 文件路径必须在工作空间内；realpath 越界会拒绝。
- 文件页操作如果目标是已注册外部仓库，只执行真实文件操作和外部项目边界校验，不创建工作空间隐藏版本点。
- 没有实际变更时返回 `hash: null`，不制造空提交。
- Git 面板只对真实 `.git` 可用；隐藏版本不能通过 `/api/git/*` 或 Git 面板暴露。
