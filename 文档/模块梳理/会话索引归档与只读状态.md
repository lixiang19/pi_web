# 会话索引归档与只读状态

## 职责

- `session_index`（`sessions` 表）是 ridge 产品层会话真源索引，保存会话元信息；消息正文仍以 Pi 原始 session 文件为真源。
- 归档 API 对会话树同时设置 `archived=true` 和 `readonly=true`；取消归档时 `archived=false` 且 `readonly` 恢复为 `false`（普通会话）。
- 只读规则：前端 composer 禁用、编辑/重试禁用；后端对 `archived` 或 `readonly` 会话发送消息返回 403。
- 设备离线时，对应桌面项目会话不可打开（hydrate 返回 403）。
- 同一个会话在标签系统中只能存在一个标签。

## 关键文件

- `packages/server/src/session-indexer.ts` — 索引 CRUD、列表查询（排除归档）。
- `packages/server/src/session-metadata.ts` — `setArchived` 同时更新 `readonly`。
- `packages/server/src/db/index.ts` — `sessions` 表列修复（含 `readonly`）。
- `packages/server/src/index.ts` — 归档 API、创建会话分叉守卫（task session 禁止）。
- `packages/server/src/session-payload.ts` — `assertSessionAvailable` 离线桌面设备守卫。
- `packages/web/src/lib/session-sidebar.ts` — 列表排除归档。
- `packages/web/src/composables/usePerSessionChat.ts` — `isReadonly` 计算属性、fork 禁用。

## 接口契约

- `GET /api/sessions` — 返回非归档会话摘要（含 `readonly` 字段）。
- `POST /api/sessions/:id/archive` — 输入 `{ archived: boolean }`，输出 `{ ok, sessionIds }`，同时设置 `readonly`。
- `POST /api/sessions` — 带 `parentSessionId` 时，若父会话关联 `taskId` 则返回 403。
- `POST /api/sessions/:id/messages` — 若会话 `archived` 或 `readonly` 则 403。
- `GET /api/sessions/:id/hydrate` — 桌面设备离线时 403。
