# 项目注册与设备绑定

## 职责边界

- **project-service.ts**：内部项目创建、外部仓库注册、GitHub 克隆、归档/删除。
- **routes/projects.ts**：REST API 路由层，参数校验，调用 service。
- **storage/index.ts**：projects 表查询（已存在，新增 deviceId/deviceName/isOnline 关联）。

## 数据结构

### projects 表（已存在，新增约束）
- `project_id` TEXT PK
- `name` TEXT
- `path` TEXT UNIQUE（同设备同路径唯一通过 `idx_projects_device_path` 实现）
- `project_type` TEXT — 'internal' | 'external'
- `external_origin` TEXT — 'folder' | 'github' | null
- `device_id` TEXT — 外部仓库绑定的桌面设备
- `archived_at` INTEGER
- `workspace_path` TEXT

### 内部项目
- 目录：`~/ridge-workspace/项目/<name>`
- projectType='internal', externalOrigin=null, device_id=null
- 受工作空间 RAG 和隐藏版本管理覆盖
- **不作为 pi cwd**（会话创建时拒绝）

### 外部仓库
- projectType='external', externalOrigin='folder' | 'github'
- device_id 可为 null（服务器本地仓库）或具体设备 ID（桌面本机仓库）
- **可作为 pi cwd**
- 会话启动时检查设备在线状态

## API

- `POST /api/workspace/projects/internal` — { name }
- `POST /api/workspace/projects/external` — { path, deviceId? }
- `POST /api/workspace/projects/github` — { url, deviceId? }
- `PATCH /api/workspace/projects/:id` — { archived?: boolean }
- `DELETE /api/workspace/projects/:id` — 删除注册记录（有会话时 409）

### 外部项目设备校验

- `deviceId` 为空时按服务器本机项目处理。
- `deviceId` 非 `server` 时必须存在且 `device_type='desktop'`。
- 设备不存在返回 404，设备类型不是桌面端返回 400，设备离线或桌面端路径校验不可达返回 409。

## 删除规则

1. 查询 session_index 确认无关联会话
2. 仅删除 projects 表记录
3. 不删除文件系统

## 归档规则

- PATCH archived=true 写入 archived_at
- 归档项目从普通列表过滤（由前端或查询参数控制）
- 归档项目下会话只读（session_index.archived 联动）
