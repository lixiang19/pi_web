# 全局搜索资产导航器模块契约

> 状态：**已实现**
> 对应功能：Task 24

## 1. 模块定位

全局搜索是 ridge 的确定性资产导航器，不启动 AI，不生成回答。

`WorkspaceSearchView.vue` 是工作台内的搜索标签组件；当前主左侧固定入口不再展示「搜索」，搜索 API 与组件保留给内部调用或后续重新设计入口使用。

## 2. API

```txt
GET /api/workspace/search?q=&type=&project=&time=&dir=&limit=&sort=
GET /api/workspace/knowledge/diagnostics
```

参数：

- `q`：必填关键词。
- `type`：逗号分隔类型，省略表示全部。
- `project`：项目 ID 筛选。
- `time`：`all | today | week | month`。
- `dir`：工作空间相对目录筛选。
- `sort`：`relevance | updated`。

返回：

```ts
{
  query: string;
  results: WorkspaceSearchResult[];
  groups: Array<{ type: WorkspaceSearchResultType; count: number }>;
  indexStatus: { pending: number; indexed: number; indexFailed: number };
}
```

诊断返回：

```ts
{
  rag: { pending; indexed; indexFailed; latestIndexedAt; failedTargets };
  memory: { memoryPath; exists; injected; dailyCount; latestDailyAt };
  wiki: { indexPath; exists; injected; indexStatus };
  graph: { graphPath; schemaExists; databaseExists; correctionsEndpoint };
  mcp: { endpoint; boundary; tools };
  backgroundJobs: { byStatus; byType; recentFailures };
  notifications: { unhandled; ragFailures; backgroundFailures };
}
```

## 3. 结果类型

第一版类型：

- `file`
- `task`
- `milestone`
- `project`
- `session`
- `memory`
- `wiki`
- `space`
- `rag`

`graph` 不在当前搜索类型中暴露。图谱节点依赖 Task 28 的 Kuzu 图谱存储和抽取完成后再接入，禁止在没有真实图谱数据源时返回占位结果。

## 4. 数据源

搜索聚合多个确定性来源：

- 工作空间可见文件名、路径和文本内容；
- `workspace_tasks`；
- `workspace_milestones`；
- `projects` 注册信息；
- `session_index` 会话索引标题；
- `记忆/`；
- `Wiki/`；
- `空间/*/index.html`；
- `search_chunks` RAG 片段。

RAG 片段包含 Markdown、空间 HTML、转换后的 Markdown 产物，以及图片原文件的 Qwen3-VL 视觉 embedding。图片结果使用原文件路径作为 `sourcePath`，点击后打开图片文件标签；若图片同时产生 OCR Markdown，OCR 产物会作为独立文本 RAG 结果参与召回。

项目筛选会根据 `projects.path` 推导项目内 file/RAG 的 `projectId`，因此项目内 Markdown 文件和 RAG chunk 不会因为自身行没有 `project_id` 字段而被过滤掉。

不搜索：

- `.ridge`；
- `.originals`；
- `.git`；
- `node_modules`；
- 外部项目文件内容；
- Pi 会话正文。

## 5. 前端行为

`WorkspaceSearchView.vue` 使用 shadcn-vue 的 `Input`、`Button`、`ScrollArea`。

用户界面只暴露一个搜索框和扁平结果列表，不展示类型筛选、类型分组或 `RAG`、`Wiki`、`记忆`、`MCP` 等内部知识系统名词。

无查询时显示极简空态，提示用户直接输入关键词。知识诊断仍通过 `GET /api/workspace/knowledge/diagnostics` 读取真实状态，但默认只折叠成用户可理解的异常提示，例如「1 个内容暂时搜不到」。失败目标可通过「重新整理」调用 `POST /api/workspace/rag/refresh`，并重新加载诊断状态。

搜索结果按后端综合排序扁平展示，不在 UI 上按类型分组。`type` 字段只作为内部打开路由使用。

结果点击行为：

- 文件、记忆、Wiki、RAG：打开文件标签；
- 任务、里程碑：打开任务页；
- 会话：打开会话标签；
- 空间：打开空间预览标签；
- 项目：打开项目主页标签；
- 普通结果不展示手动刷新索引入口。
