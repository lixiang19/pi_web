# 定时会话自动化模块 Codemap

## Responsibility

- 负责把“自动化”收敛成定时创建普通 AI 会话的规则系统。
- 负责保存定时会话规则，并由 Node 服务在到点后创建普通会话、发送配置好的用户消息。
- 负责提供自动化页的规则管理接口。
- 不负责运行记录、工作流编排、审批中心、自动化专属消息协议。

## Design

### Architecture Pattern

```text
--------------------------+
| AutomationsPage.vue     |
| - 规则列表              |
| - 规则编辑器            |
+------------+-------------+
             |
             v
+--------------------------+
| /api/automations         |
| - CRUD 规则              |
| - 立即创建一次会话       |
+------------+-------------+
             |
             v
+--------------------------+
| Node AutomationScheduler |
| - 单个 setTimeout        |
| - 查询 due rules         |
| - 计算 nextRunAt         |
+------------+-------------+
             |
             v
+--------------------------+
| 普通会话链路             |
| createSessionRecord      |
| applySessionAgentSelection|
| session.prompt           |
+--------------------------+
```

### Key Abstractions

- `AutomationRule`
  - 定时会话规则。
  - 字段包含 `name/enabled/cwd/agent/model/thinkingLevel/schedule/prompt/nextRunAt`。
  - 只描述“下次什么时候创建哪种会话并发送什么消息”。

- `AutomationSchedule`
  - 首版只支持 `daily`、`weekly`、`interval`。
  - 不暴露 cron 表达式，避免把最基础的定时会话做成调度器配置页。

- `AutomationScheduler`
  - 服务端常驻模块。
  - 只维护一个全局 timer。
  - 到点后执行所有 due 规则，每条规则创建一条普通会话。

## Flow

### Flow 1: 保存规则

```text
用户编辑规则
  -> PATCH/POST /api/automations
  -> 写入 automation_rules
  -> scheduler reschedule
```

### Flow 2: 定时触发

```text
Node timer 到点
  -> 查询 enabled 且 next_run_at <= now 的规则
  -> 创建普通会话
  -> 应用 agent/model/thinking
  -> 发送 prompt
  -> 更新 next_run_at
  -> scheduler reschedule
```

### Flow 3: 立即创建

```text
用户点击立即创建
  -> POST /api/automations/:id/run
  -> 创建普通会话并发送 prompt
  -> 前端刷新会话列表
  -> 打开新会话
```

## Boundary

- 自动化页不展示运行记录，因为普通会话列表就是结果入口。
- 自动化调度放在 Node 服务层，Tauri 只负责打包。
- 前端不伪造规则持久化，规则真源是 `ridge.db`。
- 会话执行仍使用 Pi runtime 的真实会话链路。
