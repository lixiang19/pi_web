# 文件处理状态模块契约

## 1. 职责边界

- `file_processing_status` 表负责记录工作空间内每个文件的处理生命周期。
- 只有**文件**有状态；目录没有状态。
- 状态生命周期是严格的有限状态机，只允许预定义的状态迁移。

## 2. 状态定义（`file_processing_status.status`）

| 状态值 | 含义 | 可迁移到 |
|--------|------|----------|
| `pending` | 待处理 | `converting` |
| `converting` | 转换中 | `converted`, `convert_failed` |
| `converted` | 已转换 | `indexed`, `index_failed` |
| `indexed` | 已索引 | （终态） |
| `convert_failed` | 转换失败 | 只能由 `POST /retry` 回到 `pending` |
| `index_failed` | 索引失败 | 只能由 `POST /retry` 回到 `pending` |

- `PATCH /api/workspace/files/status` 只允许上表中的正向迁移；`convert_failed`/`index_failed` 回到 `pending` 只能通过 `POST /retry`。
- 进入 `convert_failed` 或 `index_failed` 时，必须提供 `error` 字符串，并自动生成一条 `notification_events` 记录。

## 3. 路径存储规范

- `file_path` 字段**永远以 POSIX 形式存储**（`/` 分隔）。
- `workspace_path` 同样以 POSIX 形式存储。
- 所有服务端路由在查、写、更新 `file_processing_status` 时，必须先对本地文件系统路径调用 `toPosixPath(...)`，再操作数据库。
- `toPosixPath` 实现：`value.replace(/\\/g, '/')` —— 统一替换字符串中的反斜杠，不依赖运行时 `path.sep`。
- **入口层拒绝**：任何外部传入的 `path` / `root` / `directory` 参数若含 `\`，`validateSafePath()` 立即抛出 400。反斜杠路径**禁止**进入后续文件系统或 DB 操作，以防止 `toPosixPath` 将其折叠为 `/` 后与真实子目录语义冲突。

## 4. 文件系统操作同步契约

### 4.1 上传（`POST /api/files/upload`）

- Zod schema 中 `root`、`directory` 字段增加 `.refine(v => !v.includes('\\'))` 校验；含 `\` 的请求在 schema 阶段即返回 400。
- 上传完成后，对非 `.ridge` 目录下的文件，自动 `INSERT OR IGNORE` 一条 `pending` 记录。
- `file_path` 使用 `entry.path`（已是 POSIX）。

### 4.2 删除（`DELETE /api/files/entries`）

- `fileContentQuerySchema` 中 `path` / `root` 增加 backslash 拒绝 refine。
- `fileTreeQuerySchema` 中 `path` / `root` 增加 backslash 拒绝 refine。
- 删除成功后，同步清理 `file_processing_status`：
  - 精确匹配 `file_path = trashedPath`（POSIX）。
  - 前缀匹配 `file_path LIKE escapedPrefix% ESCAPE '\'`，处理目录删除时批量删除内部文件。
  - `LIKE` 特殊字符（`%`、`_`）必须转义，防止误匹配。

### 4.3 移动/重命名（`PATCH /api/files/entries/path`）

- move 成功后，必须同步更新 `file_processing_status`：
  - **文件**：`UPDATE file_processing_status SET file_path = newPosixPath WHERE file_path = oldPosixPath`
  - **目录**：前缀批量更新
    ```sql
    UPDATE file_processing_status
    SET file_path = newPrefix || substr(file_path, length(oldPrefix) + 1), updated_at = ?
    WHERE file_path LIKE escapedPrefix% ESCAPE '\'
    ```
    - `escapedPrefix` 必须对 `oldPrefix + "/"` 中的 `%`、`_` 进行 `\` 转义。
    - 注意：反斜杠本身在 SQL 字符串中需要双重转义（`'\'`），而 `ESCAPE '\'` 中的 `'\'` 在 JavaScript 模板字符串中写作 `'\\'`。

## 5. API 路由与 DB 查询一致性

- `GET /api/workspace/files/tree`：读取 `file_processing_status` 时，对 `row.file_path` 调用 `toPosixPath` 后与 `entry.path`（已是 POSIX）匹配。
- `PATCH /api/workspace/files/status`：对 `targetPath` 调用 `toPosixPath` 后查/写 DB。
- `POST /api/workspace/files/retry`：同上。重试必须要求后台任务队列可用；队列不可用时返回 503，保持 `convert_failed` / `index_failed` 原状态不变。
- `POST /api/workspace/files/retry` 与 `POST /api/workspace/files/convert` 的状态恢复、旧任务取消和新 `file.convert` 入队必须在同一 SQLite transaction 中提交，避免状态显示为 `pending` 但没有可执行后台任务。

## 6. 前端可访问性

- 文件行使用 `div role="button" tabindex="0"`，支持键盘 `Enter`/`Space` 打开。
- 重试按钮（内部 `<button>`）阻止事件冒泡，避免误触发文件打开。
- 目录行保持原生 `<button>`，同样支持键盘交互。

## 7. 测试覆盖要求

- 服务端必须覆盖反斜杠路径拒绝场景：入口层（Zod schema / `validateSafePath`）对含 `\` 的请求返回 400，禁止进入文件系统或 DB。
- 服务端必须覆盖 retry 队列缺失返回 503 且不改写失败状态。
- 必须验证 move/rename 后状态同步（文件精确更新、目录前缀更新、LIKE 特殊字符不误匹配）。
- 必须验证 `%`、`_` 源目录的移动不会误更新其他目录。
- 前端组件测试必须覆盖 `Enter`/`Space` 键盘事件及重试按钮不冒泡。
- e2e 测试 cleanup 必须记录真实 `filePath`，逐个删除并断言成功，禁止静默忽略。
