# 工作空间主页与会话创建

## 职责

工作空间主页是全局 AI 输入入口，不是固定会话。

- 用户输入前显示输入框、快捷动作和轻量摘要。
- 用户提交后，在**同一标签内**原地替换为新的工作空间会话。
- 新会话立即加入左侧“工作空间会话”列表并保持选中。
- 临时模型/思考强度/Agent 只影响本次/当前会话后续消息，不写回全局设置。

## 入口

- `packages/web/src/pages/WorkspacePage.vue`
  - `handleHomeSubmit(homeTabId, payload)`：创建 Pi 会话和 ridge 会话索引，成功后 replaceTab。如有附件，先调用 `uploadSessionAttachments` 上传，得到 `attachmentIds` 后传入 `createChatTab`。
- `packages/web/src/components/workspace/HomePage.vue`
  - 输入框、快捷动作、附件入口（`<input type="file">`）、发送按钮。附件以 `File[]` 形式随 submit payload 传出。
- `packages/web/src/components/workspace/WorkspaceChatTab.vue`
  - 接收 `initialPrompt`、`initialModel`、`initialAgent`、`initialThinkingLevel`、`initialAttachmentIds`；挂载后自动发送首条消息，并将 `initialAttachmentIds` 透传给 `chat.submit()`。
- `packages/web/src/composables/useSplitPanes.ts`
  - `createChatTab(sessionId, title, options)` 支持 `initialThinkingLevel` 和 `initialAttachmentIds`；`SplitTabItem` 序列化保留两者。
- `packages/web/src/composables/usePerSessionChat.ts`
  - `submit(attachmentIds?)` 发送消息时将 `attachmentIds` 附加到 `SendMessagePayload`。
  - 临时选择器 setter，不写全局 settingsStore。
- `packages/server/src/index.ts`
  - `POST /api/sessions`：创建 Pi 会话并 `upsertIndexedSessionRecord` 建立索引。
  - `POST /api/sessions/:id/messages`：`messageSchema` 支持 `attachmentIds`；校验归属后调用 `buildAttachmentContext` 拼接到 prompt。
- `packages/server/src/session-attachments.ts`
  - `storeSessionAttachment`：安全文件名、sha256、落盘到 `~/.pi/session-attachments/<sessionId>/`、DB 写入。
  - `validateAttachmentIds` / `buildAttachmentContext`：文本附件读取追加到 prompt（单文件上限 64KB，总量上限 192KB）；二进制附件只发引用。
  - `createSessionAttachmentsRouter()`：挂载 `POST /` 和 `GET /` 路由。

## 状态机

- `empty_home`：主页打开，不绑定任何 Pi 会话。
- `sending_first_message`：用户已提交，调用 `createSessionApi`。
- `conversation_created`：服务端返回 sessionId，原地 replaceTab，自动发送首条消息。
- `failed`：`createSessionApi` 失败，保留输入，恢复按钮，toast 错误。

## 边界

- 打开主页不调用 createSession，不产生 session index。
- 提交非空文本才调用 createSessionApi；空输入不触发 submit。
- createSession payload 必须包含：cwd、title（首句前24字）、model、agent（NO_AGENT_VALUE 转为 null）、thinkingLevel。
- replaceTab 后的 chat tab 必须携带 `initialPrompt`、`initialModel`、`initialAgent`、`initialThinkingLevel`、`initialAttachmentIds`。
- `NO_AGENT_VALUE` 传入时不能被当成真实 agent 导致 send 失败；WorkspaceChatTab 应在 `initialAgent === NO_AGENT_VALUE` 时不写入 composer.selectedAgent。
- 创建成功后必须调用 `core.refreshSessions()` 和 `core.refreshSessionContexts()`（或等效方式），让左侧工作空间会话列表立即出现新会话。
- 失败时：不替换 home tab、保留输入、发送按钮恢复可用、显示错误 toast。若附件上传失败同样不 replaceTab，并调用 `deleteSession` 清理刚创建的会话，避免孤儿会话。
- 临时模型/思考强度/Agent 切换只更新当前 composer/会话元数据，禁止写回 `useSettingsStore` 全局默认。
- 附件落盘到 `~/.pi/session-attachments/<sessionId>/`，不写入知识库/闪念/正式 `附件/` 目录。
- 文本附件内容读取后追加到 prompt（单文件上限 64KB，总量上限 192KB）；二进制附件只发文件名/MIME/大小引用。
- 快捷动作只填入输入框，不直接发送。

## 数据流

1. HomePage `handleSubmit` → emit("submit", payload 含 `attachments?: File[]`) → WorkspacePage `handleHomeSubmit`
2. `handleHomeSubmit` 查找 homeTab cwd → 调用 `createSessionApi` → 得到 sessionId
3. 如有附件：调用 `uploadSessionAttachments(sessionId, files)` → 得到 `attachmentIds`
4. `createChatTab(sessionId, title, { initialPrompt, initialModel, initialAgent, initialThinkingLevel, initialAttachmentIds })`
5. `splitPanes.replaceTab(homeTabId, chatTab)`
6. `WorkspaceChatTab` onMounted → `chat.loadSession(sessionId)` → 设置 composer 临时选择 → `chat.submit(initialAttachmentIds)` 自动发送首条消息（attachmentIds 拼入 SendMessagePayload）
7. `handleHomeSubmit` 成功后 `core.refreshSessions()` + `core.refreshSessionContexts()`

## 测试覆盖点

- HomePage：快捷动作只填入不提交；附件入口选择文件后显示待附加文件；提交 payload 含 attachments；发送中时禁用；不因 submit 立即清空 draft。
- WorkspacePage：打开主页不 createSession；submit 成功 createSession payload 正确、refreshSessions/Contexts 被调用、replaceTab 参数含 initialAttachmentIds；失败不 replaceTab 且 HomePage 保留输入/错误提示；NO_AGENT_VALUE 转 null；附件上传失败不 replaceTab；附件上传成功带 initialAttachmentIds。
- WorkspaceChatTab：initialThinkingLevel 生效；NO_AGENT_VALUE 不作为真实 agent 发送；附件随首条消息提交。
- usePerSessionChat：选择模型/agent/thinking 不写全局 settings；submit 支持 attachmentIds 并拼入 payload。
- 后端 session-attachments：store/write/validate/read/buildContext 单元测试覆盖；文件名安全化；router 上传/列表测试；文本/二进制区分。
