# 中间对话区模块 Codemap

## Responsibility

- 负责当前会话消息流的展示、渲染与滚动管理，承担多会话缓存切换的核心数据层
- 支持新会话草稿态与正式会话的无缝过渡，管理父子会话回退导航与上下文继承
- 实现历史消息的限窗加载与向上扩窗机制，承载会话级草稿持久化与恢复能力
- 提供回到底部/脱离底部的滚动状态管理，处理输入框工作台的完整编排逻辑
- **完整支持富内容渲染：文本、思考过程、工具调用、工具结果、图片**
- 不处理阻塞式交互卡片、文件mention与附件协议、虚拟列表渲染、语音输入等高级能力

### Key Abstractions

#### 1. ContentBlock 联合类型

```ts
// types.ts - 对齐 Pi SDK 原生类型
type ContentBlock =
  | { type: 'text'; text: string }
  | { type: 'thinking'; thinking: string; redacted?: boolean }
  | { type: 'image'; data: string; mimeType: string }
  | { type: 'toolCall'; id: string; name: string; arguments: Record<string, unknown> }
  | { type: 'toolResult'; toolCallId: string; toolName: string; content: (TextContent | ImageContent)[]; isError: boolean }
```

- 代表消息中的单个内容单元，支持完整 Pi SDK 内容类型
- 被 `ChatMessage.contentBlocks` 数组持有
- 由 `extractContentBlocks()` 从 StreamEvent 转换，由 SSE 增量事件更新
- 关联文件：types.ts:30-68, usePiChat.ts:74-116

#### 2. ChatMessage 消息实体

```ts
// types.ts
interface ChatMessage {
  id: string
  role: 'user' | 'assistant' | 'system' | 'tool'
  text: string              // 纯文本摘要（用于快速预览）
  contentBlocks: ContentBlock[]  // 完整富内容
  createdAt: number
  pending?: boolean
}
```

- 代表会话中的单条消息，包含完整内容块数组
- `text` 字段提供向后兼容的纯文本摘要
- `contentBlocks` 支持思考、工具调用、图片等富内容
- 关联文件：types.ts:70-78

#### 3. 流式消息构建状态

```ts
// usePiChat.ts:142-145
let currentAssistantMessageId = ''
let currentAccumulatedBlocks: ContentBlock[] = []
let currentPendingToolCall: ToolCallContentBlock | null = null
```

- 代表 SSE 流式接收过程中的增量构建状态
- `currentAccumulatedBlocks` 累积所有内容块（text/thinking/toolCall）
- `currentPendingToolCall` 跟踪正在接收参数的 tool call
- 由 `connectStream()` 中各种事件处理器（text_delta/thinking_delta/toolcall_delta）更新
- 关联文件：usePiChat.ts:537-700 (connectStream 方法)

#### 4. 会话缓存桶 (Session Cache Entry)

```ts
// usePiChat.ts
type CachedSessionEntry = {
  snapshot: SessionSnapshot  // 包含完整 ChatMessage[]
  hydratedAt: number
}

interface SessionSnapshot extends SessionSummary {
  messages: ChatMessage[]   // 现在包含 contentBlocks
  historyMeta: SessionHistoryMeta
}
```

- 代表每个会话的完整消息状态（含富内容）与时间窗口元信息
- 被 `sessionCache` (Record<string, CachedSessionEntry>) 持有
- 由 `hydrateSession()` 创建，由 SSE 事件增量更新
- 关联文件：usePiChat.ts:26-29, types.ts:80-85

### Design Patterns

#### 1. 分桶缓存 + 去重加载 (Request Deduplication)

```ts
// usePiChat.ts:277-306
const loadSessionInFlightById = new Map<string, Promise<SessionSnapshot>>()
const loadSessionRequestSeqById = new Map<string, number>()

async function hydrateSession(sessionId: string) {
  // 命中缓存直接返回（现在缓存包含完整 contentBlocks）
  const cached = getCachedSessionSnapshot(sessionId)
  if (cached) return cached
  
  // 复用进行中的请求
  const inFlight = loadSessionInFlightById.get(sessionId)
  if (inFlight) return inFlight
  // ...
}
```

- 防止并发加载同一session产生重复请求
- 缓存现在包含完整的 ContentBlock 数组，不只是纯文本
- 应用于：usePiChat.ts:277-306

#### 2. ContentBlock 累积与增量更新

```ts
// usePiChat.ts:656-740 - SSE 事件处理
case 'text_delta':
  if (newBlocks[contentIndex]?.type === 'text') {
    newBlocks[contentIndex] = {
      ...newBlocks[contentIndex],
      text: (newBlocks[contentIndex] as TextContentBlock).text + delta,
    }
  }
  break
case 'thinking_delta':
  // 累积思考内容...
case 'toolcall_delta':
  // 累积工具调用参数...
```

- SSE 流式事件直接更新 ContentBlock 数组
- 每种 delta 类型更新对应的内容块
- 关联文件：usePiChat.ts:656-740

#### 3. 纯文本摘要提取

```ts
// usePiChat.ts:98-103
const extractTextFromBlocks = (blocks: ContentBlock[]): string => {
  return blocks
    .filter((b): b is TextContentBlock => b.type === 'text')
    .map((b) => b.text)
    .join('')
}
```

- 从 ContentBlock 数组提取纯文本用于预览
- 保持 `ChatMessage.text` 字段向后兼容
- 关联文件：usePiChat.ts:98-103

#### 4. 组件化内容渲染

```vue
<!-- ChatMessageItem.vue -->
<template v-for="(block, index) in message.contentBlocks" :key="index">
  <p v-if="block.type === 'text'">{{ block.text }}</p>
  <img v-else-if="block.type === 'image'" :src="..." />
  <ThinkingCard v-else-if="block.type === 'thinking'" :block="block" />
  <ToolCallCard v-else-if="block.type === 'toolCall'" :block="block" />
  <ToolCallCard v-else-if="block.type === 'toolResult'" :block="block" is-result />
</template>
```

- 每种 ContentBlock 类型对应专用组件
- ThinkingCard 支持折叠/展开思考过程
- ToolCallCard 显示工具调用参数和执行结果
- 关联文件：ChatMessageItem.vue, ThinkingCard.vue, ToolCallCard.vue

## Flow

### Flow 1: SSE 流式富内容接收流程

```
SSE onmessage
    │
    ▼
解析 StreamEvent
    │
    ├─► type: 'message_start'
    │   currentAssistantMessageId = createLocalId()
    │   currentAccumulatedBlocks = extractContentBlocks(payload.message)
    │   appendMessageToSession({ role: 'assistant', contentBlocks, pending: true })
    │
    ├─► type: 'message_update'
    │   │
    │   ├─► assistantMessageEvent.type: 'text_delta'
    │   │   patchSessionSnapshot -> 更新对应 text block
    │   │
    │   ├─► assistantMessageEvent.type: 'thinking_delta'
    │   │   patchSessionSnapshot -> 更新/创建 thinking block
    │   │
    │   ├─► assistantMessageEvent.type: 'toolcall_start'
    │   │   currentPendingToolCall = { type: 'toolCall', ... }
    │   │   添加到 contentBlocks
    │   │
    │   ├─► assistantMessageEvent.type: 'toolcall_delta'
    │   │   累积 JSON 参数到 currentPendingToolCall.arguments
    │   │
    │   └─► assistantMessageEvent.type: 'toolcall_end'
    │       currentPendingToolCall = null
    │
    └─► type: 'message_end'
        finalBlocks = extractContentBlocks(payload.message)
        patchSessionSnapshot -> 设置 pending: false, contentBlocks: finalBlocks
        重置 currentAssistantMessageId = '', currentAccumulatedBlocks = []
```

关键特性：
- 实时累积各种内容块（text/thinking/toolCall）
- 增量更新避免全量重渲染
- 工具调用参数流式累积

### Flow 2: 消息渲染流程

```
WorkbenchMessageStream.vue 消息列表
    │
    ▼
v-for="message in messages" :key="message.id"
    │
    ▼
ChatMessageItem :message="message"
    │
    ▼
根据 message.role 确定布局（user右/assistant左）
    │
    ▼
v-for="(block, index) in message.contentBlocks"
    │
    ├─► block.type === 'text' -> 渲染纯文本
    ├─► block.type === 'image' -> 渲染图片
    ├─► block.type === 'thinking' -> ThinkingCard（可折叠）
    ├─► block.type === 'toolCall' -> ToolCallCard（显示调用参数）
    └─► block.type === 'toolResult' -> ToolCallCard（显示执行结果）
```

关键特性：
- 组件化渲染每种内容类型
- ThinkingCard 支持展开/收起
- ToolCallCard 区分调用和结果状态

### Flow 3: 历史消息加载与扩窗

```
用户点击"加载更早消息"
    │
    ▼
loadEarlierMessages()
    │
    ▼
GET /api/sessions/:id?limit=nextLimit
    │
    ▼
syncSessions(newSnapshot)
    │
    ▼
newSnapshot.messages 包含完整 ChatMessage[]
    │   每条消息包含 contentBlocks 数组
    ▼
Vue nextTick 后恢复滚动位置
    │
    ▼
UI 渲染新增消息（使用 ChatMessageItem 组件）
```

关键特性：
- 扩窗加载保留完整富内容
- 与初始加载使用相同数据格式

### 检查机制
**双 Agent 独立检查模式**：
- Agent A（功能验收）：验证消息列表、发送链路、布局稳定性、Pi SDK 边界
- Agent B（规范安全）：验证滚动性能、主题一致性、安全性、代码质量
- 互不可见对方结果，避免自查盲区
