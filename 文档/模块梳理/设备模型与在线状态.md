# 设备模型与在线状态

## 职责边界

- **devices.ts**：设备 CRUD、心跳、离线清理、token 生成、token hash 存储、**共享 token 校验 `validateDeviceToken` / `authenticateDeviceToken`**。
- **routes/devices.ts**：REST API 路由层，**所有敏感操作带 `requireDeviceToken` 中间件**。
- **desktop-bridge.ts**：**WebSocket 长连接管理**（ping/pong 心跳、请求转发、SSE 广播）。
- **index.ts**：启动时 `ensureServerDevice()` 创建内置服务器设备；WebSocket upgrade handler 挂载设备 ws 路径。

## 数据结构

### devices 表（migration v16 新增 token_hash 字段）
- `device_id` TEXT PK
- `name` TEXT
- `device_type` TEXT — 'server' | 'desktop' 等设备类型
- `token` TEXT — 已废弃，不再写入明文设备令牌
- `token_hash` TEXT — 设备注册时生成明文 token，仅在响应中返回；数据库只保存 SHA-256 hash
- `status` TEXT — 'online' | 'offline'
- `capabilities_json` TEXT — JSON { skill_mac: true, skill_chrome: true, ... }
- `last_seen_at` INTEGER
- `created_at` / `updated_at` INTEGER

### device_bundle_acks 表（migration v14）
- `device_id` TEXT PK
- `bundle_id` TEXT NOT NULL
- `acked_at` INTEGER
- `created_at` INTEGER
- 同设备多次 ack 覆盖更新（`ON CONFLICT(device_id) DO UPDATE`）

## 在线状态机制

- 桌面设备注册时 status='online'，同时轮换 `token_hash`
- 心跳双通道：
  1. **REST** `POST /api/devices/heartbeat`（需 token）刷新 `last_seen_at`
  2. **WebSocket** `ws://host/api/devices/:deviceId/ws?token=xxx`，服务器每 30s 发送 `ws.ping()`，桌面端回复 `pong` 时同时更新 DB
- `sweepOfflineDevices()`：查询 `last_seen_at < now - 60s` 的桌面设备，设为 offline
- 每次 `GET /api/devices` 前自动执行 sweep
- WebSocket 连接断开或 pong 超时自动清理 connections map

## 设备能力与 Skill 分发

capabilities_json 中的 key 决定设备能接收哪些带 tag 的 Skill：
- `skill_mac: true` → 接收 `[mac]` tag 的 Skill
- `skill_chrome: true` → 接收 `[chrome]` tag 的 Skill
- 无 tag 的通用 Skill 所有设备都接收
- Skill 路径中**任意目录名**含 `[tag]` 即视为设备专属；所有 tag 必须全部匹配才保留

## API

- `GET /api/devices` — 列表（含 sweep），无需 token
- `POST /api/devices/register` — { deviceId, name, deviceType, capabilities? }，返回含 token
- `POST /api/devices/heartbeat` — { deviceId, **token** }
- `POST /api/devices/:deviceId/rename` — { name, **token** }
- `GET /api/devices/:deviceId/bundle?token=xxx` — 获取 runtime bundle（query token）
- `POST /api/devices/:deviceId/bundle/ack` — { bundleId, **token** }，持久化 ack 到 DB

## WebSocket 协议

```
ws://host/api/devices/:deviceId/ws?token=<device_token>
```

- 连接前必须 query 校验 token，失败返回 401
- 连接后服务器每 30s 发送 `ping`
- 桌面端回复 `pong`，服务器更新 `lastPingAt` 并刷新 DB `last_seen_at`
- 桌面端可向服务器发送 `sse_event` JSON 消息，服务器通过 `forwardDesktopSseEvent` 广播给 SSE 监听者
- 服务器可通过 `forwardRunRequestToDesktop` 发送请求，桌面端按 `requestId` 回复

## 调度规则

- Web 请求桌面项目会话时，服务器校验设备在线状态
- 离线设备的项目禁止新建/继续会话（409）
- 长连接用于：实时状态、请求转发、SSE 事件回传
