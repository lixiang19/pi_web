# 工作空间模块梳理

## 概述

工作空间（`/workspace`）是 ridge 的个人内容平台，替代原 `/notes` 笔记页面。核心模型：文件树 + 标签页 + 数据库。默认用户内容目录为 `~/ridge-workspace`，当前工作空间路径记录在 `~/.pi/ridge.db`。

## 路由

- `router/index.ts`：`/` → `WorkspacePage`（工作空间为唯一根路由）
- `router/index.ts`：`/login` → `LoginPage`，未登录访问工作空间会被路由守卫跳转到登录页
- 旧的 `chat/terminal/automations/settings/search/spaces/datasets` 独立页面路由已全部删除
- 终端/自动化/设置：点击顶部系统菜单后打开为工作空间标签页
- 旧 `useWorkbenchPrimaryNavigation` composable 和 `WorkbenchSidebar` 侧边栏已删除
- 旧聊天/项目导航入口已从侧栏移除，AI 会话后续从工作空间主页标签启动

## 关键数据流

1. **仪表盘**：`useDashboard` → 调用 `getRecentFiles` + `getNoteContent` → DashboardView 渲染
2. **任务系统**：`useWorkspaceTasks`（WorkspacePage provide → DashboardView/TaskView inject 共享）→ 调用 `getWorkspaceTasks` + `getWorkspaceMilestones` → TaskView 看板/列表/日历/里程碑四视图展示
   - 新建任务必填标题、优先级、完成标准；未选里程碑时由后端归入“未归属”
   - 状态由后端统一校验：Agent 不能完成，完成必须人工确认
   - “未归属”是系统里程碑，不允许完成或删除
   - 看板支持按里程碑筛选、拖拽状态流转和同列排序；非法状态流转在前后端双重拦截
   - 日历为任务月视图，按截止日期入格，无截止日期进入侧栏，并使用里程碑颜色区分
   - 任务/里程碑详情使用 shadcn Sheet，删除走 AlertDialog 确认
3. **日历**：CalendarView → 调用 `getJournalMonths` + `getJournalEntries` → 月历网格+日记圆点
4. **闪念**：useInbox（WorkspacePage provide → InboxView/DashboardView inject 共享）→ `getFileTree` + `getNoteContent`(预览) → InboxView 列表展示
   - 搜索筛选：本地 computed 过滤 name + preview
   - 排序：修改时间(默认) / 创建时间 / 名称
   - 乐观删除：先移除本地列表，失败回滚
   - 归档：`moveFileEntry` 移到目标目录，InboxArchiveDialog 选目录
   - 重命名：`renameNote`，InboxRenameDialog 输入新名
   - 右键菜单：ContextMenu 打开/重命名/归档/删除
   - 相对时间："3分钟前" 替代绝对日期
   - 侧边栏 badge：收件箱入口显示笔记数
5. **数据库（Bases）**：BaseView → `getBaseData` 加载 .base JSON → 表格/看板/画廊/日历四视图
   - 列管理：addColumn / deleteColumn / renameColumn
   - 行管理：addRow / deleteRow / updateCell（双击编辑）
   - 视图管理：addView / deleteView / switchView
   - 排序过滤：列头点击排序、filter 系统（设计阶段）
   - 防抖保存：1s debounce → `saveBaseData`
   - 文件引用行：后端自动解析 frontmatter → fileTitle + cells 合并
6. **日记**：`handleCreateJournal(date?)` → 创建 `日记/YYYY/MM/YYYY-MM-DD.md`
7. **标签页状态**：`saveStatusMap` 跟踪每个编辑器标签的保存状态 → TabBar 显示指示点
