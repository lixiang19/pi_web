# 工作空间模块梳理

## 概述

工作空间（`/workspace`）是 ridge 的个人内容平台，替代原 `/notes` 笔记页面。核心模型：文件树 + 标签页 + 数据库。

## 路由

- `router/index.ts`：`/workspace` → `WorkspacePage`
- `useWorkbenchPrimaryNavigation.ts`：侧边栏"工作空间" → `workspace`

## 页面结构

```
WorkspacePage.vue（协调层）
├── 左侧 260px
│   ├── 4个固定视图入口（仪表盘/待办/日历/收件箱+badge计数）
│   ├── Separator
│   ├── FileTreePanel（useFileTreeData，root=workspaceDir）
│   └── 新建按钮行（笔记/文件夹/Canvas/Base）

        │   ├── TabBar（面板内独立标签栏，右键菜单支持拆分）
```

```
| `/api/workspace/journal`          | GET    | 获取月度日记列表/年度有日记月份                      |
| `/api/workspace/base`             | GET    | 读取 .base 文件（含文件行 frontmatter 解析）         |
| `/api/workspace/base`             | PUT    | 保存 .base 文件                                      |
| `/api/workspace/base`             | DELETE | 删除 .base 文件                                      |
| `/api/workspace/base/create`      | POST   | 创建新 .base 文件（含默认列和空行）                  |
| `/api/notes/content`              | GET    | 获取笔记内容                                         |
| `/api/notes/content`              | PUT    | 保存笔记内容                                         |
| `/api/notes`                      | POST   | 创建笔记                                             |
| `/api/notes/folder`               | POST   | 创建文件夹                                           |

## 数据库表

### workspace_tasks

| 字段       | 类型    | 说明                            |
| ---------- | ------- | ------------------------------- |
| task_id    | TEXT PK | 格式: task-{timestamp}-{random} |
| title      | TEXT    | 任务标题                        |
| status     | TEXT    | pending / in_progress / done    |
| priority   | TEXT    | low / medium / high             |
| due_date   | INTEGER | 截止日期时间戳                  |
| tags       | TEXT    | 标签（逗号分隔）                |
| note_path  | TEXT    | 关联笔记路径                    |
| created_at | INTEGER | 创建时间                        |
| updated_at | INTEGER | 更新时间                        |

## 关键数据流

1. **仪表盘**：`useDashboard` → 调用 `getRecentFiles` + `getNoteContent` → DashboardView 渲染
2. **待办**：`useWorkspaceTasks`（WorkspacePage provide → DashboardView/TaskView inject 共享）→ 调用 `getWorkspaceTasks` + `getCheckboxTasks` → 合并为 AggregatedTask[] → TaskView 三模式展示
   - 乐观更新：toggle/create/delete 先本地更新，失败回滚
   - Toast 反馈：操作失败显示 toast 提示
   - Checkbox 校验：toggle 带 expectedText 防止文件编辑后行号偏移
3. **日历**：CalendarView → 调用 `getJournalMonths` + `getJournalEntries` → 月历网格+日记圆点
4. **闪念**：useInbox（WorkspacePage provide → InboxView/DashboardView inject 共享）→ `getFileTree` + `getNoteContent`(预览) → InboxView 列表展示
   - 搜索筛选：本地 computed 过滤 name + preview
   - 排序：修改时间(默认) / 创建时间 / 名称
   - 乐观删除：先移除本地列表，失败回滚
   - 归档：`moveFileEntry` 移到目标目录，InboxArchiveDialog 选目录
   - 重命名：`renameNote`，InboxRenameDialog 输入新名
   - 右键菜单：ContextMenu 打开/重命名/归档/删除
   - 相对时间："3分钟前" 替代绝对日期
   - 侧边栏 badge：收件箱入口显示笔记数
5. **数据库（Bases）**：BaseView → `getBaseData` 加载 .base JSON → 表格/看板/画廊/日历四视图
   - 列管理：addColumn / deleteColumn / renameColumn
   - 行管理：addRow / deleteRow / updateCell（双击编辑）
   - 视图管理：addView / deleteView / switchView
   - 排序过滤：列头点击排序、filter 系统（设计阶段）
   - 防抖保存：1s debounce → `saveBaseData`
   - 文件引用行：后端自动解析 frontmatter → fileTitle + cells 合并
6. **日记**：`handleCreateJournal(date?)` → 创建 `日记/YYYY/MM/YYYY-MM-DD.md`
7. **标签页状态**：`saveStatusMap` 跟踪每个编辑器标签的保存状态 → TabBar 显示指示点
```
