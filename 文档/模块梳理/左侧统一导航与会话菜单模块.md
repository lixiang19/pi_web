# 左侧统一导航与会话菜单模块 Codemap

## Responsibility

- 负责把左侧区域从“会话列表”升级为“统一一级导航 + 会话高频菜单”的混合导航结构。
- 负责承载一级路由入口：`新聊天 / 搜索 / 文件 / 终端 / 自动化 / 数据集 / 空间 / 设置`。
- 负责把 `搜索` 入口稳定导向独立搜索页；真实搜索输入与结果展示在右侧主内容区完成，而不是回退为左栏本地过滤输入。
- 负责把 `终端` 入口稳定导向独立终端页；真实终端创建、标签池、分屏和侧栏全部在右侧主内容区完成，而不是回退成聊天页子面板。
- 负责承载独立的“聊天”系统项目入口，它固定绑定工作区 `chat` 目录，但不进入普通项目列表。
- 负责继续承载会话域的高频二级菜单，并把它提升到全局左侧而不是继续藏在会话页内部。
- 负责把“切会话”与“切主路由”两类动作区分清楚：
  - 切会话：回到 `/chat` 后激活会话池中的会话
  - 切主路由：只切换一级路由页面
- 负责会话树相关视图状态：项目折叠、组展开、重命名输入态、本地搜索过滤。

## Design

### Architecture Pattern

- 模式命名：统一一级导航 + 提升后的会话二级菜单

```text
WorkbenchSidebar
├── 一级动作
│   └── 新聊天
├── 一级路由入口
│   ├── 搜索
│   ├── 文件
│   ├── 终端
│   ├── 自动化
│   ├── 数据集
│   ├── 空间
│   └── 设置
├── 系统聊天区
│   └── 工作区 chat 会话树
└── 会话区
    ├── 项目分组
    ├── worktree 分组
    └── 父子会话树
```

### Key Abstractions

- `WorkbenchPrimaryRoute`
  - 左侧统一一级导航的主路由枚举
  - 当前包含 `chat/search/files/terminal/automations/datasets/spaces/settings`

- `WorkbenchNavItem`
  - 一级导航项模型
  - 包含 `route/label/icon`

- `useWorkbenchPrimaryNavigation()`
  - 负责路由动作和会话动作的边界收敛
  - `createChat()`：先回 `/chat`，再创建草稿会话；默认落在工作区 `chat` 目录，只有显式传入 `cwd` 时才落到普通项目/worktree
  - `openChatSession()`：先回 `/chat`，再激活会话
  - `navigateToRoute()`：切一级主路由

- `WorkspaceChatProject`
  - 左侧栏的系统级聊天项目视图
  - 来源于服务端 `SystemInfo.chatProject*` 配置和会话上下文归属
  - 只在“项目”标题上方渲染，不进入 `/api/projects` 返回值

## Flow

### Flow 1: 点击一级功能入口

```text
用户点击一级导航
  -> useWorkbenchPrimaryNavigation.navigateToRoute()
  -> vue-router 切到目标主路由
  -> 右侧 RouterView 切换页面
```

### Flow 1.1: 点击搜索并打开结果会话

```text
用户点击左侧“搜索”
  -> useWorkbenchPrimaryNavigation.navigateToRoute('search')
  -> 右侧进入 SearchPage
  -> 用户输入关键词并提交
  -> SearchPage 基于 core.sessions 过滤全部会话标题
  -> 用户点击结果
  -> useWorkbenchPrimaryNavigation.openChatSession(sessionId)
  -> router.push({ name: 'chat' }) + activateSession(sessionId)
```

### Flow 1.2: 点击终端并进入真实终端工作区

```text
用户点击左侧“终端”
  -> useWorkbenchPrimaryNavigation.navigateToRoute('terminal')
  -> 右侧进入 TerminalPage
  -> TerminalPage 加载服务端终端池快照
  -> 若终端池为空则创建默认终端
  -> 页面建立对应终端的 WebSocket 流连接
```

### Flow 2: 点击会话节点

```text
用户点击会话
  -> useWorkbenchPrimaryNavigation.openChatSession(sessionId)
  -> router.push({ name: 'chat' })
  -> useSessionLruPool.activateSession(sessionId)
  -> /chat 页面显示对应会话实例
```

### Flow 3: 点击新聊天

```text
用户点击新聊天
  -> useWorkbenchPrimaryNavigation.createChat()
  -> router.push({ name: 'chat' })
  -> useSessionLruPool.activateDraft({ cwd: <workspace>/chat })
  -> /chat 进入新的草稿会话
```

## Boundary

- 左侧模块不负责会话消息加载，它只负责导航和会话入口选择。
- 左侧模块不托管会话实例生命周期，会话实例仍由 `useSessionLruPool` 与 `/chat` 页面管理。
- 左侧模块不伪造一级路由内容；其它页面的数据和空态由各自路由页负责。
