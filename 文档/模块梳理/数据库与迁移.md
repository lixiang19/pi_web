# 数据库与迁移

## 职责

`~/.pi/ridge.db` 是 ridge 产品层结构化真源，保存设备、项目、任务、会话索引、闪念、自动化、后台任务、搜索状态、通知和设置。

## 入口

- `packages/server/src/db/index.ts`
- `packages/server/src/db/migrations.ts`
- `initializeRidgeDb(workspaceDir?)`

## 当前迁移机制

- `ridge_meta` 保存当前 `schema_version`，当前为 `9`。
- `ridge_schema_migrations` 逐条记录已应用的 migration 版本。
- `RIDGE_DB_MIGRATIONS` 是版本化迁移清单，启动时按 `version` 升序补齐未执行项。
- 启动时在事务中执行迁移 bookkeeping、旧库预修复、版本化迁移、二次缺列修复和 schema version 写入。
- 缺列修复使用 `PRAGMA table_info` 检查，不能只依赖 `CREATE TABLE IF NOT EXISTS`。

## 核心表

- `devices`
- `projects`
- `session_index`
- `workspace_tasks`
- `workspace_milestones`
- `fleeting_notes`
- `fleeting_attachments`

- `notification_events`
- `app_settings`
- `session_attachments`

## 边界

- Pi 原始会话正文不进入 `ridge.db`。
- 现有运行代码仍可继续使用 `sessions`、`session_contexts`、`session_selections`、`automation_rules` 等兼容表。
- 旧库存在缺列时启动自动补齐。
- 数据库迁移失败应由事务回滚，不能留下半成品 schema。
- 新增结构化表或索引时必须新增 `RIDGE_DB_MIGRATIONS` 版本，不得只改总量 bootstrap SQL。

## 当前 migration 版本变更记录

| 版本 | 说明 |
|------|------|
| 6 | session_attachments |
| 7 | clips 表；workspace_tasks / workspace_milestones 增加 `project_id` 及对应索引 |
| 8 | fleeting_notes 增加 `type`、`suggestion`；修正 `status` 默认 `pending`、`analysis_status` 默认 `unanalyzed`；修复旧数据主状态 |
| 9 | `fleeting_attachments` 表：闪念临时附件引用关系；索引 `idx_fleeting_attachments_note`；`final_path` 记录迁移后正式路径 |

## 闪念表字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| note_id | TEXT PK | - | 闪念唯一标识 |
| content | TEXT | '' | 文字内容 |
| status | TEXT | 'pending' | 主状态：pending / processing |
| analysis_status | TEXT | 'unanalyzed' | 分析状态：unanalyzed / analyzing / suggested / failed |
| type | TEXT | 'text' | 类型：text / screenshot / file / clipboard / browser / audio |
| suggestion | TEXT | null | AI 处理建议文本（分析失败时也可存原因） |
| recommendation_type | TEXT | null | 建议类型：journal / clip / task / delete |
| recommendation_text | TEXT | null | 建议说明 |
| draft | TEXT | null | AI 生成的处理草稿 |
| requires_input | INTEGER | 0 | 是否需要用户输入 |
| pi_session_id | TEXT | null | 关联 Pi 会话 |
| pi_session_file | TEXT | null | 关联 Pi 会话文件 |
| retry_count | INTEGER | 0 | 重试次数 |
| last_error | TEXT | null | 最后一次错误 |
| created_at | INTEGER | 0 | 创建时间戳 |
| updated_at | INTEGER | 0 | 更新时间戳 |

## 闪念附件表字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| attachment_id | TEXT PK | - | 附件唯一标识 |
| note_id | TEXT FK | - | 关联闪念 |
| original_name | TEXT | '' | 用户上传时的原始文件名（已安全化 basename） |
| stored_name | TEXT | '' | 存储文件名（含短 ID 防冲突） |
| temp_path | TEXT | '' | 临时目录绝对路径（`.ridge/fleeting-attachments/<noteId>/`） |
| final_path | TEXT | null | 处理成功后迁移到的正式路径（`附件/`） |
| mime_type | TEXT | 'application/octet-stream' | MIME 类型 |
| size | INTEGER | 0 | 文件大小（字节） |
| sha256 | TEXT | '' | 文件内容哈希 |
| created_at | INTEGER | 0 | 创建时间戳 |

### 边界

- 闪念表不写入 `search_index_status`；未处理闪念不进入 RAG。
- 处理成功后删除原闪念记录，临时附件迁移到正式附件目录后清理。
- 分析失败只改 `analysis_status` 和 `suggestion`，不改变主 `status`。
- 临时附件不写入 `search_index_status`，不进 RAG、不进图谱、不允许 workspace MCP 读取。
