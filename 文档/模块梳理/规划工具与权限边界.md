# 规划工具与权限边界

## 职责

为所有 Agent 会话提供 7 个任务/里程碑规划工具，并纳入 `task` 权限控制，严格禁止 Agent 将任务或里程碑标记为 completed。

## 入口

- `packages/server/src/planning-tools.ts` — 7 个规划工具的实现与 Pi Extension 注册
  - `createPlanningToolExecutors(workspaceDir)`：导出可独立单元测试的 executor 工厂
  - `createPlanningToolsExtension(workspaceDir)`：注册为 Pi Extension（`ExtensionAPI`）
- `packages/server/src/agent-permissions.ts` — 权限映射与工具移除
  - `PLANNING_TOOL_NAMES`：7 个工具名常量
  - `mapToolToLogicalPermission`：所有规划工具映射为 `task`
  - `extractPermissionSubject` / `derivePermissionPattern`：规划工具返回工具名本身
  - `compileAgentPermission`：`task: deny` 时移除规划工具
- `packages/server/src/session-context.ts` — 会话接入
  - `createSessionResourceLoader` 在 `extensionFactories` 中注册 `createPlanningToolsExtension(deps.defaultWorkspaceDir)`
  - 所有 Agent 会话（普通/任务）默认可用，是否可用由权限控制

## 工具清单

| 工具名 | 功能 | 批量支持 | completed 校验 |
|--------|------|----------|----------------|
| `create_task` | 创建单个任务 | 否 | — |
| `update_task` | 更新单个任务 | 否 | 执行层显式拒绝 `completed` |
| `create_milestone` | 创建单个里程碑 | 否 | — |
| `update_milestone` | 更新单个里程碑 | 否 | 执行层显式拒绝 `completed` |
| `move_task` | 移动任务到指定里程碑 | 否 | — |
| `set_blocked` | 将任务设为 blocked | 否 | 校验当前状态为 `in_progress` |
| `set_reviewing` | 将任务设为 reviewing | 否 | 校验当前状态为 `in_progress` |

## 权限模型

- 逻辑权限键：`task`（与 `task` 子代理工具共享同一个键）
- 默认行为：无配置时 `allow`（Pi 默认语义）
- `task: deny`：`compileAgentPermission` 遍历 `PLANNING_TOOL_NAMES`，从 `activeToolNames` 移除
- `task: ask`：permission gate 拦截时归类为 `task` 权限请求，subject/pattern 均为工具名本身（如 `create_task`）
- 子代理 `task` 工具与规划工具共享 `task` 键，permission request 统一归类

## 边界

- 规划工具只走 Pi Agent 工具链路，不替代 HTTP API（`workspace-tasks.ts` 保留）。
- `completed` 禁止在三层有校验：
  1. 规划工具 schema 中 `status` 枚举不含 `completed`。
  2. 规划工具执行层显式抛错说明“完成必须由用户人工确认”。
  3. `task-system.ts` 的 `assertStatusAllowed` 兜底（`actor === 'agent'` 时禁止）。
- 批量拒绝在执行层 `rejectBatchInput` 统一处理，拒绝任何含 `tasks`、`task_ids`、`milestones`、`milestone_ids`、数组形式 `task_id`/`milestone_id` 的输入。
- `set_blocked` / `set_reviewing` 在执行层校验当前状态必须是 `in_progress`，与 `task-system.ts` 状态机一致。
- 创建任务未指定 `milestone_id` 时挂到默认里程碑；未指定 `project_id` 时继承里程碑项目。

## 测试覆盖

- `packages/server/src/__tests__/planning-tools.test.ts`（24 项）
  - 成功创建任务/里程碑并返回必要字段（id, title, status, milestoneId/projectId）
  - `update_task` / `update_milestone` 尝试 `completed` 的明确错误
  - 批量/隐式多任务输入被拒绝（`tasks`、`task_ids`、`milestones`、`milestone_ids`、数组 task_id）
  - `set_blocked` 保存 blockedReason 且状态流转合法
  - `set_reviewing` 合法推进到 reviewing
  - 权限：规划工具映射为 `task`；`task: deny` 移除规划工具；默认无配置保留；`task: ask` 保留

## 模块关系

```
session-context.ts
  → createSessionResourceLoader
    → createPlanningToolsExtension(defaultWorkspaceDir)
      → planning-tools.ts (7 tools)
        → task-system.ts (底层读写 + assertStatusAllowed)

agent-permissions.ts
  → mapToolToLogicalPermission / compileAgentPermission
    → 控制 planning tools 可用性
```
