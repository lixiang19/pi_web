# 文件树与标签页统一模块

## 概述

统一工作空间和会话页面中重复的文件树组件和标签页组件，消除 `WorkspaceFileTree`/`NoteVaultSidebar` 和 `WorkbenchOperationPanel`/`NoteTabBar` 的双实现。

## 架构

```
components/common/
  ├── TabBar.vue          — 统一标签栏（替代 NoteTabBar）
  └── FileTreePanel.vue   — 统一文件树面板（项目文件树渲染+收藏）

composables/
  └── useFileTreeData.ts  — 文件树数据逻辑（API加载、展开/折叠、刷新）
```

## 组件关系

### 标签页系统

```
TabBar (通用标签栏UI)
  ├── WorkbenchOperationPanel (会话页文件预览) → TabBar + 文件预览内容
  └── NotesPage (笔记页) → TabBar + Milkdown编辑器内容
```

- `TabBar` 只管标签栏的渲染（选中态、关闭按钮、状态点）
- 标签内容区由各使用方通过插槽/自己的布局提供

### 文件树系统

```
useFileTreeData (composable) — 数据层（API调用、展开折叠状态）
  └── FileTreePanel (UI层) — 树形渲染、收藏切换
        └── WorkspaceFileTree (业务适配) — 适配收藏Store + 事件
              └── ProjectFilePanel — Git/文件Tab切换外壳
                    ├── SessionTabContent 右侧面板
                    └── SessionDetailPage 右侧面板
```

- `useFileTreeData` 封装文件树的数据获取和展开/折叠逻辑
- `FileTreePanel` 封装统一的树形渲染UI
- `WorkspaceFileTree` 是业务适配层，处理收藏和事件转换
- `NoteVaultSidebar` 保持独立（笔记有筛选/搜索/CRUD等特殊交互）

## 接口契约

### TabBar

```ts
interface TabItem {
  id: string;
  title: string;
  status?: 'idle' | 'saving' | 'unsaved' | 'error' | 'loading';
}

Props: { tabs: TabItem[], activeTabId: string }
Events: { select(id), close(id) }
```

### FileTreePanel

```ts
Props: {
  title: string,
  nodes: VisibleTreeNode[],  // 来自 useFileTreeData
  isRootLoading: boolean,
  error: string,
  isExpanded: (path: string) => boolean,
  isLoading: (path: string) => boolean,
}
Events: { select(entry), toggle-expand(entry), toggle-favorite(path), refresh() }
```

### useFileTreeData

```ts
(rootDir: () => string) => {
  rootPath, visibleNodes, fileTreeError,
  isDirectoryExpanded, isDirectoryLoading,
  toggleDirectory, expandToPath, refreshTree,
}
```

## 已删除

- `NoteTabBar.vue` — 被 `TabBar` 替代

## 不统一的部分

- `NoteVaultSidebar` — 笔记侧边栏有完全不同的数据源(notes API)和交互(搜索/筛选/CRUD)，强行统一会增加复杂度而非降低
