# 00 工作空间初始化与目录结构

## 目标

服务器首次启动时创建唯一 ridge 工作空间，并初始化所有预制目录和核心空文件。

## 范围

- 创建 `~/ridge-workspace`。
- 创建可见一级目录：`项目/`、`笔记/`、`日记/`、`剪藏/`、`附件/`、`记忆/`、`Wiki/`、`空间/`。
- 创建系统隐藏目录 `.ridge/`。
- 创建 `记忆/MEMORY.md`，初始内容为 `# MEMORY`。
- 创建 `Wiki/index.md`，初始内容为 `# Wiki`。
- 路径存在但不是目录时必须失败。

## 不做

- 不创建外部项目目录。
- 不创建 RAG、Kuzu、转换缓存的具体实现。
- 不迁移用户旧数据。

## 验收

- 新启动服务后工作空间结构完整。
- 重复启动幂等。
- 预制目录不能被当作普通自定义目录覆盖。
- 初始化不污染源码仓库。

## 关联设计

- `文档/项目设计/工作空间文件组织.md`
- `文档/项目设计/根设计-工作空间设备项目会话.md`
- `文档/项目设计/记忆Wiki图谱与工作空间MCP.md`

## Spec 提取点

- 工作空间路径解析。
- 目录创建幂等规则。
- 文件存在但类型错误的失败规则。
- 初始文件内容。

## Spec 草案

### 输入

- 服务器启动事件。
- 配置中的工作空间根目录，默认 `~/ridge-workspace`。

### 行为

- 启动时解析工作空间路径。
- 路径不存在时创建目录。
- 路径存在但不是目录时中止启动并返回明确错误。
- 创建八个预制一级目录和 `.ridge/`。
- 创建空标题文件 `记忆/MEMORY.md` 和 `Wiki/index.md`。
- 重复启动不得覆盖已有用户文件。

### 输出

- 完整工作空间目录。
- 初始化结果日志。

### 测试

- 空环境启动创建完整结构。
- 重复启动不改已有文件内容。
- 工作空间路径是文件时启动失败。
- 缺失单个预制目录时启动补齐。
