# 任务模块本地 DB 与 Pi 自定义任务操作设计

> 日期：2026-05-01  
> 任务：设计任务模块；任务存储为本地 DB；新增 Pi 自定义任务工具供 AI 操作；任务界面支持列表、看板、日历视图。  
> 结论：任务模块以 `~/.pi/ridge.db` 为唯一真源，通过服务端 TaskService 统一承接 Web UI 与 Pi 自定义工具写入；列表 / 看板 / 日历只是同一批任务的不同投影视图。

## 1. 当前现状

当前仓库已经存在基础待办能力：

- 前端：`packages/web/src/components/workspace/TaskView.vue`
  - 已有 `list / kanban / today` 三种本地视图模式；
  - 任务字段仍偏轻量：标题、状态、优先级、截止时间、标签；
  - `today` 不是完整日历，只是今日任务列表。
- 前端状态：`packages/web/src/composables/useWorkspaceTasks.ts`
  - 通过 provide/inject 给仪表盘和任务页共享任务列表；
  - 调用 `/api/workspace/tasks`；
  - 使用乐观更新与 toast 反馈。
- 后端 API：`packages/server/src/routes/workspace-tasks.ts`
  - 暴露 `GET / POST / PATCH / DELETE /api/workspace/tasks`；
  - 当前接口直接面向 UI CRUD。
- 后端存储：`packages/server/src/repositories/workspace-task-repository.ts`
  - 当前实现是 SQLite repository；
  - 但路径落在 `<workspace>/.ridge/ridge.db`，与项目级数据目录 `~/.pi/` 的顶层约束不一致。
- Pi 工具：`packages/server/src/subagents.ts`
  - 已经存在名为 `task` 的子代理工具；
  - 新任务模块不能复用或覆盖这个工具名，否则会和“启动子代理”的语义冲突。

## 2. 目标

### 2.1 产品目标

任务模块提供一个可被用户和 AI 同时操作的工作台：

1. 用户可以在任务页面直接创建、编辑、完成、归档任务；
2. AI 可以通过 Pi 自定义工具创建、修改、排期、完成任务；
3. 所有任务写入同一个本地 DB；
4. 任务页面提供：
   - 列表视图；
   - 看板视图；
   - 日历视图；
5. 三个视图之间无数据副本，切换视图不改变任务真源。

### 2.2 技术目标

1. DB 是唯一任务真源，不再出现任务 JSON、Markdown 任务文件、前端 localStorage 多真源；
2. Web UI 和 Pi 工具都只能调用 TaskService，不能绕过服务层直接写表；
3. Pi 自定义任务工具必须有字段白名单、操作日志和撤销基础；
4. 任务视图状态和任务业务数据分离：视图筛选、排序、当前视图可保存在 UI 状态；任务内容只在 DB；
5. 不覆盖已有 `task` 子代理工具，新工具命名为 `ridge_task`。

## 3. 非目标

本阶段不做：

- 甘特图；
- 多人协作同步；
- CalDAV / Google Calendar 外部日历同步；
- 复杂项目依赖图；
- AI 裸 SQL；
- 可编辑任务 Markdown 文件；
- 继续维护 `<workspace>/.ridge/ridge.db` 作为任务真源。

## 4. 存储设计

### 4.1 DB 路径

任务模块使用本地 DB：

```text
~/.pi/ridge.db
```

原因：

- ridge 用户数据目录已经约束为 `~/.pi/`；
- `packages/server/src/db/index.ts` 已有全局 ridge DB 打开与 bootstrap 机制；
- 自动化、会话索引、设置等平台级数据已经在全局 DB 方向收敛；
- 任务可能关联多个工作区、Pi 会话和未来 RAG 索引，不应该散落到每个 workspace 的 `.ridge` 目录。

### 4.2 任务表

```sql
CREATE TABLE tasks (
  task_id TEXT PRIMARY KEY,
  workspace_root TEXT NOT NULL,
  title TEXT NOT NULL,
  description TEXT,
  status TEXT NOT NULL,
  priority TEXT NOT NULL,
  start_at INTEGER,
  due_at INTEGER,
  completed_at INTEGER,
  archived_at INTEGER,
  tags_json TEXT NOT NULL,
  source_type TEXT NOT NULL,
  source_id TEXT,
  source_path TEXT,
  session_id TEXT,
  sort_order REAL NOT NULL,
  created_by TEXT NOT NULL,
  updated_by TEXT NOT NULL,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

CREATE INDEX idx_tasks_workspace_status ON tasks(workspace_root, status);
CREATE INDEX idx_tasks_workspace_due_at ON tasks(workspace_root, due_at);
CREATE INDEX idx_tasks_workspace_updated_at ON tasks(workspace_root, updated_at DESC);
CREATE INDEX idx_tasks_session_id ON tasks(session_id);
```

### 4.3 字段说明

| 字段             | 说明                                                           |
| ---------------- | -------------------------------------------------------------- |
| `task_id`        | 任务 ID，格式 `task_<timestamp>_<random>`                      |
| `workspace_root` | 当前任务所属工作区根目录；支持同一个本地 DB 管多个仓库/工作区  |
| `title`          | 任务标题，必填                                                 |
| `description`    | 任务说明，可空                                                 |
| `status`         | `todo / doing / done / archived / cancelled`                   |
| `priority`       | `low / medium / high`                                          |
| `start_at`       | 开始时间戳；日历视图可用                                       |
| `due_at`         | 截止时间戳；日历视图主要依据                                   |
| `completed_at`   | 完成时间                                                       |
| `archived_at`    | 归档时间；归档不是物理删除                                     |
| `tags_json`      | 标签数组 JSON                                                  |
| `source_type`    | `manual / ai / dashboard / note / journal / capture / session` |
| `source_id`      | 来源对象 ID                                                    |
| `source_path`    | 来源文件路径                                                   |
| `session_id`     | 关联 Pi 会话 ID                                                |
| `sort_order`     | 同状态列内排序，用于看板拖拽                                   |
| `created_by`     | `user / ai`                                                    |
| `updated_by`     | `user / ai`                                                    |
| `created_at`     | 创建时间                                                       |
| `updated_at`     | 更新时间                                                       |

### 4.4 操作日志表

Pi 自定义工具允许 AI 自动写任务，因此必须记录操作日志。

```sql
CREATE TABLE task_operations (
  operation_id TEXT PRIMARY KEY,
  task_id TEXT,
  workspace_root TEXT NOT NULL,
  actor TEXT NOT NULL,
  tool_name TEXT,
  action TEXT NOT NULL,
  before_json TEXT,
  after_json TEXT,
  reason TEXT,
  status TEXT NOT NULL,
  created_at INTEGER NOT NULL
);

CREATE INDEX idx_task_operations_task_id ON task_operations(task_id);
CREATE INDEX idx_task_operations_workspace_created_at ON task_operations(workspace_root, created_at DESC);
```

硬规则：**AI 通过工具写任务时，任务变更与 `task_operations` 必须在同一个 SQLite transaction 中提交。**

## 5. 状态模型

任务状态统一为：

```ts
type TaskStatus = "todo" | "doing" | "done" | "archived" | "cancelled";
```

含义：

| 状态        | 含义   | 列表       | 看板     | 日历         |
| ----------- | ------ | ---------- | -------- | ------------ |
| `todo`      | 未开始 | 显示       | 待做列   | 有日期则显示 |
| `doing`     | 进行中 | 显示       | 进行中列 | 有日期则显示 |
| `done`      | 已完成 | 可折叠显示 | 完成列   | 可低亮显示   |
| `archived`  | 已归档 | 默认隐藏   | 不显示   | 不显示       |
| `cancelled` | 已取消 | 默认隐藏   | 不显示   | 不显示       |

当前代码中的 `pending / in_progress / done` 应在实现任务中替换为上述状态，不继续扩展旧状态。

## 6. 服务端模块设计

### 6.1 分层

```text
Web UI / Pi runtime
        |
        v
TaskService
- 字段校验
- 权限边界
- 状态流转
- transaction
- operation log
        |
        v
TaskRepository
- SQLite 读写
- 查询投影
- 排序更新
        |
        v
~/.pi/ridge.db
```

### 6.2 文件建议

```text
packages/server/src/tasks/
├── task-types.ts
├── task-schema.ts
├── task-repository.ts
├── task-service.ts
├── task-tools-extension.ts
└── task-index-document.ts

packages/server/src/routes/workspace-tasks.ts
```

### 6.3 TaskService 职责

TaskService 是唯一写入口：

- `listTasks(query)`：按工作区、状态、日期、关键词查询；
- `getTask(taskId)`：读取单任务；
- `createTask(input, actor)`：创建任务；
- `updateTask(taskId, patch, actor)`：更新字段白名单；
- `moveTask(taskId, status, sortOrder, actor)`：看板拖拽；
- `scheduleTask(taskId, startAt, dueAt, actor)`：日历排期；
- `completeTask(taskId, actor)`：完成任务；
- `archiveTask(taskId, actor)`：软归档；
- `splitTask(taskId, children, actor)`：拆分任务；
- `undoOperation(operationId)`：撤销 AI 单步操作。

## 7. API 设计

### 7.1 查询任务

```http
GET /api/workspace/tasks?status=todo,doing&from=1714492800000&to=1717084800000&q=设计
```

响应：

```ts
type TaskListResponse = {
  tasks: Task[];
  updatedAt: number;
};
```

### 7.2 创建任务

```http
POST /api/workspace/tasks
```

请求：

```ts
type CreateTaskRequest = {
  title: string;
  description?: string;
  priority?: "low" | "medium" | "high";
  startAt?: number | null;
  dueAt?: number | null;
  tags?: string[];
  source?: {
    type: "manual" | "dashboard" | "note" | "journal" | "capture" | "session";
    id?: string;
    path?: string;
  };
  sessionId?: string;
};
```

### 7.3 更新任务

```http
PATCH /api/workspace/tasks/:taskId
```

允许字段：

```ts
type UpdateTaskRequest = {
  title?: string;
  description?: string | null;
  status?: "todo" | "doing" | "done" | "archived" | "cancelled";
  priority?: "low" | "medium" | "high";
  startAt?: number | null;
  dueAt?: number | null;
  tags?: string[];
  sessionId?: string | null;
  sortOrder?: number;
};
```

### 7.4 归档任务

```http
POST /api/workspace/tasks/:taskId/archive
```

归档只设置 `status=archived` 与 `archived_at`，不物理删除。

### 7.5 操作日志

```http
GET /api/workspace/tasks/operations?taskId=task_xxx
POST /api/workspace/tasks/operations/:operationId/undo
```

MVP 可以先只返回 AI 操作日志，用户操作日志后置。

## 8. Pi 自定义任务工具设计

### 8.1 命名

新增 Pi 自定义工具命名为：

```text
ridge_task
```

不使用 `task`，因为 `task` 已经是子代理工具，语义是“启动子代理”，不是“操作任务数据”。

### 8.2 注册位置

不建议作为独立项目 `.pi/extensions` 文件去访问 HTTP 接口；更稳的方式是在服务端资源加载链路里注册工具扩展：

```text
packages/server/src/tasks/task-tools-extension.ts
```

由会话创建链路注入：

```text
createSessionResourceLoader
  -> createSubagentToolExtension(...)
  -> createWorkspaceTaskToolExtension(...)
```

这样工具可以直接调用 TaskService，避免：

- AI 直接拼 HTTP；
- 本地服务端口变化；
- 工具绕过 transaction；
- 工具写入后缺少 operation log。

### 8.3 工具参数

采用单工具多动作，减少系统提示和工具列表噪音。

```ts
type RidgeTaskAction =
  | "list"
  | "get"
  | "create"
  | "update"
  | "schedule"
  | "complete"
  | "archive"
  | "split";

type RidgeTaskParams = {
  action: RidgeTaskAction;
  taskId?: string;
  query?: {
    status?: TaskStatus[];
    from?: number;
    to?: number;
    q?: string;
  };
  task?: {
    title?: string;
    description?: string;
    priority?: "low" | "medium" | "high";
    startAt?: number | null;
    dueAt?: number | null;
    tags?: string[];
    status?: TaskStatus;
  };
  children?: Array<{
    title: string;
    description?: string;
    dueAt?: number | null;
  }>;
  reason?: string;
};
```

### 8.4 工具动作边界

| action     | AI 是否允许自动执行 | 说明                             |
| ---------- | ------------------- | -------------------------------- |
| `list`     | 允许                | 只读                             |
| `get`      | 允许                | 只读                             |
| `create`   | 允许                | 必须有 title，记录 created_by=ai |
| `update`   | 限制允许            | 只能改字段白名单                 |
| `schedule` | 允许                | 只改 `startAt/dueAt`             |
| `complete` | 允许                | 只改完成状态与完成时间           |
| `archive`  | 限制允许            | 软归档，不物理删除               |
| `split`    | 允许                | 创建子任务，原任务不删除         |

禁止提供：

- `delete`；
- 裸 SQL；
- 批量删除；
- 批量覆盖日期；
- 跨工作区移动。

### 8.5 工具返回

```ts
type RidgeTaskToolResult = {
  action: RidgeTaskAction;
  task?: Task;
  tasks?: Task[];
  operationId?: string;
  updatedAt: number;
};
```

返回给 LLM 的文本要简短，例如：

```text
已创建任务：设计任务模块多视图（task_171...）。
```

`details` 返回完整结构给前端渲染和调试。

## 9. 前端任务界面设计

### 9.1 页面结构

将当前 `TaskView.vue` 拆成协调组件 + 视图组件：

```text
components/workspace/tasks/
├── TaskPage.vue
├── TaskToolbar.vue
├── TaskQuickCreate.vue
├── TaskListView.vue
├── TaskKanbanView.vue
├── TaskCalendarView.vue
├── TaskCard.vue
├── TaskEditorDrawer.vue
└── TaskOperationTimeline.vue
```

`WorkspacePage.vue` 中的固定入口仍打开任务标签页，标签页内容渲染 `TaskPage`。

### 9.2 视图切换

视图枚举：

```ts
type TaskViewMode = "list" | "kanban" | "calendar";
```

顶部工具栏：

- 左侧：标题、统计、快速新建；
- 中间：搜索、状态筛选、标签筛选；
- 右侧：列表 / 看板 / 日历切换。

视图切换只改变展示方式，不触发重新建数据源。

### 9.3 列表视图

列表视图适合快速处理：

- 默认分组：未完成、已完成；
- 默认排序：`status -> dueAt -> priority -> updatedAt`；
- 行内支持：完成、标题编辑、优先级、日期、标签；
- 点击任务打开右侧 `TaskEditorDrawer`；
- 已归档任务默认隐藏。

### 9.4 看板视图

看板列：

```text
待做(todo) | 进行中(doing) | 完成(done)
```

交互：

- 拖动卡片到另一列：调用 `moveTask(taskId, status, sortOrder)`；
- 同列上下拖动：只更新 `sortOrder`；
- 卡片展示标题、优先级、截止日期、标签；
- 归档 / 取消任务不在默认看板列展示。

### 9.5 日历视图

MVP 做月视图 + 日期任务列表：

- 月格子展示 `startAt` 或 `dueAt` 落在当天的任务；
- 任务无日期时不出现在日历网格，但可在侧边“未排期”区域显示；
- 拖动任务到某日：更新 `dueAt`；
- 点击某日：右侧列表展示当天任务；
- 点击任务：打开 `TaskEditorDrawer`。

不在 MVP 做周视图、日程时间块和外部日历同步。

## 10. Web 与 AI 写入一致性

所有写入口统一到 TaskService：

```text
用户在列表编辑
  -> PATCH /api/workspace/tasks/:taskId
  -> TaskService.updateTask(actor=user)
  -> tasks

用户拖动看板
  -> PATCH /api/workspace/tasks/:taskId
  -> TaskService.moveTask(actor=user)
  -> tasks

用户拖动日历
  -> PATCH /api/workspace/tasks/:taskId
  -> TaskService.scheduleTask(actor=user)
  -> tasks

AI 调用 ridge_task
  -> TaskService.*(actor=ai)
  -> tasks + task_operations 同事务
```

前端不直接构造 AI 操作日志；AI 工具也不直接写 repository。

## 11. RAG 投影预留

任务进入 RAG 时，不读取 DB 原始行给模型，而由纯函数生成文本投影：

```ts
type TaskIndexDocument = {
  id: string;
  sourceType: "task";
  sourceId: string;
  title: string;
  text: string;
  metadata: {
    workspaceRoot: string;
    status: TaskStatus;
    priority: TaskPriority;
    startAt?: number;
    dueAt?: number;
    tags: string[];
    sourceType: string;
    sessionId?: string;
  };
  updatedAt: number;
};
```

MVP 先实现：

```ts
taskToIndexDocument(task: Task): TaskIndexDocument
```

真正接入仓库 RAG 索引可以单独开任务。
