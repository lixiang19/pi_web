# 任务系统实施方案 v0.1

## 基准

本方案唯一产品基准为 `文档/功能开发/任务系统PRD-v0.1.md`。

不迁移旧 `.ridge/tasks.json` 数据。旧 JSON 待办实现视为废弃原型，新任务系统使用 `~/.pi/ridge.db`。

## 第一阶段目标

- 任务绑定当前 workspace。
- 里程碑绑定当前 workspace。
- 任务和里程碑使用 DB 存储。
- 系统自动提供默认里程碑 `未归属`。
- 新建任务必须归属里程碑，未指定时进入 `未归属`。
- 任务和里程碑统一状态：`pending`、`in_progress`、`blocked`、`reviewing`、`completed`。
- 用户可完成人工确认；Agent 不允许设为 `completed`。
- `未归属` 里程碑不允许完成。

## 非目标

- 不迁移 `.ridge/tasks.json`。
- 不做标签、子任务、多处理会话、活动日志。
- 第一阶段不实现任务过程展示。

## DB 模型

### workspace_milestones

- `milestone_id`：主键。
- `workspace_path`：workspace 绝对路径。
- `title`：标题。
- `goal`：目标。
- `acceptance_criteria`：完成标准。
- `status`：统一状态。
- `due_date`：截止日期，毫秒时间戳，可空。
- `is_system`：是否系统里程碑。
- `color`：日历视图颜色。
- `sort_order`：手动排序。
- `created_at` / `updated_at`。

约束：同一 workspace 只能有一个系统 `未归属` 里程碑。

### workspace_tasks

- `task_id`：主键。
- `workspace_path`：workspace 绝对路径。
- `milestone_id`：所属里程碑。
- `title`：标题。
- `status`：统一状态。
- `priority`：`normal`、`important`、`urgent`。
- `acceptance_criteria`：完成标准。
- `due_date`：截止日期，毫秒时间戳，可空。
- `blocked_reason`：阻塞原因，可空。
- `processing_session_id`：处理会话 ID，可空且唯一。
- `sort_order`：看板列内排序。
- `created_at` / `updated_at`。

## API

当前工作区继续复用服务端 `defaultWorkspaceDir` 边界：优先使用 `ridge.db` 记录的当前工作空间，首次启动默认创建 `~/ridge-workspace`。

- `GET /api/workspace/milestones`
- `POST /api/workspace/milestones`
- `GET /api/workspace/milestones/:milestoneId`
- `PATCH /api/workspace/milestones/:milestoneId`
- `DELETE /api/workspace/milestones/:milestoneId`
- `GET /api/workspace/tasks`
- `POST /api/workspace/tasks`
- `GET /api/workspace/tasks/:taskId`
- `PATCH /api/workspace/tasks/:taskId`
- `DELETE /api/workspace/tasks/:taskId`

## 状态规则

- 新建任务和里程碑默认 `pending`。
- Agent 可设置 `pending`、`in_progress`、`blocked`、`reviewing`。
- Agent 不可设置 `completed`。
- 用户可设置 `completed`。
- `未归属` 里程碑不可设置 `completed`。

## 测试

后端先覆盖：

- 首次查询自动创建 `未归属`。
- 创建里程碑默认 `pending`。
- 创建任务未指定里程碑时进入 `未归属`。
- 任务和里程碑详情查询。
- 用户可完成任务。
- Agent 不能完成任务或里程碑。
- `未归属` 里程碑不能完成。
- 删除里程碑时仍有任务则拒绝。
