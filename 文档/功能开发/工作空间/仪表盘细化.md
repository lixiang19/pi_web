# 仪表盘细化

> 状态：✅ 已修复
> 优先级：P1

## 问题

1. **今日待办卡片是死占位** — 文案写"待办视图将在后续实现"，但 M4 已做
2. **日记摘要含 frontmatter** — 取前 3 行全是 YAML 废信息
3. **"打开"按钮行为错误** — 日记已存在时应打开文件，不是触发 create-journal

## 修复方案

### 1. 今日待办卡片接入真实数据

DashboardView 引入 `useWorkspaceTasks`，取 `todayTasks` 前 3 条展示。点击跳转待办视图。

### 2. 日记摘要跳过 frontmatter

```ts
const journalPreview = computed(() => {
  const content = todayJournalContent.value;
  if (!content) return "";
  // 跳过 frontmatter
  let body = content;
  const fmEnd = content.indexOf("\n---", 4); // 从第4个字符开始找，跳过开头的 ---
  if (content.startsWith("---") && fmEnd > 0) {
    body = content.slice(fmEnd + 4).trim();
  }
  const lines = body.split("\n").filter((l) => l.trim());
  return lines.slice(0, 3).join("\n");
});
```

### 3. "打开"按钮直接打开日记

- `hasTodayJournal` 为 true 时，emit `open-file` + todayJournalPath
- `hasTodayJournal` 为 false 时，emit `create-journal`
