> 版本：v1.0  
> 日期：2026-04-29

## 一、目标

将当前工作空间中间区域从"单 TabBar + 单内容区"升级为"多分屏面板 + 每面板独立标签页"，实现类似 VS Code 编辑器分组的标签页管理能力。

### 现有问题

```
当前结构：
┌──────────────────────────────────┐
│ TabBar（所有标签页平铺一行）       │
├──────────────────────────────────┤
│ 内容区（v-show 互斥，同一时刻只一个）│
└──────────────────────────────────┘
```

- 标签页一多就挤成一排，不可见部分需要滚动
- 只能同时看一个文件，无法并排参考
- 关闭标签页后，仪表盘和其他文件之间没有分组概念

### 目标结构

```
┌─────────────────────┬───────────┐
│ 分屏组 A（左侧）       │ 分屏组 B   │
│ ┌Tab───┐┌Tab───┐    │ ┌Tab───┐  │
│ │仪表盘 ││设计md │    │ │参考md │  │
│ └──────┘└──────┘    │ └──────┘  │
│ ┌─────────────────┐ │           │
│ │  仪表盘内容       │ │ 参考文档   │
│ │  （可滚动）       │ │ 内容       │
│ └─────────────────┘ │           │
└─────────────────────┴───────────┘
```

## 二、核心概念

### 2.1 PaneGroup（分屏组）

PaneGroup 是分屏网格中的一个**叶子节点**，拥有：

- 一组标签页列表
- 一个活跃标签页 ID
- 占比（flex 比例）

### 2.2 SplitNode（分屏树）

整个中间区域是一棵**二叉树**，每个节点要么是 PaneGroup（叶子），要么是 SplitNode（分叉节点）。

```typescript
// 分屏方向
type SplitDirection = "horizontal" | "vertical";

// 分屏网格节点
type GridNode = PaneGroup | SplitContainer;

// 分屏组（叶子）
interface PaneGroup {
  id: string; // 唯一 ID
  tabs: TabItem[]; // 标签页列表
  activeTabId: string; // 当前活跃标签页
}

// 分屏容器（内部节点）
interface SplitContainer {
  id: string;
  direction: SplitDirection; // 拆分方向
  children: [GridNode, GridNode]; // 两个子节点
  sizes: [number, number]; // 两个子节点的比例，如 [60, 40]
}
```

### 2.3 操作集合

| 操作       | 触发方式                                                 | 效果                                                                       |
| ---------- | -------------------------------------------------------- | -------------------------------------------------------------------------- |
| 水平拆分   | 右键标签页 → "向右拆分" / 拖拽标签页到右边缘             | 当前 PanelGroup 变为 SplitContainer(horizontal) 的左子，右侧新建 PaneGroup |
| 垂直拆分   | 右键标签页 → "向下拆分" / 拖拽标签页到下边缘             | 当前 PanelGroup 变为 SplitContainer(vertical) 的上子，下方新建 PaneGroup   |
| 移动标签页 | 拖拽标签页到另一个 PaneGroup 的标签栏                    | 标签从一个组移到另一个组                                                   |
| 合并方向键 | 拖拽标签页到另一个 PaneGroup 的内容区（上/下/左/右边缘） | 在目标组的方向新建拆分                                                     |
| 调整大小   | 拖拽分屏之间的分割线                                     | 调整 `sizes` 比例                                                          |
| 关闭面板   | 关闭 PaneGroup 最后一个标签页                            | 删除该 PaneGroup，父 SplitContainer 替换为兄弟节点                         |
| 全关闭保护 | 拖拽标签页到另一个 PaneGroup 时保留非空                  | PaneGroup 至少保留一个标签页（仪表盘不可关闭 → 最后一个为仪表盘）          |

## 三、拖拽与分屏交互

### 3.1 拖拽标签页：三种目标区域

当用户拖拽一个标签页时，进入另一个 PaneGroup 区域，目标区域分为三种：

| 区域                              | 视觉指示       | 操作                     |
| --------------------------------- | -------------- | ------------------------ |
| **标签栏区域**（顶部 32px）       | 高亮标签栏     | 移动标签到该组的标签列表 |
| **边缘区域**（上下左右 25% 窄带） | 半透明色块覆盖 | 在该方向新建拆分         |
| **内容区中央**（其余区域）        | 无指示         | 移动标签到该组           |

```
拖拽到 PaneGroup 的热区划分：

┌───────────────────────┐
│  ▒▒ 上边缘 25% ▒▒      │  → 上方新建 SplitContainer(vertical)
│                       │
│                       │
│     内容区中央          │  → 移动标签到该组标签列表
│                       │
│                       │
│  ▒▒ 下边缘 25% ▒▒      │  → 下方新建 SplitContainer(vertical)
└───────────────────────┘

  ▒▒         ▒▒
左边缘 25%  右边缘 25%
  → 左侧新建   → 右侧新建
  SplitContainer(horizontal)
```

### 3.2 右键菜单

标签页右键菜单新增：

```
─────────────────
  向右拆分  →     水平拆分，当前标签移到左侧，右侧空白
  向下拆分  ↓     垂直拆分，当前标签移到上侧，下方空白
─────────────────
  关闭
  关闭其他
  关闭右侧的
─────────────────
```

### 3.3 分割线调整

- 分割线宽度：4px（可拖拽区域）
- 拖拽时显示半透明辅助线
- 释放时更新 SplitContainer.sizes
- 最小比例：子节点不低于 15%

### 3.4 空面板处理

- 当 PaneGroup 最后一个标签页被关闭（且非仪表盘）：
  - 若仪表盘存在 → 仪表盘移入，保持该面板存活
  - 若仪表盘已在其他面板 → 关闭该面板，合并
- 当整个树只剩一个 PaneGroup：
  - 仪表盘始终在其中且不可关闭

## 六、状态管理 — useSplitPanes

### 6.1 Composables 结构

```
composables/
├── useWorkspaceFilePreview.ts  — 保留，不变（管理文件标签页的加载状态）
└── useSplitPanes.ts            — 新增，管理分屏树
```

### 6.2 useSplitPanes 接口

```typescript
export function useSplitPanes() {
  // 分屏树根节点
  const rootNode: Ref<GridNode>;

  // 展平所有 PaneGroup 为数组（用于查找）
  const allPaneGroups: ComputedRef<PaneGroup[]>;

  // 当前活跃（最后交互的）PaneGroup
  const activePaneGroupId: Ref<string>;

  // ===== PaneGroup 操作 =====

  /** 在指定 PaneGroup 打开/激活标签页 */
  function openTab(paneGroupId: string, tab: TabItem): void;

  /** 关闭标签页 */
  function closeTab(paneGroupId: string, tabId: string): void;

  /** 在指定 PaneGroup 设置活跃标签页 */
  function setActiveTab(paneGroupId: string, tabId: string): void;

  /** 移动标签从一个 PaneGroup 到另一个 */
  function moveTab(fromPaneId: string, toPaneId: string, tabId: string): void;

  // ===== 分屏操作 =====

  /** 水平拆分（向右） */
  function splitHorizontal(paneGroupId: string, tabIdToMove?: string): void;

  /** 垂直拆分（向下） */
  function splitVertical(paneGroupId: string, tabIdToMove?: string): void;

  /** 拖拽标签到目标 PaneGroup 的边缘方向 */
  function dropTabToEdge(
    fromPaneId: string,
    tabId: string,
    targetPaneId: string,
    direction: "left" | "right" | "top" | "bottom",
  ): void;

  /** 调整分割比例 */
  function resizeSplit(splitContainerId: string, sizes: [number, number]): void;

  /** 关闭 PaneGroup（最后一个标签关闭时调用） */
  function closePaneGroup(paneGroupId: string): void;

  // ===== 快照（持久化） =====

  /** 导出可序列化的快照 */
  function exportSnapshot(): SerializableGridNode;

  /** 从快照恢复 */
  function importSnapshot(snapshot: SerializableGridNode): void;
}
```

### 6.3 核心算法

#### 拆分（Split）

```
splitHorizontal(paneGroupId):
  1. 在树中找到 target = PaneGroup(id=paneGroupId)
  2. 创建新 PaneGroup newPane = { id: uuid(), tabs: [], activeTabId: "" }
  3. 创建 SplitContainer { direction: "horizontal", children: [target, newPane], sizes: [50, 50] }
  4. 用 SplitContainer 替换原 target 在父节点中的位置
  5. 如果 tabIdToMove 指定，将标签移到新面板
  6. 更新 activePaneGroupId = newPane.id
```

#### 关闭面板（Close PaneGroup）

```
closePaneGroup(paneGroupId):
  1. target = PaneGroup(paneGroupId)
  2. parent = target 的父 SplitContainer
  3. sibling = parent.children 中另一个子节点
  4. 将 parent 替换为 sibling
  5. 如果 parent 是根节点，sibling 成为新根
```

```
└── SplitGrid（分屏网格，通过 useSplitPanes 管理）

```

### 8.3 数据流重构

```
左侧面板
  │  点击文件/视图入口
  ▼
WorkspacePage（协调层）
  │  handleOpen(entry, preferredPaneGroup?)
  ▼
useSplitPanes.openTab(activePaneGroupId, tab)
  │  在指定 PaneGroup 打开标签页
  │  view 类型 → tab = { kind: "view", viewId, ... }
  │  file 类型 → tab = { kind: "file", filePath, ... } + useWorkspaceFilePreview.openFile
  ▼
SplitGrid 响应式更新 → PaneGroup 内 TabBar + ContentArea 更新
```

### 8.4 WorkspacePage 简化后伪代码

```typescript
const splitPanes = useSplitPanes();
const preview = useWorkspaceFilePreview(workspaceDir);
const activePaneId = computed(() => splitPanes.activePaneGroupId.value);

// 左侧面板 → 中间区域：在当前活跃面板打开
function handleOpenView(viewId: string) {
  const tab: TabItem = {
    id: viewId,
    title: getViewLabel(viewId),
    kind: "view",
    viewId,
    status: "idle",
  };
  splitPanes.openTab(activePaneId.value, tab);
}

function handleOpenFile(entry: FileTreeEntry) {
  if (entry.kind === "directory") {
    toggleDirectory(entry);
    return;
  }

  // 文件类型 → useWorkspaceFilePreview 管理加载状态
  // Tab 注册到 useSplitPanes，内容由 preview.tabs 驱动
  preview.openFile(entry.path);

  const fileTab = preview.tabs.value.find((t) => t.path === entry.path);
  if (fileTab) {
    const tab: TabItem = {
      id: fileTab.id,
      title: fileTab.title,
      kind: "file",
      filePath: fileTab.path,
      status: fileTab.isLoading ? "loading" : "idle",
    };
    splitPanes.openTab(activePaneId.value, tab);
  }
}
```

## 十、布局持久化

### 10.1 持久化位置

```
~/.pi/ridge-settings.json
  └── workspace.splitLayout → SerializableGridNode
```

### 10.2 可序列化快照

```typescript
type SerializableGridNode =
  | {
      type: "pane";
      id: string;
      tabs: { id: string; kind: string; viewId?: string; filePath?: string }[];
      activeTabId: string;
    }
  | {
      type: "split";
      id: string;
      direction: "horizontal" | "vertical";
      sizes: [number, number];
      children: [SerializableGridNode, SerializableGridNode];
    };
```

### 10.3 恢复策略

1. 进入工作空间 → 读取 `ridge-settings.json` 中的 `workspace.splitLayout`
2. 若有快照 → 调用 `splitPanes.importSnapshot(snapshot)`
3. 若无快照 → 使用初始单面板布局（仅仪表盘）
4. 视图标签页立即渲染，文件标签页延迟加载内容
5. 布局变化时 300ms debounce 写入 `ridge-settings.json`

| 边界                        | 影响                                                         |
| --------------------------- | ------------------------------------------------------------ |
| **左侧面板**                | 不变。点击入口在当前活跃 PaneGroup 打开标签页                |
| **文件树**                  | 不变。useFileTreeData 完全不受影响                           |
| **视图组件**                | 不变。DashboardView/TaskView/CalendarView/InboxView 无需改造 |
| **useWorkspaceFilePreview** | 不变。文件预览加载逻辑完整保留                               |
| **TabBar 组件**             | 微调。增加拖拽事件、activePane 标记，接口向后兼容            |
| **路由**                    | 不变。依然是 `/workspace` → WorkspacePage                    |
| **其他页面**                | 不影响。chat、settings、terminal 等页面不涉及分屏            |

---

## 十四、设计原则

1. **分屏是空间分配，不是逻辑复制**：同一个文件不会在两个面板中打开两次
2. **标签页是交互单元**：一个标签页同一时刻只在一个 PaneGroup 中存在
3. **面板是可选布局**：初始单面板等价于当前体验，分屏是渐进增强
4. **不破坏现有功能**：所有 M1-M7 的视图和编辑器功能保持不变
5. **组件接口简洁**：PaneGroup 内部复用现有的 WorkspaceContentArea 和视图组件

---

## 十五、风险与注意事项

| 风险                         | 缓解                                                       |
| ---------------------------- | ---------------------------------------------------------- |
| 拖拽实现复杂度高             | 使用 HTML5 Drag and Drop API（标准、成熟），不做自定义模拟 |
| 同一文件不在两个面板重复打开 | openTab 时先检查所有 PaneGroup，若已存在则激活而非新建     |
| 布局持久化时文件可能已不存在 | 恢复时跳过无效的文件路径，只恢复视图标签页                 |
| 分屏后性能                   | reactively 已做扁平化优化，内容区 v-show 切换无额外开销    |
