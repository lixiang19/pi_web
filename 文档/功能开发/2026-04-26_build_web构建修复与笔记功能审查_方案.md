# build:web 构建修复与笔记功能审查方案

## 背景

用户要求对当前问题进行修复与审查。上一轮实现笔记页面后，验证中发现：

```bash
pnpm build:web
```

失败，错误集中在 `packages/web/vite.config.ts` 的 `test` 配置类型上。

## 探索结论

失败信息：

```text
vite.config.ts(13,3): error TS2769: No overload matches this call.
Object literal may only specify known properties, and 'test' does not exist in type 'UserConfigExport'.
```

根因：

- `packages/web/vite.config.ts` 使用 `defineConfig`，来源是 `vite`。
- 配置中包含 Vitest 的 `test` 字段。
- 当前构建链路 `vue-tsc -b` 会检查 `vite.config.ts`。
- `tsconfig.node.json` 只加载了 `node` 类型，没有激活 Vitest 对 Vite `UserConfig` 的类型扩展。
- 因此 TypeScript 认为 `test` 不是合法 Vite 配置字段。

该问题是既有问题，不是笔记功能本身引入。

## 修复方案

### 推荐修复

在 `packages/web/vite.config.ts` 顶部添加 Vitest 配置类型引用：

```ts
/// <reference types="vitest/config" />
```

理由：

- 改动最小。
- 不改变运行时逻辑。
- 明确表达该 Vite 配置文件同时承载 Vitest 配置。
- 不需要修改 tsconfig 边界。

## 审查方案

修复构建后，委派 reviewer 子 agent 对本次笔记功能进行审查，重点检查：

1. 服务端 notes API 路径安全边界
   - 是否只允许访问 `chatProjectPath` 内文件。
   - 是否阻止绝对路径和 `..` 逃逸。
   - 是否只允许 `.md` / `.markdown`。
2. 前后端协议一致性
   - `NoteListItem` / `NoteContentResponse` / `NoteSaveResponse` / `NoteCreateResponse` 与服务端响应一致。
3. Milkdown 集成正确性
   - `MilkdownProvider` 与 `useEditor` 的层级正确。
   - CSS 导入路径正确。
   - 切换文件重新挂载逻辑可用。
4. UI 与产品范围
   - 中文文案。
   - 没有重命名、删除、双链、标签、图谱等越界功能。
   - 没有硬编码颜色。
5. 导航和路由
   - 侧边栏 `notes` 入口。
   - `/notes` 路由。

## 验证命令

修复后执行：

```bash
pnpm lint
pnpm typecheck
pnpm build:server
pnpm build:web
```

如审查发现问题，再委派 implementer 继续修复。

## 影响范围

预计只修改：

- `packages/web/vite.config.ts`

审查阶段不直接修改代码，除非发现必须修复的问题并再次进入执行。

## 回退策略

如修复不符合预期，删除 `vite.config.ts` 顶部的 Vitest 类型引用即可恢复原状。
