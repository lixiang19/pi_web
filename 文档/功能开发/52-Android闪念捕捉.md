# 52 Android 闪念捕捉

## 目标

实现 Android 第一版核心能力：文字、录音、拍照和相册图片捕捉，并写入现有闪念与临时附件生命周期。

## 范围

- 捕捉页支持文字输入。
- 支持录音、预览、删除、重新录制。
- 支持拍照。
- 支持从相册选择图片。
- 文字、录音、图片可组合保存为同一条闪念。
- 保存后写入真实 `fleeting_notes`。
- 附件写入真实 `fleeting_attachments`，文件落在 `.ridge/fleeting-attachments/{noteId}/`。
- 上传失败时保留本地待发送草稿，可手动重试。

## 不做

- 不做短视频。
- 不做本地 AI 转写。
- 不做移动端独立闪念模型。
- 不绕过现有 fleeting analysis 和附件迁移流程。
- 不把临时附件暴露给文件页、RAG 或 MCP。

## TDD 与验证

先写测试：

- 移动端捕捉草稿可添加文字、音频、图片。
- 删除附件不会删除其他草稿内容。
- 录音状态机：`idle -> recording -> preview -> uploading -> done/failed`。
- 拍照和相册返回结果能转为待上传附件。
- 服务端移动捕捉成功后创建闪念和附件记录。
- 附件上传失败时不删除本地草稿。
- 部分失败不能产生孤儿附件或半条不可处理闪念。

再实现：

- 捕捉页 UI。
- 媒体采集封装。
- 本地待发送队列。
- 复用或新增窄接口 `POST /api/mobile/captures`，内部仍写 `fleeting_notes` 和 `fleeting_attachments`。

## 验收标准

- 真机可创建文字闪念。
- 真机可录音并作为闪念附件上传。
- 真机可拍照并作为闪念附件上传。
- 真机可从相册选图并作为闪念附件上传。
- 断网时录音和图片不丢失，恢复网络后可重试。
- 服务端现有闪念列表能看到移动端创建的闪念和附件。
- 修改 `.ts` / `.vue` / `.js` 后根目录 `npm run check` 通过。
