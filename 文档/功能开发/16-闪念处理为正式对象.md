# 16 闪念处理为正式对象

## 目标

实现用户确认后把闪念处理为正式对象。

## 范围

- 处理为任务。
- 处理为里程碑。
- 处理为剪藏。
- 处理为正式附件。
- 可在处理时选择项目。
- 处理成功后删除原闪念。
- 附件迁移到正式附件。

## 不做

- 不在采集阶段选择项目。
- 不允许 AI 建议自动执行。

## 验收

- 目标对象创建成功后才删除闪念。
- 处理失败保留原闪念。
- 处理为剪藏或附件时进入文件/RAG流程。
- 处理为任务或里程碑时可选择项目。

## Spec 提取点

- 各处理动作 API。
- 成功事务边界。
- 附件迁移规则。

## Spec 草案

### API

- `POST /api/workspace/fleeting/:id/process`

### 目标类型

- task
- milestone
- clipping
- attachment

### 行为

- 处理动作必须由用户触发。
- 创建目标对象和附件迁移必须在同一成功边界内。
- 成功后删除闪念记录。
- 失败时不删除闪念和临时附件。

### 测试

- 处理为任务创建任务并删除闪念。
- 处理为附件迁移文件到 `附件/`。
- 目标创建失败时闪念仍存在。
- 处理时可选择项目。
