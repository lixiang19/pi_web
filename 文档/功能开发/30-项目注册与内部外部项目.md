# 30 项目注册与内部外部项目

## 目标

实现项目创建、注册、归档和删除规则。

## 范围

- 内部项目。
- 服务器文件夹项目。
- GitHub URL 拉取项目。
- 桌面本机文件夹项目注册入口预留。
- 项目绑定设备。
- 同设备同路径唯一。
- 归档项目。
- 删除注册不删除项目文件。

## 不做

- GitHub 不做登录。
- 项目有会话时禁止删除。
- 注册后不能改设备、路径、名称。

## 验收

- 内部项目创建在 `~/ridge-workspace/项目/<name>`。
- GitHub 项目拉取到 `~/ridge-projects/<repo-name>`。
- 外部项目不进入 RAG。
- 内部项目进入 RAG 和隐藏版本管理。

## 关联设计

- `文档/项目设计/项目创建注册与归档删除.md`
- `文档/项目设计/项目管理与会话任务.md`

## Spec 提取点

- projects 表字段。
- 项目注册 API。
- 删除/归档规则。

## Spec 草案

### API

- `POST /api/workspace/projects/internal`
- `POST /api/workspace/projects/server-folder`
- `POST /api/workspace/projects/github`
- `PATCH /api/workspace/projects/:id`
- `DELETE /api/workspace/projects/:id`

### 行为

- 项目必须绑定设备。
- 同设备同路径唯一。
- GitHub 无权限时报错。
- 有会话的项目禁止删除。

### 测试

- 内部项目创建目录。
- 重复路径注册失败。
- 归档项目会话只读。
- 删除注册不删除真实文件。
