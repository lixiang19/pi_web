# 2026-04-28 文件页极简重新设计

## 用户故事

用户在 ridge 工作台点击侧边栏「文件」入口，进入一个**简洁直观的文件管理器**。浏览工作区目录结构、查看文件夹内容、点击文件快速预览。不需要编辑、不需要复杂操作，只需干净的浏览体验。

**场景**：想快速看一眼配置文件内容、看看项目有哪些文件、浏览图片。

## 界面结构

```
┌─────────────────────────────────────────────────────────┐
│  页头：Files 标题 | 统计（N 文件 / M 文件夹）｜ 面包屑     │
├───────────────────────────────────┬─────────────────────┤
│  操作栏：搜索 | 排序 | 新建/上传   │  预览区              │
├───────────────────────────────────┤  文件元信息卡片      │
│                                    │  · 文件名           │
│  文件网格                           │  · 大小 / 修改时间   │
│  ┌──────┐ ┌──────┐ ┌──────┐       │  ─────────         │
│  │  📁  │ │  📁  │ │  📄  │       │  只读内容渲染       │
│  │ src  │ │ docs │ │ read │       │  （文本/代码/图片/   │
│  │      │ │      │ │ .md  │       │   HTML/大文件）     │
│  └──────┘ └──────┘ └──────┘       │                     │
│  ┌──────┐ ┌──────┐                │                     │
│  │  🖼️  │ │  📄  │                │                     │
│  │logo  │ │config│                │                     │
│  └──────┘ └──────┘                │                     │
│                                    │                     │
└───────────────────────────────────┴─────────────────────┘
```

### 布局

- **两栏**：左侧文件管理区（flex-1）+ 右侧预览区（520px，同当前宽度）
- 左右通过 `border-l` 分割，符合 ridge 分割线规范

### 页头

- 面包屑升级为视觉主元素，使用更清晰的路径层次
- 右侧显示当前目录统计：`N 个文件 · M 个文件夹`
- 工作区根路径以辅助信息展示

### 文件卡片

- **无边框**，使用 `bg-card` + 微妙阴影区分层次
- Hover 时阴影提升 + `bg-accent/10`，符合 ridge 交互规范
- 圆角 8px（ridge 友好圆角）
- 大图标（28px），目录使用 primary 色，文件使用 muted-foreground
- 元信息行：文件名（主）+ 大小 | 修改时间（副）

### 预览区

- **极简**：不使用 WorkbenchOperationPanel 的标签系统
- 顶部：文件元信息卡片（图标 + 文件名 + 大小 + 修改时间）
- 底部：只读内容渲染，直接复用 WorkbenchReadonlyFilePreview
- 不提供编辑、下载、AI 动作等按钮
- 未选中文件时展示简洁空态

### 操作栏

- 搜索框 + 排序下拉
- 新建文件夹 / 新建文件 / 上传（保留当前功能）
- 刷新按钮

## 数据流

```
/files 路由
  -> useFilesRouteState()（不变）
  -> useFileManager(rootDir)（不变）
  -> 文件网格渲染
  -> 文件点击 -> openFile(path) -> 预览区渲染
```

数据层完全不变。只改 UI 层：

- `FilesPage.vue`：重新设计布局和样式
- `FileGrid.vue`：卡片无边框化
- 新增 `FilePreviewPanel.vue`：替代 WorkbenchOperationPanel，实现极简预览
- `FileManagerToolbar.vue`：简化视觉
- `FileBreadcrumbs.vue`：升级视觉

## 交互逻辑

1. **文件夹点击** -> enter directory，更新网格
2. **文件点击** -> 右侧预览区展示文件信息 + 只读内容
3. **面包屑点击** -> 跳转到对应目录
4. **拖拽移动**：保留拖拽到文件夹的能力
5. **右键菜单**：保留重命名 / 移入回收区

## 删减内容

- ❌ 去掉预览区的标签系统（WorkbenchOperationPanel）
- ❌ 去掉 Markdown 编辑能力
- ❌ 去掉 AI 动作（块级注入输入框）
- ❌ 去掉下载 / 外部打开等快捷操作
- ❌ 去掉预览区 header（文件路径显示）
- ✅ 保留新建文件/文件夹/上传/重命名/移入回收区/拖拽移动

## 技术方案

### 新组件

| 组件                             | 用途                                    |
| -------------------------------- | --------------------------------------- |
| `FilePreviewPanel.vue`           | 极简预览面板：文件元信息 + 只读内容渲染 |
| （保留）`FileGrid.vue`           | 重写样式，边框→无边框+阴影              |
| （保留）`FileBreadcrumbs.vue`    | 升级视觉层次                            |
| （保留）`FileManagerToolbar.vue` | 简化装饰                                |
| （保留）`FilesPage.vue`          | 重新编排布局                            |

### 预览内容复用

预览区的内容渲染直接复用现有组件链：

- `WorkbenchReadonlyFilePreview` — 代码/图片/HTML/大文件只读预览
- `WorkbenchMarkdownPreview` — Markdown 只读渲染（关闭 AI 动作）

不再使用 `WorkbenchOperationPanel` 的标签系统。
