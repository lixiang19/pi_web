# 55 Android 真机闭环与发布准备

## 目标

完成 Android 第一版端到端真机验证、构建产物、文档同步和归档。

## 范围

- 在真实 Android 设备上验证捕捉、对话、任务三条主路径。
- 验证低配 VPS 构建 APK 的流程。
- 建立 APK debug 构建说明。
- 建立真机测试 checklist。
- 更新模块梳理文档。
- 将 50-55 任务完成后归档。

## 不做

- 不做应用商店上架。
- 不做 iOS。
- 不做完整自动化云真机矩阵。
- 不加入第一版范围外功能。

## 依赖

- 任务 50 Android 移动端工程骨架。
- 任务 51 Android 设备注册与服务连接。
- 任务 52 Android 闪念捕捉。
- 任务 53 Android 轻对话。
- 任务 54 Android 任务查看与轻操作。

## TDD 与验证

必须执行：

- 服务端相关测试。
- 移动端单元与组件测试。
- 根目录 `npm run check`。
- Android debug APK 构建。
- 真机手动 E2E checklist。

真机 checklist：

- 配置 VPS ridge 服务地址。
- 注册 Android 设备。
- 创建文字闪念。
- 录音并上传为闪念附件。
- 拍照并上传为闪念附件。
- 断网捕捉后恢复网络重试。
- 新建普通对话并收到流式回复。
- 发送带图片或录音附件的对话。
- 查看任务列表和详情。
- 修改任务状态。
- 打开或继续任务处理会话。

## 验收标准

- Android 第一版只暴露捕捉、对话、任务和必要设置。
- 捕捉、对话、任务三条主路径在真机通过。
- APK debug 构建可复现。
- 低配 VPS 不需要跑 Android 模拟器。
- 文档更新到相关模块梳理。
- 任务文档归档到 `文档/功能开发/归档/`。
- 根目录 `npm run check` 通过。
