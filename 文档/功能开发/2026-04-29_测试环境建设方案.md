# 测试环境建设方案

## 现状

| 维度             | 前端 (web)                           | 后端 (server)                  | 协议 (protocol) |
| ---------------- | ------------------------------------ | ------------------------------ | --------------- |
| 框架             | Vitest 4.1.4 + @vue/test-utils 2.4.6 | node:test + node:assert/strict | ❌ 无           |
| 环境             | jsdom                                | Node 原生                      | —               |
| 已有测试         | 7 个                                 | 1 个                           | 0               |
| test script      | `"test": "vitest run"`               | ❌ 无                          | ❌ 无           |
| 全局 test script | ❌ 无                                | —                              | —               |
| CI               | ❌ 无                                | —                              | —               |

## 问题

1. **后端无 npm test script** — 无法 `pnpm test` 统一执行
2. **根目录无统一 test script** — 无法一键跑全量
3. **后端框架分散** — node:test 手动运行，缺 mock/stub 能力
4. **无共享 test utils** — mock 工厂、fixture 全部内联
5. **无 API 集成测试** — 文件树/笔记等核心 API 无自动化测试
6. **前端 setup 极简** — 缺少 router/store/plugin 注入

## 方案

### 一、统一后端测试框架为 Vitest

**理由**：前后端统一一套框架，共享断言/mock 生态，且 Vitest 对 ESM + TypeScript 支持优于 node:test。

**改动**：`packages/server/package.json`

```json
{
  "devDependencies": {
    "vitest": "^3.1.4"
  },
  "scripts": {
    "test": "vitest run",
    "test:watch": "vitest"
  }
}
```

**配置**：`packages/server/vitest.config.ts`

```ts
import { defineConfig } from "vitest/config";
export default defineConfig({
  test: {
    globals: true,
  },
});
```

### 二、根目录统一 test script

**改动**：根 `package.json`

```json
{
  "scripts": {
    "test": "pnpm -r run test",
    "test:watch": "pnpm -r run test:watch"
  }
}
```

### 三、后端测试基础设施

#### 目录结构

```
packages/server/
├── vitest.config.ts
├── src/
│   ├── __tests__/
│   │   ├── file-manager.test.ts      (已有，迁移到 vitest)
│   │   ├── file-tree-api.test.ts      (新增：API 路由集成测试)
│   │   └── notes-api.test.ts          (新增：笔记 API 测试)
│   └── test/
│       ├── helpers.ts                 (新增：createTempDir, createTestServer 等)
│       └── fixtures/                  (新增：测试用文件)
```

#### helpers.ts

```ts
import fs from "node:fs/promises";
import os from "node:os";
import path from "node:path";
import { createFileManager } from "../file-manager.js";

export const createTempDir = async (prefix = "ridge-test-") => {
  const dir = await fs.mkdtemp(path.join(os.tmpdir(), prefix));
  return dir;
};

export const createTempFileManager = async () => {
  const root = await createTempDir("ridge-fm-");
  const manager = createFileManager({
    defaultWorkspaceDir: root,
    ensureManagedProjectScope: async () => undefined,
  });
  return {
    manager,
    root,
    cleanup: () => fs.rm(root, { recursive: true, force: true }),
  };
};
```

### 四、前端测试基础设施

#### 补充 setup.ts

```ts
// packages/web/src/test/setup.ts
import { afterEach, vi } from "vitest";

afterEach(() => {
  document.body.innerHTML = "";
  vi.restoreAllMocks();
});

// Mock fetch globally
globalThis.fetch = vi.fn();
```

#### 新增 test-utils

```ts
// packages/web/src/test/test-utils.ts
import { render, type RenderOptions } from "@vue/test-utils";
import { createRouter, createMemoryHistory } from "vue-router";

export const createTestRouter = () =>
  createRouter({
    history: createMemoryHistory(),
    routes: [{ path: "/", component: { template: "<div/>" } }],
  });

export const renderWithRouter = (component: any, options?: RenderOptions) => {
  const router = createTestRouter();
  return render(component, { ...options, global: { plugins: [router] } });
};
```
