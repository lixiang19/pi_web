# 子代理能力补全计划

- 日期：2026-04-28
- 状态：规划中
- 关联分析：[子代理系统对比分析](./2026-04-28_子代理系统对比分析_pi-subagents-vs-task.md)
- 关联方案：[内嵌子代理运行时方案](./2026-04-11_内嵌子代理运行时_方案.md)
- 涉及文件：`packages/server/src/subagents.ts`

## 背景

对比 `@tintinweb/pi-subagents` 后，项目 task 子代理在权限体系上有明显优势，但在运行时管理与输出质量上存在 4 项短板。本次补全聚焦这 4 项，不引入过度设计。

## 引入清单

| #   | 能力            | 优先级 | 说明                                                     |
| --- | --------------- | ------ | -------------------------------------------------------- |
| 1   | **优雅关闭**    | P0     | max_turns 到达时先 steer 提醒收尾，给宽限期，再 abort    |
| 2   | **会话恢复**    | P1     | 已完成子代理可 resume 复用 session 继续追问              |
| 3   | **并发队列**    | P1     | 可配置并发上限，超出排队                                 |
| 4   | **prompt_mode** | P1     | 支持 append 模式，子代理继承父系统 prompt 成为"父双胞胎" |

## 详细设计

### 1. 优雅关闭（Graceful Max Turns）

**现状**：到达 max_turns 直接 `session.abort()`

```
while (true) {
  turnCount++
  if (maxTurns && turnCount >= maxTurns) {
    session.abort()  // 一刀切
  }
}
```

**目标**：三阶段关闭

```
turnCount >= maxTurns
  → steer("请立即收尾，给出当前结果")     // 第1阶段
  → 继续等待最多 3 轮                    // 第2阶段（graceTurns=3）
  → 仍未完成则 abort                      // 第3阶段
```

**改动点**：

- `ChildSessionEntry` 增加 `graceTurnsUsed: number` 字段
- `runChildSession` 中修改 abort 逻辑为三阶段
- 允许在 agent frontmatter 中配置 `grace_turns`（默认 3）

### 2. 会话恢复（Resume）

**现状**：`task` 工具无 `resume` 参数

**目标**：传入 `resume: "sessionId"` 复用已有 child session

```
Agent: task({
  agent: "explore",
  prompt: "再补充一下搜索结果",
  resume: "abc-123"   // 指向已完成子代理
})
```

**改动点**：

- `task` 工具增加 `resume` 参数
- 校验：sessionId 存在、且属于当前父会话
- 若 session 已完成，通过 `session.prompt()` 继续
- session 状态从 `completed` 变为 `running`

### 3. 并发队列

**现状**：无并发控制

**目标**：可配置上限，超出自动排队

```ts
const queue = new Map()           // waiting queue
const running = new Set()         // currently running
const MAX_CONCURRENT = 4          // configurable

spawn(entry):
  if running.size >= MAX_CONCURRENT:
    queue.set(entry.sessionId, entry)  // 排队
  else:
    running.add(entry.sessionId)
    execute(entry)

onComplete(entry):
  running.delete(entry.sessionId)
  if queue.size > 0:
    next = queue第一个
    move from queue to running
    execute(next)
```

**改动点**：

- `childRegistry` 旁边加 `running` 计数
- `task` 后台模式返回 `queued` 状态
- 允许通过环境变量或配置覆盖 `MAX_CONCURRENT`

### 4. prompt_mode（append 模式）

**现状**：子代理系统 prompt 固定为 agent 自己的

```ts
const childSystemPrompt = buildSubagentSystemPrompt(
  agent,
  parentRecord,
  inheritContext,
);
// = "You are an internal subagent..." + agent.systemPrompt + context
```

**目标**：agent frontmatter 增加 `prompt_mode` 字段

| 值                | 行为                                              |
| ----------------- | ------------------------------------------------- |
| `replace`（默认） | 仅使用 agent 自己的 systemPrompt                  |
| `append`          | agent.systemPrompt **追加到父** systemPrompt 之后 |

**改动点**：

- `AgentConfig` 类型增加 `promptMode?: 'replace' | 'append'`
- `buildSubagentSystemPrompt` 支持 `append` 模式，读取父 session 的 system prompt 前缀
- frontmatter 新增 `prompt_mode` 字段解析

## 实施顺序

```
M1: 优雅关闭 → 缓解 max_turns 截断，最小改动最大收益
M2: 会话恢复 → 解决已完成子代理不可追问的问题
M3: 并发队列 + prompt_mode → 架构补强
```

## 验收标准

### M1 验收

- [ ] 子代理到达 max_turns 后收到 steer 消息
- [ ] 宽限期内可继续执行并正常返回结果
- [ ] 超出宽限期后 abort，状态标记为 `steered`（非 `error`）
- [ ] agent frontmatter 支持 `grace_turns` 配置

### M2 验收

- [ ] `task({ resume: sessionId })` 复用已有 child session
- [ ] 非亲生子代理 session 不可 resume（403）
- [ ] 运行中子代理不可 resume（与 steer 区分）
- [ ] 恢复后 child session 状态回到 running

### M3 验收

- [ ] 后台子代理超过并发上限时进入排队
- [ ] 运行中完成一个后自动启动下一个排队任务
- [ ] 最大并发数可配置
- [ ] `prompt_mode: append` 子代理收到父系统 prompt + 自身 prompt
- [ ] `prompt_mode: replace` 行为不变（默认）
