## M1 — 骨架跑通

**目标**：打开 `/workspace` 能看到完整的左侧面板+中间标签页，点击文件能预览。

### Spec

#### 1.1 左侧面板

- 搜索栏：静态 UI，输入框，暂不接搜索逻辑
- 4 个固定视图入口：仪表盘/待办/日历/收件箱，点击打开标签页（内容区显示占位文字）
- 分割线
- 文件树：使用 `useFileTreeData` + `FileTreePanel`，根目录 = `core.info.value.workspaceDir`
- 底部新建按钮：静态 UI，暂不接逻辑

#### 1.2 标签页系统

- `TabBar` 渲染标签页列表
- 仪表盘标签页默认存在且不可关闭
- 点击文件树文件 → 新建标签页或激活已有
- 点击固定视图 → 新建标签页或激活已有
- 关闭标签页（仪表盘除外）

#### 1.3 文件内容渲染

| 文件类型   | 渲染方式                          | 组件                                                     |
| ---------- | --------------------------------- | -------------------------------------------------------- |
| `.md`      | 只读 Markdown 预览（M1 不做编辑） | 复用 `WorkbenchMarkdownPreview`（enableAiActions=false） |
| 文本/代码  | 只读语法高亮                      | 复用 `WorkbenchReadonlyFilePreview`                      |
| 图片       | 图片渲染                          | 复用 `WorkbenchReadonlyFilePreview`                      |
| HTML       | 沙箱 iframe                       | 复用 `ReadonlyHtmlPreview`                               |
| 不支持类型 | 「使用默认应用打开」按钮          | 新建 `OpenWithDefaultApp.vue`                            |
| 大文件     | 分页加载                          | 复用 `WorkbenchReadonlyFilePreview`                      |

**`OpenWithDefaultApp.vue`**：

- Props：`filePath: string`, `fileName: string`
- 渲染：居中显示文件图标+文件名+按钮「使用默认应用打开」
- 点击按钮：调用 `POST /api/files/open` → 用系统默认应用打开

#### 1.4 文件内容加载

- 点击文件 → 调用 `getFilePreview(path, root)` 获取内容
- 将结果适配为标签页的 tab state
- 复用 `useWorkbenchFilePreview` 的数据逻辑，或新建 `useWorkspaceFilePreview`

### 文件变更

| 操作 | 文件                                                                              |
| ---- | --------------------------------------------------------------------------------- |
| 修改 | `pages/WorkspacePage.vue`（已有骨架，充实内容渲染）                               |
| 新建 | `components/workspace/WorkspaceContentArea.vue`（标签页内容路由）                 |
| 新建 | `components/workspace/OpenWithDefaultApp.vue`                                     |
| 新建 | `composables/useWorkspaceFilePreview.ts`                                          |
| 复用 | `WorkbenchReadonlyFilePreview`、`WorkbenchMarkdownPreview`、`ReadonlyHtmlPreview` |

## M2 — 编辑器闭环

**目标**：.md 文件可编辑、可保存，体验完整。

### Spec

#### 2.1 Milkdown 编辑器

- 复用 `NoteMilkdownEditor` + `NoteMilkdownEditorInner`
- 适配为工作空间标签页的内容组件
- .md 标签页状态：`idle | loading | unsaved | saving | error`
- 标签页有未保存指示圆点

#### 2.2 自动保存

- 编辑后 2s debounce 自动保存
- 调用 `PUT /api/notes/content`（已有 API）
- 关闭标签页时 flush 保存
- 保存失败保持 dirty 标记

#### 2.3 文件内容加载

- 打开 .md 文件 → 调用 `GET /api/notes/content`（已有 API）
- 将内容传入 Milkdown 编辑器

#### 2.4 新建文件

- 底部「新建」菜单可用
- 「新笔记」→ 在 `笔记/` 目录新建 .md，打开编辑
- 「闪念笔记」→ 在 `收件箱/` 目录新建 .md，打开编辑
- 「今日日记」→ 按日期路径新建 .md，打开编辑
- 「新文件夹」→ 新建目录
- 调用 `POST /api/notes`（已有 API）

#### 2.5 文件操作

- 重命名：右键菜单 → 输入框 → `PATCH /api/notes/rename`
- 删除：右键菜单 → 确认 → `DELETE /api/notes`
- 拖拽移动：拖到文件夹 → `PUT /api/files/entries/path`

### 文件变更

| 操作 | 文件                                                                |
| ---- | ------------------------------------------------------------------- |
| 新建 | `components/workspace/WorkspaceMarkdownEditor.vue`（封装 Milkdown） |
| 修改 | `pages/WorkspacePage.vue`（.md 标签页走编辑器路径）                 |
| 修改 | 新建菜单可用                                                        |
| 移动 | `components/notes/NoteMilkdownEditor.vue` → `components/workspace/` |

## M4 — 待办系统

**目标**：工作空间有自己的待办管理，聚合两个来源。

### Spec

#### 4.1 独立任务 DB

- 后端新建 `workspace_tasks` 表：
  ```
  id, title, status(pending/in_progress/done),
  priority(low/medium/high), due_date, tags, note_path(可空),
  created_at, updated_at
  ```
- API：`GET/POST /api/workspace/tasks`，`PATCH /api/workspace/tasks/:id`，`DELETE /api/workspace/tasks/:id`

#### 4.2 Checkbox 聚合引擎

- 扫描工作空间下所有 .md 文件
- 正则提取 `- [ ] xxx` 和 `- [x] xxx` 行
- 解析行内元数据：`截止: YYYY-MM-DD`、`优先级: 高/中/低`、`#标签`
- 返回：`{ text, done, dueDate, priority, tags, sourcePath, lineNumber }`
- 后端 API：`GET /api/workspace/tasks/checkboxes`

#### 4.3 聚合接口

- 前端合并两个来源：
  ```
  allTasks = [...independentTasks, ...checkboxTasks]
  ```
- 每个 task 统一属性：`{ id, title, status, priority, dueDate, tags, source }`
- `source` 标注来源：`{ type: 'checkbox', path, line }` 或 `{ type: 'independent' }`

#### 4.4 待办视图

- **清单模式**（默认）：
  - 分组：今日到期 / 已逾期 / 无截止日
  - 每项显示：标题 + 优先级色标 + 截止日 + 标签 + 来源
  - 点击 checkbox 切换完成状态
  - 点击来源 → 打开对应 .md 文件标签页并跳转到行

- **看板模式**：
  - 三列：待做 | 进行中 | 完成
  - 独立任务可拖拽改列
  - Checkbox 来源的任务只能改完成/未完成

- **今日模式**：
  - 过滤：今日截止 + 今日创建 + 已逾期未完成

#### 4.5 新建独立任务

- 待办视图顶部有「+ 新建任务」按钮
- 输入标题即可创建，存入 DB
- 可选填：截止日、优先级、标签

## M7 — 数据库 Bases ⭐

**目标**：`.base` 文件可在文件树创建，点击打开数据库视图，支持表格/看板/画廊/日历。

### Spec

#### 7.1 .base 文件格式

```json
{
  "name": "项目追踪",
  "columns": [
    { "id": "c1", "name": "名称", "type": "text" },
    {
      "id": "c2",
      "name": "状态",
      "type": "select",
      "options": ["待做", "进行中", "完成"]
    },
    { "id": "c3", "name": "优先级", "type": "number" },
    { "id": "c4", "name": "截止日", "type": "date" }
  ],
  "sources": [{ "type": "folder", "path": "项目/" }],
  "rows": [
    {
      "id": "r1",
      "type": "independent",
      "cells": { "c1": "买域名", "c2": "待做", "c3": 3, "c4": "2026-04-30" }
    },
    { "id": "r2", "type": "file", "path": "项目/工作空间设计.md" }
  ],
  "views": [
    {
      "id": "v1",
      "name": "表格",
      "type": "table",
      "sort": { "column": "c3", "direction": "asc" },
      "filters": []
    },
    { "id": "v2", "name": "看板", "type": "kanban", "groupColumn": "c2" }
  ],
  "activeViewId": "v1"
}
```

#### 7.2 后端 API

- `GET /api/workspace/base?path=xxx.base` — 读取 .base 文件
- `PUT /api/workspace/base` — 保存 .base 文件
- `POST /api/workspace/base` — 创建 .base 文件
- `DELETE /api/workspace/base?path=xxx.base` — 删除

文件引用行：打开时后端自动读取引用 .md 的 frontmatter，合并为行数据返回。

#### 7.3 列类型

M7 首版实现：

| 类型        | 渲染        | 编辑               |
| ----------- | ----------- | ------------------ |
| text        | 文本        | input              |
| number      | 数字        | input[type=number] |
| date        | 日期字符串  | date picker        |
| select      | 下拉标签    | select             |
| multiselect | 标签列表    | 多选下拉           |
| checkbox    | ✅/☐        | toggle             |
| file        | 文件名+链接 | 点击打开           |

M7 暂不做：longtext、url、formula

#### 7.4 表格视图

- 行 = rows，列 = columns
- 列头可排序
- 行内双击可编辑单元格
- 行头可拖拽排序
- 底部有「+ 新增行」按钮
- 新增行可选「关联文件」或「独立行」
- 右键行：删除行

#### 7.5 看板视图

- 按 select 类型列分组
- 每组一列
- 卡片显示：标题 + 关键字段
- 独立任务可拖拽改组

#### 7.6 画廊视图

- 点击卡片 → 展开详情（侧滑面板或模态框）

#### 7.7 日历视图

- 按 date 类型列在月历上展示
- 每天显示该日期的行列表
- 点击 → 打开详情

#### 7.8 视图管理

- 数据库标签页顶部有视图切换标签
- 可新增/删除/重命名视图
- 每个视图独立的 sort + filter 配置

## M8 — 白板 Canvas ⭐

**目标**：`.canvas` 文件可在文件树创建，点击打开白板画布。

### Spec

#### 8.1 .canvas 文件格式

```json
{
  "nodes": [
    {
      "id": "1",
      "type": "card",
      "x": 100,
      "y": 200,
      "width": 300,
      "height": 200,
      "content": "# 标题\n内容"
    },
    {
      "id": "2",
      "type": "note",
      "x": 500,
      "y": 200,
      "width": 320,
      "height": 400,
      "path": "笔记/xxx.md"
    },
    {
      "id": "3",
      "type": "group",
      "x": 50,
      "y": 100,
      "width": 800,
      "height": 600,
      "label": "分组"
    }
  ],
  "edges": [
    {
      "id": "e1",
      "fromNode": "1",
      "fromSide": "right",
      "toNode": "2",
      "toSide": "left"
    }
  ],
  "viewport": { "x": 0, "y": 0, "zoom": 1 }
}
```

#### 8.2 后端 API

- `GET /api/workspace/canvas?path=xxx.canvas` — 读取
- `PUT /api/workspace/canvas` — 保存
- `POST /api/workspace/canvas` — 创建
- `DELETE /api/workspace/canvas?path=xxx.canvas` — 删除

#### 8.3 画布渲染

- 基于 HTML Canvas 或 SVG 渲染（推荐 SVG，交互更简单）
- 无限画布：支持平移和缩放
- 视口状态持久化到 .canvas 文件

#### 8.4 节点类型（M8 首版）

| 类型  | 渲染                         | 交互                   |
| ----- | ---------------------------- | ---------------------- |
| card  | 文本卡片，Markdown 渲染      | 双击编辑文本           |
| note  | 嵌入 .md 笔记，显示标题+摘要 | 点击标题打开笔记标签页 |
| group | 分组框+标题                  | 拖入其他节点           |

M8 暂不做：图片节点、数据库嵌入、手绘

#### 8.5 连线

- 从节点边缘拖出连线
- 连线类型：直线
- 箭头指示方向
- 点击连线 → 删除

#### 8.6 交互

- 鼠标拖拽移动节点
- 框选多个节点
- 双击空白 → 新建 card
- 右键菜单：删除/锁定/复制/新建连线
- Ctrl+滚轮缩放
- 空格+拖拽平移
- 从文件树拖拽 .md 文件 → 创建 note 节点

#### 8.7 保存

- 操作后 1s debounce 自动保存
- 关闭标签页时 flush

### 验收标准

- [ ] 新建 .canvas 文件，出现在文件树
- [ ] 点击 .canvas → 白板画布打开
- [ ] 双击空白新建 card，可输入文本
- [ ] 拖拽移动节点
- [ ] 节点间可连线
- [ ] 从文件树拖 .md 文件创建 note 节点
- [ ] 缩放/平移画布
- [ ] 保存后 .canvas 文件更新

## M9 — 链接+标签

**目标**：`[[双向链接]]` 和 `#标签` 串联所有内容。

### Spec

#### 9.1 双向链接

- Milkdown 编辑器中输入 `[[` → 弹出搜索补全菜单
- 补全来源：工作空间下所有 .md 文件名
- 选中 → 插入 `[[文件名]]`
- 渲染 `[[xxx]]` 为可点击链接，点击 → 打开对应 .md 标签页
- 后向引用：打开笔记时底部显示「引用此笔记的其他笔记」列表

#### 9.2 标签系统

- 编辑器中输入 `#` → 弹出已有标签补全
- 标签来源：扫描所有 .md 的 frontmatter `tags` 字段
- 点击 `#标签` → 搜索所有含该标签的笔记
- 标签云视图（可选）

#### 9.3 跨模块关联

- 待办任务可以打标签，和笔记标签统一
- 数据库行可以引用笔记
- 标签成为跨模块的搜索/过滤纽带

### 验收标准

- [ ] `[[` 触发搜索补全
- [ ] `[[xxx]]` 渲染为可点击链接
- [ ] 点击链接打开对应笔记
- [ ] `#` 触发标签补全
- [ ] 点击标签搜索相关内容
- [ ] 笔记底部显示反向引用

## M10 — 右侧 AI 会话

**目标**：工作空间右侧有 AI 对话面板，可和当前打开的内容联动。

### Spec

#### 10.1 右侧面板

- 固定宽度面板，默认折叠
- 展开/折叠按钮在中间区域右侧边缘
- 内容：复用 `WorkbenchChatPanel` 核心组件
- 一个工作空间 = 一个 AI 会话

#### 10.2 内容联动（v2 预留）

- 暂不做自动上下文注入
- 用户可手动在输入框中 `@文件名` 引用当前打开的文件
- 后续版本考虑自动附带当前标签页的文件路径

### 验收标准

- [ ] 右侧面板可展开/折叠
- [ ] AI 会话可发送消息
- [ ] 会话上下文为工作空间目录
