# 笔记编辑体验 Level 2 范围与验收标准 v0.1

> 任务：task #3 非 AI 会话部分方向收敛  
> 日期：2026-05-01  
> 产品决策：先做笔记编辑体验，目标到 Level 2（日常可用），但拆成 B1/B2 两个可开发切片。  
> 约束：不占用 `项目/` 目录语义，不改 Dashboard 新设计，不扩 AI 会话链路。

## 1. 目标

让用户敢把真实内容写进 ridge：能稳定新建、打开、编辑、保存 markdown 笔记，并在日常使用中完成目录选择、文件树管理、多标签切换和保存状态感知。

Level 2 的定义不是“高级知识库”，而是：**日常笔记工作可以放心使用**。

## 2. 分层范围

### B1：可靠写作基础

目标：确保写进去的内容不会丢，保存状态真实可感知。

开发侧核查：B1 已有 60–70% 基础能力，因此开发任务不应重写编辑器，而应聚焦“可靠性保护 + 测试”。

范围：
- 打开 `.md` 文件进入 markdown 编辑器。
- 编辑后自动保存。
- 显示保存状态：未保存 / 保存中 / 已保存 / 保存失败。
- 刷新或重新打开后内容仍存在。
- 多标签切换不丢当前编辑状态。
- 保存失败时有明确 toast 或错误提示，并提供重试路径。
- 关闭标签时如果处于 `unsaved/saving/error`，需要 flush 成功再关闭，或阻止/确认，不能静默丢内容。
- 组件卸载前清理自动保存 timer，必要时 flush 待保存内容。
- 不支持的文件只预览，不误进入编辑。

### B2：日常文件组织与多标签体验

目标：让用户能在 `笔记/` 等非项目目录里顺畅组织笔记。

范围：
- 新建笔记时可以选择目录和标题。
- 默认新建笔记进入 `笔记/`，不进入根目录，也不进入 `项目/`。
- 文件树里新建文件夹、重命名、删除、打开笔记的反馈清楚。
- 多标签打开多个笔记，标签标题与文件名同步。
- 编辑器保存状态与标签页状态同步，例如未保存/保存中标记。
- 暗色/亮色主题下编辑区、占位、错误态均可读。

## 3. 暂不做

- 不做 Dashboard 新设计。
- 不改 AI 会话、Agent、Pi session 接线。
- 不占用 `项目/` 目录；`项目/` 保留给未来特殊“创建项目”能力。
- 不做 markdown checkbox 聚合到待办。
- 不做双链、反链、知识图谱。

- 不做双栏编辑/预览作为第一批目标。

## 4. 信息架构与目录策略

### 目录语义

- `笔记/`：普通长期笔记，B1/B2 默认新建位置。
- `收件箱/`：临时捕捉，后续可整理到 `笔记/`、`阅读/`、`归档/` 等。
- `日记/`：按日期生成的日记。
- `阅读/`：阅读笔记、资料摘录。
- `归档/`：不活跃但保留的内容。
- `项目/`：冻结，后续留给特殊“创建项目”能力。

### 新建笔记规则

- 默认目录：`笔记/`。
- 用户可选目录：`笔记/`、`阅读/`、`归档/`、用户自建非冻结目录。
- 禁止默认落到 `项目/`。
- 如果用户手动选择 `项目/`，本阶段应避免暴露该入口；如已有文件树可进入，至少不把它作为推荐/默认项。

## 5. 关键用户故事

### Story 1：新建一篇可保存的笔记

作为用户，我可以新建一篇笔记，输入标题和正文，系统自动保存，并且刷新后内容还在。

验收：
- B1 不要求新增新建笔记流程；新建默认 `笔记/` 放到 B2。
- 编辑正文后出现保存状态变化。
- 刷新或重新打开后，正文内容与保存后内容一致。

### Story 2：打开已有 markdown 并继续写

作为用户，我可以从文件树打开已有 `.md` 文件，继续编辑，不会因为切换标签或刷新丢内容。

验收：
- 点击 `.md` 文件打开编辑器。
- 切换到其他标签再回来，内容仍是当前编辑内容。
- 自动保存完成后，文件内容持久化。
- 不支持的文件类型不会误用 markdown 编辑器。

### Story 3：知道保存是否成功

作为用户，我需要知道当前内容是否已经保存，保存失败时不能静默丢失。

验收：
- 编辑后能看到未保存/保存中状态。
- 保存成功后显示已保存状态或状态消退但可感知。
- 保存失败时显示错误提示。
- 保存失败后用户仍能看到当前编辑内容，并能重试或继续触发保存。

### Story 4：日常组织笔记

作为用户，我可以在非 `项目/` 目录里创建和组织笔记。

验收：
- 新建笔记时可选择非冻结目录。
- 重命名后文件树和标签标题同步。
- 删除有确认或足够明确的危险操作反馈。
- 新建文件夹后文件树刷新并可继续创建笔记。

## 6. B1 验收标准

- [ ] 打开 `.md` 文件进入 markdown 编辑器。
- [ ] 编辑内容触发自动保存。
- [ ] 保存状态至少覆盖：保存中、已保存、保存失败。
- [ ] 保存失败有可见提示，不静默失败。
- [ ] 保存失败后当前编辑内容仍保留。
- [ ] 保存失败后存在重试路径，或用户再次编辑/触发保存时可恢复。
- [ ] 切换标签不丢当前内容。
- [ ] 刷新后内容仍存在。
- [ ] 关闭 `unsaved/saving/error` 标签前有保护，不静默丢内容。
- [ ] 组件卸载前清理自动保存 timer，避免悬挂保存。
- [ ] 不支持文件类型只读预览或外部打开，不进入编辑器。
- [ ] 不改 Dashboard / AI session / `项目/` 默认规则。

## 7. B2 验收标准

- [ ] 新建笔记默认进入 `笔记/`。
- [ ] 新建笔记可填写标题，标题为空时阻止创建或给出提示。
- [ ] 新建笔记可选择目录，但不推荐/默认选择 `项目/`。
- [ ] 文件树支持新建文件夹后的刷新与继续操作。
- [ ] 重命名笔记后，文件树和标签页标题同步。
- [ ] 删除笔记有确认或明确反馈。
- [ ] 多标签打开多个笔记时，活跃标签与内容对应正确。
- [ ] 标签页能反映编辑器保存状态，至少能看到未保存/保存中风险。
- [ ] 暗色/亮色主题下编辑器正文、placeholder、错误态可读。
