# 非 Git 项目内置 Git 隔离方案

## 背景

ridge 的右侧 Git 面板当前依赖系统 git CLI + 项目目录 `.git`。两个前提都有问题：

1. **用户可能没装 git** —— `execFile('git', ...)` 直接失败。
2. **非 Git 项目无法使用 Git 面板** —— `isGitRepository()` 返回 false 就隐藏面板。

新目标：无论用户是否安装 git、项目是否初始化 git，ridge 都能提供本地变更查看与提交能力，且不污染用户项目目录。

## 目标

- 非 Git 项目也能使用 Git 状态、文件选择和本地提交。
- 用户未安装 git 也能使用内置 Git。
- 内置 Git 不在用户项目目录创建 `.git`。
- 项目后来变成真实 Git 项目后，ridge 自动切换到真实 Git。
- 内置 Git 的历史提交不进入真实 Git 的提交记录。

## 非目标

- 不把内置 Git 做成真实 remote Git 的替代品。
- 不为内置 Git 支持 push、pull、fetch、worktree。
- 不把内置 Git 注入终端或 agent shell 环境。
- 不提供兼容旧静态 `is_git` 状态的双路径逻辑。
- 不在第一版实现 branch / merge / rebase 等分支操作（后续迭代扩展）。

## 核心设计

### 双引擎架构

| 引擎         | 实现                                          | 适用场景                   | 能力边界              |
| ------------ | --------------------------------------------- | -------------------------- | --------------------- |
| **CLI 引擎** | `execFile('git', ...)`，不改动                | 系统有 git + 项目有 `.git` | 全部 Git 能力         |
| **Iso 引擎** | [isomorphic-git](https://isomorphic-git.org/) | 系统无 git 或项目无 `.git` | status / add / commit |

解析优先级：

1. 系统有 git 且项目目录存在 `.git` → 使用 CLI 引擎（现有行为不变）
2. 否则 → 使用 Iso 引擎 + 内置 gitdir

### Iso 引擎的 gitdir 分离

isomorphic-git 原生支持 `dir`（工作目录）与 `gitdir`（元数据目录）分离，不需要裸仓库技巧：

```ts
import * as isoGit from "isomorphic-git";

// 内置 Git 初始化
await isoGit.init({
  dir: projectPath, // 用户项目目录
  gitdir: path.join(dataDir, "ridge-git", projectId), // 元数据在 ridge 数据域
  defaultBranch: "main",
});

// status — 不碰项目目录的 .git
const mat = await isoGit.statusMatrix({ dir: projectPath, gitdir });
// commit
await isoGit.add({ dir: projectPath, gitdir, filepath: "src/index.ts" });
await isoGit.commit({ dir: projectPath, gitdir, message, author });
```

项目目录不会出现 `.git`，git 元数据全部在 `~/.pi/ridge-git/<project-id>/`。

### 不改造 `git-service.ts`

CLI 引擎代码路径**零修改**。`gitService` 的方法签名保持 `(cwd: string, ...)` 不变。

引擎分发在路由层完成：

```ts
// 路由层
app.get("/api/git/status", async (req, res) => {
  const cwd = resolveGitCwd(req.query.cwd);
  const ctx = resolveGitContext(cwd); // 解析仓库上下文

  if (ctx.engine === "cli") {
    res.json(await gitService.getStatus(cwd)); // CLI 引擎原样调用
  } else {
    res.json(await isoGitService.getStatus(ctx)); // Iso 引擎
  }
});
```

理由：

- `gitService` 和 `worktreeService` 只需 `cwd`，不需要 `GitRepositoryContext` 的其他字段。
- 引擎调度信息（`canPushPull`、`canWorktree`）是路由层的事，不该污染 git 执行层。
- 改 `gitService` 签名会连锁波及 `worktreeService` 和所有调用方，纯属噪声。

### Git 仓库解析（两个函数）

不做独立服务接口，就是两个普通函数：

```ts
// ---- server/src/git-resolver.ts ----

import fs from "node:fs/promises";
import path from "node:path";
import { execFile } from "node:child_process";
import { promisify } from "node:util";

const execFileAsync = promisify(execFile);

type EngineKind = "cli" | "iso";

interface GitContext {
  engine: EngineKind;
  workTree: string; // = 项目目录
  gitdir: string; // .git 或 ~/.pi/ridge-git/<id>
  canCommit: boolean; // 两种引擎都 true
  canPushPull: boolean; // 仅 cli true
  canWorktree: boolean; // 仅 cli true
  label: string; // "Git" | "ridge 内置"
}

// ===== 系统 git 可用性（进程级缓存） =====

let systemGitAvailable: boolean | null = null;

async function detectSystemGit(): Promise<boolean> {
  try {
    await execFileAsync("git", ["--version"]);
    return true;
  } catch {
    return false;
  }
}

export async function hasSystemGit(): Promise<boolean> {
  if (systemGitAvailable === null) {
    systemGitAvailable = await detectSystemGit();
  }
  return systemGitAvailable;
}

// ===== 解析仓库上下文 =====

export async function resolveGitContext(cwd: string): Promise<GitContext> {
  const projectPath = cwd;

  // CLI 引擎：系统有 git 且项目有 .git
  if (await hasSystemGit()) {
    const dotGit = path.join(projectPath, ".git");
    try {
      await fs.access(dotGit);
      return {
        engine: "cli",
        workTree: projectPath,
        gitdir: dotGit,
        canCommit: true,
        canPushPull: true,
        canWorktree: true,
        label: "Git",
      };
    } catch {
      // .git 不存在，fall through to iso
    }
  }

  // Iso 引擎
  const projectId = await resolveProjectId(projectPath);
  const isoGitdir = path.join(getDataDir(), "ridge-git", projectId);
  return {
    engine: "iso",
    workTree: projectPath,
    gitdir: isoGitdir,
    canCommit: true,
    canPushPull: false,
    canWorktree: false,
    label: "ridge 内置",
  };
}

// ===== 从 cwd 反查 projectId（服务端内部，不暴露给前端） =====

import { getProjects } from "./storage/index.js";

async function resolveProjectId(cwd: string): Promise<string> {
  const state = await getProjects();
  const normalized = path.resolve(cwd);
  const project = state.projects.find(
    (p) => path.resolve(p.path) === normalized,
  );
  // 找不到匹配项目时用目录名 hash 作为 fallback
  return project?.id || Buffer.from(normalized).toString("hex").slice(0, 16);
}
```

解析流程简单直接：检查系统 git → 检查项目 `.git` → 有则 cli，无则 iso。每次 API 调用时实时检测，因为 `fs.access` 是毫秒级的，不需要缓存。

### commit author 身份

CLI 引擎沿用系统 `git config user.name/user.email`。

Iso 引擎硬编码 ridge 身份：

```ts
const RIDGE_AUTHOR = {
  name: "ridge",
  email: "ridge@local",
};
```

内置 Git 的提交历史不考虑与真实 Git 互操作，固定身份最简洁。

### engine 切换机制

用户外部执行 `git init` 后：

1. 点击 Git 面板 refresh 按钮
2. 触发 `/api/git/is-repo` → `resolveGitContext` 检测到 `.git` 存在 → 返回 `engine: "cli"`
3. 前端 `useGitRepositoryStatus` 响应式更新 → 面板切换到 CLI 模式（显示 push/pull 等）

无需轮询，无需自动检测。与现有手动 refresh 行为一致。

## 服务端改动

### 新增 `iso-git-service.ts`

第一版仅实现 status / add / commit：

```ts
import * as isoGit from "isomorphic-git";
import fs from "node:fs";

// 注册 node:fs 后端
isoGit.plugins.set("fs", fs);

const RIDGE_AUTHOR = { name: "ridge", email: "ridge@local" };

export function createIsoGitService() {
  return {
    /** 初始化内置 gitdir */
    async init(gitdir: string): Promise<void> {
      await fs.promises.mkdir(gitdir, { recursive: true });
      await isoGit.init({ dir: "/", gitdir, defaultBranch: "main" });
    },

    /** 确保 gitdir 已初始化（幂等） */
    async ensureInit(gitdir: string): Promise<void> {
      try {
        await fs.promises.access(path.join(gitdir, "HEAD"));
      } catch {
        await this.init(gitdir);
      }
    },

    async getStatus(ctx: GitContext): Promise<GitStatusResult> {
      await this.ensureInit(ctx.gitdir);

      const matrix = await isoGit.statusMatrix({
        dir: ctx.workTree,
        gitdir: ctx.gitdir,
      });

      const branch = await isoGit.currentBranch({
        dir: ctx.workTree,
        gitdir: ctx.gitdir,
      });

      return convertStatusMatrix(matrix, branch, ctx);
    },

    async add(ctx: GitContext, filepath: string): Promise<void> {
      await isoGit.add({ dir: ctx.workTree, gitdir: ctx.gitdir, filepath });
    },

    async commit(
      ctx: GitContext,
      message: string,
      files: string[],
    ): Promise<{ hash: string }> {
      // 逐文件 add，支持 paths 参数
      for (const f of files) {
        await isoGit.add({
          dir: ctx.workTree,
          gitdir: ctx.gitdir,
          filepath: f,
        });
      }
      const sha = await isoGit.commit({
        dir: ctx.workTree,
        gitdir: ctx.gitdir,
        message,
        author: RIDGE_AUTHOR,
      });
      return { hash: sha.slice(0, 12) };
    },
  };
}
```

### Iso status 转换

isomorphic-git 的 `statusMatrix` 返回 `[filepath, headStatus, workdirStatus, stageStatus][]`。

值与含义对照：

| 值  | HEAD 中       | workdir 中    | staging 中    |
| --- | ------------- | ------------- | ------------- |
| `0` | absent        | absent        | absent        |
| `1` | unchanged     | unchanged     | unchanged     |
| `2` | modified      | modified      | modified      |
| `3` | added/deleted | added/deleted | added/deleted |

映射到 `GitFileStatus` 的 XY 码：

```ts
function convertStatusMatrix(
  matrix: string[][],
  branch: string,
  ctx: GitContext,
): GitStatusResult {
  const files: GitFileStatus[] = [];

  for (const [filepath, head, workdir, stage] of matrix) {
    // 三状态都是 1（未改）→ 跳过
    if (head === "1" && workdir === "1" && stage === "1") continue;

    // 排除 .git 目录自身
    if (filepath.startsWith(".git/") || filepath === ".git") continue;

    let index = " ";
    let working_dir = " ";

    // head: 0=absent, 1=unchanged, 2=modified, 3=added/deleted
    // workdir: 同上
    // stage: 0=not staged, 1=unchanged, 2=modified, 3=staged addition/deletion

    // staging 状态 → index 列
    if (stage === "3") {
      index = head === "0" ? "A" : head === "2" ? "M" : "?";
    } else if (stage === "0") {
      index = head === "0" ? "A" : head === "2" ? "M" : " ";
    }

    // workdir 状态 → working_dir 列
    if (workdir === "2") working_dir = "M";
    if (workdir === "0") working_dir = "D";
    if (head === "0" && stage === "0") working_dir = "?";

    files.push({ path: filepath, index, working_dir });
  }

  return {
    current: branch || "main",
    tracking: null, // iso 无 remote，无 tracking
    files,
    ahead: 0, // iso 无 remote，无 ahead/behind
    behind: 0,
  };
}
```

### 路由改造

```ts
// ---- server/src/index.ts Git 路由部分 ----

// 注入 isoServices
const isoGitService = createIsoGitService();

// GET /api/git/is-repo — 返回仓库上下文全量
app.get("/api/git/is-repo", async (req, res, next) => {
  try {
    const query = gitCwdQuerySchema.parse(req.query ?? {});
    const cwd = resolveGitCwd(query.cwd);
    const ctx = await resolveGitContext(cwd);

    // iso 引擎下确保初始化
    if (ctx.engine === "iso") {
      await isoGitService.ensureInit(ctx.gitdir);
    }

    res.json({
      engine: ctx.engine,
      canCommit: ctx.canCommit,
      canPushPull: ctx.canPushPull,
      canWorktree: ctx.canWorktree,
      label: ctx.label,
    });
  } catch (error) {
    next(error);
  }
});

// GET /api/git/status
app.get("/api/git/status", async (req, res, next) => {
  try {
    const query = gitCwdQuerySchema.parse(req.query ?? {});
    const cwd = resolveGitCwd(query.cwd);
    const ctx = await resolveGitContext(cwd);

    if (ctx.engine === "cli") {
      res.json(await gitService.getStatus(cwd));
    } else {
      res.json(await isoGitService.getStatus(ctx));
    }
  } catch (error) {
    next(error);
  }
});

// GET /api/git/branches — iso 模式返回当前分支名，其余为空
app.get("/api/git/branches", async (req, res, next) => {
  try {
    const query = gitCwdQuerySchema.parse(req.query ?? {});
    const cwd = resolveGitCwd(query.cwd);
    const ctx = await resolveGitContext(cwd);

    if (ctx.engine === "cli") {
      res.json(await gitService.getBranches(cwd));
    } else {
      const branch = await isoGitService.getCurrentBranch(ctx);
      res.json({
        current: branch,
        all: [branch],
        branches: { [branch]: { current: true } },
      });
    }
  } catch (error) {
    next(error);
  }
});

// POST /api/git/commit
app.post("/api/git/commit", async (req, res, next) => {
  try {
    const { cwd, message, files } = commitSchema.parse(req.body ?? {});
    const ctx = await resolveGitContext(resolveGitCwd(cwd));

    if (ctx.engine === "cli") {
      res.json(await gitService.commit(cwd, message, files));
    } else {
      res.json(await isoGitService.commit(ctx, message, files));
    }
  } catch (error) {
    next(error);
  }
});

// POST /api/git/push — 仅 CLI，iso 返回 400
app.post("/api/git/push", async (req, res, next) => {
  try {
    const { cwd, remote, branch, force } = pushSchema.parse(req.body ?? {});
    const ctx = await resolveGitContext(resolveGitCwd(cwd));

    if (!ctx.canPushPull) {
      return res.status(400).json({ error: "内置 Git 不支持远程操作" });
    }

    await gitService.push(cwd, { remote, branch, force });
    res.json({ ok: true });
  } catch (error) {
    next(error);
  }
});

// fetch / pull / worktree 同理，加 canPushPull / canWorktree 守卫
```

## 前端改动

### Git 能力探测

`useIsGitRepo` 升级为 `useGitRepositoryStatus`：

```ts
// ---- composables/useGitRepositoryStatus.ts ----

import { ref, watch, type Ref } from "vue";
import { getGitRepositoryStatus } from "@/lib/api";

interface GitRepositoryStatus {
  engine: "cli" | "iso";
  canCommit: boolean;
  canPushPull: boolean;
  canWorktree: boolean;
  label: string;
}

export function useGitRepositoryStatus(cwd: Ref<string>) {
  const status = ref<GitRepositoryStatus | null>(null);
  const isChecking = ref(false);
  let requestToken = 0;

  watch(
    cwd,
    async (nextCwd) => {
      const token = ++requestToken;
      if (!nextCwd) {
        status.value = null;
        isChecking.value = false;
        return;
      }

      isChecking.value = true;
      try {
        const result = await getGitRepositoryStatus(nextCwd);
        if (token !== requestToken) return;
        status.value = result;
      } catch {
        if (token === requestToken) status.value = null;
      } finally {
        if (token === requestToken) isChecking.value = false;
      }
    },
    { immediate: true },
  );

  return { status, isChecking };
}
```

## 验收标准

- 非 Git 项目添加后，右侧显示 Git 面板。
- 非 Git 项目可以查看变更、选择文件并提交。
- 内置 Git commit 后，项目目录中不存在 `.git`。
- 用户未安装 git 时，内置 Git 正常工作。
- 项目执行真实 `git init` 后，用户点击 refresh，ridge 切换到 CLI 引擎。
- 真实 Git 的 `git log` 不包含内置 Git 的历史提交。
- 内置 Git 模式下不出现 fetch、pull、push、worktree 操作按钮。
- CLI 引擎模式保持现有 Git 能力不变。
- Git API 前端签名不变（不新增 `projectId` 参数）。

## 测试计划

### 服务端测试

- 系统无 git 时，解析为 `engine: "iso"`。
- 系统有 git + 项目无 `.git` 时，解析为 `engine: "iso"`。
- 系统有 git + 项目有 `.git` 时，解析为 `engine: "cli"`。
- Iso 引擎 commit 不创建项目目录 `.git`。
- 项目出现 `.git` 后，解析结果从 `iso` 切换为 `cli`。
- Iso 引擎调用 push/pull/fetch/worktree 返回 400。
- CLI 引擎下现有 Git API 行为不回退。
- `resolvedProjectId` 从 cwd 正确反查。
- 无匹配项目时 fallback projectId 稳定一致。

### 前端测试

- Iso 模式显示 Git 面板。
- Iso 模式显示"ridge 内置"标识。
- Iso 模式隐藏远程操作按钮（fetch/pull/push）。
- CLI 模式显示完整 Git 操作。
- 从 Iso 切换为 CLI 后，面板状态随 refresh 更新。
- Git API 调用不传 `projectId`。

## 风险与约束

- isomorphic-git 不支持 worktree，内置 Git 模式下 worktree 相关功能不可用。
- 内置 Git 不能配置 remote，不做 push/pull/fetch。
- 内置 Git 不能写入项目 `.git`。
- 真实 Git 一旦出现，必须立即优先使用 CLI 引擎。
- 不做兼容式双写，不同时提交到 iso 和 cli。
- isomorphic-git 的 `statusMatrix` 对大项目性能可能低于 CLI，但 iso 模式只适用于非 Git 项目，目录通常较小。
- engine 切换依赖用户手动点击 refresh，属于有意识操作而非自动触发。
- `git-resolver.ts` 和 `iso-git-service.ts` 需要单独在 Node 22 上验证 isomorphic-git 兼容性。
