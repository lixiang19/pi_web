# 内嵌子代理运行时 方案

- 状态：草案
- 日期：2026-04-11
- Owner：AI Agent
- 目标版本/里程碑：子代理能力第一阶段

## 1. 背景与问题

- 背景：当前 `pi_web` 已经具备主会话 agent 发现、选择、权限注入与会话级持久化能力，但仍停留在“主会话 agent 配置系统”阶段，尚未形成真正可用的子代理任务执行体系。
- 现象/痛点：用户已经明确需要“内嵌在当前项目里的子代理运行时”，并明确排除 worktree、RPC、memory、TUI；现状下虽然 `mode: task` 已进入 schema，但服务端仍会在主会话列表中过滤 task-only agent，且不存在子代理生命周期、并发队列、结果回收、steer/resume 等运行时闭环。
- 成功定义：在不引入 worktree、RPC、memory、TUI 的前提下，让当前 `pi_web` 支持项目内嵌子代理任务系统：可启动任务、前后台执行、受控并发、查询结果、回收结果、发送 steer，并在 Web 中可观察。

## 2. 目标与非目标

- 目标：
  - 建立统一的 agent 注册与配置体系，覆盖主会话 agent 与子代理 agent 的共同需求。
  - 在 `packages/server` 内新增内嵌子代理运行时，支持前台/后台执行、并发限制、队列与结果状态管理。
  - 在 `packages/web` 提供与主会话解耦的子代理任务状态展示与结果消费入口。
  - 保留当前主会话 agent 能力，不把子代理能力做成独立扩展，而是集成进当前项目。
- 非目标：
  - 不实现 worktree 隔离执行。
  - 不实现跨扩展 RPC、事件总线开放接口。
  - 不实现 persistent memory。
  - 不复用或移植 TUI widget / overlay 交互。
  - 不做“兼容所有现有 agent frontmatter”的兜底解析；新 schema 一次性明确，旧的不符合就视为待迁移配置。

## 5. 方案概览（主方案）

### 5.1 方案摘要

- 一句话：在当前 `pi_web` 内新增“内嵌子代理运行时层”，把现有主会话 agent 能力扩展为“主会话 + 子代理任务系统”的双层结构。
- 取舍：不做外部扩展化能力，不引入 worktree、RPC、memory、TUI，只保留最核心的子代理执行、编排、回收与 Web 可视化；这样可以沿用当前 Node 服务桥和 Web 工作台结构，避免架构漂移。
- 影响面：会修改 `packages/server` 的 agent 解析、任务管理、API 层；会修改 `packages/web` 的任务状态层与会话页 UI；会改变用户对 agent 的使用方式，从“只在主会话里切人格”升级为“在当前项目内部派生子任务”。

### 5.2 风险与回滚

- 风险：
  - agent schema 升级导致现有主会话 agent 行为变化：通过分层 schema 和默认 agent 注册表缓解，先保证主会话行为可回归验证。
  - 子代理运行时与主会话状态耦合过深：通过独立 `SubagentManager` / `SubagentRecord` 与独立前端 composable 缓解。
  - 后台任务过多拖垮服务端：通过固定并发上限、队列和状态回收机制缓解。
  - group join 过早引入导致调试复杂：第一阶段先支持基础后台通知，group join 放在第二阶段打开。
- 回滚：
  - 回滚策略：先以独立服务端模块和独立前端任务面板落地；若运行时不稳定，直接关闭子代理入口与相关 API，保留当前主会话 agent 系统不动。

## 6. 详细设计（Spec）

### 6.1 交互/行为（如适用）

- 用户流程：
  - 用户在当前会话中选择或触发一个子代理任务。
  - 前端调用服务端创建子代理任务，指定 agent type、prompt、描述、前台/后台模式。
  - 服务端创建 `SubagentRecord`，根据配置决定 prompt mode、tools、skills、max turns，并启动运行时。
  - 若为前台任务：结果直接挂回当前会话消息流。
  - 若为后台任务：任务进入任务面板；完成后可查看结果摘要、手动拉取完整结果、发送 steer。
  - 若用户显式消费结果，则后续不重复提示。
- 边界条件：
  - 仅允许当前项目内部调用，不暴露跨扩展能力。
  - task-only agent 不进入主会话 agent 选择器，但可进入子代理类型选择器。
  - 运行中任务可 steer；非运行中任务禁止 steer。
  - 超过并发上限进入 queued；前台任务不走后台队列。
  - 旧 agent frontmatter 中与新 schema 不匹配的字段不做兜底兼容，统一迁移到新 schema。

### 6.2 接口与契约

- API/函数签名：
  - `discoverAgents(cwd): AgentDefinition[]`：返回统一后的 agent 定义，包含主会话可选与 task 可选能力标签。
  - `createSubagentTask(sessionId, payload): SubagentTaskSnapshot`：创建并启动子代理任务。
  - `getSubagentTask(taskId, options): SubagentTaskSnapshot | SubagentTaskResult`：查看状态或结果。
  - `steerSubagentTask(taskId, message): { ok: true }`：向运行中的子代理发送 steering message。
  - `listSubagentTasks(sessionId): SubagentTaskSummary[]`：返回当前主会话关联的任务列表。
  - `SubagentManager.spawnAndWait(...) -> SubagentRecord`：前台执行。
  - `SubagentManager.spawn(...) -> taskId`：后台执行。
- 错误码/异常：
  - `SUBAGENT_TYPE_NOT_FOUND`：子代理类型不存在或被禁用。
  - `SUBAGENT_NOT_RUNNING`：steer 目标不在运行态。
  - `SUBAGENT_RESULT_NOT_READY`：请求完整结果时任务尚未完成。
  - `SUBAGENT_QUEUE_LIMIT_REACHED`：超过系统上限且当前策略不允许继续入队。
  - `INVALID_AGENT_SCHEMA`：agent frontmatter 不符合统一 schema。

### 6.3 数据与存储（如适用）

- Schema/字段：
  - `AgentDefinition`：`name`、`description`、`displayName`、`mode`、`tools`、`disallowedTools`、`skills`、`model`、`thinking`、`maxTurns`、`promptMode`、`inheritContext`、`runInBackground`、`isolated`、`enabled`、`sourceScope`。
  - `SubagentRecord`：`id`、`parentSessionId`、`type`、`description`、`status`、`result`、`error`、`toolUses`、`startedAt`、`completedAt`、`sessionRef`、`resultConsumed`、`pendingSteers`。
  - `SubagentTaskSummary`：给 Web 列表展示的轻量视图，不暴露运行时对象。
- 迁移策略：
  - 不做旧 frontmatter 字段的运行时兼容分支；统一升级到新 schema。
  - 先引入默认 agent 注册表，保证即使用户目录配置未迁移，系统也有可用默认类型。
  - 对当前已存在但无效的 agent 文件，按日志与文档方式提示迁移，不做 silent fallback。

### 6.4 兼容性与发布

- 兼容策略：
  - 保持当前主会话 agent 选择与消息发送链路可用；子代理能力作为新增模块接入，不替换现有主消息流逻辑。
  - 主会话 agent 与子代理 agent 共用统一 schema，但在 UI 和 API 上明确分层，不混用同一选择入口。
- 发布步骤：
  - 第一步：只落地服务端 registry 与 manager，不开放 UI 入口。
  - 第二步：开放最小前端任务入口与任务面板。
  - 第三步：补齐结果回收、steer、join 策略与默认 agent 管理。

### 6.5 可观测性

- 日志：
  - agent schema 解析失败日志。
  - 子代理任务创建、入队、开始、完成、失败、steer 日志。
  - 结果被消费、通知被抑制的关键日志。
- 指标：
  - 当前 running / queued / completed / failed 任务数。
  - 平均任务时长、失败率、结果拉取率。
  - 按 agent type 统计调用次数与平均 toolUses。
- 告警：
  - 队列长度持续高于阈值。
  - 失败率异常升高。
  - 某类 agent 长时间卡在 running 未完成。
