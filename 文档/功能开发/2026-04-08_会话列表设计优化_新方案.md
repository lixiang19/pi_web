# 会话列表设计优化方案
## 1. 背景与问题
- 当前会话列表透明度使用过多，影响可读性
- 三层结构（项目→分组→会话）视觉复杂
- 需要更清晰的分组边界和视觉层次
## 2. 目标与非目标
**目标**：
- 极简干净的Linear/GitHub风格
- 所有文字使用不透明色，提升可读性
- 保留分组层级，优化分组样式
- **新增"新建worktree"功能** - 在worktree分组快速创建会话
- 悬停显示完整标题tooltip
- 明暗主题兼容
- **新增"添加项目"功能** - 纯前端界面，Node服务层持久化
**非目标**：
- 不删除最近访问区域
- 不改变会话数据结构
- **Tauri不做任何功能开发**（只打包）

#### 3. webkitdirectory文件选择（网络搜索确认）
**浏览器支持度**: 92.75%全球支持（Baseline 2025）
- ✅ Chrome 7+, Edge 13+, Firefox 50+, Safari 11.1+
- ⚠️ Safari iOS 18.4+才支持目录选择（之前只能选文件）
**关键限制**: 
- `webkitdirectory`选择目录后，`File.webkitRelativePath`只包含**相对路径**（如`folder/file.txt`）
- **无法直接获取绝对路径**（如`/Users/admin/projects/my-project`）
- 这是浏览器安全限制，无法绕过
**解决方案**: 用户选择文件夹内任意一个文件，服务端提取父目录
```
用户选择: /Users/admin/projects/my-project/README.md
前端发送: fileName="README.md", relativePath="my-project/README.md"
服务端提取: parentPath = path.dirname(filePath) → "/Users/admin/projects/my-project"
```

### 3.3 关键问题与决策
#### 问题1: webkitdirectory无法直接获取文件夹绝对路径
**影响**: 用户选择文件夹后，前端只能获得相对路径
**解决方案**: 用户选择文件夹内任意一个文件，服务端提取父目录作为项目路径

**影响**: 添加项目仅保存路径信息，该项目可能没有会话
**解决方案**:
- 添加项目仅保存路径信息到`~/.ridge/setting.json`

- 点击项目时，切换`cwd`到项目路径，创建新会话草稿
- 项目列表与会话列表独立，但点击项目可快速切换到该目录创建会话

#### 5.3.1 数据结构设计
```typescript
// ~/.ridge/setting.json (Node服务层读写)
// 路径: os.homedir() + '/.ridge/setting.json'
interface RidgeSettings {
  version: number;        // 配置版本，便于后续迁移
  projects: Array<{
    id: string;           // 项目唯一标识（路径md5 hash前8位）
    name: string;         // 项目名称（文件夹名，如"my-project"）
    path: string;         // 绝对路径（如"/Users/admin/projects/my-project"）
    addedAt: number;      // 添加时间戳
  }>;
}
```

#### 5.3.3 服务端配置持久化实现
```typescript
// packages/server/src/project-settings.js
import fs from "node:fs/promises";
import path from "node:path";
import os from "node:os";
import crypto from "node:crypto";
const RIDGE_DIR = path.join(os.homedir(), ".ridge");
const SETTINGS_FILE = path.join(RIDGE_DIR, "setting.json");
const SETTINGS_VERSION = 1;
const createDefaultSettings = () => ({
  version: SETTINGS_VERSION,
  projects: [],
});
// 生成项目ID: 路径的md5 hash前8位
const generateProjectId = (projectPath) => {
  return crypto.createHash("md5").update(projectPath).digest("hex").slice(0, 8);
};
// 加载配置（带缓存）
let cachedSettings = null;
let writeQueue = Promise.resolve();
const loadSettings = async () => {
  if (cachedSettings) return cachedSettings;
  try {
    const content = await fs.readFile(SETTINGS_FILE, "utf8");
    cachedSettings = { ...createDefaultSettings(), ...JSON.parse(content) };
  } catch (error) {
    if (error?.code !== "ENOENT") throw error;
    cachedSettings = createDefaultSettings();
  }
  return cachedSettings;
};
// 保存配置（队列化写入）
const saveSettings = async (settings) => {
  cachedSettings = settings;
  writeQueue = writeQueue.then(async () => {
    await fs.mkdir(RIDGE_DIR, { recursive: true });
    await fs.writeFile(SETTINGS_FILE, JSON.stringify(settings, null, 2) + "\n", "utf8");
  });
  await writeQueue;
};
export const getProjects = async () => {
  const settings = await loadSettings();
  return settings.projects;
};
export const addProject = async (fileName, relativePath) => {
  // 从relativePath提取项目路径
  // relativePath格式: "project-folder/sub-folder/file.txt"
  const projectName = relativePath.split('/')[0];
  const projectPath = "/" + relativePath.split('/').slice(0, -1).join('/');
  
  const settings = await loadSettings();
  
  // 检查是否已存在（通过路径匹配）
  const existing = settings.projects.find(p => p.path === projectPath);
  if (existing) return existing;
  
  const project = {
    id: generateProjectId(projectPath),
    name: projectName,
    path: projectPath,
    addedAt: Date.now(),
  };
  
  settings.projects = [project, ...settings.projects];
  await saveSettings(settings);
  return project;
};
export const deleteProject = async (id) => {
  const settings = await loadSettings();
  settings.projects = settings.projects.filter(p => p.id !== id);
  await saveSettings(settings);
  return { success: true };
};
```
#### 5.3.4 文件选择实现方案（纯前端）
```vue
<!-- 隐藏的文件选择input -->
<input
  ref="fileInput"
  type="file"
  style="display: none"
  @change="handleFileSelect"
>
<!-- 点击按钮触发文件选择 -->
<button @click="fileInput?.click()">添加项目</button>
```
```typescript
// 前端处理文件选择
const handleFileSelect = async (event: Event) => {
  const input = event.target as HTMLInputElement;
  const file = input.files?.[0];
  if (!file) return;
  
  // 获取文件相对路径（如 "my-project/src/main.ts"）
  const relativePath = file.webkitRelativePath || file.name;
  
  // 发送给服务端
  const project = await fetch('/api/projects', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ 
      fileName: file.name,
      relativePath 
    })
  }).then(r => r.json());
  
  // 刷新项目列表
  projects.value.push(project);
};
```
#### 5.3.5 UI交互流程
```
1. 用户点击"添加项目"按钮
2. 前端触发 <input type="file"> 点击，打开文件选择器
3. 用户导航到目标文件夹，选择文件夹内任意一个文件

5. 发送 POST /api/projects { fileName, relativePath }
6. 服务端提取文件所在目录，作为项目路径保存到 ~/.ridge/setting.json
7. 返回项目信息，Sidebar刷新显示项目列表
```
