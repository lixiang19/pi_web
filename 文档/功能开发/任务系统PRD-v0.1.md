# ridge 任务系统 PRD v0.1

## 1. 产品定位

任务是工作空间内的人机协作工作项。

里程碑是工作空间内的阶段性目标容器，用于把一组任务组织到同一个目标下。

第一版不做复杂项目管理，只解决：

- 用户创建和管理任务；
- 用户把任务推进到进行中；
- 任务 Agent 接管处理；
- Agent 创建处理会话；
- Agent 可通过规划工具创建/更新里程碑和任务；
- 用户最终验收完成。

---

## 2. 范围

### 做

- 任务绑定 workspace；
- 里程碑绑定 workspace；
- 任务使用 DB 存储；
- 里程碑使用 DB 存储；
- 任务看板；
- 任务列表；
- 任务日历月视图；
- 里程碑管理页；
- 专用任务 Agent；
- 一个任务最多一个处理会话；
- Agent 可通过统一规划工具创建/更新任务和里程碑；
- 完成必须人工确认。

### 不做

- 从对话创建任务；
- 子任务；
- 标签；
- 任务级工具权限；
- 多处理会话；
- 里程碑处理会话；
- 任务过程展示；
- 活动日志；
- 日期自动触发 Agent；
- 多人协作。

---

## 3. 通用状态

任务和里程碑统一使用 5 个状态：

```txt
待处理 → 进行中 → 审核中 → 完成
             ↓
            阻塞
阻塞 → 进行中
```

状态说明：

| 状态 | 说明 |
|---|---|
| 待处理 | 新建后的默认状态 |
| 进行中 | 用户已决定开始处理 |
| 阻塞 | Agent 或用户判断暂时无法继续 |
| 审核中 | Agent 认为已完成，等待用户审核 |
| 完成 | 用户人工确认完成 |

约束：

- 新建任务和里程碑默认都是待处理；
- Agent 可以把任务或里程碑改为阻塞；
- Agent 判断完成后最多只能提交到审核中；
- Agent 不能把任务或里程碑改为完成；
- 完成只能人工操作。

---

## 4. 新建任务

新建任务字段：

| 字段 | 是否必填 |
|---|---|
| 标题 | 必填 |
| 优先级 | 必填 |
| 完成标准 | 必填 |
| 截止日期 | 选填 |
| 所属里程碑 | 必填，未选择时进入系统默认“未归属”里程碑 |

优先级：

```txt
普通 / 重要 / 紧急
```

默认状态：

```txt
待处理
```

任务必须归属一个里程碑。

系统提供默认里程碑：

```txt
未归属
```

规则：

- 用户未选择里程碑时，任务进入“未归属”；
- “未归属”是系统里程碑；
- “未归属”不允许被改为完成。

---

## 5. 里程碑

里程碑字段：

| 字段 | 是否必填 |
|---|---|
| 标题 | 必填 |
| 目标 | 必填 |
| 完成标准 | 必填 |
| 状态 | 必填 |
| 截止日期 | 选填 |

新建里程碑默认状态：

```txt
待处理
```

里程碑约束：

- 里程碑属于 workspace；
- 里程碑下可以有多个任务；
- 里程碑没有处理会话；
- 里程碑可以由用户手动创建；
- 里程碑也可以由 Agent 通过规划工具创建；
- Agent 可更新里程碑，但不能改为完成；
- 完成只能人工确认。

---

## 6. 任务详情

任务详情字段：

- 标题；
- 状态；
- 优先级；
- 截止日期；
- 完成标准；
- 所属里程碑；
- 阻塞原因；
- 处理会话。

不包含：

- 描述；
- 标签；
- 子任务；
- 活动记录；
- 多会话列表。

## 7. 任务 Agent

### 定位

系统提供一个专门处理任务的默认 Agent。

它属于现有 agent 配置体系：

- 作为默认 agent 配置存在；
- 用户可以覆盖；
- 后续所有任务默认使用它。

### 权限

第一版先不细分任务级权限。

原则：

- 任务 Agent 是完整 agent；
- 默认可使用现有 agent 工具体系；
- 默认允许调用规划工具；
- 权限细节后续单独设计。

---

## 8. Agent 处理流程

### 触发条件

“Agent 处理”按钮只在任务进入 **进行中** 状态后显示。

### 点击后行为

如果任务没有处理会话：

```txt
点击 Agent 处理
→ 创建一个新 AI 会话
→ 会话标题为「处理任务：任务标题」
→ 使用任务 Agent
→ 把任务、所属里程碑和 workspace 信息作为初始上下文传给 Agent
→ 保存为该任务的处理会话
→ 不自动打开新标签页
```

如果任务已有处理会话：

```txt
不再显示 Agent 处理按钮
只在任务详情中显示“处理会话”入口
```

### 一个任务限制

```txt
一个任务最多一个处理会话
```

---

## 9. 会话上下文

创建处理会话时，应向 Agent 提供尽可能完整的任务信息，包括：

- 任务 ID；
- 所属 workspace；
- 所属里程碑 ID；
- 所属里程碑标题；
- 所属里程碑目标；
- 所属里程碑完成标准；
- 所属里程碑当前状态；
- 标题；
- 当前状态；
- 优先级；
- 完成标准；
- 截止日期；
- 阻塞原因；
- 处理会话 ID；
- 可用规划工具说明。

核心目的：

> Agent 后续可以根据任务 ID 和里程碑 ID 调用规划工具更新当前任务、更新相关里程碑，或继续拆解规划。

---

## 10. 规划工具

第一版只设计一个统一工具：

```txt
规划工具
```

能力：

- 允许 Agent 在当前 workspace 内创建里程碑；
- 允许 Agent 在当前 workspace 内创建任务；
- 允许 Agent 建立任务和里程碑的归属关系；
- 允许 Agent 更新当前任务；
- 允许 Agent 更新相关里程碑；
- 允许 Agent 移动任务到不同里程碑；
- 允许 Agent 一次请求创建多个里程碑和多个任务；
- 不允许跨 workspace 操作；
- 不允许 Agent 将任务或里程碑改为完成；
- Agent 最多只能把任务或里程碑改为审核中。

创建规则：

- Agent 创建任务时，必须挂到一个里程碑；
- 如果没有合适的已有里程碑，Agent 可以先创建新里程碑，再把任务挂进去；
- 用户和 Agent 新建任务、里程碑时，默认状态都固定为待处理。

更新限制：

- Agent 不能批量修改某个里程碑下所有任务状态；
- 如果要修改多个任务，必须逐个任务明确更新；
- Agent 不能把“未归属”里程碑改为完成。

返回规则：

- 成功时只返回成功，不返回完整任务/里程碑列表；
- 失败时必须返回详细失败原因，方便 Agent 自我修正。

约束：

```txt
任务和里程碑的完成都必须由用户人工确认。
```

## 11. 任务视图

第一版包含 4 种视图：

- 看板；
- 列表；
- 日历月视图。
- 里程碑。

---

## 12. 看板视图

看板列固定：

```txt
待处理 / 进行中 / 阻塞 / 审核中 / 完成
```

约束：

- 第一版不支持自定义列；
- 顶部提供里程碑筛选；
- 用户可拖动任务改变状态；
- 任务拖到进行中后，任务详情出现 Agent 处理按钮；
- 列内排序使用手动排序。

---

## 13. 列表视图

列表默认排序：

```txt
优先级 → 截止日期
```

规则：

- 紧急 > 重要 > 普通；
- 同优先级下，截止日期越近越靠前；
- 没有截止日期的任务放在同优先级最后。

里程碑呈现：

- 任务按里程碑分组显示。

列表不显示处理会话入口。

处理会话只在任务详情里显示。

---

## 14. 日历月视图

日历按截止日期展示任务。

规则：

- 有截止日期的任务显示在对应日期；
- 没有截止日期的任务显示在“无截止日期”侧栏；
- 任务按所属里程碑颜色区分；
- 不做计划日期；
- 不做到点自动触发 Agent。

---

## 15. 里程碑视图

里程碑视图是任务模块内的一个标签页。

展示方式：

```txt
卡片列表
```

每张里程碑卡片显示：

- 标题；
- 状态；
- 截止日期；
- 任务数。

里程碑详情显示：

- 标题；
- 目标；
- 完成标准；
- 状态；
- 截止日期；
- 其下任务列表。

里程碑详情中的任务列表：

- 不允许直接拖动任务状态；
- 点击任务后打开任务详情。

## 17. 成功标准

第一版完成后，用户可以：

1. 在某个 workspace 内创建里程碑；
2. 在某个 workspace 内创建任务；
3. 任务必须归属里程碑，未选择时进入“未归属”；
4. 任务和里程碑默认进入待处理；
5. 在看板、列表、日历、里程碑视图中查看任务和里程碑；
6. 将任务拖到进行中；
7. 点击 Agent 处理创建处理会话；
8. 在任务详情进入处理会话；
9. Agent 可通过规划工具创建/更新任务和里程碑；
10. Agent 可把任务或里程碑提交到审核中或阻塞；
11. Agent 不能把任务或里程碑改为完成；
12. 用户人工确认任务或里程碑完成。
