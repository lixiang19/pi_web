# 新建按钮逻辑修复

## 问题

工作空间四个新建按钮有两个 bug：

1. **"新建"按钮创建的是文件，不是文件夹** — 当前 `handleCreateNote('笔记')` 创建 `笔记/未命名.md` 文件，用户期望创建子文件夹
2. **点击第二次不递增** — 四个按钮硬编码名称，重复点击无递增

## 方案

### 核心思路

1. "新建"按钮改为创建文件夹（不是文件）
2. 四个按钮统一采用前端递增策略——创建前扫描目标目录下已有条目，生成不冲突的名称
3. 在当前选中的目录下创建（有选中文件取其父目录，有选中目录取该目录，否则取工作空间根目录）

### 具体修改

#### 前端 `packages/web/src/pages/WorkspacePage.vue`

1. **新增** `selectedDirPath` ref — 跟踪当前选中的目录路径
   - 在 `handleSelectFile` 中更新：选中文件时取其父目录，选中目录时取该目录
2. **新增** `getNextName(entries, baseName, ext)` 辅助函数 — 从目录已有条目中找出不冲突的名称
3. **修改** `handleCreateNote` — 改为创建文件夹：调用 `createFileEntry({ root, directory: selectedDirPath, name: nextName, kind: "directory" })`
4. **修改** `handleCreateFolder` — 使用 `selectedDirPath` + 递增名称，调用 `createFileEntry`
5. **修改** `handleCreateCanvas` — 使用递增名称（`未命名.canvas` → `未命名1.canvas`）
6. **修改** `handleCreateBase` — 使用递增名称（`新数据库` → `新数据库1`）

#### 递增规则

| 按钮           | 基础名         | 扩展名    | 递增示例                                              |
| -------------- | -------------- | --------- | ----------------------------------------------------- |
| 新建（文件夹） | `未命名`       | 无        | `未命名` → `未命名1` → `未命名2`                      |
| 文件夹         | `未命名文件夹` | 无        | `未命名文件夹` → `未命名文件夹1` → `未命名文件夹2`    |
| Canvas         | `未命名`       | `.canvas` | `未命名.canvas` → `未命名1.canvas` → `未命名2.canvas` |
| Base           | `新数据库`     | `.base`   | `新数据库.base` → `新数据库1.base` → `新数据库2.base` |

#### 不修改服务端

服务端已有冲突检测（`assertTargetAbsent`、409 错误），前端递增后冲突概率极低。

## 影响范围

- `packages/web/src/pages/WorkspacePage.vue` — 四个 handler + 新增辅助函数

## 风险

- 递增依赖目录下已有条目的可见数据，如果文件树未展开可能不完整——但递增后若仍冲突，服务端报错只是不创建，不会丢数据

## 回退策略

直接 revert 即可，无数据迁移风险。
