# 15 fleeting agent 分析建议

## 目标

实现闪念后台分析 Agent，给未处理闪念生成一个最佳处理建议。

## 范围

- 闪念保存后触发。
- 同一个闪念只允许一个分析任务。
- 分析文字内容。
- 分析图片内容。
- 生成一个最佳建议。
- 写入分析状态和建议。
- 用户提前处理成功时取消分析。

## 不做

- 不自动执行处理。
- 不解析 PDF、Word、长文档。
- 不向通知中心写普通成功事件。

## 验收

- 建议不自动沉淀为正式对象。
- 分析失败不阻塞固定处理按钮。
- 已处理或删除的闪念不继续分析。
- 需要用户处理的建议可进入通知中心。

## Spec 提取点

- 后台任务类型。
- 分析输入输出。
- 取消条件。

## Spec 草案

### 后台任务

- 类型：`fleeting_analysis`
- 关联：`fleeting_note_id`

### 输入

- 闪念文字。
- 图片临时附件。
- 闪念创建时间。

### 输出

- 一个最佳处理建议。
- 建议目标类型。
- 简短理由。

### 测试

- 新闪念触发分析任务。
- 同一闪念不会并发两个分析任务。
- 用户处理成功后取消未完成分析。
- 分析失败写入 `analysis_failed` 且不打扰用户。
