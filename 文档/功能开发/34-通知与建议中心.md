# 34 通知与建议中心

## 目标

实现左侧固定入口的通知与建议中心。

## 范围

- `notification_events`。
- 左侧通知入口。
- 未处理数量或红点。
- 通知列表。
- 建议、待确认、失败、警告、信息类型。
- 查看、忽略、重试、接受建议、拒绝建议、打开对象、标记已处理动作。

## 不做

- 不做系统级推送。
- 不做邮件通知。
- 不做聊天式通知助手。
- 不做通知规则编辑器。

## 验收

- 后台成功且无用户动作不产生通知。
- 文件转换失败、RAG 失败、后台 Agent 失败进入通知。
- 接受建议后才写入正式对象。
- 通知事件不进入 RAG。

## 关联设计

- `文档/项目设计/通知与建议中心.md`

## Spec 提取点

- notification_events 字段。
- actions schema。
- 状态流转。

## Spec 草案

### 数据

- id
- type
- source
- severity
- status
- title
- body
- related_type
- related_id
- actions
- handled_at

### 行为

- 左侧入口显示未处理数量或红点。
- 默认显示未处理事件。
- 接受建议才写正式对象。
- 已处理通知不进入 RAG。

### 测试

- 后台失败生成通知。
- 接受任务建议后更新任务。
- 忽略事件后状态为 dismissed。
- summary/memory 成功不生成通知。
