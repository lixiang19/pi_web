## 一、核心模型

工作空间是 **一个以文件系统为基底、支持多种文件类型视图、通过标签页交互的个人内容平台**。

### 三区布局

```
┌──────────────────────┬────────────────────────────┐
│   左侧（260px）        │      中间（flex-1）          │
│                      │                            │
│  🔍 搜索...           │  ┌Tab1──┐┌Tab2──┐┌Tab3─┐ │
│  ─────────────────   │  │仪表盘 ││周报md││追踪  │ │
│  📊 仪表盘            │  └──────┘└──────┘│.base │ │
│  ✅ 待办              │                    └──────┘ │
│  📅 日历              │                            │
│  📥 收件箱            │  ◄ 当前标签页的内容：         │
│  ─────────────────   │    .md  → Milkdown 编辑器    │
│  📁 日记/             │    .canvas → 白板画布        │
│    📄 2026-04-28.md   │    .base → 数据库视图        │
│  📁 笔记/             │    其他  → 只读预览          │
│  📁 项目/             │                            │
│  📁 卡片/             │                            │
│  📁 收件箱/           │                            │
│  📁 白板/             │                            │
│    📄 规划.canvas     │                            │
│  📁 数据库/           │                            │
│    📄 项目追踪.base   │                            │
│    📄 读书清单.base   │                            │
│  📄 README.md        │                            │
│  ─────────────────   │                            │
│  [+ 新建 ▾]          │                            │
└──────────────────────┴────────────────────────────┘
```

### 没有"模块"

工作空间没有"切换模块"的概念。所有东西都在标签页里打开。每个标签页可以是一个文件或一个视图。左侧提供入口，中间展示内容。

## 二、左侧面板

### 结构：固定视图 + 文件树

```
左侧面板 =
    搜索栏（固定顶部）
  + 固定视图入口（仪表盘/待办/日历/收件箱）
  + 分割线
  + 文件树（实际文件系统）
  + 新建按钮（固定底部）
```

### 搜索栏

- 位置：面板顶部
- 行为：输入时实时搜索
- 范围：文件名 + .md 全文 + frontmatter 字段 + 数据库行
- 结果：下拉面板，按类型分组，点击 → 打开标签页

### 固定视图入口

| 入口      | 打开的标签页  | 说明                                               |
| --------- | ------------- | -------------------------------------------------- |
| 📊 仪表盘 | 今日总览      | 日记卡片 + 待办 + 最近编辑 + 日历小部件 + 快捷操作 |
| ✅ 待办   | 待办看板/清单 | 聚合所有任务（笔记 checkbox + 独立任务）           |
| 📅 日历   | 日历视图      | 月/周/日视图，按日期浏览日记和日程                 |
| 📥 收件箱 | 闪念笔记列表  | `收件箱/` 目录下的 .md 文件                        |

### 文件树

- 显示工作空间目录的实际文件系统
- 分层树形结构，支持展开/折叠
- 四种核心文件类型：

| 后缀                        | 文件类型      | 点击行为                                                                          |
| --------------------------- | ------------- | --------------------------------------------------------------------------------- |
| `.md`                       | Markdown 笔记 | 在中间标签页打开，Milkdown 编辑器（可编辑）                                       |
| `.canvas`                   | 白板文件      | 在中间标签页打开，白板画布视图                                                    |
| `.base`                     | 数据库文件    | 在中间标签页打开，数据库视图（表格/画廊/看板/日历）                               |
| `.ts .js .json .yaml` 等    | 代码/配置     | 只读语法高亮预览                                                                  |
| `.png .jpg .gif .webp` 等   | 图片          | 只读渲染预览                                                                      |
| `.html`                     | HTML          | 沙箱 iframe 预览                                                                  |
| `.docx .xlsx .pptx .pdf` 等 | 办公文档      | 不支持内嵌预览/编辑。标签页显示「使用默认应用打开」按钮，调用系统默认应用打开文件 |
| 其他未知类型                | —             | 同上，显示「使用默认应用打开」按钮                                                |

- 文件树交互：
  - 新建文件/文件夹
  - 重命名
  - 删除（移入回收区）
  - 拖拽移动
  - 点击星标收藏（在文件树内标记，不独立为路由）
  - 右键菜单

### 新建按钮

点击 `[+ 新建 ▾]` 弹出：

```
📝 新笔记       → 在"笔记/"下新建 .md 文件
📅 今日日记     → 打开/新建今天的日记
💡 闪念笔记     → 在"收件箱/"下新建 .md 文件
✅ 新待办       → 新建独立任务
🗃️ 新数据库     → 在"数据库/"下新建 .base 文件
🎨 新白板       → 在"白板/"下新建 .canvas 文件
📁 新文件夹     → 在当前位置新建目录
```

## 三、标签页系统

### 标签页类型

| 打开的内容     | 标签页标题 | 内容渲染                            |
| -------------- | ---------- | ----------------------------------- |
| `.md` 文件     | 文件名     | Milkdown 编辑器（可编辑，自动保存） |
| `.canvas` 白板 | 文件名     | 白板画布（无限画布 + 卡片 + 连线）  |
| `.base` 数据库 | 文件名     | 数据库视图（表格/画廊/看板/日历）   |
| 📊 仪表盘      | "仪表盘"   | 仪表盘组件                          |
| ✅ 待办        | "待办"     | 待办看板/清单组件                   |
| 📅 日历        | "日历"     | 日历视图组件                        |
| 📥 收件箱      | "收件箱"   | 闪念笔记列表组件                    |
| 非 md 文本文件 | 文件名     | 只读语法高亮预览                    |
| 图片文件       | 文件名     | 图片渲染预览                        |

### 标签页行为

- 多标签页并存，点击切换
- 点击左侧入口 → 已有则激活，否则新建
- 点击文件 → 总是激活已有或新建
- `.md` 标签页有未保存状态指示（圆点）
- 关闭标签页 → .md 有未保存内容时提示
- 仪表盘标签页始终存在（固定首页）
- 关闭工作空间后，标签页持久化恢复

### 不可关闭的标签页

- 📊 仪表盘 — 始终存在，着陆页

```

│   └── 2026/04/

│       └── 2026-04-29.md
├── 笔记/
│   ├── 工作空间设计.md
│   └── ...
├── 项目/
│   └── 项目名/
│       └── ...
├── 阅读/
│   └── 书名.md
├── 卡片/
│   └── 概念名.md
├── 收件箱/
│   └── 临时想法.md
├── 白板/
│   ├── 项目规划.canvas
│   └── 头脑风暴.canvas
├── 数据库/
│   ├── 项目追踪.base
│   └── 读书清单.base
└── README.md
```

### 文件类型定义

| 后缀      | 类型   | 存储格式                           | 打开方式        |
| --------- | ------ | ---------------------------------- | --------------- |
| `.md`     | 笔记   | Markdown + YAML frontmatter        | Milkdown 编辑器 |
| `.canvas` | 白板   | JSON（节点/连线/位置数据）         | 白板画布        |
| `.base`   | 数据库 | JSON（列定义+视图配置+独立行数据） | 数据库视图      |

### 结构化数据存储（不在文件系统里）

以下数据存在服务端 DB：

- 独立任务（不绑定 .md 文件的待办）
- 日程事件（日历上的日程，不同于日记）
- 外部书签（URL + 元信息）

但 —— **这些数据在界面上无缝融入**。用户在待办视图看到聚合后的所有任务，不知道也不关心哪些来自文件、哪些来自 DB。

## 五、数据库（Bases）⭐

### 概述

数据库是工作空间最重要的特色功能之一。灵感来自 Obsidian Bases 和 Notion Database，但有自己的设计。

一个数据库：

- 以 `.base` 文件形式存在文件树中
- 点击打开 → 在标签页渲染为数据库视图
- 支持多种视图布局：表格、画廊、看板、日历
- 支持过滤、排序、分组
- 每行可以关联 .md 文件（继承其 frontmatter 属性），也可以是独立数据行

### 与 Obsidian Bases 的关键差异

|            | Obsidian Bases            | ridge 数据库                          |
| ---------- | ------------------------- | ------------------------------------- |
| 行数据来源 | 必须是 .md 文件           | 可以是 .md 文件引用 **或** 独立数据行 |
| 存储位置   | 数据分散在 .md 文件中     | 独立数据行存在 .base 文件内           |
| 视图配置   | `.base` 文件（YAML 语法） | `.base` 文件（JSON 格式）             |

### 列（字段）类型

| 类型     | 说明                | 示例                     |
| -------- | ------------------- | ------------------------ |
| 文本     | 单行文本            | 任务名称                 |
| 长文本   | 多行文本            | 备注                     |
| 数字     | 整数或小数          | 优先级、评分             |
| 日期     | 日期选择器          | 截止日期、创建日期       |
| 选择     | 下拉单选            | 状态：待做/进行中/完成   |
| 多选     | 标签多选            | 标签：design, bug        |
| 复选框   | 是/否               | 已完成                   |
| URL      | 链接                | 参考资料链接             |
| 文件引用 | 关联已有的 .md 笔记 | 关联笔记                 |
| 公式     | 基于其他列的计算    | 剩余天数 = 截止日 - 今天 |

### 视图类型

| 视图     | 说明                                 | 适用场景                 |
| -------- | ------------------------------------ | ------------------------ |
| **表格** | 行 = 数据行，列 = 字段，经典电子表格 | 项目管理、清单、数据浏览 |
| **看板** | 按某一"选择"列的值分列，卡片拖拽     | 任务管理、工作流         |
| **画廊** | 卡片网格，适合展示封面图             | 读书清单、灵感收集       |
| **日历** | 按日期列在月历上展示                 | 排期、内容日历           |

### 数据库示例：项目追踪.base

```
视图：表格
数据源：项目/ 目录下的所有 .md 文件 + 独立行

列：
| 项目名(文件引用) | 状态(选择)    | 优先级(数字) | 截止日(日期) |
|------------------|--------------|-------------|-------------|
| 工作空间设计.md   | 进行中        | 1           | 2026-05-01  |
| 终端重构.md      | 待做          | 2           | 2026-05-15  |
| [独立行] 买域名   | 待做          | 3           | 2026-04-30  |
```

切换为看板视图：按"状态"列分三列（待做/进行中/完成），卡片可拖拽。

### 数据库示例：读书清单.base

```
视图：画廊
数据源：阅读/ 目录下的所有 .md 文件 + 独立行

每张卡片显示：封面图、书名、评分、状态
```

## 六、白板（Canvas）⭐

### 概述

白板是可视化思维的无限画布。以 `.canvas` 文件形式存在文件树中，点击打开后在标签页渲染。

### 元素类型

- **卡片**：文本卡片（直接在画布上输入文字，支持 Markdown）
- **笔记嵌入**：引用已有的 .md 笔记，内容同步显示
- **数据库嵌入**：引用已有的 .base 数据库，内嵌视图
- **连线**：元素之间的箭头/直线/曲线
- **图片**：拖入或粘贴图片
- **分组框**：用边框框起一组元素，附加标题

### 交互

- 双指缩放/平移
- 拖拽移动元素
- 框选多个元素
- 双击空白处 → 新建卡片
- 右键菜单 → 连线/删除/锁定/复制
- 从文件树拖拽文件 → 创建笔记嵌入

### 存储格式

`.canvas` 文件为 JSON：

```json
{
  "nodes": [
    {
      "id": "1",
      "type": "card",
      "x": 100,
      "y": 200,
      "width": 300,
      "content": "# 想法"
    },
    {
      "id": "2",
      "type": "note",
      "x": 500,
      "y": 200,
      "path": "笔记/工作空间设计.md"
    }
  ],
  "edges": [{ "id": "e1", "from": "1", "to": "2", "type": "arrow" }]
}
```

## 七、待办系统

### 双来源聚合

待办视图聚合两个来源的任务，用户不需要区分：

**来源 1：笔记内的 checkbox**

```markdown
- [ ] 写完设计文档
- [ ] 给小李回邮件 截止: 2026-04-30
```

自动解析，显示在待办视图中，标注来源笔记。

**来源 2：独立任务**
在待办视图中直接新建，存在 DB，不绑定 .md 文件。

### 待办视图

- **清单模式**：按优先级/日期/标签排序的筛选列表
- **看板模式**：待做 | 进行中 | 完成，拖拽改状态
- **今日模式**：只显示今天截止或今天创建的任务

### 任务属性

```
☐ 写完设计文档
   优先级：🔴 高
   截止日：2026-04-30
   标签：design workspace
   来源：📝 笔记/工作空间设计.md  ← 可点击跳转
   创建于：2026-04-28 10:30
```

## 八、日历系统

### 两种内容

- **日记**：以 .md 文件存在，`日记/YYYY/MM/YYYY-MM-DD.md`
- **日程事件**：存在 DB，有具体时间+标题+描述

### 视图

- 月视图（默认）：有日记的日期标圆点，有日程的日期标小标签
- 周视图：7 天水平排列
- 日视图：当天时间轴

### 交互

- 点击有日记的日期 → 打开对应日记标签页
- 点击空白日期 → 新建日记
- 点击/拖拽时间段 → 新建日程事件

## 十、编辑器

### Markdown（.md 文件）

- Milkdown 所见即所得编辑器
- `[[双向链接]]`：输入时实时搜索补全
- `#标签`：自动补全已有标签
- 图片拖入/粘贴
- 代码块语法高亮
- 表格编辑
- Frontmatter 可视化面板

### 非 Markdown 文件

- 文本/代码：只读语法高亮
- 图片：渲染预览
- HTML：沙箱 iframe
- 大文件：分页加载
- 办公文档（.docx .xlsx .pptx .pdf）及其他未知类型：
  - 标签页不试图内嵌渲染
  - 标签页内容区显示文件图标+文件名+「使用默认应用打开」按钮
  - 点击按钮调用系统默认应用打开文件
  - Word/Excel/PPT 不做内嵌预览或编辑（工程代价过高）

## 十一、收藏系统

### 不独立为路由

收藏不作为一个独立的"视图入口"。它集成在文件树中：

- 文件树中的每个文件/文件夹有星标按钮
- 已收藏的文件在文件树中有视觉标记（星标）
- 数据库可以创建一个"收藏.base"来展示所有收藏内容，如果需要更丰富的视图

### 书签

外部网页书签作为独立数据存在 DB，可以通过数据库（创建一个"书签.base"）来管理。

## 十二、与系统其他部分的边界

```
ridge
│
├── 工作空间（内容平台）← 我们正在设计的
│   文件：.md .canvas .base
│   视图：仪表盘、待办、日历、收件箱
│   标签页：所有内容的交互容器
│
├── AI 会话（/chat）—— 独立的对话工作台
│
├── 系统工具（独立入口）
│   ├── 终端
│   ├── 自动化
│   └── 设置
│
└── 全局搜索（/search）—— 跨所有模块搜索
```

## 十三、设计原则

1. **文件即真理**：.md 文件是内容的真实载体，可被任何编辑器打开
2. **视图是透镜**：仪表盘/待办/日历只是看同一份数据的不同方式
3. **数据库不强绑 .md**：每行可以关联文件，也可以独立存在
4. **收藏不独立**：收藏是文件树的属性，不是独立路由
5. **白板是文件**：`.canvas` 文件和其他文件一样存在文件树里
6. **标签页是交互单元**：一切都在标签页里打开
7. **不替代现有工具**：不做 IDE、不做项目管理、不做协作平台
