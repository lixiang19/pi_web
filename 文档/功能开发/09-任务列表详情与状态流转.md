# 09 任务列表详情与状态流转

## 目标

实现任务页、任务详情和任务状态流转。

## 范围

- 任务列表。
- 里程碑列表。
- 项目筛选。
- 创建、更新、删除任务。
- 创建、更新、删除里程碑。
- 阻塞原因。
- 完成标准。
- 截止日期。
- 排序。

## 不做

- 不做会话内任务面板。
- 不自动完成任务。

## 验收

- 用户可以把任务和里程碑改为 completed。
- Agent/API 普通规划路径不能改为 completed。
- 项目筛选包含全部、无项目、具体项目。
- 新任务默认继承里程碑项目，用户可覆盖。

## 关联设计

- `文档/项目设计/任务里程碑与处理会话.md`

## Spec 提取点

- 任务 API。
- 里程碑 API。
- 状态流转权限。
- 筛选和排序。

## Spec 草案

### API

- `GET /api/workspace/tasks`
- `POST /api/workspace/tasks`
- `PATCH /api/workspace/tasks/:id`
- `DELETE /api/workspace/tasks/:id`
- `GET /api/workspace/milestones`
- `POST /api/workspace/milestones`
- `PATCH /api/workspace/milestones/:id`
- `DELETE /api/workspace/milestones/:id`

### 行为

- 用户 API 可设置 completed。
- Agent/规划工具不能设置 completed。
- 项目筛选支持全部、无项目、具体项目。
- 新任务默认继承里程碑项目。

### 测试

- 用户完成任务成功。
- Agent 完成任务失败。
- 无项目筛选只返回 `project_id is null`。
- 更新里程碑项目后新任务继承。
