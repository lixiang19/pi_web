# 11 规划工具与完成确认边界

## 目标

实现 Agent 可用的规划工具，并严格禁止 Agent 完成任务或里程碑。

## 范围

- 创建任务。
- 更新任务。
- 创建里程碑。
- 更新里程碑。
- 移动任务到里程碑。
- 设置阻塞。
- 设置 reviewing。
- 工具权限纳入 `task` 权限。

## 不做

- 不支持 Agent 设置 completed。
- 不支持批量隐式修改多个任务状态。

## 验收

- Agent 尝试完成任务时失败并返回明确原因。
- 工具成功只返回必要结果。
- 工具失败返回可自我修正的原因。
- 多任务更新必须逐项明确。

## Spec 提取点

- 工具 schema。
- 权限校验。
- completed 禁止规则。

## Spec 草案

### 工具能力

- create_task
- update_task
- create_milestone
- update_milestone
- move_task
- set_blocked
- set_reviewing

### 行为

- 工具受 `task` 权限控制。
- 工具不能设置任务或里程碑为 completed。
- 批量修改必须拆成逐项明确调用。
- 失败返回可修正原因。

### 测试

- 无 `task` 权限时工具不可用或需审批。
- `status=completed` 返回错误。
- 多任务隐式更新请求失败。
- 成功创建任务返回任务 ID 和必要字段。
