# 2026-04-12 项目真源与会话文件树收敛

## 背景
用户新增项目后，左侧项目、会话归属、右侧文件树三条链路没有共用同一真源。
现状问题：
- 左侧项目会从 session 反推出项目节点
- 文件树受 server 启动目录推导出的静态 workspace scope 限制
- 新会话和资源发现会默认回退到 `workspaceDir`

结果就是：
- 新增项目后文件树报 `Requested root is outside the allowed workspace scope`
- 左栏项目心智和实际数据源不一致

## 目标
- 已添加项目成为唯一真源
- 左侧“浏览项目”只显示手动添加项目
- 每个会话都必须属于某个已添加项目
- 文件树只允许访问已添加项目及其 worktree
- 未选项目时保持空态，不自动落回 `workspaceDir`

## 实施
### 服务端
- 删除 `resolveWorkspaceScope / WorkspaceScope`
- 新增 `buildManagedProjectScopes()`，基于已添加项目构建允许目录集合
- `/api/sessions` 改为只返回命中已添加项目作用域的会话
- `SessionSummary.projectId/projectRoot/projectLabel` 改为对齐已添加项目
- `/api/files/tree` 改为校验 root 是否属于已添加项目或其 worktree
- `POST /api/sessions` 改为必须落在已添加项目目录内
- `GET /api/agents`、`GET /api/resources` 不再默认把 `workspaceDir` 当项目上下文

### 前端
- `useEffectiveDirectory` 取消 `workspaceDir` 回退
- `usePiChat` 取消新会话、资源、agent 的 `workspaceDir` 默认目录
- 未选项目时发送前先报错“请先选择项目”
- `buildSessionProjects()` 改为只从 `storedProjects` 构造左栏项目
- `SessionSidebar.vue` 空态改为“无已添加项目”
- `WorkbenchPage.vue` 未选项目时显示“未选择项目”

## 验证
已执行：
```bash
npm run lint
npm run typecheck
npm run build --workspace @pi/server
```

## 结果
项目列表、会话归属、文件树边界现在统一收敛到“已添加项目”一条主线。
用户新增项目后，文件树不再被 server 启动目录错误拦截。
