# 66-全局权限基线

## 目标

为 ridge 的 Pi Agent 权限系统增加服务器级全局权限基线。

## 背景

原权限只读取每个 Agent 的 `permission`。这导致两类需求无法稳定表达：

- 默认关闭正式任务规划工具，只给少数 Agent 显式开启；
- 某些敏感文件永远禁止读取，即使 Agent 或运行时授权尝试放开。

## 实现

- 新增 `~/.pi/agent/permissions.json` 作为可选全局权限配置。
- 未配置 `permissions.json` 或未配置某个权限键时，仍沿用默认 `allow`。
- `default` / `defaults` 是全局默认权限，可被 Agent 的 `permission` 显式覆盖。
- `locked` 是全局硬拒绝层，只接受 `deny`，优先于 Agent 权限和运行时 `always` 授权。
- 普通会话和子代理会话都加载同一份全局权限。

## 配置示例

```json
{
  "default": {
    "task": "deny"
  },
  "locked": {
    "read": {
      ".env": "deny",
      ".env.*": "deny",
      "secrets/*": "deny"
    }
  }
}
```

某个 Agent 需要任务工具时，在该 Agent frontmatter 中显式开启：

```yaml
permission:
  task: allow
```

## 验收

- 全局未配置时任务工具保持默认可用。
- 配置 `default.task=deny` 后，规划工具默认从 active tools 移除。
- Agent 配置 `permission.task=allow` 后，可重新启用规划工具。
- 配置 `locked.read` 后，命中文件路径即使 Agent 和运行时规则都 allow 也会被拒绝。
