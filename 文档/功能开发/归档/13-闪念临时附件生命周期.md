# 13 闪念临时附件生命周期

## 目标

实现闪念临时附件目录和生命周期。

## 范围

- `.ridge/fleeting-attachments`。
- 截图和文件副本上传。
- 未处理闪念引用临时附件。
- 删除未处理闪念时清理未引用临时附件。
- 处理成功后迁移到正式附件目录。

## 不做

- 临时附件不进入 RAG。
- 临时附件不进入图谱。
- 临时附件不允许 workspace MCP 读取。

## 验收

- 临时附件属于未处理闪念生命周期。
- 处理失败不删除临时附件。
- 只有目标系统确认创建成功后才迁移附件。
- 正式附件保留原文件名，冲突时追加短 ID。

## 关联设计

- `文档/项目设计/闪念系统与桌面采集.md`
- `文档/项目设计/文件处理流程.md`

## Spec 提取点

- 临时附件路径规则。
- 迁移事务。
- 清理规则。

## Spec 草案

### 路径

- 临时目录：`.ridge/fleeting-attachments`
- 正式目录：`附件/`

### 行为

- 闪念附件先保存副本到临时目录。
- 临时附件记录引用计数或引用关系。
- 处理成功后迁移到正式附件。
- 删除未处理闪念时清理未被引用的临时附件。

### 测试

- 创建带附件闪念生成临时文件。
- 未处理附件不出现在文件树和 RAG。
- 处理成功迁移到 `附件/`。
- 处理失败临时附件保留。

## 完成记录

### 已实现内容

- 数据库：schema version 9；新增 `fleeting_attachments` 表（`attachment_id`、`note_id`、`original_name`、`stored_name`、`temp_path`、`final_path`、`mime_type`、`size`、`sha256`、`created_at`）及索引；`RIDGE_DB_BOOTSTRAP_SQL` / `CORE_TABLE_COLUMNS` 同步。
- 后端路由：
  - `POST /api/workspace/fleeting` 同时支持 JSON 和 multipart；`upload.fields([{ attachments }, { files }])` 兼容两个文件字段；body `.strict()` 始终校验，拒绝 `projectId` 等未声明字段；允许“有附件但文字为空”；无文字无附件仍 400。
  - 附件保存到 `.ridge/fleeting-attachments/<noteId>/`，DB 记录引用；返回 note 包含 `attachments` 数组。
  - `GET /api/workspace/fleeting` 批量查询附件并按 noteId 分组。
  - `DELETE` 删除 note + 附件 DB 记录 + 临时目录清理（路径安全校验）。
  - `process/journal/clip`：先创建目标，再 `migrateAttachmentsToFinal`，最后删除 note + 清理临时目录。
  - `process/task`：未接入，保留原闪念和临时附件。
  - `migrateAttachmentsToFinal`：`fs.copyFile` 迁移到 `附件/`，冲突时循环生成 `base-短ID.ext` 直到唯一；外层 `try/catch` 捕获中途失败，调用 `rollbackMigratedAttachments` 删除已复制的正式文件并将 DB `final_path` 重置为 NULL，不删除 note/临时附件。
- 前端：`createFleetingNote(content, files?)` 支持 `File[]`；`InboxView.vue` 附件选择/移除/保存；列表展示附件数量/名称；保存成功清空附件，失败保留。
- 路径安全：`sanitizeFileName` 先归一化反斜杠再取 basename，空/`.`/`..`/全点下划线返回 `"unnamed"`；`getSafeNoteAttachmentDir` 与正式目录均使用 `fs.realpath` + `path.relative` + `path.isAbsolute` 多层校验。

### 测试代码

- `fleeting-api.test.ts`（29 tests）：
  - 纯文字创建、带附件创建、无文字但有附件成功、空内容无附件失败。
  - strict 拒绝 projectId（JSON + multipart 均覆盖）；`files` 字段兼容测试。
  - 列表返回附件元数据、删除清理附件 DB + 临时目录、路径安全不越界。
  - journal/clip 成功迁移附件到 `附件/` 并清理临时目录。
  - 附件名称冲突循环去重直到唯一。
  - process 失败（clip schema 校验失败）保留原闪念和临时附件；task process 保留原闪念。
  - 迁移中途失败：真实制造 copyFile 失败（篡改第二条 temp_path 为不存在路径），验证已复制的正式附件被删除、DB `final_path` 重置为 null、note 保留、临时目录保留。

- `InboxView.test.ts`：新增带附件保存调用、附件选择后保存、保存失败保留附件、列表展示附件数量/名称、纯附件无文字可保存；修复 Vue unmount 内部错误（添加 `attachTo: document.body` 和 `afterEach` 统一 `unmount()`）。
