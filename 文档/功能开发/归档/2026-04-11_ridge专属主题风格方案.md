# ridge专属主题风格方案

- 状态：已确认
- 日期：2026-04-11
- Owner：AI Agent
- 目标版本：v0.1.0

## 1. 背景与问题

- 背景：ridge是AI工作平台，需要长时间使用的舒适视觉体验。现有default主题使用蓝紫色调(#4c6ef5)，过于冷峻，不符合"温暖人文+精密现代"的品牌定位。
- 现象/痛点：
  - 当前默认主题冷色调，长时间使用视觉疲劳
  - 缺乏独特的品牌识别色
  - 控件风格与AI工作平台定位不完全匹配
- 成功定义：用户在使用ridge进行1小时以上连续工作时，视觉舒适度提升，品牌识别度增强

## 2. 目标与非目标

- 目标：
  - G1：创建ridge专属主题文件(ridge.css)，定义温暖米白+青色强调配色
  - G2：更新registry.ts，注册ridge主题并设为DEFAULT_THEME_NAME
  - G3：确保所有UI组件自动应用新配色
- 非目标：
  - NG1：不删除现有其他主题(amber, amethyst等)
  - NG2：不改写组件逻辑代码，仅通过CSS变量调整视觉

## 3. 现状探索（Explorer）

- 入口与调用链：
  - packages/web/src/assets/registry.ts:1-30 - 主题注册中心，导入CSS文件并导出themes对象
  - packages/web/src/assets/default.css - 现有默认主题CSS变量定义
  - packages/web/src/lib/theme.ts:1-50 - 主题应用逻辑，通过applyTheme函数切换
  - packages/web/src/stores/settings.ts - 主题持久化存储
- 关键数据结构/状态：
  - registry.ts中`themes`对象映射themeName到CSS字符串
  - `DEFAULT_THEME_NAME: ThemeName = 'default'` 决定默认主题
  - CSS变量--background/--foreground等控制组件颜色
- 约束与坑点：
  - 必须使用Tailwind v4的@theme inline语法映射变量
  - 颜色需转换为hsl格式供shadcn组件使用
  - 深色/浅色切换依赖.dark类，需正确设置:root和.dark变量

## 4. 资料与依据（Librarian）

- 关键结论：
  - 用户确认风格方向：温暖人文 + 精密现代 + 青色强调
  - 浅色背景：#faf9f6（温暖米白），深色背景：#1a1a1a（柔和深灰）
  - 强调色：#2a9d8f（青色），主文字：#2d3436（深灰），次文字：#78716c（暖灰）
  - 控件风格：小圆角8-12px，弱化边框，扁平无阴影
  - 技术路径：参考default.css结构创建ridge.css，更新registry.ts设置DEFAULT_THEME_NAME='ridge'

## 5. 方案概览（主方案）

### 5.1 方案摘要

- 一句话：创建ridge.css主题文件，以温暖米白/柔和深灰为底色、青色为强调色，设为系统默认主题
- 取舍：
  - 直接替换default为ridge：简化用户选择，统一品牌体验
  - 保留其他主题：给用户更多选择余地
  - CSS变量方案：无需修改组件代码，纯样式层改造
- 影响面：
  - 新增：packages/web/src/assets/ridge.css
  - 修改：packages/web/src/assets/registry.ts（导入ridge.css，更新DEFAULT_THEME_NAME）
  - 影响：所有页面自动应用新配色

### 5.2 风险与回滚

- 风险：
  - 颜色对比度不足：需验证所有文字/背景组合满足WCAG 4.5:1
  - 用户习惯冲突：部分用户可能偏好原有蓝紫色调
- 回滚：
  - 在registry.ts保留原default主题，用户可通过设置切换回default
  - 紧急回滚：修改registry.ts中DEFAULT_THEME_NAME回'default'

## 6. 详细设计（Spec）

### 6.1 交互/行为

- 用户流程：
  1. 首次打开ridge → 自动应用ridge主题
  2. 打开SettingsPage → 外观设置 → 主题下拉显示"ridge"为选中
  3. 可切换其他主题(amber, amethyst等)或回到ridge
- 边界条件：
  - 升级用户：保留原有主题设置，新用户默认ridge
  - 系统主题跟随：dark/light模式自动切换对应配色

### 6.2 接口与契约

- API/函数签名：

  - 新增ridge.css导出CSS字符串，格式与其他主题一致
- 错误码/异常：

### 6.3 数据与存储

- Schema/字段：

- 迁移策略：

  - 新用户：默认使用ridge
