# 16 闪念处理为正式对象

## 目标

用户确认后把闪念处理为正式对象：任务、里程碑、剪藏、正式附件。

## 范围

- 处理为任务（task）
- 处理为里程碑（milestone）
- 处理为剪藏（clip / clipping）
- 处理为正式附件（attachment）
- 处理为日记（journal）——既有兼容路径，任务16不新增但代码保留
- 可在处理时选择项目（task、milestone）
- 处理成功后删除原闪念
- 附件迁移到正式附件目录

## 不做

- 不在采集阶段选择项目
- 不允许 AI 建议自动执行

## API

- `POST /api/fleeting/:noteId/process/journal`
- `POST /api/fleeting/:noteId/process/clip`
- `POST /api/fleeting/:noteId/process/task`
- `POST /api/fleeting/:noteId/process/milestone`
- `POST /api/fleeting/:noteId/process/attachment`

### 目标类型

- **任务16 核心类型**：task、milestone、clip、attachment（四种）
- **既有兼容路径**：journal（闪念早期已实现，任务16代码保留但不做为主推目标类型）

### 行为

- 处理动作必须由用户触发
- 附件先复制到正式目录，复制失败即中止
- 多附件场景：若部分附件复制失败，已复制附件必须回滚删除，确保不留半成品
- 创建目标对象和删除闪念记录必须在同一事务边界内（better-sqlite3 `db.transaction()`）
- 成功后删除闪念记录和附件 DB 引用，再异步清理临时文件
- 失败时不删除闪念和临时附件，不保留已复制的正式文件

## 测试

- 处理为任务创建任务并删除闪念
- 处理为里程碑创建里程碑并删除闪念
- 处理为剪藏创建 clip 并删除闪念
- 处理为附件迁移文件到 `附件/`
- 处理为日记追加到今日日记并删除闪念
- 目标创建失败时闪念仍存在
- 附件复制失败时闪念仍存在，临时附件（DB + 文件）完整保留
- 多附件部分迁移失败时：闪念保留、所有临时附件保留、已复制的正式文件被清理、不留半成品
- 处理 task / milestone 时可选择项目（projectId）

## 实现方案

### 后端

1. `routes/fleeting.ts` 中把 `process/task` 从占位改为真实实现
2. 新增 `process/milestone`、`process/clip`、`process/attachment` 路由
3. 所有 process 路由统一为三步安全边界：
   - 第一步：copyFleetingAttachmentsToFormal(db, workspaceDir, noteId) — 仅复制到正式目录，不删临时
   - 第二步：db.transaction() 包裹：删附件 DB 记录 + INSERT 目标对象 + 删闪念记录
   - 第三步：transaction 成功后异步 deleteFleetingTempFiles(workspaceDir, noteId)
4. copyFleetingAttachmentsToFormal 在部分复制失败时，回滚已复制的正式文件
5. 任务/里程碑支持 `projectId` 参数

### 前端

1. `lib/api.ts` 新增 `processFleetingToMilestone`、`processFleetingToAttachment`，更新 `processFleetingToTask` 签名
2. `composables/useInbox.ts` 新增 `processToMilestone`、`processToAttachment`，更新 `processToTask`
3. `InboxView.vue` 为闪念卡片提供处理操作 UI（选择目标类型、选择项目）
4. 任务/里程碑对话框中包含项目 Select（可选）

### 事务边界

- 使用 better-sqlite3 的 `db.transaction()` 包裹：
  1. 删除附件 DB 记录（deleteFleetingAttachmentRecords）
  2. 创建目标对象（INSERT）
  3. 删除闪念记录（DELETE FROM fleeting_notes）
- 附件复制在事务外先执行；复制失败直接抛错，不进入事务，闪念保留
- 多附件复制：按顺序复制，任一个失败时回滚已复制的文件，再抛错
- 如果目标 INSERT 失败（抛出异常），事务回滚，闪念保留；此时临时附件文件仍保留（因为清理在事务外/后执行）

## 验收

- 处理为任务时，创建 workspace_tasks 记录并删除闪念
- 处理为里程碑时，创建 workspace_milestones 记录并删除闪念
- 处理为剪藏时，创建 clips 记录并删除闪念
- 处理为附件时，仅迁移附件到正式目录并删除闪念
- 处理为日记时，追加到今日日记并删除闪念
- 失败时闪念和临时附件均保留
- 多附件部分迁移失败时，已复制的正式文件被清理，不留半成品
- 可选择项目（task、milestone）
- npm run check 通过
