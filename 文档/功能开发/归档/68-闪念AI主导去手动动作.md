# 68 闪念 AI 主导去手动动作

## 目标

把闪念从“用户手动选择处理动作”收口为“AI 主导分析队列”。后端不再暴露 `process/*` 人工处理动作；前端不再展示按建议、日记、任务、里程碑、剪藏、附件、删除等手动按钮，只保留分析失败时的重试入口。

## 完成内容

- 删除 `POST /api/fleeting/:noteId/process/journal|clip|task|milestone|attachment` 后端路由。
- 删除前端 `processFleetingTo*` API 封装和 `useInbox` 中的手动处理函数。
- `InboxView.vue` 仅展示闪念内容、附件、状态和失败重试；不展示 AI 建议文案或人工处理按钮。
- 临时附件仍保存在 `.ridge/fleeting-attachments/<noteId>/`，等待后续 AI 主导链路消费；手动 process 路由不会迁移或清理附件。
- 保留 `POST /api/fleeting/:noteId/analyze` 作为重试分析入口。
