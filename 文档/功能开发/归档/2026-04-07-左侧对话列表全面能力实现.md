# 左侧对话列表全面能力实现

## 任务目标

- 根据 openchamber 左侧会话列表能力梳理文档，重做当前项目的左侧对话列表。
- 左栏不能继续只展示运行时内存里的简单双列列表，必须接入真实持久化 session。
- 左栏能力至少覆盖项目分组、worktree 分组、父子会话树、recent、搜索、pin、归档、重命名、子会话和删除。

## 本次实现

### 服务端能力

- 会话列表从当前进程内存切换为基于 `SessionManager.listAll()` 的真实历史恢复。
- 新增 `project-context.js`，为每个 session 解析 `projectId/projectRoot/worktreeRoot/branch`。
- 新增 `session-metadata.js`，把归档状态和空会话最小索引持久化到工作区 `.pi-web/session-sidebar.json`。
- 显式从左栏新建但还没有 assistant 回复的空会话，会立即把 session header 落盘，因此刷新或重启后不会丢失。
- 新增和扩展的接口包括：
  - `GET /api/sessions`
  - `GET /api/sessions/:sessionId`
  - `PATCH /api/sessions/:sessionId`
  - `POST /api/sessions/:sessionId/archive`
  - `DELETE /api/sessions/:sessionId`
- `POST /api/sessions` 支持 `parentSessionId`，可直接从左栏创建子会话。

### 前端左栏能力

- 新增独立组件 `SessionSidebar.vue`，不再把左栏逻辑硬塞在 `App.vue`。
- 新增 `session-sidebar.ts` 负责构造：
  - 项目分组
  - `project root / worktree / archived` 三类 group
  - 父子会话树
  - pin + 时间排序
  - 搜索过滤
- 新增 `SessionSidebarSessionNode.vue` 递归渲染会话树节点。
- 左栏已支持：
  - recent 区
  - 项目折叠
  - group 折叠
  - 父会话展开/收起
  - 当前项目最后活跃 session 恢复
  - pin 置顶
  - 重命名
  - 归档/取消归档
  - 创建子会话
  - 已归档会话硬删除

### 右侧文件树联动

- 文件树边界不再只允许单一工作区目录。
- 同仓库 worktree 路径也会被服务端识别为允许浏览的 root。
- 切换到其他 worktree 下的 session 时，右侧文件树可正常跟随目录切换。
