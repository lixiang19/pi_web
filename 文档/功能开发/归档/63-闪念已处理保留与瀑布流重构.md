# 63 闪念已处理保留与瀑布流重构

## 目标

按 `_temp/fleeting-design-c.html` 的闪念时间线样式重构 Web 闪念页；处理后的闪念不删除，继续保留在列表中并标记为「已处理」。

## 范围

- `GET /api/fleeting` 继续返回未处理和已处理闪念。
- 五种处理动作成功后创建正式对象，并把 `fleeting_notes.status` 更新为 `processed`。
- 临时附件在处理成功后仍迁移到正式 `附件/` 并清理临时记录/文件。
- 前端列表增加全部、未处理、已处理过滤。
- 已处理闪念不再展示处理动作，仍允许手动删除。
- `InboxView.vue` 使用设计稿的顶部标题、过滤条、时间线卡片和底部捕捉栏样式。

## 不做

- 不恢复旧的 `收件箱/*.md` 文件模型。
- 不给已处理闪念自动恢复/反处理。
- 不新增 DB 列；复用 `fleeting_notes.status='processed'`。

## 验收

- `fleeting-api.test.ts` 覆盖日记、剪藏、任务、里程碑、附件处理后保留闪念并标记已处理。
- `fleeting-attachments.test.ts` 覆盖附件迁移成功后闪念保留为已处理。
- `useInbox.test.ts` 覆盖处理成功后本地列表保留闪念，待处理数下降。
- `InboxView.test.ts` 覆盖已处理状态展示和处理动作隐藏。
- `npm run check` 通过。
