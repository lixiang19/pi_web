# 实现文件树内文件夹和文件都能拖动到输入框，在输入框内填入路径 方案

- 状态：已确认
- 日期：2026-04-09
- Owner：admin
- 目标版本/里程碑：M1

## 1. 背景与问题

- **背景**：用户在使用文件树浏览项目结构时，经常需要将文件或文件夹路径引用到聊天输入框中
- **现象/痛点**：目前只能手动复制粘贴路径，操作繁琐，打断工作流程
- **成功定义**：用户可以直接从文件树拖动文件/文件夹到输入框，自动填充绝对路径

## 2. 目标与非目标

### 目标
- 支持从文件树拖动文件或文件夹到 WorkbenchComposer 输入框
- 拖放后在输入框中插入绝对路径文本
- 拖动时显示半透明预览（ghost image）
- 支持文件和文件夹两种类型
- 单文件/单文件夹拖放（不支持多选）

### 非目标
- 不支持多文件同时拖放
- 不支持文件夹递归展开填充所有子文件
- 不将拖放内容作为附件/资源引用格式
- 不扩展其他页面的输入框组件
- 不支持从系统文件管理器拖入（仅内部文件树）

### 关键数据结构

```typescript
interface FileTreeEntry {
  name: string;
  path: string;        // 绝对路径
  kind: "file" | "directory";
```

### 约束与坑点

1. 文件树采用懒加载，需处理好目录展开状态
2. 输入框使用 shadcn-vue Textarea 组件
3. 无现有拖放基础设施，需从零实现
4. 三栏布局中文件树和输入框位于不同区域

## 4. 资料与依据（Librarian）

- Vue 3 原生 HTML5 Drag and Drop API 支持
- shadcn-vue Textarea 组件可包装扩展
- 无需额外第三方库

## 5. 方案概览（主方案）

### 5.1 方案摘要

- **一句话**：使用原生 HTML5 Drag and Drop API，在 WorkspaceFileTree 中启用 draggable，在 WorkbenchComposer 中处理 drop 事件
- **取舍**：不使用第三方拖拽库，保持轻量；仅用原生 API 足够满足简单拖放需求
- **影响面**：修改 WorkspaceFileTree.vue（添加拖拽源）和 WorkbenchComposer.vue（添加放置目标）

### 5.2 风险与回滚

- **风险**：拖拽事件与现有点击/折叠事件冲突
  - **缓解**：使用 event.stopPropagation() 和适当的拖拽状态判断
- **回滚**：功能开关由 v-if 控制，可快速禁用

## 6. 详细设计（Spec）

### 6.1 交互/行为

**用户流程：**
1. 用户在文件树中按住文件/文件夹项开始拖动
2. 显示半透明预览（使用浏览器原生 ghost image）
3. 拖动到输入框区域时，输入框可接受放置
4. 释放后，在输入框当前光标位置插入绝对路径
5. 路径后自动添加一个空格

**边界条件：**
- 输入框未聚焦时拖放：聚焦并追加到末尾
- 输入框已有内容时拖放：插入到光标位置
- 拖放到非输入框区域：取消操作，无反馈

### 6.2 接口与契约

**WorkspaceFileTree 修改：**
- 为每个文件/文件夹项添加 `draggable="true"`
- 处理 `dragstart` 事件，设置 DataTransfer 数据

```typescript
// dragstart handler
const handleDragStart = (event: DragEvent, entry: FileTreeEntry) => {
  event.dataTransfer?.setData('text/plain', entry.path);
  event.dataTransfer?.setData('application/json', JSON.stringify(entry));
};
```

**WorkbenchComposer 修改：**
- 处理 `drop` 事件，读取 DataTransfer 数据
- 在 Textarea 当前光标位置插入路径

```typescript
// drop handler
const handleDrop = (event: DragEvent) => {
  event.preventDefault();
  const path = event.dataTransfer?.getData('text/plain');
  if (path) {
    insertAtCursor(path + ' ');
  }
};
```

### 6.3 数据与存储

无需新增数据结构，复用现有 FileTreeEntry。

- 浏览器兼容：所有现代浏览器支持 HTML5 Drag and Drop
