# 14 桌面采集入口（已归档）

## 目标

实现桌面端菜单栏采集入口，包括文字、截图、文件、剪贴板、选区、浏览器网址、录音等采集方式。

## 范围

- 桌面端必须登录服务器。
- 菜单栏常驻入口。
- 文字采集。
- 区域截图、窗口截图、全屏截图。
- 文件采集。
- 剪贴板采集。
- 当前选区采集。
- 浏览器网址采集。
- 录音采集。

## 不做

- 采集时不选择项目。
- 录音采集不立即转写。
- 采集内容不直接进 RAG。

## 验收

- 采集结果统一进入服务器闪念。
- 浏览器网址权限不足时明确失败。
- 截图或文件允许没有文字说明。
- 桌面离线不可用。

## 采集类型

| 类型 | 实现方式 | 保存内容 |
|------|----------|----------|
| text | 用户手动输入 | 文本内容 |
| clipboard | `pbpaste` (macOS) / `navigator.clipboard` (Web) | 剪贴板文本 |
| selection | Sentinel + Cmd+C + `pbpaste` | 前台应用选区文本 |
| browser_url | System Events + AppleScript + base64 编码 | URL、title、选中文本 |
| screenshot_region | `screencapture -i -s -x -o` | PNG 二进制 |
| screenshot_window | `screencapture -i -w -x -o` | PNG 二进制 |
| screenshot_fullscreen | `screencapture -x -o` | PNG 二进制 |
| file | `<input type="file">` | 文件 base64 |
| audio | `MediaRecorder` + `getUserMedia` | WebM 音频 |

## 实现细节

### 1. Tauri Rust 端（`packages/desktop/src-tauri/src/main.rs`）

#### 浏览器网址采集 `capture_browser_url`
- 通过 **System Events** 检测前台应用名称
- 只查询当前 **前台浏览器**（Safari / Chrome / Edge / Firefox）
- Safari/Chrome/Edge 通过 `do JavaScript` / `execute javascript` 读取页面选中文本
- Firefox 通过 `URL of front document` 读取 URL
- **base64 编码输出**：每个字段单独 base64 后用 tab 分隔，完全无损传输（支持逗号、tab、换行等特殊字符）
- Rust 端用 `base64` crate 解码

#### 剪贴板采集 `capture_clipboard`
- 使用 `pbpaste` 命令（比 AppleScript `get the clipboard` 更可靠）
- 空剪贴板时返回明确错误

#### 当前选区采集 `capture_selection`
- **1. 保存旧剪贴板**（`pbpaste`）
- **2. 写入 sentinel**（唯一时间戳标记 `__ridge_sentinel_<millis>__`）到剪贴板（`pbcopy`）
- **3. 发送 Cmd+C**（AppleScript System Events）
- **4. 等待 400ms**
- **5. 读取新剪贴板**（`pbpaste`）
- **6. 判断**：
  - 若新剪贴板仍是 sentinel → 前台应用没有可复制的选区 → 报错
  - 若新剪贴板为空 → 报错
  - 否则返回选区文本
- **7. 恢复旧剪贴板**并**验证恢复成功**（再次读取确认）
- 剪贴板恢复失败时返回警告错误（不静默忽略）

#### 截图
- 区域：`screencapture -i -s -x -o /tmp/file.png`
- 窗口：`screencapture -i -w -x -o /tmp/file.png`
- 全屏：`screencapture -x -o /tmp/file.png`
- 返回 `Vec<u8>` PNG 二进制，前端通过 `Blob` 转为 base64

#### 状态同步 `set_desktop_status`
- 前端登录/登出/网络变化时调用，同步 `server_online` 和 `authenticated`
- 菜单点击时检测，未登录或离线时阻止并 toast 错误

### 2. 前端桥接（`packages/web/src/lib/desktop-bridge.ts`）

- `isTauri()` 运行时检测（检查 `window.__TAURI_INTERNALS__`）
- 所有函数在 Web 环境下静默返回 `null`（不报错）
- 7 个桥接函数：`syncDesktopStatus`、`tauriCaptureClipboard`、`tauriCaptureBrowserUrl`、`tauriCaptureSelection`、`tauriCaptureScreenshotRegion`、`tauriCaptureScreenshotWindow`、`tauriCaptureScreenshotFullscreen`

### 3. 前端 UI（`packages/web/src/components/workspace/FleetingCaptureButton.vue`）

- `nativeTitle` / `nativeSelectedText` ref 保存浏览器采集的原生数据
- **Tauri 下无回退**：失败时 toast 明确错误，不会偷偷使用 webview 数据
- Web 端保留原有行为（`navigator.clipboard`、`window.location.href`、`window.getSelection()`）
- 保存时 metadata 使用原生返回的 title/selectedText（不是 `document.title`）
