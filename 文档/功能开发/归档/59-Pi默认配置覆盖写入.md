# 59 Pi 默认配置覆盖写入

## 状态

已完成。

## 目标

ridge 不再另造配置根，server 和桌面设备端都使用 Pi 默认配置根 `~/.pi/agent`，由 ridge 覆盖式写入。

## 实现

- 固定配置根：`~/.pi/agent`
- 仓库内置配置目录：`packages/server/pi-default-config/`，当前 `settings.json` 在这里手动维护。
- 服务启动时把内置配置目录非破坏性复制到 `~/.pi/agent`，覆盖内置同名文件，保留 Pi sessions 和用户自定义 skills/prompts 等目标侧资源，最后初始化 Pi 的 auth/model/settings。
- `auth.json`、`models.json`、`settings.json`、agents、skills 都从 Pi 默认配置根解析。
- 会话、临时资源 catalog、子代理和后台记忆会话共用同一配置入口。
- 桌面端 bundle 同步也直接覆盖物化到 `~/.pi/agent`。
- 删除旧资源 scope 模块，不再保留 ridge 私有配置根。
