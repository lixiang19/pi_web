# 会话区顶部 Header 方案

- 状态：已确认
- 日期：2026-04-19
- Owner：AI Agent

## 1. 背景与问题

- 背景：参考 pi 原生界面风格，会话顶部需要一个 header 栏
- 痛点：当前会话区没有 header，面板折叠控制分散（操作区用竖条按钮，文件面板无折叠控制）
- 成功定义：会话区顶部有 header，左侧显示可编辑标题，右侧两个按钮控制两个面板折叠

## 2. 目标与非目标

- 目标：
  - 新建 `ConversationHeader` 组件，左侧标题（草稿时只读，已保存会话可内联编辑重命名），右侧两个 icon 按钮
  - 折叠操作区文件按钮（移除原有竖条箭头，统一到 header）
  - 折叠/展开 git/文件面板按钮（新增折叠状态）
- 非目标：
  - 不改动操作区内部逻辑
  - 不改动文件面板内部逻辑

## 3. 现状探索

- `SessionTabContent.vue`：三栏布局，有 `isOperationPanelCollapsed` ref，操作区竖条在此
- `WorkbenchChatPanel.vue`：只是对话流+输入框，无 header
- `usePerSessionChat.ts`：`currentSessionTitle` computed、`isDraftSession` computed、`core.renameSessionTitle(sessionId, title)`
- 右侧文件面板 `ProjectFilePanel`：固定 `w-80`，没有折叠控制

## 5. 方案概览

在 `SessionTabContent.vue` 的 `<main>` 区域顶部插入 `ConversationHeader`（不改动 WorkbenchChatPanel），由 SessionTabContent 管理折叠状态并传入 props。

## 6. 详细设计

### 6.2 接口与契约

**ConversationHeader.vue props/emits:**
```typescript
defineProps<{
  title: string
  isDraftSession: boolean
  isOperationPanelCollapsed: boolean
  isFilePanelCollapsed: boolean
}>()
defineEmits<{
  'update:title': [title: string]        // 重命名成功后
  'toggle-operation-panel': []
  'toggle-file-panel': []
}>()
```

**SessionTabContent.vue 变更:**
- 新增 `isFilePanelCollapsed` ref (默认 false)
- 在 main 顶部添加 `ConversationHeader`
- 删除操作区竖条 `<div v-if="isOperationPanelCollapsed">` 部分
- 对右侧 `<aside>` 添加 `v-show="!isFilePanelCollapsed"`
- 处理 `update:title` → 调用 `chat.core.renameSessionTitle(chat.sessionId.value, title)`

### 6.1 交互/行为

- 标题：草稿会话不可编辑（cursor-default）；已保存会话点击进入内联编辑，Enter/失焦保存，Escape 取消
- 操作区按钮：`PanelRight` 图标，折叠时高亮（muted 状态），toggle 操作区
- 文件面板按钮：`FolderTree` 图标，折叠时高亮，toggle 文件面板
