# 08 任务里程碑数据模型

## 目标

实现任务和里程碑的 DB 模型。

## 范围

- `workspace_tasks`。
- `workspace_milestones`。
- 系统默认里程碑。
- 状态：pending、in_progress、blocked、reviewing、completed。
- 优先级：normal、important、urgent。
- 可选项目绑定。
- 任务必须挂到里程碑。

## 不做

- 不做多工作空间。
- 不允许 Agent 完成任务。

## 验收

- 首次查询可创建系统默认里程碑。
- 创建任务未指定里程碑时进入系统默认里程碑。
- 删除仍有任务的里程碑必须失败。
- 任务和里程碑都可绑定项目。

## 关联设计

- `文档/项目设计/任务里程碑与处理会话.md`
- `文档/项目设计/数据库与持久化.md`

## Spec 提取点

- 表字段。
- 状态迁移规则。
- 默认里程碑创建规则。

## Spec 草案

### 表

- `workspace_milestones`
- `workspace_tasks`

### 状态

- pending
- in_progress
- blocked
- reviewing
- completed

### 行为

- 首次查询任务系统时确保系统默认里程碑存在。
- 任务必须有里程碑。
- 删除仍有任务的里程碑失败。
- Agent 路径不得写入 completed。

### 测试

- 空库查询后存在默认里程碑。
- 创建任务不传 milestone 时挂默认里程碑。
- 删除非空里程碑返回错误。
- Agent 更新 completed 被拒绝。

## 验收验证

- 空库查询后创建并返回系统默认里程碑（`GET /api/workspace/milestones`）。
- 创建任务未传 `milestoneId` 自动挂默认里程碑；传不存在 `milestoneId` 返回 404。
- 删除仍有任务的里程碑返回 409；删除系统里程碑返回 400。
- 任务和里程碑创建/更新均支持 `projectId: string | null`；任务未显式 `projectId` 继承所属里程碑，显式 `null` 不继承。
- Agent 不能把任务或里程碑状态改为 `completed`；用户可按合法状态流转完成。
- 优先级仅允许 `normal/important/urgent`；非法值返回 400。
- 状态非法流转返回 400。
