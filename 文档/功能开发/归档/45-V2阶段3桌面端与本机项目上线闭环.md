# 45 V2 阶段 3 桌面端与本机项目上线闭环（已归档）

## 目标

完成 V2 阶段 3：桌面端不只是壳，而是可靠连接服务器的本机运行设备。

## 完成口径

- 设备注册、token、心跳、离线清理、bundle ack 在服务端真实实现。
- runtime bundle 支持下载、校验、物化到 ridge 管理目录，并能作为 Pi resource loader 真源。
- Web 对桌面本机项目创建会话、发送消息、读取消息/runtime、ask、permission、cancel、events 均走桌面 WebSocket 转发。
- 服务器只保存桌面会话轻量 `session_index`，不镜像桌面项目消息正文。
- 离线桌面设备禁止新建、继续和查看需要设备的项目会话。
- 桌面采集入口覆盖文字、区域/窗口/全屏截图、文件、剪贴板、当前选区、浏览器网址、录音。
- runtime bundle 与普通会话注入均读取 ridge 真源，不读取用户真实 `~/.pi` 作为 ridge 全局配置。

## 手工验收清单

- 新桌面设备首次启动后注册并出现在 `GET /api/devices`。
- Web 对在线桌面项目发起会话后，`session_index.run_location='desktop'` 且消息流回当前标签。
- 关闭桌面端或断开 WebSocket 后，项目新建/继续/查看会话返回明确离线原因。
- 桌面采集在未登录或服务器离线时显示明确不可用原因；在线时统一写入闪念。
