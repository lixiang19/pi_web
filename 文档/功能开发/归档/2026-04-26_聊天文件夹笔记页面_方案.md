# 聊天文件夹笔记页面方案

## 背景

用户希望在左侧侧边栏加入「笔记」页面，笔记内容直接来源于主聊天文件夹，并具备类似 Obsidian 的基础体验。

本次已确认首版范围：

- 范围：基础笔记库，不做双链、反向链接、标签和关系图。
- 数据来源：读取整个主聊天文件夹 `chat` 下所有 Markdown 文件，包括子目录。
- 编辑器：使用 Milkdown，编辑和预览实时一体化，不做独立预览区。
- 操作：支持新建和保存，不做重命名和删除。

## 现状结论

### 侧边栏与路由

- 一级导航入口定义在 `packages/web/src/composables/useWorkbenchPrimaryNavigation.ts`。
- 侧边栏渲染在 `packages/web/src/components/workbench/WorkbenchSidebar.vue`。
- 页面路由注册在 `packages/web/src/router/index.ts`。
- 页面壳由 `packages/web/src/layouts/PlatformShell.vue` 提供，右侧通过 `RouterView` 切换。

### 主聊天文件夹

- 服务端启动时会引导工作区 `chat` 目录。
- 系统信息已暴露：`chatProjectId`、`chatProjectPath`、`chatProjectLabel`。
- 当前文件接口已有 `/api/files/tree`、`/api/files/content`、Markdown 保存能力，但文件页边界服务整个工作区或项目文件树，不应直接把笔记语义混入现有文件页。

### Milkdown 调研结论

推荐使用：

- `@milkdown/vue`
- `@milkdown/crepe`

理由：

- `@milkdown/vue` 是官方 Vue 3 集成。
- `@milkdown/crepe` 提供开箱即用编辑器，适合首版基础笔记库。
- 内容变化可通过 Crepe listener 的 `markdownUpdated` 获取。
- 切换文件时用 Vue `key` 重新挂载编辑器，避免复杂运行时替换文档。

## 目标

在工作台左侧一级导航中增加「笔记」入口，进入后展示 `chat` 文件夹下的 Markdown 笔记库：

```text
左侧导航
  -> 笔记
    -> NotesPage
      -> 左侧 Markdown 文件列表
      -> 右侧 Milkdown 实时编辑器
      -> 新建笔记
      -> 保存笔记
```

## 非目标

首版不做：

- `[[wikilink]]` 双链解析。
- 反向链接。
- 标签系统。
- 图谱视图。
- 重命名。
- 删除。
- JSONL 会话文件转笔记。
- 把会话消息直接变成可编辑笔记。

## 设计方案

### 1. 新增一级导航入口

修改 `packages/web/src/composables/useWorkbenchPrimaryNavigation.ts`：

- `WorkbenchPrimaryRoute` 增加 `notes`。
- `workbenchPrimaryNavItems` 增加：`笔记`。
- 图标建议使用 `BookOpen` 或 `Notebook`。

`WorkbenchSidebar.vue` 不需要新增特殊逻辑，现有一级导航渲染会自动显示非 `settings` 项。

### 2. 新增路由和页面

修改 `packages/web/src/router/index.ts`：

- 增加 `/notes` 子路由。
- 页面组件为 `packages/web/src/pages/NotesPage.vue`。
- `meta.title` 为 `笔记`。

新增 `NotesPage.vue`：

- 使用工作台页面结构，不绕开 `PlatformShell`。
- 左侧为 Markdown 文件列表。
- 右侧为 Milkdown 编辑区。
- 顶部提供当前笔记名称、保存状态、新建按钮、保存按钮。

建议布局：

```text
NotesPage
├── header：笔记标题 / 新建 / 保存状态
└── main
    ├── 左侧：chat 下 Markdown 文件列表
    └── 右侧：Milkdown 编辑器
```

### 3. 新增笔记 API

在服务端新增独立 notes 接口，不把笔记语义塞进现有文件页 API：

- `GET /api/notes`
  - 列出 `chatProjectPath` 下所有 `.md` / `.markdown` 文件。
  - 返回相对路径、文件名、更新时间、大小。
- `GET /api/notes/content?path=...`
  - 读取指定 Markdown 文件。
  - `path` 是相对 `chatProjectPath` 的路径。
- `PUT /api/notes/content`
  - 保存指定 Markdown 文件内容。
  - 只允许写 `.md` / `.markdown`。
- `POST /api/notes`
  - 新建 Markdown 文件。
  - 默认写入 `chatProjectPath` 根目录或前端传入的相对路径。
  - 文件名由用户输入，服务端规范化为 Markdown 文件名。

安全边界：

- 所有笔记路径都以 `chatProjectPath` 为根。
- 路径必须经过 `isPathInsideRoot` 校验。
- 禁止绝对路径。
- 禁止 `..` 逃逸。
- 禁止非 Markdown 扩展名。

### 4. 前端 API 封装

修改 `packages/web/src/lib/api.ts`：

- 增加 `listNotes()`。
- 增加 `getNoteContent(path)`。
- 增加 `saveNoteContent(path, content)`。
- 增加 `createNote(name)`。

修改 `packages/web/src/lib/types.ts`：

- 增加 `NoteListItem`。
- 增加 `NoteContentResponse`。
- 增加 `CreateNoteRequest/Response` 相关类型。

### 5. Milkdown 编辑器组件

新增组件建议：

- `packages/web/src/components/notes/NoteMilkdownEditor.vue`

职责：

- 接收 `content` 与 `notePath`。
- 内部使用 `MilkdownProvider`、`Milkdown`、`useEditor`、`Crepe`。
- `markdownUpdated` 时向父组件发出最新 Markdown。
- `notePath` 或初始内容变化时通过 `key` 重新挂载编辑器。
- 样式通过 ridge 主题变量覆盖 Crepe CSS 变量。

组件边界：

- 只负责编辑器实例。
- 不负责拉取文件列表。
- 不负责保存请求。
- 不负责新建笔记。

### 6. 页面状态流

```text
NotesPage mounted
  -> GET /api/notes
  -> 默认选择第一篇笔记
  -> GET /api/notes/content?path=...
  -> NoteMilkdownEditor 渲染
  -> 用户编辑
  -> markdownUpdated 更新 dirty content
  -> 用户点击保存
  -> PUT /api/notes/content
  -> 保存成功后刷新列表元数据
```

新建流程：

```text
用户点击新建
  -> 输入文件名
  -> POST /api/notes
  -> 服务端创建 .md
  -> 前端刷新列表
  -> 选中新文件
  -> 打开 Milkdown 编辑器
```

## 风险

1. Milkdown 样式可能与 Tailwind/shadcn 主题存在冲突。
   - 处理：只在编辑器容器内覆盖 Crepe CSS 变量，不改全局 Tailwind reset。
2. Milkdown 动态切换内容没有稳定 `setMarkdown` API。
   - 处理：用 `notePath + loadedAt` 作为 key 重新挂载编辑器。
3. 读取整个 `chat` 子树可能扫描到非笔记 Markdown。
   - 这是用户确认的数据来源，首版全部展示，不做隐藏规则。
4. 服务端文件写入存在路径逃逸风险。
   - 处理：所有路径都相对 `chatProjectPath`，并强制校验 inside root 与扩展名。

## 回退策略

- 删除 `notes` 路由和侧边栏导航项即可从 UI 隐藏功能。
- 删除 notes API 不影响现有 `/api/files/*` 文件接口。
- Milkdown 依赖只用于 Notes 页面，不影响聊天主链路。
- 笔记文件写入 `chat` 文件夹内的普通 Markdown 文件，无额外数据库迁移。

## 验收标准

- 左侧一级导航出现「笔记」。
- 点击「笔记」进入 `/notes` 页面。
- 页面能列出 `chat` 文件夹及子目录下所有 Markdown 文件。
- 点击 Markdown 文件后，右侧 Milkdown 编辑器显示内容。
- 编辑内容后点击保存，文件真实写回主聊天文件夹。
- 可以新建 Markdown 笔记，并自动出现在列表中。
- 不展示重命名和删除功能。
- 运行检测通过：
  - `pnpm lint`
  - `pnpm typecheck`
  - 必要时执行 `pnpm build`
