# 23 RAG 更新删除移动规则

## 目标

实现 RAG 的更新、删除、移动和手动刷新规则。

## 范围

- 上传后立即转换和索引。
- 普通文件变更夜间索引。
- 用户手动立即索引。
- 内容 hash 判定。
- 文件删除删除 chunk。
- 文件移动或改名更新 metadata。
- 索引失败状态。

## 不做

- 上传后不立即更新图谱和 Wiki。
- MEMORY/daily 修改不立即 RAG。

## 验收

- 上传后 RAG 可检索。
- 普通 Markdown 编辑等待夜间或手动刷新。
- 夜间 worker 入口会消费 deferred pending。
- 内容 hash 未变时跳过 chunk 重建。
- 删除文件后检索不到旧 chunk。
- 索引失败进入通知中心。

## Spec 提取点

## Spec 草案

### 触发

- 上传正式文件：立即索引。
- 普通文件变更：夜间索引。
- 用户手动刷新：立即索引。

### 行为

- 内容 hash 未变则跳过重建。
- 文件删除删除对应 chunk。
- 文件移动更新 `source_path`。
- 索引失败生成通知。

### 测试

- 上传文件后可立即检索。
- 修改 Markdown 后未手动刷新前不重建。
- 夜间入口后 Markdown deferred 变更会重建。
- 内容未变重复索引不会重建 chunk。
- 删除文件后旧结果消失。
- 手动刷新成功更新 chunk。
