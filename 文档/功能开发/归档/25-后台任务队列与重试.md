# 25 后台任务队列与重试

## 目标

实现后台 Agent 和系统维护任务的队列、状态和重试。

## 范围

- `background_jobs`。
- 任务类型。
- 关联对象。
- 状态。
- 失败原因。
- 重试次数。
- 自动有限重试。
- 并发控制。

## 不做

- 不让后台任务阻塞主对话。
- 不把成功且无用户动作的结果写通知。

## 验收

- 同一天 summary agent 按 daily 串行。
- memory agent 全局串行。
- 同一个闪念只允许一个分析任务。
- 失败需要用户处理时生成通知事件。

## 关联设计

- `文档/项目设计/后台Agent系统.md`
- `文档/项目设计/通知与建议中心.md`

## Spec 提取点

- `background_jobs` 字段。
- 重试策略。
- 并发锁。

## Spec 草案

### 数据

- id
- type
- related_type
- related_id
- status
- error
- retry_count
- next_retry_at
- created_at
- updated_at

### 行为

- 后台任务不阻塞主对话。
- 失败按退避有限重试。
- 达到上限后保留 failed。
- 需要用户处理时创建通知。

### 测试

- 失败任务自动重试。
- 超过重试上限后不再执行。
- memory agent 全局串行。
- 同一闪念分析任务去重。
