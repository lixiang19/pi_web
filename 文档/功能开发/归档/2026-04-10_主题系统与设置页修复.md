# 2026-04-10 主题系统与设置页修复

## 背景

设置页修改主题后界面没有明显变化，且刷新后无法恢复用户选择。

## 根因

1. 构建期没有任何主题契约文件进入 Tailwind v4 编译链路，`bg-background`、`text-foreground` 等语义类没有稳定生成。
2. 运行时主题注册表错误地把主题文件当成“纯 token 文本”处理，主题值和构建期契约没有清晰分层。
3. `themeName` 没有进入服务端设置存储，启动时永远回落到默认主题。
4. 设置页 Select 绑定使用了错误的受控属性，当前值展示不稳定。

## 方案

### 1. 拆分构建期主题契约与运行时主题变量

- 新增 `packages/web/src/assets/theme-contract.css`
- 在 `packages/web/src/style.css` 中静态导入该文件
- 让 Tailwind 在构建期生成 `background/foreground/sidebar` 等语义工具类

### 2. 收敛运行时主题注入职责

- `packages/web/src/assets/registry.ts` 只移除浏览器无法执行的 `@theme inline` 代码块
- 保留各主题文件中的真实颜色值，直接写入运行时 `<style>`

### 3. 主题偏好进入统一设置模型

- 服务端 `Settings` 新增 `themeName`
- Web 端 `Settings`、`settings store`、`lib/theme.ts`、`useThemePreferences.ts` 全链路改为从统一设置源读取与写回
- 启动初始化改为读取已加载设置后的 `themeName + resolvedThemeMode`

### 4. 设置页修正

- `SettingsPage.vue` 改用 `model-value`
- 直接消费 `useThemePreferences()` 返回的 `themeName`

## 验证

- 已确认开发态 CSS 产物中重新生成 `text-foreground`、`text-sidebar-foreground`、`data-[state=active]:bg-background` 等语义主题类
- 本次修改涉及文件已通过 ESLint 与编辑器类型检查
- 当前仓库仍存在与本任务无关的全局 `vue-tsc` 失败，未在本次修复中处理
