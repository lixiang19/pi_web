# 2026-04-11 边框分隔治理与 ridge 分层重构

## 背景

工作台右侧 `Project Files` 标题栏出现了意外的深色下边框。根因不是单个组件配色错误，而是项目里存在裸 `border-*` 用法：

- Tailwind v4 的 `border-b` / `border-t` 只设置宽度与样式
- 未显式指定边框颜色时，会回退到 `currentColor`
- 外层壳 `text-foreground` 会把边框染成深色前景色

这与 ridge 主题“弱化边框、强调层次与留白”的设计目标冲突。

## 目标

1. 消灭核心工作台与对话区中的裸 `border-*`
2. ridge 主题下优先使用背景层次与留白，而不是细线分隔
3. 把常见头部/内嵌区域收敛为语义化 surface 类，避免再次出现隐式黑边

## 实施方案

### 1. 新增语义化 surface 类

在 `packages/web/src/style.css` 中新增：

- `ridge-panel-header`
- `ridge-panel-inset`

二者均使用 `background` 的 `color-mix()` 与主题变量组合，不依赖边框线做分隔。

### 2. 核心区域改造

- `ProjectFilePanel.vue`：标题栏从裸 `border-b` 改为 `ridge-panel-header`
- `WorkspaceFileTree.vue`：头部从 `border-b` 改为 surface header
- `WorkbenchChatHeader.vue`：顶部条从边框分隔改为 surface header
- `WorkbenchComposer.vue`：输入区上方从 `border-t border-border` 改为 surface 容器
- `CommandInput.vue`：命令输入头部从 `border-b` 改为 inset surface
- `ThinkingCard.vue` / `ToolCallCard.vue`：展开区从 `border-t` 改为 inset surface
- `SheetContent.vue`：侧边/上下弹出层去掉裸 `border-l/r/t/b`
- `SettingsPage.vue`：示例页头部去掉 `border-b`，改为 surface header

### 3. 规则收口

本次不通过修改主题默认边框色来掩盖问题，而是直接清理错误用法：

- 禁止依赖裸 `border-*` 做结构分隔
- 需要边界时优先用语义类或组件级明确样式
- 错误/状态性边框允许保留，但必须显式表达语义

## 验证

- `npm run lint`
- `npm run check`

## 结果预期

- ridge 主题下不再出现 `currentColor` 驱动的意外黑边
- 工作台/对话区分层更统一
- 后续新增头部与内嵌区可以直接复用语义 surface 类
