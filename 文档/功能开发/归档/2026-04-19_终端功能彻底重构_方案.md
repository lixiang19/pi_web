# 终端功能前端完全重构 方案

- 状态：已确认
- 日期：2026-04-19
- Owner：ridge 前端

## 1. 背景与问题

- 背景：终端页面是 ridge 的核心功能，但当前设计臃肿
- 现象/痛点：
  - 页面顶部有大标题 + 多状态徽章，占据大量空间，实际价值低
  - 标签采用宽卡片（固定 220px），横向滚动体验差，视觉嘈杂
  - 侧边栏与标签切换操作分离，操作流割裂
  - 分屏功能无关闭按钮，体验不完整
  - `TerminalPage.vue` 承担太多职责，难以维护
- 成功定义：终端页面变成 IDE 风格，95% 空间给终端，标签极简紧凑，操作通过右键菜单内聚

## 2. 目标与非目标

- 目标：
  - 重新设计布局为 IDE 风格（标签栏 + 工具栏行 + 终端区，无大标题）
  - 标签栏类似浏览器标签（标题 + × 关闭按钮）
  - 新建终端弹出工作目录选择 Dialog
  - 标签右键菜单：重命名 / 重启（选择新目录）/ 关闭
  - 删除侧边栏
  - 移除分屏功能（同时清理 `useTerminalPool` 中的 split 逻辑）
- 非目标：
  - 不改动服务端代码
  - 不改动 `useTerminalContextOptions.ts`
  - 不改动 `@wterm/dom` 的使用方式

## 3. 现状探索

- 入口：`packages/web/src/pages/TerminalPage.vue`（约 320 行）
- `TerminalViewport.vue`：终端渲染器，顶部有冗余标题区
- `TerminalSidebar.vue`：侧边栏，含工作目录选择 + 重命名 + 重启 + 关闭
- `useTerminalPool.ts`：含 split 逻辑（splitTerminalId / visibleTerminals / openSplit / closeSplit / setSplitTerminal）
- 约束：
  - 标签右键菜单用 shadcn ContextMenu 组件
  - Dialog 用 shadcn Dialog 组件
  - 重命名/重启通过 Dialog 多步流程完成

## 4. 方案概览

### 4.1 新布局结构

```
┌─────────────────────────────────────────────────────┐
│ [+]  Terminal1 ×  Terminal2 ×  Terminal3 ×           │  ← TerminalTabBar（紧凑）
├─────────────────────────────────────────────────────┤
│                                                     │
│              TerminalViewport (全屏占满)              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 4.2 组件拆分

| 文件 | 操作 | 说明 |
|------|------|------|
| `TerminalPage.vue` | 完全重写 | 极简布局容器，只组合子组件 |
| `TerminalTabBar.vue` | 新增 | 标签栏：[+] 新建 + 标签列表（右键菜单） |
| `TerminalViewport.vue` | 重写 | 移除顶部标题栏，纯渲染区 |
| `TerminalCreateDialog.vue` | 新增 | 新建终端的工作目录选择 Dialog |
| `TerminalRenameDialog.vue` | 新增 | 重命名 Dialog（单个输入框） |
| `TerminalRestartDialog.vue` | 新增 | 重启 Dialog（工作目录选择） |
| `TerminalSidebar.vue` | 删除 | 功能迁移到右键菜单 + Dialog |
| `useTerminalPool.ts` | 修改 | 删除 split 相关：splitTerminalId / visibleTerminals / openSplit / closeSplit / setSplitTerminal |

### 4.3 风险与回滚

- 风险：删除 split 导致现有 `visibleTerminals` 引用报错 →  TerminalPage 中已移除用法，风险低
- 回滚：git revert 即可

## 5. 详细设计

### 5.1 TerminalTabBar 行为

- 左侧固定 `[+]` 按钮，点击触发 `TerminalCreateDialog`
- 标签列表：水平排列，`overflow-x-auto`，每个标签：
  - 显示 terminal.title（truncate + max-w-[160px]）
  - `×` 关闭按钮（点击 stop，emit close）
  - 激活态：bg-muted/60 border-b-2 border-primary
  - 右键 ContextMenu：重命名 / 重启 / 分隔线 / 关闭
- 空态（无终端）：只显示 `[+]` 按钮

### 5.2 TerminalViewport 变更

- 删除顶部 `.flex.items-center.justify-between.border-b` 整个 div（标题 + 状态徽章）
- 终端区域 `div.ref=hostRef` 直接铺满 section
- 保留 error overlay
- 保留 `terminalStyle` CSS 变量（主题色）

### 5.3 useTerminalPool 变更

移除：
- `splitTerminalId`
- `splitTerminal`
- `visibleTerminals`
- `openSplit()`
- `closeSplit()`
- `setSplitTerminal()`

调整：
- `activeTerminal.value` 就是显示的那个终端
- return 中移除 split 相关导出

### 5.4 Dialog 规格

**TerminalCreateDialog**:
- props: `options: TerminalCwdOption[]`, `open`
- emits: `update:open`, `create(cwd)`
- UI: Select 工作目录 + 创建按钮

**TerminalRenameDialog**:
- props: `terminal: TerminalSnapshot | null`, `open`
- emits: `update:open`, `rename(id, title)`
- UI: Input + 保存按钮

**TerminalRestartDialog**:
- props: `terminal: TerminalSnapshot | null`, `options: TerminalCwdOption[]`, `open`
- emits: `update:open`, `restart(id, cwd)`
- UI: Select 工作目录 + 重启按钮

## 6. 接口与契约

`useTerminalPool` 返回（精简后）:
```ts
{
  terminals: Ref<TerminalSnapshot[]>;
  activeTerminalId: Ref<string>;
  activeTerminal: ComputedRef<TerminalSnapshot | null>;
  isLoading: Ref<boolean>;
  isMutating: Ref<boolean>;
  error: Ref<string>;
  load(options?: { defaultCreatePayload?: TerminalCreateRequest }): Promise<...>;
  createNewTerminal(payload?: TerminalCreateRequest): Promise<TerminalSnapshot>;
  activateTerminal(id: string): void;
  renameTerminalTitle(id: string, title: string): Promise<TerminalSnapshot>;
  restartTerminalSession(id: string, payload: TerminalRestartRequest): Promise<TerminalSnapshot>;
  closeTerminal(id: string): Promise<boolean>;
}
```
