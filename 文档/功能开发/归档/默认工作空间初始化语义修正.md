# 默认工作空间初始化语义修正

## 背景

启动 ridge 时，普通用户不应该理解或填写环境变量。工作空间是 ridge 管理的用户内容根目录，服务启动时应自动解析并创建默认目录。

## 目标

- 删除环境变量工作空间覆盖逻辑。
- 默认工作空间为 `~/ridge-workspace`。
- 当前工作空间路径写入 `~/.pi/ridge.db` 的 `ridge_meta.current_workspace_dir`。
- 服务启动时确保默认工作空间目录存在。
- 首次启动时创建基础模板目录，保证文件树、收件箱、日记、数据库入口有稳定根目录。

## 输入

- 用户主目录：`os.homedir()`

## 输出

- 后端 `defaultWorkspaceDir` 解析为绝对路径。
- 默认工作空间目录及基础模板目录已创建。

## 边界

- 不再把 npm 启动目录、源码目录或隐藏应用数据目录当作默认工作空间。
- 不迁移已有用户数据。
- 不改变 API 契约，只修正默认目录解析和启动初始化。

## 模板目录

- `收件箱/`
- `日记/`
- `笔记/`
- `项目/`
- `阅读/`
- `数据库/`
