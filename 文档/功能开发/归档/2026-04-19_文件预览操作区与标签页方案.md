# 文件预览操作区与标签页方案

- 状态：已完成
- 日期：2026-04-19
- Owner：ridge
- 目标版本/里程碑：Workbench File Preview Phase 1

## 1. 背景与问题

- 背景：当前工作台已经具备会话区与右侧文件树/Git 功能区，但缺少独立操作区，无法承载文件预览、Markdown 编辑以及后续更多标签式能力。
- 现象/痛点：
  - 当前布局仍是“会话区 + 右侧功能区”两列，无法在中间承载独立操作区。
  - 文件树只能展开目录和拖拽路径，点击文件不会打开内容。
  - 服务端只有目录树接口，没有文件内容读取、图片流、Markdown 保存这条完整链路。
  - 后续该区域会承载多种功能，如果继续把文件预览塞进文件树面板，会把职责边界做脏。
- 成功定义：用户点击文件树中的文件后，会话区与文件树之间出现独立操作区标签并打开对应文件；同一路径重复点击复用同一标签；Markdown 支持源码编辑、组件化块渲染和块级内容发给 AI；代码文件支持语法高亮、行号和语言标签；code/text 在超过 5MB 时进入大文件模式并首屏展示前 1000 行；HTML 文件支持基础 sandbox 预览；图片/普通文本按类型预览；预览标签按会话独立保留，操作区宽度可拖拽。

## 2. 目标与非目标

- 目标：
  - 将当前工作台升级为“会话区 | 操作区 | 文件树/Git 面板”三列结构。
  - 新增独立的操作区模块与状态编排，后续可复用到其它入口，不把预览逻辑耦合进文件树组件。
  - 打通文件树点击 -> 打开标签 -> 拉取内容 -> 按类型预览的完整链路。
  - 为 Markdown 文件提供源码编辑、组件化块渲染和块级内容发给 AI 的动作能力，并按自动保存语义写回磁盘。
  - 为代码文件提供只读语法高亮、行号和语言标签，不再与普通文本共用同一个纯文本预览器。
  - 为 code/text 大文件提供独立的大文件模式：5MB 触发、首屏前 1000 行、按需继续加载。
  - 为 HTML 文件提供基础 sandbox 预览，不再继续混入代码高亮分支。
  - 保持预览标签按当前会话独立维护，与现有 LRU 会话实例生命周期一致。
  - 支持操作区宽度拖拽调整，并与现有主题体系保持一致。
- 非目标：
  - 不做预览状态持久化。
  - 不做代码预览工具化（搜索、outline、复制工具栏等）。
  - 不做评论持久化。
  - 不在本阶段接入“其它入口打开文件”的具体业务，只把能力抽离为独立模块。
  - 不引入完整 IDE 级代码编辑器；Markdown 以轻量编辑为目标，其它类型只读。
  - 不做旧两列布局兼容，也不保留旧结构分支。
  - 不支持任意二进制文件编辑；非文本且非图片文件只展示不支持预览状态。

## 4. 资料与依据（Librarian）

- 关键结论：
  - 前端已具备 `vue-stream-markdown`、`Textarea`、`Separator` 等基础能力，可直接复用 Markdown 渲染、轻量编辑和分隔条视觉，不需要引入非常见第三方库。
  - `packages/web/package.json` 已有 `shiki` 依赖但尚未用于文件预览，代码文件高亮应直接复用它，而不是再引入 `highlight.js` / `Prism` / 重型编辑器。
  - 当前大文件体验的根因在服务端：`MAX_FILE_PREVIEW_BYTES` 超过 5MB 直接返回 `unsupported`，必须收敛成大文件模式而不是继续在前端兜底。
  - HTML 预览不能直接混进现有代码高亮链路，必须升级为独立 `previewKind` 和 sandbox 容器。
  - Markdown “评论”在参考实现里的真实价值是块级锚点和发给 AI 的上下文，不是本地注释卡片；在当前仓库里最短接线点应直接接到当前会话 `composer.draftText`。
  - 当前仓库没有现成的可拖拽分栏组件，因此拖拽宽度需要围绕现有 shadcn-vue/基础 UI 组件补一层交互逻辑，而不是再塞回固定宽度面板。
  - 文件预览问题的根源在服务端缺少内容接口，而不是前端少一个组件，所以必须同时补上读取、图片流和保存契约。
  - 操作区应独立成模块，由上游通过事件打开文件，而不是把状态硬塞进 `WorkspaceFileTree` 或 `ProjectFilePanel` 内部。

## 5. 方案概览（主方案）

### 5.1 方案摘要

- 一句话：在保持独立操作区架构不变的前提下，把 Markdown 块级评论重构成发给 AI 的动作锚点，为 code/text 引入大文件模式，并将 HTML 拆到独立 sandbox 预览分支。
- 取舍：不再把 Markdown 评论当注释系统继续补功能，而是直接改成面向 AI 的块级动作；大文件问题从服务端读法和协议边界根治，不做前端纯样式兜底；HTML 不直接插入主 DOM，而是走 sandbox 容器隔离。
- 影响面：
  - Web：工作台三列布局、操作区模块、文件树事件链路、预览状态编排。
  - Server：文件内容读取接口、图片流接口、Markdown 保存接口。
  - Protocol：文件预览载荷与保存请求的共享类型。

### 5.2 风险与回滚

- 风险：
  - 自动保存存在并发覆盖风险：通过每个标签独立的保存序号与最新写入优先策略收口，避免旧请求回写覆盖新内容。
  - 图片/大文件预览可能带来加载抖动：由服务端先识别文件类型和可读性，前端只渲染明确支持的预览模式。
  - 三列布局引入拖拽后，可能影响会话区和右侧面板最小宽度：通过宽度上下限约束保证布局稳定。
- 回滚：
  - 若开发期发现链路不可用，可回退新增操作区与文件内容接口，但不保留长期兼容分支。
  - 回滚只删除本次新增模块与接口，不改动既有 Git/文件树能力边界。

## 6. 详细设计（Spec）

### 6.1 交互/行为（如适用）

- 用户流程：
  1. 用户在文件树或收藏列表点击文件。
  2. `WorkspaceFileTree` 发出 `select-file` 事件，`ProjectFilePanel` 向上继续转发。
  3. `SessionTabContent` 把文件路径交给 `WorkbenchOperationPanel` 的预览状态管理器。
  4. 若该路径已存在标签，则直接激活已有标签；否则创建新标签并请求服务端读取内容。
  5. 操作区顶部展示文件标签，正文区域按文件类型渲染：Markdown 源码编辑 + 组件化块预览 + 块级 AI 动作、代码高亮只读预览（含行号和语言标签）、code/text 大文件模式、HTML sandbox 预览、普通文本只读预览、图片预览、其它类型显示不支持状态。
  6. Markdown 编辑时自动触发保存；关闭标签、切换标签或切换会话前会先冲刷当前待保存内容。
  7. 切换会话时，各会话保留自己的操作区标签与当前活动项。
- 边界条件：
  - 同一路径重复点击不新增重复标签，只切换到已有标签。
  - 关闭活动标签后自动切到相邻标签；没有标签时展示空态。
  - Markdown 保存失败时必须在标签内显示错误，并保留当前编辑内容。
  - code/text 超过 5MB 时不再直接显示 unsupported，而是进入大文件模式：首屏前 1000 行，只读，允许继续加载更多。
  - HTML 预览必须在 sandbox 容器内渲染，不能直接把正文插入应用 DOM。
  - Markdown 块级动作只负责把块内容和用户输入送入当前会话 composer，不做本地持久化评论列表。
  - 图片通过独立 blob 接口展示，不把二进制内容塞进 JSON。
  - 不支持预览的文件显示文件类型、路径和“不支持预览”说明。

### 6.2 接口与契约

- API/函数签名：
  - `GET /api/files/content?root=<root>&path=<path>`：`{ root, path } -> FilePreviewPayload`
  - `GET /api/files/content-window?root=<root>&path=<path>&startLine=<n>&lineCount=<m>`：`{ root, path, lines, startLine, nextStartLine?, totalLines? }`
  - `GET /api/files/blob?root=<root>&path=<path>`：返回图片或其它二进制流，供 `<img>` 等只读预览使用
  - `PUT /api/files/content`：`{ root, path, content } -> FileSaveResponse`
  - `useWorkbenchFilePreview(rootDir)`：`openFile(path) / closeFile(tabId) / activateFile(tabId) / flushActiveTab() / loadMore(tabId) / tabs / activeTabId / panelWidth`
  - `ProjectFilePanel`：新增 `open-file` 事件，向父级抛出所选文件信息
- 错误码/异常：
  - 读取或保存目标超出 `root` 边界时返回 400。
  - 请求目录、非文件路径或不存在文件时返回 400/404，并带明确错误信息。
  - `PUT /api/files/content` 只允许 Markdown 文件写入，其它类型返回 400。
  - 无法按 UTF-8 读取的非图片文件返回 `unsupported` 预览类型，不伪造文本内容。

### 6.3 数据与存储（如适用）

- Schema/字段：
  - `FilePreviewPayload`
    - `root: string`
    - `path: string`
    - `name: string`
    - `extension: string`
    - `mimeType: string`
    - `size: number`
    - `previewKind: "markdown" | "code" | "text" | "html" | "image" | "unsupported"`
    - `content?: string`
    - `isLargeFile?: boolean`
    - `previewLineCount?: number`
    - `nextStartLine?: number`
    - `readOnly: boolean`
  - `PreviewTabState`
    - `id: string`（使用路径去重）
    - `path: string`
    - `title: string`
    - `previewKind: string`
    - `content: string`
    - `savedContent: string`
    - `isLoading: boolean`
    - `isSaving: boolean`
    - `error: string`
    - `comments: WorkbenchMarkdownComment[]`（待重构为块级 AI 动作临时态）
    - `extension: string` / `mimeType: string`（供代码高亮语言识别）
  - `OperationPanelState`
    - `activeTabId: string`
    - `tabs: PreviewTabState[]`
    - `panelWidth: number`
- 迁移策略：
  - 不做持久化迁移。
  - 操作区标签状态只驻留在当前 `SessionTabContent` 实例，由现有会话保活机制自然保留。

### 6.4 兼容性与发布

- 兼容策略：直接升级为三列布局，不保留旧两列结构。
- 发布步骤：
  1. 先补 protocol 类型与 server 文件内容接口。
  2. 再实现操作区状态与预览组件。
  3. 最后接入三列布局、文件树事件和宽度拖拽。
  4. 完成后执行 eslint 与 TypeScript 检查。

### 6.5 可观测性

- 日志：
  - 文件内容读取日志（仅路径、类型、大小，不记录正文）
  - Markdown 保存日志（路径、大小、成功/失败）
- 指标：
  - 操作区打开文件成功率
  - Markdown 自动保存成功率
- 告警：
  - 连续读取失败或保存失败时在界面内显式提示
