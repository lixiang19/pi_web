# 17 文件页与正式附件目录 — 实现规格

## 目标

实现工作空间文件页和正式附件目录展示，替代 `WorkspacePage.vue` 中的 `WorkspaceFeaturePlaceholder`。

## 范围

- 文件树展示工作空间可见目录。
- 隐藏 `.ridge`。
- 正式附件目录 `附件/` 可见并可打开。
- 文件打开与 Markdown 文件查看。
- 文件处理状态展示（`pending | converting | converted | indexed | convert_failed | index_failed`）。
- 正式附件可被引用（通过文件页交互）——**注意：引用协议与RAG索引属于后续任务（任务22-24），当前仅展示文件与处理状态，不做入队/索引。**

## 不做

- 不展示临时附件（`.ridge/fleeting-attachments` 等）。
- 不做复杂文件编辑器（沿用现有 Markdown 预览 + 只读代码预览）。
- 不读取外部项目文件（仅服务器工作空间根目录为边界）。

## 验收

- `附件/` 是用户可见长期资产。
- 正式附件可进入 RAG（DB 状态表 `file_processing_status` 已就位，支持后续 RAG 索引链路读取状态；**RAG 入队/索引/检索属于任务 22-24，本任务不做**）。
- `.ridge` 不作为普通目录展示。
- 文件页以服务器工作空间为边界。

## API

- `GET /api/workspace/files/tree?path=<dir>`  
  返回 `FileTreeResponse`，其中 `entries[*].processingStatus` 为文件处理状态；目录节点无此字段。
- `GET /api/workspace/files/read?path=<file>`  
  返回 `FilePreviewPayload`；拒绝 `.ridge` 内文件。

## 数据模型

- `file_processing_status(file_path PK, workspace_path, status, content_hash, converted_at, indexed_at, error, updated_at)`

## 测试要点

1. 文件树包含八个预制目录（项目、笔记、日记、剪藏、附件、记忆、Wiki、空间）。
2. 文件树不包含 `.ridge`。
3. 正式附件（`附件/` 下文件）可读取。
4. 临时附件（`.ridge` 下文件）读取返回 400。
5. 文件节点带有 `processingStatus` 字段。
6. 前端 `FilesView.vue` 渲染文件树并展示状态徽章。
