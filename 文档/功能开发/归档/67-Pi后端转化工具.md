# 67-Pi 后端转化工具

## 背景

Python Converter 已承载文档/URL、图片视觉 OCR、音频 Groq 转写等重型转换能力，但 Pi Agent 会话目前只能通过文件处理队列或闪念流程间接使用这些能力。需要给后端 Pi 增加自定义工具，让 Agent 可按需调用转换服务。

## 目标

- 新增 Pi 自定义工具 `convert_file_to_markdown`。
- 工具只调用 Python Converter，不在 Node 内部自研解析。
- 文件路径必须限制在 workspace 内。
- 工具纳入 `read` 权限控制。

## 验收

- 单元测试覆盖文件转换、workspace 越界拒绝、权限映射。
- 会话资源 loader 注册 conversion tools extension。
- 更新模块梳理与记忆。
- 修改 `.ts` 后根目录运行 `npm run check`。
