# 文件树功能修复与完善方案

## 一、崩溃 Bug 修复（阻断性）

### Bug 1：session-payload.ts fileManager 初始化时序

**文件**: `packages/server/src/session-payload.ts`

`fileManager` 在模块加载时创建，此时 `deps.defaultWorkspaceDir` 为 `undefined`（`initSessionPayload` 还没调用）。导致 `resolveManagedRoot` 中 `isPathInsideRoot` 永远 false，每个请求都走 `ensureManagedProjectScope`。

**修复**：改为懒初始化函数

```ts
let fileManagerInstance: ReturnType<typeof createFileManager> | null = null;
export const getFileManager = () => {
  if (!fileManagerInstance) {
    fileManagerInstance = createFileManager({
      defaultWorkspaceDir: deps.defaultWorkspaceDir,
      ensureManagedProjectScope: async (candidatePath: string) => {
        await ensureManagedProjectScope(candidatePath);
      },
    });
  }
  return fileManagerInstance;
};
```

## 二、功能完善设计

### 1. 右键菜单（精简版）

**位置**：FileTreePanel.vue 文件/目录节点

**菜单项**：
| 条目 | 文件 | 目录 | 说明 |
|------|------|------|------|
| 新建文件夹 | ❌ | ✅ | 在选中目录下创建 |
| 重命名 | ✅ | ✅ | 触发行内编辑 |
| 删除 | ✅ | ✅ | 移到回收站，需确认弹窗 |
| 收藏/取消收藏 | ✅ | ❌ | 切换收藏状态 |

**实现**：使用已安装的 shadcn-vue `ContextMenu` 组件，参考 InboxView 的实现。

**菜单分组**：

```
--- 操作组 ---
新建文件夹    (仅目录)
---
重命名
删除          (destructive 色)
--- 分隔 ---
收藏/取消收藏  (仅文件)
```

### 2. 重命名（行内编辑）

**触发方式**：

- 右键菜单「重命名」
- 键盘 F2（当选中节点时）

**交互**：

1. 文件名处变为 input 输入框，自动选中文件名（不含扩展名）
2. Enter 确认 → 调用 `moveFileEntry` API（改 name）
3. Esc 取消 → 恢复原显示
4. 点击外部 → 等同 Enter 确认
5. 输入为空或与原名相同 → 取消
6. 重名 → 后端返回 409，前端 toast 提示

**API**：`moveFileEntry({ root, path, name })` — 已有

### 3. 删除（确认弹窗）

**触发方式**：右键菜单「删除」

**交互**：

1. 弹出 AlertDialog 确认对话框
2. 提示：「确定将 {name} 移到回收站？」
3. 确认 → 调用 `trashFileEntry` API
4. 刷新文件树

**API**：`trashFileEntry(root, path)` — 已有（使用 trash 库移到系统回收站）

### 4. 新建文件夹（右键创建）

**触发方式**：右键目录 → 「新建文件夹」

**交互**：

1. 在选中目录下新增一个行内编辑节点，默认名「未命名文件夹」
2. 自动进入行内编辑模式
3. Enter 确认 → 调用 `createFileEntry({ root, directory, name, kind: 'directory' })`
4. 创建失败 → toast 提示

**API**：`createFileEntry({ root, directory, name, kind: 'directory' })` — 已有

### 5. 文件类型图标（公共函数）

### 6. 服务端搜索 API

**新增端点**：`GET /api/files/search?root=...&query=...&limit=50`

**后端实现**：

- 递归遍历 root 目录
- 按文件名（basename）模糊匹配 query（大小写不敏感）
- 返回 `FileTreeEntry[]`，按修改时间降序
- 忽略 `.git`、`node_modules` 等目录
- 限制 50 条结果

**前端**：

- 搜索 Tab 输入时，debounce 300ms 调用搜索 API
- 搜索结果展示文件名 + 相对路径
- 点击结果打开文件
