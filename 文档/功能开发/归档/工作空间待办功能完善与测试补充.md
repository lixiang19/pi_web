## 三、详细设计

### F1：待办搜索/筛选

**界面**：TaskView 顶部加一行工具栏

```
[🔍 搜索待办...] [状态 ▾] [优先级 ▾] [标签 ▾]
```

- 搜索框：实时过滤 title（computed 过滤，不走 API）
- 状态下拉：全部 / 待做 / 进行中 / 已完成
- 优先级下拉：全部 / 高 / 中 / 低
- 标签下拉：动态取已有标签列表，支持多选

**数据流**：

- 新增 `searchQuery`、`filterStatus`、`filterPriority`、`filterTags` 四个 ref
- 新增 `filteredTasks` computed，在原 `pendingTasks`/`completedTasks` 基础上二次过滤
- 清单/看板/今日三视图统一消费 `filteredTasks`

### F2：清单拖拽排序

**方案**：引入 `vuedraggable-next`（Vue 3 专用 sortablejs 封装）

- 清单视图的任务行用 `<draggable>` 包裹
- 拖拽结束回调更新 `tasks` 数组顺序
- 需在 `TaskItem` 中新增 `order` 字段（number）作为排序依据
- 后端 PATCH 支持批量排序更新（`PATCH /api/workspace/tasks/reorder`）

**影响**：

- 数据结构变化：TaskItem 新增 `order?: number`
- 后端新增 API：`PATCH /api/workspace/tasks/reorder`
- 向后兼容：`order` 可选，旧数据无 order 字段时按 `createdAt` 排序

### F3：看板拖拽跨列

**方案**：同样使用 `vuedraggable-next`

- 看板三列（待做/进行中/完成）各自用 `<draggable>` 包裹
- `group="tasks"` 共享组名，允许跨列拖拽
- 拖拽跨列时自动更新 `status` 字段
- 拖拽同列内调整顺序时更新 `order` 字段
- 拖拽结束触发 `updateTask()` API 同步

**关键逻辑**：

```ts
// onEnd 回调
if (fromColumn !== toColumn) {
  task.status = toColumnStatus; // 跨列→更新状态
}
// 统一更新 order（取目标列内新位置索引）
await updateTask(task.id, { status: task.status, order: newIndex });
```
