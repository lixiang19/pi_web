# 2026-04-13 会话历史按轮分页与分段展开修复

## 背景
- 旧会话续聊后，前端会让旧消息“看起来消失”。
- 根因不止渲染层隐式折叠。
- 更深层问题是 server / web 都把历史窗口建立在“消息条数”上：`80/400`、`+80`。
- 一旦消息里混入 thinking、toolCall、toolResult，消息条数和用户理解的“对话轮次”就彻底脱钩。

## 目标
- 后端历史分页单位改成 **user 轮次**。
- 前端默认只显示最近 **3 轮**。
- 前端提供三种显式动作：
  - `展开更早 3 轮`
  - `折叠旧轮次`
  - `加载更早历史`
- 删除旧的 80/400 条数截窗语义。

## 方案
### 后端
- `packages/server/src/index.ts`
  - 删除 `DEFAULT_SESSION_MESSAGE_LIMIT / MAX_SESSION_MESSAGE_LIMIT`
  - `GET /api/sessions/:id` 与 `GET /api/sessions/:id/stream` 改读 `rounds`
  - `toSessionSnapshot()` 改为：
    - 找出全部 `user` 消息索引
    - 以 user 为轮次边界
    - 返回最近 N 轮对应消息
- `packages/server/src/types/index.ts`
  - `historyMeta` 改成：`loadedRounds / totalRounds / hasMoreAbove / roundWindow`

### 前端
- `packages/web/src/lib/types.ts`
  - 对齐新的 `historyMeta`
- `packages/web/src/lib/api.ts`
  - `getSession()` 参数从 `limit` 改成 `rounds`
- `packages/web/src/composables/usePiChat.ts`
- `packages/web/src/composables/usePiChatCore.ts`
- `packages/web/src/composables/usePerSessionChat.ts`
  - 历史窗口常量从消息数改成轮次数
  - SSE reconnect 改传 `rounds`
  - `loadEarlier()` 改成 `+3 轮`
  - `expandVisibleHistoryMeta()` 改按 user 轮次重算
- `packages/web/src/components/workbench/chat/WorkbenchMessageStream.vue`
  - 默认只展示最近 3 轮
  - `展开更早 3 轮` 只放开当前已加载窗口
  - `加载更早历史` 才触发后端继续取更早轮次

## 验证
- `npm run check`
- `npm run build --workspace @pi/server`

均通过。

## 结果
- 历史窗口语义从“消息条数”收敛到“对话轮次”。
- 旧消息不再因为续聊而像被删掉。
- 长会话不会被“展开全部”一次性冲垮，而是按 3 轮逐段展开。
