# 75 闪念附件成功提示收敛

## 背景

网页端保存带附件闪念时，会连续执行创建闪念、上传附件、触发重新分析。此前每个步骤都会展示成功 toast，用户一次操作会看到多条绿色提示堆叠。

## 实现

- `packages/web/src/composables/useInbox.ts`
  - `uploadAttachments` 和 `retryAnalysis` 增加 `successToast` 选项。
  - 默认保持原有独立动作成功提示。
  - 复合流程可传 `successToast: false` 静默内部成功提示。
- `packages/web/src/components/workspace/InboxView.vue`
  - 带附件保存时静默上传和触发分析成功提示。
  - 外层保存流程只展示一次 `已保存闪念，N 个附件已进入分析`。
- `packages/web/src/components/workspace/__tests__/InboxView.test.ts`
  - 覆盖带附件保存只出现一次成功 toast。

## 验收

- `pnpm --dir packages/web exec vitest run src/components/workspace/__tests__/InboxView.test.ts`
- `pnpm --dir packages/web exec vitest run src/composables/__tests__/useInbox.test.ts`
- `npm run check`
