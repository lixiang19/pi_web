# 72 闪念状态事件驱动更新

## 背景

闪念页为了展示分析状态，旧实现会在存在 `unanalyzed` 或 `analyzing` 闪念时定时刷新 `GET /api/fleeting`。这会让“等待分析”或“分析中”变成前端无限请求，且旧版本还会把每次列表刷新放大成所有闪念的附件请求。

## 改动

- Web 端删除所有闪念状态轮询；`unanalyzed` 和 `analyzing` 只展示状态，不驱动网络请求。
- `GET /api/fleeting` 只在页面进入、创建闪念、用户显式刷新等明确动作触发。
- `POST /api/fleeting/:noteId/analyze` 返回的 note 直接更新本地列表，不再追加一次列表刷新。
- 新增 `GET /api/fleeting/events` SSE；worker 进入 `analyzing`、完成 `processed`、失败 `failed` 或写入重试错误时推送 `fleeting.note.updated`。
- 前端收到 `fleeting.note.updated` 后按 note id 替换本地记录；SSE 断开时关闭连接，不启动轮询兜底。
