## 生产启动检查

- `NODE_ENV=production` 时必须配置 `RIDGE_ADMIN_PASSWORD`，否则服务端拒绝启动。
- 默认服务端端口为 `3000`，端口占用时输出明确错误，可用 `PORT=3001 npm run dev` 切换。
- 数据目录统一为 `~/.pi/`，运行状态存储在 `~/.pi/ridge.db`。
- 默认工作空间目录统一为 `~/ridge-workspace`，启动时会创建并补齐预制目录。
- 数据库启动时执行版本化迁移与旧表缺列修复，迁移失败不得静默继续。

## 桌面打包检查

- Tauri 只打包 Web 与本机采集/桥接能力。
- 业务功能仍由 Node 服务层、Web 前端和桌面 runtime bridge 承担。
- 桌面设备通过 token 鉴权、心跳、WebSocket、runtime bundle 和 ack 与服务端协作。
- 本机项目离线时不得伪造可运行状态，必须明确拒绝或记录跳过原因。

## 验收清单

- 安装：使用 `pnpm install`，原生依赖由根 `package.json` 的 `pnpm.onlyBuiltDependencies` 放行。
- 首次启动：创建 `~/ridge-workspace`、`~/.pi/ridge.db`、`记忆/MEMORY.md`、`Wiki/index.md`。
- 备份：`GET /api/workspace/backup` 生成含 manifest/checksum 的 ZIP。
- 恢复：`POST /api/workspace/restore` 校验包后整包恢复，失败保留恢复前快照。
- 桌面注册：设备 token 鉴权、在线心跳、runtime bundle 物化和 ack 生效。
- 主会话：Agent、模型、思考级别、prompt、skill、command 和附件入口可用。
- 任务：任务处理会话只能由 `task-agent` 驱动，正式完成必须用户确认。
- 文件：上传、移动、删除、重命名、预览、Markdown 保存、转换重试和隐藏版本点可用。
- 搜索：文件、任务、会话、记忆、Wiki、空间和 RAG 可检索，知识诊断显示失败原因。
- 通知：文件处理、RAG、后台任务、自动化失败或跳过都能进入通知中心并关联对象。
