# V2 阶段 2 文件、附件、空间与隐藏版本

## 目标

完成 V2 阶段 2：文件资产从上传、文件页操作、编辑保存、处理状态、RAG 刷新到空间保存点和隐藏版本形成闭环。

## 实现

- `FilesView` 增加上传、新建文件夹、重命名、移动、删除入口，并保留重试转换、重新转换入口。
- `useWorkspaceFiles` 接入 `uploadFiles`、`createFileEntry`、`moveFileEntry`、`trashFileEntry`，所有操作后刷新当前目录。
- `WorkspaceMarkdownEditor` 改走 `/api/files/content`，Markdown 保存写入 `search_index_status` 的 `deferred` 编辑状态。
- `WorkspaceTextFileEditor` 支持编辑 `空间/<作品名>/index.html`，保存仍走 `/api/files/content`。
- `/api/files/content` 允许 Markdown 与空间 `index.html` 保存；Markdown 保存为 deferred RAG，空间 HTML 保存为 immediate RAG。
- 新增 `workspace-version.ts`，使用内置 isomorphic-git 在数据目录创建隐藏版本点，排除 `.ridge`。
- 上传、创建、移动、删除、Markdown 保存、空间 HTML 保存都会尝试创建隐藏版本点。
- `iso-git-service` 支持删除文件提交，并在状态结果中排除 `.ridge`。

## 验收

- 文件页可以完成上传、创建目录、重命名、移动、删除、失败重试和重新转换。
- Markdown 编辑保存后不会立即丢弃旧 RAG chunk，等待手动刷新或夜间刷新。
- 空间作品 `index.html` 可编辑保存，保存后进入 RAG pending，并形成隐藏版本点。
- 文件处理状态、通知中心、搜索页继续使用同一批后端状态与 RAG 真源。
