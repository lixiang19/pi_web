# 内部项目/外部仓库语义改造

## 背景

当前代码库中"内部项目/外部项目"语义混乱：
- `projectType: 'internal' | 'external'` 但 external 实际叫法不统一（外部项目、代码库、源码、资源）
- `source: 'github' | 'server-folder' | 'internal'` 与 `projectType` 语义重叠且命名不清
- `createWorkspaceChatProject` 错误标记为 `projectType: 'external'`（工作空间不是外部仓库）

## 目标语义

1. **内部仍叫"项目"**：工作空间内的组织/关注对象，不作为 pi 的运行目录。
2. **外部统一叫"外部仓库"**：可以是普通文件夹，也可以是 Git 仓库；不再叫外部项目、代码库、源码、资源等。
3. **pi/agent 运行模型不被改坏**：会话 cwd 仍是运行作用域；工作空间会话 cwd=工作空间目录，外部仓库会话 cwd=外部仓库路径。内部项目不作为 pi cwd。

## 改造范围

### 协议层
- `ProjectItem.source` → `ProjectItem.externalOrigin` (`'github' | 'folder' | null`)
- `ProjectItem.projectType` 扩展为 `'internal' | 'external' | 'workspace'`
- `SessionSummary` 中的 `projectId`/`projectRoot`/`projectLabel` 保持（运行上下文语义，不改坏运行模型）

### 后端
- DB `projects.source` → `external_origin` (TEXT, nullable)
- Migration v11 处理列改名
- `storage/index.ts` 中 `createProjectRecord` 默认 `externalOrigin: 'folder'`
- `workspace-chat.ts` 中 `createWorkspaceChatProject` 改为 `projectType: 'workspace'`
- API `/api/projects` 序列化字段改名

### 前端
- `session-sidebar.ts`：`origin` → `externalOrigin`
- `WorkspacePage.vue`：侧边栏标签文案调整
  - `'workspace'` → 不显示内外标签（或"工作空间"）
  - `'external'` → "外部仓库"
  - `'internal'` → "项目"
- 来源标签：GitHub / 本地文件夹（不再显示"服务器文件夹"）
