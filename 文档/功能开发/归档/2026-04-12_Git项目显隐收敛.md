# 2026-04-12 Git项目显隐收敛

## 背景
用户要求：
- 左侧会话列表里的 Git 按钮，只在项目是 Git 项目时显示
- 最右侧 Git 操作面板，只在项目是 Git 项目时显示
- 非 Git 项目右侧保留文件面板，隐藏 Git Tab

## 根因
当前实现有两个问题：
1. `SessionSidebar.vue` 里的 Git worktree 按钮、删除 worktree 按钮，只判断项目是否存在于 stored projects，不判断项目是否为 Git 仓库
2. `ProjectFilePanel.vue` 总是渲染 Git Tab，非 Git 项目只能等 `WorkbenchGitPanel` 内部请求 `getGitStatus()` 报错后显示错误态

这属于职责混乱：
- **Git 能力探测** 被错误塞进了 **Git 状态读取**
- UI 显隐逻辑依赖错误分支，而不是依赖明确能力信号

## 方案
### 1. 服务端新增 Git 仓库探测接口
新增：
- `GET /api/git/is-repo?cwd=...`

服务端直接复用已有 `gitService.isGitRepository(cwd)`，返回：
```ts
{
  isGitRepo: boolean
}
```

### 2. Web 侧新增轻量探测 composable
新增：
- `packages/web/src/composables/useIsGitRepo.ts`

职责：
- 监听 `cwd`
- 调用 `/api/git/is-repo`
- 返回 `isGitRepo`
- 用请求 token 避免目录快速切换时旧请求回写新状态

### 3. 右侧面板改成“先探测，再决定是否渲染 Git Tab”
修改：
- `packages/web/src/components/workbench/ProjectFilePanel.vue`

行为：
- Git 项目：显示 `Git + 文件` 两个 Tab，默认切到 Git
- 非 Git 项目：只显示 `文件` Tab，不渲染 Git Tab

### 4. 左侧 Sidebar 改成显式依赖真实 `project.isGit`
修改：
- `packages/server/src/index.ts`
- `packages/server/src/types/index.ts`
- `packages/web/src/lib/types.ts`
- `packages/web/src/lib/session-sidebar.ts`
- `packages/web/src/components/chat/SessionSidebar.vue`
行为：
- 服务端 `SessionSummary` 新增 `isGit`
- `buildSessionProjects()` 不再把“有 session”硬编码成 `isGit: true`
- `新建 worktree` 按钮：`findStoredProjectId(project.projectRoot) && project.isGit`
- `删除 worktree` 按钮：同样加 `project.isGit`

## 结论
本次改动把 Git UI 显隐从“错误驱动”改成“能力驱动”：
- 非 Git 项目不再出现 Git 操作入口
- Git 状态读取和 Git 仓库探测彻底拆开
- 左右两侧显隐规则一致
