# 06 会话索引归档与只读状态

## 目标

实现 ridge 会话索引，支持会话打开、归档和只读状态。

## 范围

- `session_index` CRUD。
- 工作空间/项目上下文。
- 任务关联。
- 设备关联。
- 运行位置。
- 归档状态。
- 只读状态。
- 最近更新时间。

## 不做

- 不复制 Pi 原始消息正文。
- 不做桌面离线会话正文读取。

## 验收

- 消息正文仍以 Pi 原始会话为真源。
- 归档会话只读。
- 桌面设备离线时不打开其项目历史。
- 同一个会话只能存在于一个标签。

## 关联设计

- `文档/项目设计/数据库与持久化.md`
- `文档/项目设计/左侧导航与项目会话列表.md`

## Spec 提取点

- `session_index` 字段。
- 归档 API。
- 只读状态判定。

## Spec 草案

### 数据

- `session_id`
- `title`
- `context_type`
- `project_id`
- `task_id`
- `device_id`
- `run_location`
- `archived`
- `readonly`
- `created_at`
- `updated_at`

### 行为

- 会话正文不存入 `ridge.db`。
- 归档会话设为只读。
- 设备离线时对应桌面项目会话不可打开。

### 测试

- 归档 API 设置 `archived=true` 和 `readonly=true`。
- 查询普通列表排除归档会话。
- 离线桌面项目历史打开失败。
- 数据库无消息正文列。
