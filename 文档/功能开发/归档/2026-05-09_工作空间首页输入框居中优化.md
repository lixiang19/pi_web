# 工作空间首页输入框居中优化

## 背景

用户希望首页上半屏留白，把“问我任何事”输入框放到视觉中心，其它信息区整体下移；同时模型、Agent、思考级别默认可见，并呈现得更像 AI 对话输入框的工具栏。

## 范围

- 调整 `HomePage.vue` 首页布局和输入框视觉。
- 保留现有 `submit/open-file/open-session/open-tasks` 事件语义。
- 保留模型、Agent、思考级别的真实 `Select` 控件与提交字段；无 Agent 继续使用既有 `NO_AGENT_VALUE`，由 `WorkspacePage` 转成 `null`。

## 输入

- 用户在首页输入框输入首条消息。
- 当前默认模型、默认 Agent、默认思考级别。
- 最近事情、最近文件、AI 建议占位数据。

## 输出

- 输入框在首页上半屏居中，上方有明显留白。
- 下方信息区在输入框之后展示。
- 模型、Agent、思考级别默认显示在输入框底部工具栏。

## 边界

- 不新增会话能力或后端接口。
- 不改变提交字段和无 Agent 语义。
- 不新增基础 UI 组件。

## 实现

- `HomePage.vue` 增加独立 `home-ai-hero` 区域，用上半屏高度把输入框推到视觉中心。
- 输入框改为更接近 AI 对话 composer 的大圆角卡片、默认多行输入和圆形发送按钮。
- 模型、Agent、思考级别改为默认可见的底部工具栏 Select chip。
- 默认 Agent 为空时使用现有 `NO_AGENT_VALUE` 选中“无 Agent（直接对话）”，避免 Select 只能显示占位。
- `SelectContent` 的 viewport 不再固定为 trigger 高度，避免下拉菜单被压成一条线；首页 SelectContent 额外限制 `max-h-72`。
- 第二轮视觉调整把 composer 从大圆角重阴影胶囊改为更窄、更轻的中性卡片，工具栏 chip 去掉重复标签。
- 下方最近事情、最近文件、AI 建议整体放在 `home-info-grid` 中，位于输入框之后。

## 验证

- `npm run test --workspace @pi/web -- HomePage.test.ts` 通过。
- `npm run check` 通过；保留既有 `useFileTreeActions` 相关 `any` warning，无新增 error。
