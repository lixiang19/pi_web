# 2026-04-13 旧会话续聊后历史消息隐藏修复

## 背景
- 用户反馈：打开已存在对话继续输入时，旧消息在 UI 中彻底消失。
- 真实 session jsonl 仍完整存在，说明不是持久化丢失。
- 典型样例：`/Users/lixiang/.pi/agent/sessions/--Users-lixiang-Documents-myCode-AuroraPlatformWeb--/2026-04-13T18-01-56-476Z_efbcbff9-78f8-45a8-bf5e-771952047570.jsonl`

## 根因
- 根因在前端渲染层，不在存储层。
- `packages/web/src/components/workbench/chat/WorkbenchMessageStream.vue` 原逻辑默认从“最后一条 user 消息”开始切片渲染。
- 会话一旦继续发送新消息，最后一条 user 向后移动，之前所有轮次就会被前端自动隐藏。
- 这属于隐式折叠，不是删除；因此 jsonl 仍完整。

## 用户确认后的目标行为
- 保留折叠能力。
- 旧轮次默认折叠。
- 消息区顶部提供显式开关：`展开全部 / 折叠旧轮次`。
- 不能再在发送新消息后偷偷让旧消息消失。

## 实施
### 代码
- 修改 `packages/web/src/components/workbench/chat/WorkbenchMessageStream.vue`
  - 新增 `isHistoryExpanded`
  - 新增 `firstUserIndex / lastUserIndex / hiddenRoundCount / hasCollapsibleHistory`
  - 折叠改为显式 `toggleHistoryExpansion()`
  - 顶部显示 `展开全部（含 N 段更早轮次）`
  - 仅在展开态下显示 `加载更早消息`
  - 会话切换时重置为默认折叠

### 文档
- 更新 `文档/模块梳理/中间对话区模块.md`
  - 补充“默认折叠旧轮次 + 顶部显式展开/折叠开关”职责
  - 修正历史消息加载流程说明

## 验证
- `npm run check`
  - 通过

## 结果
- 旧消息不再因为继续输入而“看起来被删掉”。
- 折叠成为显式交互，用户可理解、可控制。
- 数据层、sessionCache、jsonl 持久化语义保持不变。
