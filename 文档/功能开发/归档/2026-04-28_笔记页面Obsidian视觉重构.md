# 笔记页面 Obsidian 视觉重构

## 背景

用户反馈当前笔记页面简陋、视觉粗糙，也不像 Obsidian。既有首版能力已经具备 Markdown 列表、Milkdown 编辑、新建与保存，本轮只重构视觉和布局，不扩展 notes API 与知识图谱能力。

## 范围

- 重构 `NotesPage.vue` 为状态容器。
- 新增笔记 vault 侧栏组件，负责搜索、新建、列表、空态与选中态。
- 新增编辑器工作区组件，负责当前文件标题、路径、保存状态、加载态、空态、底部状态栏。
- 调整 Milkdown/Crepe 局部主题，让编辑区居中、宽度收敛、行距更适合长文编辑。

## 非目标

- 不做双链、标签、反向链接、关系图。
- 不做删除、重命名、自动保存。
- 不新增服务端接口，不改变 Markdown 文件来源。

## 完成记录

- `NotesPage.vue` 已收缩为数据流容器。
- `NoteVaultSidebar.vue` 已提供 Obsidian 式 vault 文件区。
- `NoteEditorShell.vue` 已提供编辑标题栏、保存状态、空态、加载态和底部状态栏。
- `NoteMilkdownEditorInner.vue` 已改为居中阅读宽度和主题变量驱动的编辑体验。
- 新增 `NotesPage.test.ts` 覆盖加载首篇笔记、搜索、新建和保存编辑内容。
