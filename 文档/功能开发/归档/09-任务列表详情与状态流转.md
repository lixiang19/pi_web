# 09 任务列表详情与状态流转

## 目标

实现任务页、任务详情和任务状态流转。

## 范围

- 任务列表。
- 里程碑列表。
- 项目筛选。
- 创建、更新、删除任务。
- 创建、更新、删除里程碑。
- 阻塞原因。
- 完成标准。
- 截止日期。
- 排序。

## 不做

- 不做会话内任务面板。
- 不自动完成任务。

## 验收

- 用户可以把任务和里程碑改为 completed。
- Agent/API 普通规划路径不能改为 completed。
- 项目筛选包含全部、无项目、具体项目。
- 新任务默认继承里程碑项目，用户可覆盖。

## 关联设计

- `文档/项目设计/任务里程碑与处理会话.md`

## Spec 提取点

- 任务 API。
- 里程碑 API。
- 状态流转权限。
- 筛选和排序。

## Spec 草案

### API

- `GET /api/workspace/tasks`
- `POST /api/workspace/tasks`
- `PATCH /api/workspace/tasks/:id`
- `DELETE /api/workspace/tasks/:id`
- `GET /api/workspace/milestones`
- `POST /api/workspace/milestones`
- `PATCH /api/workspace/milestones/:id`
- `DELETE /api/workspace/milestones/:id`

### 行为

- 用户 API 可设置 completed。
- Agent/规划工具不能设置 completed。
- 项目筛选支持全部、无项目、具体项目。
- 新任务默认继承里程碑项目。

### 测试

- 用户完成任务成功。
- Agent 完成任务失败。
- 无项目筛选只返回 `project_id is null`。
- 更新里程碑项目后新任务继承。

## 实现记录

### 2026-05-11

#### 服务端

- `task-system.ts` 已有状态流转校验 `assertStatusAllowed`：pending→in_progress→reviewing→completed，Agent 不能完成，系统里程碑不能完成。
- `workspace-tasks.ts` 路由已透传 `projectId=none` 和具体项目筛选。
- `createTask` 未显式传 `projectId` 时自动继承所选里程碑的 `projectId`。

#### 前端

- `TaskView.vue` 四个视图（看板、列表、日历、里程碑）均可打开详情。
- 详情支持编辑标题、完成标准、优先级、状态、里程碑、项目、截止日期、阻塞原因。
- 看板拖拽只允许合法状态流转；非法拖拽不调用更新。
- 项目筛选 UI 包含全部/无项目/具体项目，联动 `projectFilter` 触发重新加载。
- 新任务默认 `projectId: undefined` 让后端继承里程碑；用户可选 `null` 无项目或具体项目覆盖。
- 修复关键 Bug：
  - `saveSelectedTask` 成功后更新 `selectedTask` 为后端返回结果，避免用户无法连续推进到 completed。
  - `saveSelectedMilestone` 成功后更新 `selectedMilestone`。
  - 创建/更新失败时不再无条件清空表单。

#### `useWorkspaceTasks.ts` 修复

- `addTask/addMilestone/updateTask/updateMilestone/removeTask/removeMilestone` 全部改为返回 `{ success, task?/milestone?/error }`，调用方可判断成功/失败。
- 失败时本地状态回滚，toast 显示错误描述。

## 风险与后续注意

- 看板拖拽排序字段 `sortOrder` 使用 `Date.now()`，在极端并发下可能冲突；后续如需严格排序可改为显式拖拽排序 API。
- 里程碑删除时前端先乐观移除再调用后端，后端 409 时前端会回滚，但 toast 提示已显示；体验可接受。
- 当前未做可视化验收（启动长期服务+Playwright），因 worktree 环境无长期运行条件，已通过单元/集成测试覆盖核心交互。
