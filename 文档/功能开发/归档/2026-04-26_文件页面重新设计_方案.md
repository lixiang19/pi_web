# 文件页面重新设计方案

- 状态：已完成
- 日期：2026-04-26
- Owner：ridge

## 背景

一级 `文件` 页面目前只有工作区文件树，点击文件没有独立预览区；而 `/chat` 会话内已经存在可复用的文件预览标签、Markdown 自动保存、图片/HTML/代码/大文件预览能力。重复实现会造成接口和状态分裂。

## 目标

- 将 `/files` 改成独立文件工作区：左侧工作区文件树与收藏，右侧文件预览标签。
- 文件页固定使用 `SystemInfo.workspaceDir` 作为根目录，不跟随当前会话 cwd。
- 点击文件树或收藏中的文件后，在右侧打开或复用预览标签。
- 复用现有 `/api/files/content`、`/api/files/blob`、`/api/files/content-window` 与 Markdown 保存接口。
- 文件页不显示“发给 AI / 加入输入框”动作，避免非会话页面出现会话输入框语义。

## 非目标

- 不新增文件写入类型，仍只允许 Markdown 编辑。
- 不改变聊天内文件预览的会话级状态生命周期。
- 不新增后端文件接口。

## 验收

- `/files` 页面展示工作区根路径、左侧文件树、右侧预览区。
- 点击文件后右侧打开标签；重复点击同一文件复用标签。
- Markdown 文件仍支持源码编辑与自动保存。
- 图片、代码、文本、HTML、大文件预览继续走已有能力。
- 文件页 Markdown 预览不出现发给 AI 的按钮。
- 通过 eslint 与 TypeScript 检查。

## 完成记录

- `FilesPage.vue` 改为左侧文件树/收藏、右侧预览工作区的双栏结构。
- 文件选择通过 `WorkspaceFileTree.select-file` 进入 `WorkbenchOperationPanel.openFile()`，复用同一套标签去重、读取、Markdown 保存和大文件加载状态。
- `WorkbenchOperationPanel` 增加空态文案和 Markdown AI 动作开关；聊天页保持默认开启，文件页显式关闭。
- `WorkbenchMarkdownPreview` 与 `MarkdownPreviewBlock` 支持隐藏块级 AI 动作。
- 新增 `FilesPage.test.ts` 覆盖文件选择和文件页关闭 AI 动作。
