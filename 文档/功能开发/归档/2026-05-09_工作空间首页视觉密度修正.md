# 工作空间首页视觉密度修正

## 背景

用户反馈当前工作空间首页视觉高度不满意。截图显示首页在大屏下存在大面积空白、内容过度居中、卡片层级不清、信息密度低和左侧按钮拥挤问题。

## 范围

- 调整 `HomePage.vue` 的首页视觉布局、容器宽度、间距、卡片边界和列表点击区域。
- 微调 `WorkspacePage.vue` 左侧入口与新建按钮的操作密度。
- 不改变 props、emit、接口、数据流和业务逻辑。

## 输入

- `recentActivity`、`recentFiles`、模型与 Agent 列表等现有首页数据。
- 用户在首页输入框提交首条消息的现有交互。

## 输出

- 首页内容在桌面视口下更靠上，横向空间利用更充分。
- 最近事情、最近文件、AI 建议三个区块具备清晰视觉层级。
- 左侧固定入口和新建按钮拥有更稳定的点击体感。

## 边界

- 不新增功能入口。
- 不改变首页提交、打开文件、打开会话、打开待办的事件语义。
- 不引入新基础组件。

## 验证

- 修改后运行 `npm run check`。
- 视觉改动无新增业务逻辑测试；依靠类型检查和 ESLint 确保模板与导入一致。
