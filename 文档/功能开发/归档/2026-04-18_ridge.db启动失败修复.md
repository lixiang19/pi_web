# 2026-04-18 ridge.db 启动失败修复

- 日期：2026-04-18
- 状态：已完成
- 目标：修复 server 启动时 `refreshSessionCatalog` 导致的 ridge.db 初始化失败。

## 1. 现象

开发环境启动时，server 在初始化 ridge.db 后立刻退出，错误为：

`SqliteError: no such column: "" - should this be a string literal in single-quotes?`

调用链：

- `packages/server/src/index.ts`
- `refreshSessionCatalog`
- `refreshCatalogEntries`
- `packages/server/src/session-indexer.ts`

## 2. 根因

`refreshCatalogEntries` 在清理无效 `session_contexts` 时，SQL 使用了：

`context_id != ""`

在 SQLite 里，双引号会把内容解释为标识符而不是字符串字面量，因此 `""` 被当成列名处理，最终触发：

- `no such column: ""`

这不是数据库数据异常，也不是迁移问题，而是 SQL 字面量写法错误。

## 3. 修复

把错误 SQL 改为单引号字符串字面量：

- `context_id != ''`

修复后：

- `refreshSessionCatalog` 可以正常完成
- `session_contexts` 清理逻辑保持原设计，不需要额外兼容分支
- ridge.db 初始化链路恢复正常
