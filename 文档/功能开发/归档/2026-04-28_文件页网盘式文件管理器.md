# 文件页网盘式文件管理器

## 目标

将一级 `/files` 从“文件树 + 预览面板”改为当前工作区内的网盘式文件管理器，并保留现有文件预览、Markdown 自动保存、图片/HTML/代码/大文件读取能力。

## 实现

- 服务端新增 `file-manager.ts`，集中处理 root 校验、realpath 校验、目录元数据、创建、移动、上传与移入系统回收区。
- `/api/files/tree` 返回扩展后的 `FileTreeEntry`，包含 `size / modifiedAt / extension`，用于网格卡片展示。
- 新增 `/api/files/entries`、`/api/files/entries/path`、`/api/files/upload` 和 `DELETE /api/files/entries`，所有写操作都复用同一套工作区 root 安全边界。
- 前端新增 `useFileManager(rootDir)`，管理当前目录、搜索、排序、刷新、创建、重命名、拖拽移动、上传和回收区删除。
- `/files` 页面拆为工具栏、面包屑、网格、输入弹窗、回收区确认弹窗；右侧继续复用 `WorkbenchOperationPanel` 做预览。

## 经验

- 文件写操作必须和预览读取共享 root/realpath 校验，不能让新建、移动、上传单独绕出一套路径判断。
- 删除使用系统回收区时，仍然必须先完成工作区边界校验，再调用 trash。
