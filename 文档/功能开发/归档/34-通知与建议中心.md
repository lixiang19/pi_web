# 34 通知与建议中心

## 目标

实现左侧固定入口的通知与建议中心。

## 范围

- `notification_events`。
- 左侧通知入口。
- 未处理数量。
- 通知列表。
- 建议、待确认、失败、警告、信息类型。
- 查看、忽略、重试、接受建议、拒绝建议、打开对象、标记已处理动作。

## 不做

- 不做系统级推送。
- 不做邮件通知。
- 不做聊天式通知助手。
- 不做通知规则编辑器。

## 实现

### 数据

`notification_events` 扩展为通知中心真实事件表：

- `event_id`
- `event_type`
- `source`
- `severity`
- `status`
- `title`
- `body`
- `related_type`
- `related_id`
- `actions_json`
- `payload_json`
- `created_at`
- `updated_at`
- `read_at`
- `handled_at`

合并后迁移版本：`RIDGE_DB_SCHEMA_VERSION = 22`。

### API

- `GET /api/notifications?filter=unhandled|all|failed|suggestions|handled`
  - 默认显示未处理事件。
  - 信息类通知默认不进入未处理列表，只在全部中可查。
  - 返回通知列表和各筛选计数。
- `POST /api/notifications/:eventId/actions`
  - `dismiss`
  - `mark_handled`
  - `retry`
  - `accept_suggestion`
  - `reject_suggestion`
  - `open_related`

接受建议时才写入正式对象，当前支持 `task.update`、`task.create`、`milestone.update`、`milestone.create`。
动作接口会先校验动作是否存在于当前通知可用动作中；非法建议 payload 返回错误且不标记通知已处理。
接受建议的正式对象写入与通知标记已处理在同一数据库事务中提交。
服务端拥有安全生命周期动作，非终态通知默认补充「标记已处理」与「忽略」；领域动作由事件来源或类型推导。

### 事件来源

- 文件转换/索引失败写入 `file_processing.*_failed` 通知，关联文件，提供重试和打开对象动作。
- RAG 索引失败写入 `rag.index_failed` 通知，关联文件，提供重试和打开对象动作。
- 后台任务最终失败写入 `background_job.failed` 通知，关联后台任务，提供重试动作；后台任务当前没有独立详情页，不返回无效打开动作。
- 关联对象回退推导只按 `event_type` 白名单读取 payload，普通信息 payload 中的同名字段不会生成打开对象动作。
- 后台成功且无用户动作不产生通知。
- 重试动作必须确认对应失败状态记录存在；需要队列的重试在队列不可用时返回错误且不标记通知已处理。
- 文件/RAG 重试把失败状态恢复与重新入队放在同一事务中，避免状态恢复成功但队列未提交。

### 前端

- 左侧固定入口包含「通知」，显示未处理数量。
- `NotificationCenterView.vue` 提供筛选、列表、状态/类型展示、动作按钮。
- 动作按钮按通知与动作维度禁用 pending 状态，避免快速连点重复提交。
- 关联对象导航：
  - 文件：打开文件标签。
  - 任务/里程碑：打开任务页。
  - 会话：打开会话标签。
  - 项目：打开项目主页。
  - 自动化：打开自动化页。

## 验收

- 后台成功且无用户动作不产生通知。
- 文件转换失败、RAG 失败、后台 Agent/后台任务失败进入通知。
- 接受建议后才写入正式对象。
- 通知事件不进入 RAG。
- 左侧入口不再是占位页。
