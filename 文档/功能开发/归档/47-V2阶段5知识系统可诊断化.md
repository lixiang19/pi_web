# V2 阶段 5 知识系统可诊断化（已归档）

## 目标

完成 V2 阶段 5：RAG、记忆、Wiki、图谱、workspace MCP、后台任务和通知不只在后台运行，用户能从 UI 判断知识系统是否健康，并能对失败索引发起刷新。

## 已完成

- 新增 `GET /api/workspace/knowledge/diagnostics`，聚合知识系统诊断状态。
- RAG 诊断返回 `indexed/pending/indexFailed`、最近索引时间、失败目标、错误原因和关联通知。
- 记忆诊断返回 `记忆/MEMORY.md` 是否存在、是否会注入、daily 数量和最近 daily 时间。
- Wiki 诊断返回 `Wiki/index.md` 是否存在、是否会注入以及对应 RAG 索引状态。
- 图谱诊断返回 `.ridge/graph.kuzu` schema/database 是否存在、最近更新时间和纠错 API 入口。
- workspace MCP 诊断返回只读边界和真实可用工具：`rag_search`、`graph_search`、`file_search`、`read_workspace_file`。
- 后台任务诊断返回按状态/类型聚合的任务数，以及最近失败任务、错误和关联通知。
- 搜索页无查询时展示知识诊断面板；失败 RAG 目标可直接点击刷新并同步诊断状态。
