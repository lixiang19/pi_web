# 终端视口满高修复

- 状态：已完成
- 日期：2026-04-27
- Owner：ridge 前端

## 背景

用户反馈 `/terminal` 页面的终端黑色区域高度不对，底部出现大块页面背景色；终端应该占满剩余工作区高度。

## 根因

`TerminalViewport.vue` 把 `hostRef` 放在 `absolute inset-0` 的子 `div` 上，再把同一个元素传给 `new WTerm(host)`。`@wterm/dom` 初始化时会给该元素添加 `.wterm`，而库样式 `.wterm { position: relative; ... }` 会覆盖绝对定位布局，导致终端 host 回到内容高度，外层剩余区域露出页面背景。

同时库默认 `.wterm` 有 `border-radius` 和 `box-shadow`，会让终端表现为悬浮卡片，不符合当前 IDE 风格页面。

## 方案

- 让 `TerminalViewport` 的根 `section` 本身成为 `WTerm` host，直接参与父级满高布局。
- 覆盖 `.ridge-terminal-host.wterm` 的圆角和阴影，使终端成为铺满工作区的黑色画布。
- 不改 PTY、WebSocket 和输入协议。

## 验收标准

- [x] `/terminal` 下方不再露出白色或页面背景色。
- [x] 终端黑色区域铺满标签栏以下的剩余高度。
- [x] 终端输入仍正常。
- [x] `pnpm lint` 通过。
- [x] `pnpm typecheck` 通过。

## 执行结果

- 修改 `TerminalViewport.vue`，让根 `section` 直接作为 `WTerm` host，不再依赖会被 `.wterm` 覆盖的 `absolute inset-0` 子节点。
- 覆盖 `.ridge-terminal-host.wterm` 的 `border-radius` 和 `box-shadow`，移除悬浮卡片视觉。
- 终端背景改为基于主题明暗模式的专用变量：浅色模式用 `--foreground` 做深色画布，暗色模式用 `--background` 做深色画布。
- 浏览器验证输入 `echo RIDGE_HEIGHT_OK` 正常显示命令和输出。
