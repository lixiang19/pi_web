# 36 自动化规则运行与跳过

## 目标

实现自动化规则、运行记录和离线跳过。

## 范围

- 自动化列表。
- 自动化创建和编辑。
- 定时触发普通 AI 会话。
- 绑定工作空间或项目。
- 自动化运行记录。
- 项目设备离线时跳过。
- 失败或跳过进入通知中心。

## 不做

- 不做 DAG 工作流。
- 不做复杂编排。
- 不做多用户通知。

## 验收

- 自动化本质是定时创建 AI 会话。
- 项目自动化在项目设备运行。
- 离线设备跳过并记录原因。
- 自动化失败可打开详情或重试。

## 完成记录

- `automation_rules` 增加 `scope`、`project_id`，支持 workspace/project 两类运行上下文。
- 新增 `automation_runs`，记录 success/failed/skipped、reason、session_id 和触发时间。
- 调度器和立即运行 API 均写运行记录；失败或跳过写通知中心事件。
- workspace 自动化在服务端创建普通会话并发送 prompt。
- server 本地项目自动化在服务端运行。
- 桌面项目自动化在线时通过 desktop bridge 发送 `create_session` + `send_message`，并写 server 侧 desktop `session_index`。
- 桌面项目离线或项目归档时不转移设备，写 skipped run 和通知。
- 通知中心支持 `automation.failed` / `automation.skipped` 重试。
- `automation.skipped` 通知即使没有生产方动作，也会由通知中心默认派生重试动作。
- 自动化页展示运行上下文、项目绑定和运行历史，失败/跳过历史可直接重试。
- `automation_runs` 启动时具备缺表/缺列自修复路径，覆盖迁移元信息与实际表结构不一致的开发库状态。
- 桌面自动化和普通桌面项目会话在 `session_index.workspace_path` 写入项目实际运行路径，便于会话列表与排障定位。
- 相关模块梳理见 `文档/模块梳理/自动化规则运行与跳过.md`。

## Spec 提取点

- automations 字段。
- automation_runs 字段。
- 调度和跳过规则。

## Spec 草案

### 数据

- automations: id、name、scope、project_id、schedule、prompt、status。
- automation_runs: id、automation_id、status、reason、session_id、created_at。

### 行为

- 定时创建普通 AI 会话。
- workspace 自动化在服务器运行。
- 项目自动化在项目设备运行。
- 设备离线时跳过并记录原因。

### 测试

- 到点创建会话。
- 离线项目自动化写 skipped run。
- 失败 run 生成通知。
- 自动化不是 DAG 工作流。
