# 31 设备注册在线状态与调度（已归档）

## 状态

已完成并纳入 V2 阶段 3 桌面端与本机项目上线闭环。

联合实现与测试证据见：

- `文档/功能开发/归档/45-V2阶段3桌面端与本机项目上线闭环.md`
- `文档/功能开发/归档/30-31-32-联合开发计划.md`

## 目标

实现桌面设备注册、在线状态和 Web 调度在线设备项目。

## 范围

- 服务器设备内置。
- 桌面设备注册。
- 持久设备 ID。
- 长连接和心跳。
- 在线/离线状态。
- Web 请求转发到桌面。
- 桌面 Pi SSE 转发回服务器。

## 不做

- 第一版不做独立设备管理页。
- 设备不允许删除。
- 桌面离线不能运行本机项目 Agent。

## 验收

- 离线项目进入离线区。
- 离线项目不展开会话列表。
- Web 可操作在线桌面项目。
- 服务器不镜像桌面项目消息正文。

## 关联设计

- `文档/项目设计/设备模型与在线状态.md`
- `文档/项目设计/桌面端工作台与本机AI.md`

## Spec 提取点

- 设备注册协议。
- 长连接消息格式。
- SSE 转发协议。

## Spec 草案

### 数据

- device_id
- name
- type
- token
- status
- last_seen_at

### 行为

- 桌面首次登录注册设备。
- 长连接维持在线状态。
- Web 请求桌面项目时由服务器转发。
- 桌面消费本机 Pi SSE 并回传服务器。

### 测试

- 心跳超时设备变离线。
- 离线设备项目不能新建会话。
- 在线桌面项目可启动会话。
- 服务器不保存桌面 Pi 正文。
