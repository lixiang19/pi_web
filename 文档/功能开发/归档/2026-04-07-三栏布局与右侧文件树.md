# 三栏布局与右侧文件树

## 任务目标

- 删除右侧辅助区，不再展示状态卡片、上下文卡片等信息。
- 将首页布局收敛为左侧对话列表、中间会话消息流、右侧文件树。
- 右侧文件树必须使用真实数据，不允许继续使用占位结构。

## 本次实现

### 前端布局

- 首页继续保留工作台形态，但右侧只保留文件树组件。
- 左侧会话列表保持按工作目录聚合的分组方式。
- 中间区域继续负责消息流、模型切换和输入框。
- 右侧新增独立文件树组件，当前会话切换时自动切换目录根。

### 数据链路

- 服务端新增 `GET /api/files/tree` 接口。
- 接口接收 `root` 与 `path`，只允许访问工作区根目录下的路径。
- 前端文件树按目录懒加载，展开文件夹时才拉取子目录内容。

### 约束处理

- 服务端对 `.git`、`node_modules`、`dist`、`target` 等生成或无效目录做过滤。
- 文件树按“目录优先、名称排序”输出，保证右侧结构稳定。

## 涉及文件

- `packages/server/src/index.js`
- `packages/web/src/lib/api.ts`
- `packages/web/src/lib/types.ts`
- `packages/web/src/components/WorkspaceFileTree.vue`
- `packages/web/src/App.vue`

## 验证要求

- 前端 `check` 通过。
- 前端 `build` 通过。
- 服务端语法检查通过。

## 后续建议

- 增加文件树当前选中状态与文件预览联动。
- 给文件树补目录折叠状态持久化。
- 后续如果会话可切到非工作区路径，需要同步调整服务端文件树根目录策略。

# Web工作台与Pi服务桥接 Module Codemap

## Responsibility

- 负责提供 Web 端 Pi 工作台，统一组织左侧会话列表、中间消息流、右侧文件树。
- 负责把 `@mariozechner/pi-coding-agent` 的 SDK 会话能力桥接成当前前端可消费的 HTTP 与 SSE 接口。
- 负责将会话摘要、消息流、目录树这些运行时数据投影成前端状态，而不是让组件直接操作 SDK 对象。
- 负责限制文件树访问边界，只暴露工作区内允许读取的目录内容。
- 不负责桌面壳原生交互、持久化会话历史、复杂工具执行面板，这些仍在后续迭代范围内。
- 服务对象包括 Web 最终用户、Tauri 桌面壳中的前端运行时，以及本仓库的开发构建流程。

## Design

### Architecture Pattern

- 模式命名：前后端分层 + 事件流桥接 + 懒加载目录树

```text
+---------------------------+
| Vue App                   |
| App.vue                   |
| - 会话列表                |
| - 消息流                  |
| - 文件树                  |
+-------------+-------------+
              |
              v
+---------------------------+
| 前端状态与 API 包装层     |
| usePiChat.ts / api.ts     |
| - 会话摘要维护            |
| - SSE 订阅                |
| - 目录树请求              |
+-------------+-------------+
              |
              v
+---------------------------+
| Express 服务桥接层        |
| packages/server/src/index.js |
| - Pi SDK 会话桥接         |
| - 文件树目录接口          |
| - 输入校验                |
+-------------+-------------+
              |
              v
+---------------------------+
| Pi SDK + 本地文件系统     |
| createAgentSession        |
| fs.readdir / fs.stat      |
+---------------------------+
```

- Vue 层只消费投影后的会话与目录数据，不直接知道 SDK 内部实现。
- API 包装层负责把 REST/SSE 转成前端可复用的类型和状态更新函数。
- 服务层同时承担两类出口：Pi 会话桥接与工作区文件树读取。

### Key Abstractions

- 抽象名称：会话摘要与快照

```ts
export interface SessionSummary {
  id: string
  title: string
  cwd: string
  status: 'idle' | 'streaming' | 'error'
  createdAt: number
  updatedAt: number
}

export interface SessionSnapshot extends SessionSummary {
  messages: ChatMessage[]
}
```

- 这是前端会话列表和会话详情的统一投影模型。
- `usePiChat` 创建、更新、切换这些数据；`App.vue` 消费这些数据做左栏分组和中间消息流渲染。
- 依据：`packages/web/src/lib/types.ts`，`packages/web/src/composables/usePiChat.ts`

- 抽象名称：文件树响应模型

```ts
export interface FileTreeEntry {
  name: string
  path: string
  kind: 'file' | 'directory'
  relativePath: string
}

export interface FileTreeResponse {
  root: string
  directory: string
  entries: FileTreeEntry[]
}
```

- 这是右侧文件树唯一依赖的数据契约，服务端按目录返回一层节点，前端按需递归展开。
- `WorkspaceFileTree.vue` 消费该抽象；`index.js` 生产该抽象。
- 依据：`packages/web/src/lib/types.ts`，`packages/web/src/components/WorkspaceFileTree.vue`，`packages/server/src/index.js`

- 抽象名称：服务端输入校验 Schema

```ts
const createSessionSchema = z.object({
  cwd: z.string().optional(),
  title: z.string().optional(),
  model: z.string().optional(),
});

const fileTreeQuerySchema = z.object({
  root: z.string().optional(),
  path: z.string().optional(),
});
```

- 这些 Schema 把前端传入的建会话参数和文件树查询参数约束在稳定的接口边界内。
- 由 Express 路由解析时创建和消费，避免组件对后端容错逻辑形成依赖。
- 依据：`packages/server/src/index.js`

- 抽象名称：会话编排器

```ts
const {
  activeSession,
  createSession,
  loadSession,
  messages,
  sessions,
  submit,
  abort,
} = usePiChat()
```

- `usePiChat` 是当前 Web 工作台的主编排器，统一负责初始化、会话切换、SSE 流更新和显式新建会话。
- 页面组件只调用其暴露的动作，不再分散地直接请求 API。
- 依据：`packages/web/src/composables/usePiChat.ts`，`packages/web/src/App.vue`

### Design Patterns

- 模式名：会话摘要投影
  - 实现位置：`syncSessions`、`patchSessionSummary`、`upsertSessionSummary`，位于 `packages/web/src/composables/usePiChat.ts`
  - 原因：让左侧列表只依赖稳定摘要，而不是每次用完整消息列表做排序和渲染。

- 模式名：SSE 驱动的增量消息流
  - 实现位置：`connectStream()`，位于 `packages/web/src/composables/usePiChat.ts`
  - 原因：服务端按事件推送 assistant 增量文本，前端把 `message_start`、`message_update`、`message_end` 投影为单条消息状态变化。

- 模式名：目录懒加载
  - 实现位置：`loadDirectory()` 与 `toggleDirectory()`，位于 `packages/web/src/components/WorkspaceFileTree.vue`
  - 原因：只在用户展开目录时才请求子节点，避免一次性把整个工作区树打平加载。

- 模式名：工作区边界约束
  - 实现位置：`ensureWithinRoot()`，位于 `packages/server/src/index.js`
  - 原因：文件树接口只允许访问工作区根目录内部路径，从后端根源约束访问边界。

## Flow

- 场景：会话启动与消息发送流程

```text
[1] App.vue submit()
  -> [2] usePiChat.ensureSession()
  -> [3] POST /api/sessions
  -> [4] createAgentSession()
  -> [5] POST /api/sessions/:id/messages
  -> [6] session.prompt()
  -> [7] SSE /stream 回推消息增量
```

- 第 1 步：页面从输入框提交 prompt。
- 第 2 步：如果没有当前会话，先在前端显式创建新会话。
- 第 3 到 4 步：服务端创建 Pi SDK 会话并返回会话快照。
- 第 5 到 7 步：服务端发起 prompt，前端通过 SSE 持续接收 assistant 文本增量并更新消息流。

- 场景：左侧对话列表切换流程

```text
[1] 左栏点击某个 session
  -> [2] usePiChat.loadSession(sessionId)
  -> [3] GET /api/sessions/:id
  -> [4] 更新 activeSessionId/messages
  -> [5] 重新建立该会话的 SSE 订阅
```

- 输入是会话 ID，输出是新的活动会话和消息流。
- 关键分支：如果点击的是当前会话，则直接返回，不重复请求。

- 场景：右侧文件树加载流程

```text
[1] App.vue 计算 fileTreeRoot
  -> [2] WorkspaceFileTree watch(rootDir)
  -> [3] GET /api/files/tree?root=...&path=...
  -> [4] Express 校验 root/path 边界
  -> [5] fs.readdir() 返回节点
  -> [6] 前端渲染可展开目录树
```

- 第 1 步根目录来自当前会话的 `cwd`，没有会话时退回工作区根目录。
- 第 4 步若路径越界或不是目录，服务端直接拒绝。
- 第 6 步只渲染当前层；用户展开目录时会再次触发同一流程。
