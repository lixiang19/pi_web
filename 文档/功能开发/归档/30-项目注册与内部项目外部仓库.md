## 目标

实现项目（内部组织对象）和外部仓库（外部运行目录）的注册、归档和删除规则，语义彻底分离。

## 范围

- **内部项目**：工作空间内的组织/关注对象，创建在 `~/ridge-workspace/项目/<name>`，**不作为 pi 运行目录**。
- **外部仓库**：外部文件夹或 Git 仓库，可作为 pi 运行目录；普通文件夹或 Git 均可。
- GitHub URL 拉取外部仓库。
- 桌面本机文件夹通过桌面端路径确认注册。
- 绑定设备（外部仓库）。
- 同设备同路径唯一（外部仓库）。
- 归档项目（只读）。
- 删除注册不删除真实文件。

## 不做

- GitHub 不做登录。
- 项目有会话时禁止删除。
- 注册后不能改设备、路径、名称。

## 验收

- 内部项目创建在 `~/ridge-workspace/项目/<name>`，projectType='internal'，externalOrigin=null。
- 外部仓库注册时 projectType='external'，externalOrigin='folder' 或 'github'。
- 外部仓库不进入 RAG（工作空间内文件才进入）。
- 内部项目目录受工作空间 RAG 和隐藏版本管理覆盖（因为它是工作空间子目录）。
- 内部项目不作为 pi cwd；相关会话 cwd = 工作空间目录。
- 外部仓库可作为 pi cwd；外部仓库会话 cwd = 外部仓库路径。

## Spec 草案

### API

- `POST /api/workspace/projects/internal` — 创建工作空间内项目
- `POST /api/workspace/projects/external` — 注册外部仓库（文件夹/Git）
- `PATCH /api/workspace/projects/:id` — 更新（仅名称/归档状态）
- `DELETE /api/workspace/projects/:id` — 删除注册（不删文件）

### 行为

- 外部仓库必须绑定设备（服务器自身或桌面设备）。
- 同设备同路径唯一。
- GitHub 无权限时报错。
- 有会话的项目/外部仓库禁止删除。

### 测试

- 内部项目创建目录且 projectType='internal'。
- 外部仓库注册 projectType='external'、externalOrigin='folder'。
- 重复路径注册失败。
- 归档后只读。
- 删除注册不删除真实文件。
- 内部项目不作为 pi cwd（后端/前端双重防线）。
