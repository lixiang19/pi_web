# 2026-04-21 node-pty 安装契约收口

- 日期：2026-04-21
- 状态：已完成
- 目标：修复开发环境启动时 `@pi/server` 因缺失 `node-pty` 包而无法启动的问题，并把安装说明与仓库真实包管理器契约收敛一致。

## 1. 现象

启动 `pnpm run dev` 时，Web 正常启动，但 server 直接崩溃：

```text
Error [ERR_MODULE_NOT_FOUND]: Cannot find package 'node-pty'
imported from packages/server/src/terminal-runtime.ts
```

## 2. 根因

问题不在终端运行时代码，而在本地依赖安装链路：

- `packages/server/package.json` 已经正确声明 `node-pty`
- 根 `package.json` 也已经把 `node-pty` 放进 `pnpm.onlyBuiltDependencies`
- 但仓库根 `README.md` 仍写着 `npm install`
- 本仓库实际锁定 `pnpm`，原生依赖安装契约也是围绕 `pnpm` 建立的

结果就是：文档给出的安装方式和仓库真实契约不一致，容易出现本地依赖未正确落位，最终在 server 启动时表现为 `node-pty` 缺包。

## 3. 修复

本次收口分两部分：

1. **修当前环境**
   - 执行 `pnpm install`
   - 让 `packages/server/node_modules/node-pty` 正确链接到工作区

2. **修仓库说明**
   - 更新 `README.md`：本地开发命令改为 `pnpm install` / `pnpm run dev`
   - 明确补充说明：不要再用 `npm install`，因为仓库依赖 `pnpm.onlyBuiltDependencies` 管理 `better-sqlite3`、`node-pty` 这类原生依赖
   - 更新 `文档/模块梳理/Web工作台与Pi服务桥接模块.md`，把服务层原生依赖安装契约描述补全到 `better-sqlite3 + node-pty`
   - 更新 `文档/记忆/MEMORY.md`，记录“包管理器一致性”教训

## 4. 验证

已执行并通过：

- `pnpm install`
- `pnpm --filter @pi/server exec node -e "import('node-pty').then(()=>console.log('server-import-ok'))"`
- `pnpm --filter @pi/server exec tsx -e "import('./src/terminal-runtime.ts').then(()=>console.log('terminal-runtime-import-ok'))"`
- `pnpm run dev:server`

结果：server 已可正常启动，日志输出 `Pi server listening on http://127.0.0.1:3000`。
