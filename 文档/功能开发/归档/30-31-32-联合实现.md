## Reviewer 复查修复（全部完成）

### 1. 路径安全
- 内部项目名称增加 `validateInternalProjectName`：禁止 `/`、`\`、`..`、控制字符（0x00-0x1F）、绝对路径前缀
- 创建后 `fs.realpath` 校验路径仍在 `~/ridge-workspace/项目/` 内；若逃脱则删除目录并返回 400
- `project_id` 从 `hashPath`（MD5 易碰撞）改为 `crypto.randomUUID()`（稳定随机 ID）

### 2. 项目唯一性模型
- 移除 `projects.path UNIQUE` 约束
- 保留 `(COALESCE(device_id, 'server'), path) UNIQUE` 索引（migration v13）
- 同设备同路径 → 409；不同设备同路径 → 201
- 新增测试覆盖：不同设备可注册同一路径

### 3. 桌面外部仓库注册
- `registerExternalProject` 区分 server path 与 desktop path
- server-bound：继续 `fs.stat` 校验目录存在
- desktop-bound：跳过 `fs.stat`（路径在桌面端），但校验 deviceId 存在、device_type === 'desktop'
- 未传 deviceId 默认绑定 'server'

### 4. 归档只读
- 会话创建路由增加 `matchingProject?.archivedAt` 检查 → 403 "项目已归档"
- 消息发送已有 `session.archived` 检查 → 403 "归档会话不可发送消息"
- 测试覆盖：归档项目禁止新建会话、归档会话禁止发消息

### 5. 桌面调度链路
- `desktop-forward router` 已挂载到 `/api/devices`
- `GET /api/devices/:deviceId/sse`：SSE 管道，桌面 Pi 事件回传 Web 客户端
- `POST /api/devices/:deviceId/forward`：Web 请求转发到桌面设备
- `forwardRunRequestToDesktop` 通过 WebSocket 发送请求，按 `requestId` 等待桌面回复
- 服务器只做管道，不保存桌面 Pi 消息正文

### 6. 前端设备 API token 支持
- `heartbeatDevice(deviceId, token)`：body 传 token
- `renameDevice(deviceId, name, token)`：body 传 token
- `getDeviceBundle(deviceId, token)`：query `?token=`
- `ackDeviceBundle(deviceId, bundleId, token)`：body 传 token
- 前端测试增加函数签名断言（arity）

### 7. Bundle 物化与 Pi 启动
- `materializeBundle(bundle, targetDir)`：支持 base64 二进制文件正确落盘
- Pi 启动参数 `--agent-dir <targetDir>/agents --skill-dir <targetDir>/skills`
- Bundle ack 持久化到 DB（`device_bundle_acks`），覆盖更新

### 8. Bundle 配置安全
- `loadJsonConfig` 不再静默 `catch → {}`；无效 JSON 抛 400，ENOENT 返回 `{}`
- 二进制文件（`.png`, `.pdf`, `.zip` 等）用 base64 编码，文本文件用 utf-8
- `BundleResource` 新增 `encoding: 'utf-8' | 'base64'` 字段

### 9. 项目级 agents/skills 覆盖全局
- `generateRuntimeBundle` 新增可选 `projectPath` 参数
- 项目 `.pi/agents` 和 `.pi/skills` 覆盖全局同名文件
- 测试覆盖：同路径 project skill 优先于 global skill

### 10. 文档更新
