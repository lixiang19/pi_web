# 长期数据储存 方案

- 状态：草案
- 日期：2025-04-08
- 目标版本：v1.0

## 1. 背景与问题

- **背景**：当前项目使用 localStorage 存储用户偏好，缺乏统一的数据持久化架构
- **现象/痛点**：
  - 文件树收藏功能待实现，需要确定存储方案
  - 用户设置分散在各组件
  - 未来需要跨设备同步能力
- **成功定义**：建立统一的 JSON 文件存储架构，支持本地存储并预留同步接口

## 2. 目标与非目标

**目标：**
- 建立纯服务端 JSON 文件存储（~/.ridge/）
- 实现原子写入 + 版本迁移机制
- 前端内存状态管理（启动时加载，修改时保存）
- 完成后应用到文件树收藏功能

**非目标：**
- 不使用 localStorage
- 不使用 SQLite/数据库
- 不处理敏感数据加密

## 3. 现状探索

**参考项目（openchamber）关键模式：**
- `~/.ridge/settings.json` - 服务端配置存储
- `writeJsonFileAtomic()` - 原子写入（tmp + rename）
- `migrateSettings()` - 版本迁移链
- `persistSettingsLock` - 并发锁控制

**当前项目存储现状：**
- `lib/theme.ts` - 主题 localStorage（需要迁移）
- `session-metadata.js` - 服务端会话元数据（JSON 文件）

## 4. 资料与依据

**参考模式：**
- openchamber 原子写入：tmp文件 + rename，防写入中断
- openchamber 版本迁移：migrate 函数链，平滑升级

## 5. 方案概览

### 5.1 方案摘要

- **一句话**：纯服务端 JSON 文件存储，原子写入 + 版本迁移，存储目录 `~/.ridge/`
- **取舍**：JSON 文件而非数据库（简单可靠），服务端集中存储而非 localStorage（统一架构）
- **影响面**：新增存储模块，移除 localStorage 使用

### 5.2 风险与回滚

- **风险1**：服务端文件损坏 → **缓解**：原子写入 + 备份
- **风险2**：版本迁移失败 → **缓解**：保留旧版本读取逻辑
- **风险3**：并发写入冲突 → **缓解**：Promise 链锁控制

### 6.3 服务端实现

**原子写入：**
```javascript
// server/src/utils/fs.js
async function writeJsonFileAtomic(filePath, data) {
  const tmpFile = `${filePath}.${process.pid}.${Date.now()}.tmp`;
  await fs.mkdir(path.dirname(filePath), { recursive: true });
  await fs.writeFile(tmpFile, JSON.stringify(data, null, 2), 'utf8');
  await fs.chmod(tmpFile, 0o600);
  await fs.rename(tmpFile, filePath);  // 原子替换
}
```

**版本迁移：**
```javascript
// server/src/utils/migrations.js
const migrations = [
  { version: 1, migrate: (data) => data },
  { version: 2, migrate: (data) => ({ ...data, newField: true }) }
];

function migrate(data, targetVersion) {
  const currentVersion = data.version || 1;
  for (let v = currentVersion; v < targetVersion; v++) {
    data = migrations.find(m => m.version === v + 1)?.migrate(data) || data;
  }
  return { ...data, version: targetVersion };
}
```

**并发锁：**
```javascript
// server/src/utils/lock.js
const locks = new Map();

async function withLock(key, fn) {
  const current = locks.get(key) || Promise.resolve();
  const next = current.then(fn).catch(fn);
  locks.set(key, next);
  return next;
}
```
