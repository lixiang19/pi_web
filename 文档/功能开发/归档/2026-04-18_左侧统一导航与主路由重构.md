# 左侧统一导航与主路由重构

## 目标

- 把左侧“会话列表”升级为统一路由菜单。
- 明确 `会话` 才是一个主功能路由，当前聊天区 + 文件/Git 属于会话主路由内容。
- 把 `搜索 / 文件 / 终端 / 自动化 / 数据集 / 空间 / 设置` 全部落成一级路由页面。
- 保持会话高频切换的活跃状态不被路由生命周期破坏。

## 最终实现

### 1. 路由收口

- 根路由统一收口到 `PlatformShell.vue`。
- `/` 默认重定向到 `/chat`。
- 主路由收敛为：
  - `/chat`
  - `/search`
  - `/files`
  - `/terminal`
  - `/automations`
  - `/datasets`
  - `/spaces`
  - `/settings`
- 删除旧的 `design`、`session-detail` 在主工作流中的路由职责，不再把它们当统一工作台结构的一部分。

### 2. 左侧导航重构

- 新增 `WorkbenchSidebar.vue` 作为统一左侧导航。
- 顶部固定一级入口：
  - 新聊天
  - 搜索
  - 文件
  - 终端
  - 自动化
  - 数据集
  - 空间
  - 设置
- 下部继续保留会话树能力：
  - 项目分组
  - worktree 分组
  - 搜索会话
  - 父子会话展开
  - 重命名
  - 删除
  - worktree 新建/删除

### 3. 会话与路由边界

- 会话不进入 URL 参数。
- 左侧点击会话时：
  - 先切回 `/chat`
  - 再激活会话池中的目标会话
- `新聊天` 是动作，不是页面：
  - 先切回 `/chat`
  - 再创建草稿会话
- `PlatformShell.vue` 对 `/chat` 使用 `KeepAlive`。
- 会话真正的活跃态仍然由 `useSessionLruPool.ts` 托管，而不是交给路由。

### 4. 新页面

- `/files`
  - 复用 `WorkspaceFileTree.vue`
  - 目录根优先取当前活跃会话的 worktree/cwd，无活跃会话回退 `workspaceDir`
- `/settings`
  - 删除旧设置页自带内部侧栏
  - 保留主题/明暗模式等真实设置内容
- `/search`、`/terminal`、`/automations`、`/datasets`、`/spaces`
  - 全部落成独立页面
  - 只提供真实页面骨架与清晰职责说明
  - 不伪造数据或 mock 列表

## 测试

- 新增测试覆盖：
  - 统一导航动作会先回到 `/chat` 再创建/切换会话
  - 文件页根目录解析优先级
  - `PlatformShell` 在跨路由切换时保活 `/chat`
- 通过：
  - `npm run lint`
  - `npm run typecheck`
  - `npm run test --workspace @pi/web`

## 结论

- 会话是一级主功能，但单个会话不是单独路由。
- 左侧会话列表本质上是“被提升到一级导航中的高频二级菜单”。
- 路由负责表达当前功能域；会话池负责保存会话生命状态。
