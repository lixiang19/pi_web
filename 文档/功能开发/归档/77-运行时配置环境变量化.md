# 77 运行时配置环境变量化

## 目标

把外部服务密钥、服务地址和部署参数统一收口到 `.env` / 环境变量，避免 Exa、Python Converter、SiliconFlow 这类运行时配置继续进入 `app_settings`。

## 实现

- 新增 `packages/server/src/env.ts`：
  - 解析 `.env` / `.env.local`。
  - 从仓库根目录到 server 运行目录依次加载。
  - shell 环境变量优先级最高。
  - `.env.local` 可覆盖 `.env`。
- `packages/server/src/index.ts` 首行导入 `./env.js`。
- `packages/server/src/exa-tools.ts` 只读取：
  - `EXA_API_KEY`
  - `EXA_BASE_URL`
- `packages/server/src/conversion-service-client.ts` 只读取：
  - `PYTHON_CONVERTER_BASE_URL`
  - `PYTHON_CONVERTER_API_KEY`
  - `PYTHON_CONVERTER_CALLBACK_TOKEN`
  - `PYTHON_CONVERTER_CALLBACK_BASE_URL`
- `packages/server/src/siliconflow-embedding-client.ts` 只读取：
  - `SILICONFLOW_EMBEDDING_API_KEY` / `SILICONFLOW_API_KEY`
  - `SILICONFLOW_EMBEDDING_BASE_URL` / `SILICONFLOW_BASE_URL`
  - `SILICONFLOW_EMBEDDING_MODEL`
  - `SILICONFLOW_EMBEDDING_DIMENSIONS`
  - `SILICONFLOW_EMBEDDING_TIMEOUT_MS`
  - `SILICONFLOW_EMBEDDING_MAX_RETRIES`
- 新增 `.env.example`，`.gitignore` 忽略真实 `.env`。

## 边界

- `app_settings` 继续保存主题、语言、默认模型等产品状态。
- 外部服务密钥和部署参数不写入数据库设置表。
- `.env` 不提交真实值，部署环境可直接注入同名变量。
