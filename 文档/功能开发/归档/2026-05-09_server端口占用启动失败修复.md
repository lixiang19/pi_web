# 2026-05-09 server 端口占用启动失败修复

- 日期：2026-05-09
- 状态：已完成
- 目标：修复 server 端口占用时未捕获 `error` event 直接崩溃，并让 web proxy 可随 `PORT` 对齐。

## 现象

server 启动时报错：

`Error: listen EADDRINUSE: address already in use :::3000`

同时主入口 catch 文案仍是 `Failed to initialize ridge.db`，容易误判为数据库问题。

## 修复

- server 监听封装为 `listenHttpServer`，监听 `error` 并 reject。
- 主入口对 `EADDRINUSE` 输出明确提示：端口被占用，可停止旧进程或用其他 `PORT`。
- web Vite proxy target 改为读取 `process.env.PORT || "3000"`，与 server 默认端口保持一致。

## 验证

- 端口占用单测通过。
- `npm run check` 通过。
