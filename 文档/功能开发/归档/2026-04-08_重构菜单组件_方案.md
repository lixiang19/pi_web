# PlatformShell 导航组件重构方案

- 状态：计划中
- 日期：2026-04-08
- Owner：admin
- 目标版本：v1.0

## 1. 背景与问题

### 背景
PlatformShell 是应用的顶级布局组件，提供左侧导航栏 + 右侧内容区的整体框架。当前实现将所有逻辑集中在单个文件中，存在状态管理和可维护性问题。

### 当前问题
| 问题 | 影响 | 位置 |
|------|------|------|
| 状态孤岛 | `isCollapsed` 使用本地 ref，与 settings store 不同步，刷新后状态丢失 | PlatformShell.vue:15 |
| 组件臃肿 | 130+ 行包含布局、导航、状态、样式多重职责 | PlatformShell.vue |
| 样式混合 | 样式与逻辑混杂，不便于主题切换和样式覆盖 | PlatformShell.vue |

### 成功定义
PlatformShell 拆分为可复用的原子组件，状态与 settings store 同步，样式参考 Linear.app 简洁现代风格。

## 2. 目标与非目标

### 目标
- [G1] 状态同步：PlatformShell 的 sidebarCollapsed 与 settings.ts store 同步
- [G2] 组件拆分：将 PlatformShell 拆分为 Sidebar、NavItem、Logo 等独立组件
- [G3] 样式重构：采用 Linear.app 简洁风格，使用 shadcn-vue 主题变量
- [G4] 保持兼容：现有 API 和路由行为不变

### 非目标
- [NG1] 不重构 SessionSidebar（超出本次范围）
- [NG2] 不新增导航项或路由
- [NG3] 不添加动画效果以外的交互功能

## 3. 现状探索（Explorer）

### 入口与调用链
| 文件:行号 | 说明 |
|-----------|------|
| [router/index.ts:8](packages/web/src/router/index.ts:8) | PlatformShell 作为根布局组件包裹所有页面 |
| [PlatformShell.vue:1](packages/web/src/layouts/PlatformShell.vue:1) | 当前实现：本地 isCollapsed ref，未使用 store |
| [settings.ts:22](packages/web/src/stores/settings.ts:22) | 已定义 sidebarCollapsed 计算属性，但未被 PlatformShell 消费 |

### 关键数据结构
```ts
// settings.ts
const sidebarCollapsed = computed(() => settings.value?.sidebarCollapsed ?? false)
function toggleSidebar(): void {
  setSidebarCollapsed(!sidebarCollapsed.value)
}

// PlatformShell.vue - 当前问题
const isCollapsed = ref(false) // 本地 ref，未与 store 同步
const toggleSidebar = () => {
  isCollapsed.value = !isCollapsed.value
}
```

### 约束与坑点
- settings store 已提供完整的 sidebarCollapsed 状态和 toggleSidebar 方法
- 路由配置固定，不能改变组件结构导致路由失效
- 必须使用 shadcn-vue 主题变量（sidebar-* 系列）

## 4. 资料与依据（Librarian）

### Linear.app 风格特征
- 极简边框，1px 细线分隔
- 图标 + 文字紧凑布局（gap-2 或 gap-3）
- Active 状态使用 subtle background + left border accent
- 圆角使用 rounded-md（6px）而非大圆角
- 过渡动画 150ms ease

### shadcn-vue Sidebar 规范
```css
--sidebar: 侧边栏背景
--sidebar-foreground: 文字色
--sidebar-accent: 悬停/激活背景
--sidebar-border: 边框色
```

## 5. 方案概览（主方案）

### 5.1 方案摘要
**一句话**：将 PlatformShell 重构为"容器组件 + 原子组件"结构，状态下沉至 settings store，样式升级为 Linear 简洁风格。

**取舍**：
- 选择组件拆分而非单文件优化：提高可测试性和复用性
- 选择 store 状态而非 props 传递：避免多层传递，符合 Vue 生态最佳实践

**影响面**：
- 修改 [PlatformShell.vue](packages/web/src/layouts/PlatformShell.vue)
- 新建 [components/layout/Sidebar.vue](packages/web/src/components/layout/Sidebar.vue)
- 新建 [components/layout/SidebarNavItem.vue](packages/web/src/components/layout/SidebarNavItem.vue)
- 新建 [components/layout/SidebarLogo.vue](packages/web/src/components/layout/SidebarLogo.vue)

### 5.2 风险与回滚

**风险**：
| 风险 | 缓解措施 |
|------|---------|
| 状态同步引入 Bug | 保留原本地 ref 作为 fallback，逐步迁移 |
| 组件拆分后样式不一致 | 统一使用 shadcn-vue sidebar-* token |

**回滚**：
- 代码层面：单个文件回退，无数据迁移
- 功能开关：如 store 同步出现问题，可切回本地 ref

## 6. 详细设计（Spec）

### 6.1 交互/行为（如适用）
- 点击导航项：路由切换，active 状态自动更新
- 点击折叠按钮：切换 sidebarCollapsed 状态，同步到 store
- Hover 导航项：subtle background 变化，150ms 过渡

### 6.2 接口与契约

**Sidebar.vue Props**
```ts
interface SidebarProps {
  collapsed?: boolean
  navItems: NavItem[]
}

interface NavItem {
  name: string
  label: string
  to: string
  icon: Component
}
```

**SidebarNavItem.vue Props**
```ts
interface SidebarNavItemProps {
  item: NavItem
  active: boolean
  collapsed: boolean
}
```

### 6.3 数据与存储（如适用）

```ts
// 重构前
const isCollapsed = ref(false) // 本地状态

// 重构后
import { useSettingsStore } from '@/stores/settings'
const settings = useSettingsStore()
// 使用 computed 绑定 store 状态
const isCollapsed = computed(() => settings.sidebarCollapsed)
const toggleSidebar = () => settings.toggleSidebar()
```

### 6.4 兼容性与发布
- 保持现有路由结构不变
- 保持现有 navItems 配置不变
- 无破坏性变更，用户无感知

### 6.5 可观测性
**日志点**：
- toggleSidebar 调用时输出新状态
- 状态与 store 同步失败时警告
