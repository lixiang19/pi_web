# 搜索前端重构 方案

- 状态：候选
- 日期：2026-04-19
- Owner：前端

## 1. 背景与问题

- 背景：当前搜索功能有两套实现（独立搜索页 + 侧栏过滤框），UI 陈旧，体验感差
- 痛点：WorkbenchFeatureShell 包装导致页面结构臃肿；侧栏搜索框与全局搜索职责重叠；无搜索词高亮；空态设计不友好
- 成功定义：搜索页视觉焦点清晰、操作路径直接，侧栏只做会话导航

## 2. 目标与非目标

- 目标：
  - 彻底重设计 SearchPage.vue，现代化简洁布局
  - 删除 SessionSidebar.vue 和 SessionSidebarSlack.vue 中的侧栏搜索框
  - 搜索结果高亮关键词
  - 优雅的空态与无结果态
- 非目标：
  - 不扩展消息搜索、文件搜索
  - 不改为实时搜索（保持 Enter 提交式）
  - 不加模糊匹配、搜索历史等高级功能

## 3. 现状探索

- 入口：`useWorkbenchPrimaryNavigation.ts` 的导航菜单 → `router /search` → `SearchPage.vue`
- 关键文件：
  - [SearchPage.vue](packages/web/src/pages/SearchPage.vue) — 独立搜索页（WorkbenchFeatureShell + 2张Card）
  - [SessionSidebar.vue](packages/web/src/components/chat/SessionSidebar.vue) — 侧栏搜索框 L63/L91-92/L307
  - [SessionSidebarSlack.vue](packages/web/src/components/chat/SessionSidebarSlack.vue) — 侧栏搜索框 L41/L63/L169-171
  - [session-sidebar.ts](packages/web/src/lib/session-sidebar.ts) — `buildSessionProjects` 支持 `query` 可选参数
- 约束：
  - `buildSessionProjects` 的 `query` 字段为可选，删除搜索框后不传即可，无需修改 lib

## 4. 方案概览

### 4.1 方案摘要

- 一句话：删除侧栏搜索框，用现代全版搜索布局重写 SearchPage
- 取舍：不引入额外依赖（无 fuse.js），保持现有数据链路不变，仅重构视图层
- 影响面：SearchPage.vue（全量重写）、SessionSidebar.vue（删成员）、SessionSidebarSlack.vue（删成员）

### 4.2 风险与回滚

- 风险：删除侧栏过滤框后，在会话很多时侧栏浏览变难 → 缓解：侧栏保持折叠/展开的层级导航，本身已足够
- 回滚：git revert 即可，无数据结构变更

## 5. 详细设计

### 5.1 新 SearchPage 布局

```
┌─────────────────────────────────────────┐
│  页面顶部留白（pt-16）                     │
│                                           │
│  [🔍  输入会话标题关键词...        搜索 ]  │  ← 大号搜索框，h-12，无 Card 包裹
│                                           │
│  ── 3 条结果  "关键词" ──────────────────  │  ← 分割线 + 统计文字（有搜索时显示）
│                                           │
│  标题（<mark>关键词</mark>高亮）   [归档]  │  ← 结果行
│  项目/路径                         时间   │
│  ─────────────────────────────────────   │
│  另一个会话标题               [会话]      │
│  ...                                      │
└─────────────────────────────────────────┘
```

**空态（未搜索）**：居中图示 + "搜索你的所有会话 / 输入关键词按 Enter"
**无结果态**：居中灰色提示 "没有找到关于 "xxx" 的会话"

### 5.2 关键词高亮算法

```typescript
const highlight = (text: string, query: string): string => {
  if (!query) return escapeHtml(text);
  const idx = text.toLowerCase().indexOf(query);
  if (idx === -1) return escapeHtml(text);
  return (
    escapeHtml(text.slice(0, idx)) +
    `<mark class="bg-primary/20 text-primary rounded-[2px] px-0.5">${escapeHtml(text.slice(idx, idx + query.length))}</mark>` +
    escapeHtml(text.slice(idx + query.length))
  );
};
```
（使用 `v-html` 渲染，输入来自已加载的会话数据，无 XSS 风险）
