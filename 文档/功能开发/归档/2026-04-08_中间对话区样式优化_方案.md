# 中间对话区样式优化方案
- 状态：已确认
- 日期：2026-04-08
- Owner：GitHub Copilot
- 目标版本：v1
## 1. 背景与问题
- **背景**：当前中间对话区顶部(WorkbenchChatHeader)有会话标题+3个Select选择器，占据过多视觉空间且喧宾夺主
- **现象/痛点**：
  - 顶部区域显示会话标题、模型选择、思考级别、Agent选择，视觉层次混乱
  - 输入框与顶部分离，操作流不够连贯
  - 整体设计缺少极简Linear风格的精致感
- **成功定义**：会话区顶部完全干净，输入框底部集成工具栏，整体呈现清爽、极简、有设计感的视觉效果
## 2. 目标与非目标
- **目标**：
  - 移除顶部WorkbenchChatHeader组件
  - 将模型/思考/Agent选择器移入输入框底部工具栏
  - 采用小图标+文字缩写的紧凑展示形式
  - 实现极简Linear风格（清爽留白、精致排版、微妙过渡）
  - 保持明暗主题兼容性
- **非目标**：
  - 不改动侧边栏SessionSidebar的样式
  - 不改动消息流MessageStream的渲染逻辑
  - 不添加动画特效或复杂交互
## 3. 现状探索
- **入口与调用链**：
  - [WorkbenchPage.vue](packages/web/src/pages/WorkbenchPage.vue:24) -> WorkbenchChatPanel -> WorkbenchChatHeader（删除目标）/ WorkbenchComposer（改造目标）
  - WorkbenchChatHeader 位于 packages/web/src/components/workbench/chat/WorkbenchChatHeader.vue
  - WorkbenchComposer 位于 packages/web/src/components/workbench/chat/WorkbenchComposer.vue
- **关键数据结构/状态**：
  - `composer.selectedModel` / `composer.selectedThinkingLevel` / `composer.selectedAgent` — 通过props传入WorkbenchComposer
  - 选择器通过`@select-*`事件回调更新状态，已在useWorkbenchPage中处理
- **约束与坑点**：
  - WorkbenchComposer已接收`commands/skills/prompts`用于资源选择器，需确保底部工具栏不冲突
  - 需要处理无选项时的"Auto"默认值显示
  - 父会话返回按钮(parentSessionId)需要找到新位置或移除
## 4. 资料与依据
- **关键结论**：
  - 使用shadcn-vue Select组件作为选择器基础，可复用现有代码
  - 图标使用lucide-vue-next（项目已引入）
  - Linear风格特征：充足留白、微妙阴影、圆角一致、色彩克制、排版精致
## 5. 方案概览
### 5.1 方案摘要
- **一句话**：删除顶部头组件，在输入框底部添加水平工具栏，用图标+文字缩写展示当前选项，点击弹出选择器
- **取舍**：牺牲顶部的即时可读性（标题/选项状态），换取更干净的视觉层次和更紧凑的操作流
- **影响面**：
  - WorkbenchChatPanel.vue — 移除WorkbenchChatHeader引用和props传递
  - WorkbenchComposer.vue — 新增底部工具栏，集成3个选择器
  - useWorkbenchPage.ts — 无需改动，状态管理已支持
### 5.2 风险与回滚
- **风险**：
  - 父会话返回按钮失去位置 — 缓解：暂时移除（父会话功能可后续重新设计入口）
  - 选项选择器UI拥挤 — 缓解：使用Popover弹层，保持底部工具栏仅展示当前值
- **回滚**：git回退 WorkbenchChatPanel.vue 和 WorkbenchComposer.vue 的修改即可恢复
## 6. 详细设计
### 6.1 交互/行为
- **输入框底部工具栏布局**：
  - 左侧：模型选择器（Brain图标+缩写如"GPT-4o"）+ 思考级别（Lightbulb图标+"Think"）+ Agent选择（Bot图标+名称缩写）
  - 右侧：资源选择按钮（/图标）+ 发送按钮 — 保持原位但样式统一
  - 整体一行，高度32px，与输入框间距12px
- **选择器交互**：
  - 点击任一选项触发Popover弹层，内含Select选择列表
  - 选择后Popover自动关闭，底部工具栏实时更新显示
- **边界条件**：
  - 值为"Auto"时显示为"Auto"或图标状态指示
  - 长Agent名称截断显示前8字符+省略号
### 6.2 接口与契约
- **WorkbenchComposer 新增Props**：
  ```ts
  modelOptions: Array<{ label: string; value: string }>
  thinkingOptions: Array<{ value: string; label: string }>
  agents: AgentSummary[]
  autoModelValue: string
  autoThinkingValue: string
  noAgentValue: string
  ```
- **WorkbenchComposer 新增Events**：
  ```ts
  selectModel: [value: unknown]
  selectThinking: [value: unknown]  
  selectAgent: [value: unknown]
  returnToParent: []  // 如有需要
  ```
### 6.3 兼容性与发布
- **兼容策略**：纯UI改动，无数据或API变更，向后兼容
- **发布步骤**：
 1. 修改WorkbenchChatPanel移除Header
 2. 修改WorkbenchComposer添加工具栏
 3. 调整WorkbenchPage传递props给Composer
 4. 验证明暗主题下工具栏样式正确
### 6.4 视觉设计规范
- **Linear风格参考**：
  - 底部工具栏背景透明，与页面背景融合
  - 选项按钮：hover时显示`bg-muted`背景，圆角`rounded-md`
  - 文字：`text-xs font-medium text-muted-foreground`，选中状态`text-foreground`
  - 图标：`size-3.5`，颜色与文字同步
  - 按钮间距：`gap-1`，padding `px-2 py-1`
  - 选项之间分隔：`border-l border-border mx-2`
