# 文件预览操作区与标签页方案

- 状态：已确认
- 日期：2026-04-19
- Owner：ridge
- 目标版本/里程碑：Workbench File Preview Phase 1

## 1. 背景与问题

- 背景：当前工作台已经具备会话区与右侧文件树/Git 功能区，但缺少独立操作区，无法承载文件预览、Markdown 编辑以及后续更多标签式能力。
- 现象/痛点：
  - 当前布局仍是“会话区 + 右侧功能区”两列，无法在中间承载独立操作区。
  - 文件树只能展开目录和拖拽路径，点击文件不会打开内容。
  - 服务端只有目录树接口，没有文件内容读取、图片流、Markdown 保存这条完整链路。
  - 后续该区域会承载多种功能，如果继续把文件预览塞进文件树面板，会把职责边界做脏。
- 成功定义：用户点击文件树中的文件后，会话区与文件树之间出现独立操作区标签并打开对应文件；同一路径重复点击复用同一标签；Markdown 可编辑且自动保存；图片/代码/普通文本按类型预览；预览标签按会话独立保留，操作区宽度可拖拽。

## 2. 目标与非目标

- 目标：
  - 将当前工作台升级为“会话区 | 操作区 | 文件树/Git 面板”三列结构。
  - 新增独立的操作区模块与状态编排，后续可复用到其它入口，不把预览逻辑耦合进文件树组件。
  - 打通文件树点击 -> 打开标签 -> 拉取内容 -> 按类型预览的完整链路。
  - 为 Markdown 文件提供编辑能力，并按自动保存语义写回磁盘。
  - 保持预览标签按当前会话独立维护，与现有 LRU 会话实例生命周期一致。
  - 支持操作区宽度拖拽调整，并与现有主题体系保持一致。
- 非目标：
  - 不在本阶段接入“其它入口打开文件”的具体业务，只把能力抽离为独立模块。
  - 不引入完整 IDE 级代码编辑器；Markdown 以轻量编辑为目标，其它类型只读。
  - 不做旧两列布局兼容，也不保留旧结构分支。
  - 不支持任意二进制文件编辑；非文本且非图片文件只展示不支持预览状态。

## 3. 现状探索（Explorer）

- 入口与调用链：
  - `packages/web/src/components/workbench/SessionTabContent.vue:18` - 当前单会话工作台通过 `usePerSessionChat` 组织状态。
  - `packages/web/src/components/workbench/SessionTabContent.vue:134` - 当前模板只挂载右侧 `ProjectFilePanel`，整体仍是两列布局。
  - `packages/web/src/components/workbench/ProjectFilePanel.vue:15` - 右侧功能区只管理 `git/files` 两个标签。
  - `packages/web/src/components/WorkspaceFileTree.vue:96` - 文件树点击逻辑只处理目录展开，文件点击不会向上游发出事件。
  - `packages/server/src/index.ts:1772` - 服务端目前只有 `/api/files/tree`，没有文件内容读写契约。
- 关键数据结构/状态：
  - `packages/protocol/src/index.ts:283` - `FileTreeEntry` 当前只包含名称、路径、类型和相对路径，不承载预览元数据。
  - `useSessionLruPool + SessionTabContent` - 会话实例通过 `v-show` 保活，天然适合作为“按会话保留预览标签”的宿主。
  - `chat.fileTreeRoot` - 当前右侧目录根已经由会话上下文统一派生，不应在新预览链路中重复计算。
- 约束与坑点：
  - 文件读写必须继续遵守 worktree/root 安全边界，不能绕开现有服务端约束。
  - 预览状态不能做成全局单例，否则会破坏“每个会话独立保留标签”的要求。
  - 用户明确要求不做兼容方案，必须直接把工作台结构提升为三列。
  - Markdown 选择自动保存，意味着关闭标签和切换会话前必须保证最新内容已落盘或显式报错，不能静默丢失。

## 4. 资料与依据（Librarian）

- 关键结论：
  - 前端已具备 `vue-stream-markdown`、`Textarea`、`Separator` 等基础能力，可直接复用 Markdown 渲染、轻量编辑和分隔条视觉，不需要引入非常见第三方库。
  - 当前仓库没有现成的可拖拽分栏组件，因此拖拽宽度需要围绕现有 shadcn-vue/基础 UI 组件补一层交互逻辑，而不是再塞回固定宽度面板。
  - 文件预览问题的根源在服务端缺少内容接口，而不是前端少一个组件，所以必须同时补上读取、图片流和保存契约。
  - 操作区应独立成模块，由上游通过事件打开文件，而不是把状态硬塞进 `WorkspaceFileTree` 或 `ProjectFilePanel` 内部。

## 5. 方案概览（主方案）

### 5.1 方案摘要

- 一句话：在 `SessionTabContent` 内新增独立的 `WorkbenchOperationPanel` 中列，用会话级 `useWorkbenchFilePreview` 管理标签与保存状态；文件树点击通过事件链路打开标签；服务端新增安全的文件内容读取、图片流和 Markdown 保存接口。
- 取舍：不把预览做进右侧文件树/Git 面板内部，而是直接建立独立操作区，从结构上为后续更多操作标签保留边界；Markdown 不引入重型编辑器，只做轻量编辑 + 实时预览。
- 影响面：
  - Web：工作台三列布局、操作区模块、文件树事件链路、预览状态编排。
  - Server：文件内容读取接口、图片流接口、Markdown 保存接口。
  - Protocol：文件预览载荷与保存请求的共享类型。

### 5.2 风险与回滚

- 风险：
  - 自动保存存在并发覆盖风险：通过每个标签独立的保存序号与最新写入优先策略收口，避免旧请求回写覆盖新内容。
  - 图片/大文件预览可能带来加载抖动：由服务端先识别文件类型和可读性，前端只渲染明确支持的预览模式。
  - 三列布局引入拖拽后，可能影响会话区和右侧面板最小宽度：通过宽度上下限约束保证布局稳定。
- 回滚：
  - 若开发期发现链路不可用，可回退新增操作区与文件内容接口，但不保留长期兼容分支。
  - 回滚只删除本次新增模块与接口，不改动既有 Git/文件树能力边界。

## 6. 详细设计（Spec）

### 6.1 交互/行为（如适用）

- 用户流程：
  1. 用户在文件树或收藏列表点击文件。
  2. `WorkspaceFileTree` 发出 `select-file` 事件，`ProjectFilePanel` 向上继续转发。
  3. `SessionTabContent` 把文件路径交给 `WorkbenchOperationPanel` 的预览状态管理器。
  4. 若该路径已存在标签，则直接激活已有标签；否则创建新标签并请求服务端读取内容。
  5. 操作区顶部展示文件标签，正文区域按文件类型渲染：Markdown 编辑+预览、图片预览、代码只读预览、普通文本只读预览、其它类型显示不支持状态。
  6. Markdown 编辑时自动触发保存；关闭标签、切换标签或切换会话前会先冲刷当前待保存内容。
  7. 切换会话时，各会话保留自己的操作区标签与当前活动项。
- 边界条件：
  - 同一路径重复点击不新增重复标签，只切换到已有标签。
  - 关闭活动标签后自动切到相邻标签；没有标签时展示空态。
  - Markdown 保存失败时必须在标签内显示错误，并保留当前编辑内容。
  - 图片通过独立 blob 接口展示，不把二进制内容塞进 JSON。
  - 不支持预览的文件显示文件类型、路径和“不支持预览”说明。

### 6.2 接口与契约

- API/函数签名：
  - `GET /api/files/content?root=<root>&path=<path>`：`{ root, path } -> FilePreviewPayload`
  - `GET /api/files/blob?root=<root>&path=<path>`：返回图片或其它二进制流，供 `<img>` 等只读预览使用
  - `PUT /api/files/content`：`{ root, path, content } -> FileSaveResponse`
  - `useWorkbenchFilePreview(rootDir)`：`openFile(path) / closeFile(tabId) / activateFile(tabId) / flushActiveTab() / tabs / activeTabId / panelWidth`
  - `ProjectFilePanel`：新增 `open-file` 事件，向父级抛出所选文件信息
- 错误码/异常：
  - 读取或保存目标超出 `root` 边界时返回 400。
  - 请求目录、非文件路径或不存在文件时返回 400/404，并带明确错误信息。
  - `PUT /api/files/content` 只允许 Markdown 文件写入，其它类型返回 400。
  - 无法按 UTF-8 读取的非图片文件返回 `unsupported` 预览类型，不伪造文本内容。

### 6.3 数据与存储（如适用）

- Schema/字段：
  - `FilePreviewPayload`
    - `root: string`
    - `path: string`
    - `name: string`
    - `extension: string`
    - `mimeType: string`
    - `size: number`
    - `previewKind: "markdown" | "code" | "text" | "image" | "unsupported"`
    - `content?: string`
    - `readOnly: boolean`
  - `PreviewTabState`
    - `id: string`（使用路径去重）
    - `path: string`
    - `title: string`
    - `previewKind: string`
    - `content: string`
    - `savedContent: string`
    - `isLoading: boolean`
    - `isSaving: boolean`
    - `error: string`
  - `OperationPanelState`
    - `activeTabId: string`
    - `tabs: PreviewTabState[]`
    - `panelWidth: number`
- 迁移策略：
  - 不做持久化迁移。
  - 操作区标签状态只驻留在当前 `SessionTabContent` 实例，由现有会话保活机制自然保留。

### 6.4 兼容性与发布

- 兼容策略：直接升级为三列布局，不保留旧两列结构。
- 发布步骤：
  1. 先补 protocol 类型与 server 文件内容接口。
  2. 再实现操作区状态与预览组件。
  3. 最后接入三列布局、文件树事件和宽度拖拽。
  4. 完成后执行 eslint 与 TypeScript 检查。

### 6.5 可观测性

- 日志：
  - 文件内容读取日志（仅路径、类型、大小，不记录正文）
  - Markdown 保存日志（路径、大小、成功/失败）
- 指标：
  - 操作区打开文件成功率
  - Markdown 自动保存成功率
- 告警：
  - 连续读取失败或保存失败时在界面内显式提示
